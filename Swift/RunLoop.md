**`RunLoop`** — это **цикл обработки событий**, который постоянно работает в каждом потоке ([[thread]]).

- Основная задача — **обработка событий, таймеров и источников ввода**.
    
- Поддерживает поток **UI** (главный поток) и фоновый поток.
    
- Все **UI-таймеры и жесты** работают через `RunLoop`.
    

> Проще: `RunLoop` = цикл жизни потока, который ждет события и выполняет код по мере поступления.

---

## 2. Основные потоки RunLoop

- **Главный поток (Main RunLoop)**
    
    - Отвечает за UI.
        
    - Всегда существует.
        

```swift
let mainRunLoop = RunLoop.main
```

- **Фоновый поток**
    
    - Для работы с асинхронными задачами.
        
    - Создается автоматически при создании потока.
        

```swift
let currentRunLoop = RunLoop.current
```

---

## 3. Источники событий RunLoop

RunLoop может обрабатывать:

1. **Таймеры ([[Timer]])**
    
    ```swift
    let timer = Timer.scheduledTimer(withTimeInterval: 2.0, repeats: true) { _ in
        print("Сработал таймер через RunLoop")
    }
    ```
    
2. **События ввода/вывода (Input Sources)**
    
    - Сокеты, порты, пользовательские события.
        
3. **Диспетчеризация событий [[UIKit]]**
    
    - Все жесты, касания, анимации проходят через RunLoop.
        

---

## 4. Простейшее использование

```swift
let runLoop = RunLoop.current
let distantFuture = Date.distantFuture

while true {
    // Запускаем цикл обработки событий
    runLoop.run(mode: .default, before: distantFuture)
}
```

> В обычном [[iOS]]-приложении такой цикл **не пишем вручную**, RunLoop уже управляет главным потоком.

---

## 5. Режимы RunLoop

- **Modes** — режимы, в которых RunLoop обрабатывает события.
    
- Основные режимы:
    

|Mode|Описание|
|---|---|
|`.default`|Основной режим обработки событий|
|`.common`|Общий режим для нескольких источников|
|`.tracking`|Используется при прокрутке UIScrollView|

Пример добавления таймера в режим `.common`:

```swift
let timer = Timer(timeInterval: 1.0, repeats: true) { _ in
    print("Таймер в common mode")
}
RunLoop.current.add(timer, forMode: .common)
```

---

## 6. Особенности

- `RunLoop` существует **для каждого потока**, но UI работает только на главном.
    
- Таймеры и события **не сработают**, если RunLoop не запущен или поток занят.
    
- Позволяет создавать **асинхронные очереди событий** без блокировки основного потока.
    

---

## 7. Итог

- **RunLoop** — основной механизм для обработки событий, таймеров и ввода.
    
- Главный поток всегда имеет RunLoop для UI.
    
- Таймеры, события ввода, жесты — всё проходит через него.
    
- Можно добавлять свои таймеры и источники событий в нужный **режим**.
