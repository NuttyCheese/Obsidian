**Immutable** (неизменяемый) — это объект или переменная, **значение которого нельзя изменить после создания**.

- В [[Swift]] immutable создаётся с помощью ключевого слова [[let]]
    
- ==Противоположность — **mutable** (`var`)==
    
- Применяется к **примитивным типам, структурам, коллекциям и объектам**
    

> Проще говоря: immutable = «значение создаётся один раз и больше не меняется».

---

## 2. Основные термины

| Термин                 | Описание                                                                        |
| ---------------------- | ------------------------------------------------------------------------------- |
| **Immutable**          | Значение или объект, которое нельзя изменить после создания                     |
| **Mutable**            | Значение или объект, который можно изменять                                     |
| **[[let]]**            | Ключевое слово для создания immutable переменной или константы                  |
| **[[Value Type]]**     | Структуры, перечисления (изменяемость зависит от переменной, которая их хранит) |
| **[[Reference Type]]** | Классы (объект сам mutable, но ссылку можно хранить в let)                      |

---

## 3. Основной синтаксис

```swift
let pi = 3.1415
// pi = 3.14 // Ошибка: нельзя изменить значение
```

- `let` → immutable
    
- Попытка присвоить новое значение вызывает **ошибку компиляции**
    

---

## 4. Примеры от простого к сложному

### Пример 1. Примитивные типы

```swift
let age = 25
// age = 30 // Ошибка: нельзя изменить
```

- Числа, строки, булевы значения могут быть immutable
    

---

### Пример 2. Строки

```swift
let name = "Alice"
// name += " Smith" // Ошибка: нельзя изменить
```

- Строки immutable, если объявлены через `let`
    

---

### Пример 3. Массивы

```swift
let numbers = [1, 2, 3]
// numbers.append(4) // Ошибка: нельзя изменить массив
```

- Коллекции ([[Swift/Теория/Swift/Standart Library/Array]], [[Dictionary]], [[Set]]) immutable, если объявлены через `let`
    
- Если через `var` → mutable
    

---

### Пример 4. Структуры (Value Types)

```swift
struct Point {
    var x: Int
    var y: Int
}

let p = Point(x: 10, y: 20)
// p.x = 15 // Ошибка: нельзя изменить, потому что p immutable
```

- Даже если структура имеет **var свойства**, сама переменная immutable → свойства нельзя менять
    

---

### Пример 5. Классы (Reference Types)

```swift
class Person {
    var name: String
    init(name: String) { self.name = name }
}

let alice = Person(name: "Alice")
alice.name = "Alicia" // Разрешено
```

- Класс сам mutable, но ссылка на объект immutable (`let alice`)
    
- Нельзя присвоить другой объект (`alice = Person(name: "Bob")`), но можно менять **свойства объекта**
    

---

### Пример 6. Immutable + функции

```swift
func increase(number: Int) -> Int {
    return number + 1
}

let a = 10
let b = increase(number: a)
print(a, b) // 10 11
```

- Функции не изменяют исходное значение immutable
    
- Value types + immutable = безопасная работа с данными
    

---

## 5. Особенности immutable

1. Используется для **безопасного кода и предотвращения ошибок изменения данных**
    
2. Свойство `let` = immutable, `var` = mutable
    
3. Структуры и перечисления — value type → immutable зависит от переменной
    
4. Классы — reference type → ссылка immutable, объект mutable
    
5. Применяется для **констант, параметров функции, временных значений**
    

---

## 6. Итог

- **Immutable** = значение нельзя изменить после создания
    
- Создаётся через `let`
    
- Повышает **безопасность, предсказуемость и чистоту кода**
    
- Различается для **value type и reference type**
