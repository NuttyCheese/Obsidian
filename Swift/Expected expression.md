Компилятор [[Swift]] сообщает, что **он ожидал выражение (expression), но встретил что-то другое**.

- В Swift **expression** — это любая единица, которая вычисляет значение:
    
    - литералы (`1`, `"Hello"`, `true`)
        
    - вызовы функций (`foo()`)
        
    - операции (`a + b`)
        
    - обращения к переменным/константам (`x`, `name`)
        
- Ошибка обычно возникает, когда:
    
    1. Пропущено значение в выражении.
        
    2. Используется лишний оператор.
        
    3. Нарушен синтаксис вызова функции или литерала.
        

---

### Примеры кода/сценариев возникновения

**Пример 1: Пропущено значение**

```swift
let a = // ❌ Expected expression
```

- Компилятор ожидает справа выражение для присвоения.
    

---

**Пример 2: Лишний оператор**

```swift
let sum = 5 + 
// ❌ Expected expression
```

- После `+` должно быть значение или выражение.
    

---

**Пример 3: Вызов функции с ошибкой**

```swift
print()
// ✅ правильно
print( ) // ✅ правильно
print // ❌ Expected expression, пропущены скобки
```

---

**Пример 4: Литералы и массивы**

```swift
let array = [1, 2, ]
// ✅ допустимо, но если вдруг написать просто [ , ] → ❌ Expected expression
```

---

### Как исправить

#### 1️⃣ Проверить наличие значения или выражения там, где оно ожидается

```swift
let a = 10 // ✅ исправлено
```

---

#### 2️⃣ Проверить синтаксис операторов и вызовов функций

```swift
let sum = 5 + 3 // ✅ исправлено
print("Hello") // ✅ исправлено
```

---

#### 3️⃣ Проверить литералы и конструкции

```swift
let array = [1, 2, 3] // ✅ исправлено
```

---

### Резюме

- **Expected expression** возникает, когда компилятор ожидает вычисляемое значение, но его нет или оно написано неверно.
    
- Исправляется через:
    
    1. Добавление пропущенного выражения.
        
    2. Проверку правильности синтаксиса операторов и вызовов функций.
        
    3. Проверку литералов и структур данных.
        
- Позволяет компилятору правильно построить AST и успешно собрать проект.
