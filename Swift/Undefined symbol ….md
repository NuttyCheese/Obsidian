Ошибка появляется на этапе **линковки проекта**, когда **линковщик не может найти реализацию указанного символа** (функции, переменной или класса).

- Компилятор [[Swift]]/[[Objective-C]] успешно скомпилировал файлы, но **линковщик не может собрать все объекты в исполняемый файл**.
    
- Обычно сообщение содержит тип символа и его местоположение.
    

Основные причины:

1. **Функция или метод объявлен, но не реализован.**
    
2. **Отсутствует подключённая библиотека или фреймворк.**
    
3. **Дублирование заголовков/модулей**, когда один символ не совпадает между объектными файлами.
    
4. **Разные target’ы** — функция существует в другом target’е, который не линкован.
    
5. **Архитектурные несоответствия** между библиотекой и проектом (x86_64 vs arm64).
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Метод объявлен, но не реализован**

```swift
// MyClass.swift
class MyClass {
    func doSomething() // ⚠️ объявлен, но нет реализации
}

// main.swift
let obj = MyClass()
obj.doSomething() // ⚠️ Undefined symbol: _TtC7MyApp7MyClass10doSomething
```

---

**Пример 2: Отсутствует библиотека**

```swift
import Alamofire

Alamofire.request("https://example.com") // ⚠️ Undefined symbol: _TFC9Alamofire7request
```

- Alamofire не добавлен в **Link Binary With Libraries** или **Embed Frameworks**.
    

---

**Пример 3: Разные target’ы**

- Файл с функцией находится в target `MyFramework`, но вызывается из `MyApp`, который не линкован с этим target.
    

---

### Как исправить

#### 1️⃣ Реализовать недостающий метод/функцию

```swift
class MyClass {
    func doSomething() {
        print("Done")
    }
}
```

---

#### 2️⃣ Добавить необходимые фреймворки

- **TARGETS → Build Phases → Link Binary With Libraries** → добавить недостающий framework.
    

---

#### 3️⃣ Проверить target membership

- Файл с функцией должен быть включён в **правильный target** (проверяется в File Inspector).
    

---

#### 4️⃣ Проверить архитектуры

- **Build Settings → Architectures** → убедиться, что библиотека поддерживает архитектуру вашего проекта.
    

---

#### 5️⃣ Для Objective-C/Swift взаимодействия

- Если используете **Objective-C код**, убедиться, что:
    
    - Файл включён в target
        
    - Заголовки импортированы через `#import` или Bridging Header
        
    - Методы объявлены [[@objc]] при необходимости
        

---

### Резюме

- `Undefined symbol` — это **линковочная ошибка**, указывающая на отсутствие реализации функции, класса или переменной.
    
- Исправляется через:
    
    1. Реализацию символа
        
    2. Добавление нужных фреймворков
        
    3. Проверку target membership
        
    4. Проверку архитектур и совместимости Swift/Objective-C
