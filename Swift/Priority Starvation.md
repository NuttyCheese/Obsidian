**Priority Starvation** — это ситуация, когда **низкоприоритетный поток** (или задача) **очень долго (или вообще никогда) не получает процессорное время**, потому что **высокоприоритетные потоки постоянно занимают CPU**.

В отличие от **Priority Inversion** (где high-priority ждёт low-priority), здесь **low-priority просто не может пробиться** к процессору.

**Коротко и по-человечески**:
- Важные задачи (анимация, реакция на касание) — выполняются быстро.
- Фоновые задачи (индексация, аналитика, кэширование, загрузка контента) — ждут бесконечно.
- Приложение кажется «залипшим» в фоне, хотя UI отзывчивый.

### 2. Самые частые сценарии Priority Starvation в [[iOS]]-приложениях 2026

| №   | Сценарий                                                         | Как проявляется                                                          | Частота |
| --- | ---------------------------------------------------------------- | ------------------------------------------------------------------------ | ------- |
| 1   | Перегруженная очередь `.userInitiated` / `.userInteractive`      | Фоновые задачи (`.background`, `.utility`) выполняются через минуты/часы | ★★★★★   |
| 2   | Бесконечный цикл / тяжёлая обработка в high-priority Task        | Аналитика, логирование, синхронизация не доходят                         | ★★★★☆   |
| 3   | Множество параллельных `Task.detached(priority: .userInitiated)` | Фоновые индексации, кэширование, загрузка изображений тормозят           | ★★★★☆   |
| 4   | Долгая работа в `.userInitiated` без ограничения                 | Приложение «зависает» в фоне при открытом экране                         | ★★★★☆   |
| 5   | Несбалансированное использование [[QoS]] в [[GCD]]               | `.background` задачи почти не запускаются                                | ★★★☆☆   |

### 3. Классические примеры Priority Starvation

#### Пример 1 — Перегруженная high-priority очередь

```swift
// 1000 высокоприоритетных задач
for _ in 0..<1000 {
    Task(priority: .userInitiated) {
        // Тяжёлая работа (например, обработка изображения)
        for _ in 0..<1_000_000 { _ = $0 * $0 }
    }
}

// Фоновая задача
Task(priority: .background) {
    print("Я фоновая задача и я выполняюсь ОЧЕНЬ долго...")
    // Может ждать минуты или часы
}
```

**Результат**: фоновая задача может ждать **очень долго**, пока все 1000 high-priority задач не завершатся.

#### Пример 2 — Бесконечный цикл в high-priority

```swift
Task(priority: .userInitiated) {
    while true {
        // Бесконечная обработка (например, мониторинг)
        Thread.sleep(forTimeInterval: 0.001)
    }
}

Task(priority: .background) {
    print("Эта задача почти никогда не запустится")
}
```

**Результат**: низкоприоритетная задача практически **не получает CPU**.

### 5. Самые безопасные конструкции 2026 (рекомендуемые Apple)

#### Правильное использование приоритетов в Task

```swift
// Критическая задача (анимация, реакция на касание)
Task(priority: .userInteractive) {
    await animateSomething()
}

// Важная, но не критичная (загрузка данных для UI)
Task(priority: .userInitiated) {
    let data = await fetchData()
    await updateUI(data)
}

// Фоновая обработка (аналитика, кэширование, индексация)
Task(priority: .utility) {
    await processAnalytics()
}

// Очень низкий приоритет (резервное копирование, очистка кэша)
Task(priority: .background) {
    await cleanupCache()
}
```

#### Ограничение concurrency для high-priority задач

```swift
await withTaskGroup(of: Void.self) { group in
    for url in urls.prefix(4) {  // Ограничиваем до 4 параллельных загрузок
        group.addTask(priority: .userInitiated) {
            let image = try await downloadImage(url)
            await MainActor.run {
                images.append(image)
            }
        }
    }
}
```

#### actor для защиты состояния + правильный приоритет

```swift
actor AnalyticsCollector {
    private var events: [Event] = []
    
    func addEvent(_ event: Event) {
        events.append(event)
    }
}

let collector = AnalyticsCollector()

Task(priority: .utility) {  // не .userInitiated!
    for event in eventsBatch {
        await collector.addEvent(event)
    }
}
```

### 6. Как найти Priority Starvation в 2026

| Инструмент / Способ               | Что ловит                              | Где включается                  | Эффективность |
|-----------------------------------|----------------------------------------|----------------------------------|---------------|
| **Instruments → Thread Analyzer** | Долгое ожидание CPU низкоприоритетными потоками | Instruments                      | ★★★★★         |
| **os_signpost + Instruments**     | Задержки выполнения фоновых задач      | Instruments → Points of Interest | ★★★★☆         |

### 7. Лучшие практики 2026 (Swift 6+)

- **Правильно распределяйте приоритеты**:
  - `.userInteractive` — анимации, реакция на касание, скролл
  - `.userInitiated` — загрузка данных для текущего экрана
  - `.utility` — аналитика, кэширование, индексация
  - `.background` — резервное копирование, очистка, долгие фоновые задачи
- **Ограничивайте concurrency** для `.userInitiated` и выше (TaskGroup с лимитом)  
- **Не перегружайте high-priority очереди** — это главная причина starvation  
- **Используйте actor** для защиты состояния — он сериализует доступ без явных lock  
- **Swift 6 strict concurrency** — помогает косвенно, требуя Sendable и изоляцию  
- **Для тестов** — используйте `Task(priority:)` и проверяйте время выполнения  
- **Для legacy-кода** — постепенно переводите на `Task` + правильные приоритеты  
- **Мониторинг** — добавляйте os_signpost с QoS и временем выполнения

**Короткий девиз 2026**:
> «Priority Starvation — это когда фоновые задачи умирают от голода, потому что важные задачи съедают весь CPU.  
> В 2026 году ответ один: правильные приоритеты + ограничение concurrency + actor.  
> Не перегружай .userInitiated и .userInteractive — иначе фоновая работа встанет навсегда.»
