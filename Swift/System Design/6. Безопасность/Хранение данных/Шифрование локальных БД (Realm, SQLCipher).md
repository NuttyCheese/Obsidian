#system_design
## Определение

**Шифрование локальных баз данных** — это процесс защиты данных, хранящихся на устройстве пользователя, с использованием **криптографических алгоритмов**.

Цель: **обеспечить конфиденциальность данных** даже при физическом доступе к устройству или его взломе.

Основные инструменты в iOS:

- **[[Swift/Realm]]** → популярная база данных с возможностью встроенного шифрования.
    
- **SQLCipher** → расширение [[SQLite]] с поддержкой AES-256 для шифрования всех данных.
    

---

## 1. Realm с шифрованием

### Включение шифрования

```swift
import RealmSwift

let key = Data(count: 64) // 512 бит случайного ключа
_ = key.withUnsafeMutableBytes { SecRandomCopyBytes(kSecRandomDefault, 64, $0.baseAddress!) }

let config = Realm.Configuration(encryptionKey: key)

do {
    let realm = try Realm(configuration: config)
    // работа с зашифрованной базой
} catch {
    print("Ошибка при открытии зашифрованной БД: \(error)")
}
```

### Особенности

- **AES-256** → симметричное шифрование.
    
- **Ключ хранится отдельно** (лучше в Keychain).
    
- **Все данные и индексы зашифрованы**, включая метаданные.

## 2. SQLCipher

### Подключение

- SQLCipher — форк SQLite с поддержкой шифрования.
    
- Шифрование осуществляется **на уровне всего файла базы данных**.
    

### Пример использования в [[Swift]]

```swift
import SQLite3

var db: OpaquePointer?
let dbPath = "encrypted.sqlite"

if sqlite3_open(dbPath, &db) == SQLITE_OK {
    // Установка ключа шифрования
    if sqlite3_key(db, "supersecretpassword", Int32("supersecretpassword".utf8.count)) == SQLITE_OK {
        print("БД открыта и зашифрована")
    }
} else {
    print("Ошибка при открытии БД")
}
```

### Особенности

- **AES-256** для всего файла.
    
- Возможность **обновления пароля шифрования**.
    
- Полная совместимость с SQL-запросами SQLite.

## 3. Хранение ключей шифрования

| Способ                | Описание                                         | Рекомендации                          |
| --------------------- | ------------------------------------------------ | ------------------------------------- |
| **[[KeyChain]]**      | Безопасное место хранения ключей на устройстве   | Лучший вариант для секретов           |
| **Secure Enclave**    | Генерация и хранение ключей на аппаратном уровне | Для критически чувствительных данных  |
| **Встроенные строки** | Хранение ключа в коде                            | Не рекомендуется — риск компрометации |

---

## 4. Best Practices

1. **Использовать уникальный ключ на устройство** → не хранить универсальные ключи в приложении.
    
2. **Хранить ключи в Keychain или Secure Enclave** → максимальная безопасность.
    
3. **Регулярно обновлять ключи шифрования** → при изменении политики безопасности.
    
4. **Объединять с другими методами защиты** → TLS, OAuth/JWT, certificate pinning.
    
5. **Минимизировать чувствительные данные** → хранить только необходимое.
    

---

## 5. Итог

- **Realm и SQLCipher** позволяют эффективно шифровать локальные данные на iOS.
    
- **AES-256** — стандартная схема для симметричного шифрования.
    
- **Ключ шифрования** должен храниться безопасно (Keychain / Secure Enclave).
    
- Комбинация шифрования базы + безопасного хранения ключей + защиты сети создаёт **надежный стек безопасности мобильного приложения**.
