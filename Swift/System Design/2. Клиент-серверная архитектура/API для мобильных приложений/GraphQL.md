#system_design
## Определение

**GraphQL** — это язык запросов для API и среда выполнения, которая позволяет клиенту **запрашивать ровно те данные, которые ему нужны**.

> В отличие от REST, где сервер определяет структуру данных, в GraphQL клиент контролирует, какие поля получать.

---

## Основные компоненты GraphQL

1. **Схема (Schema)**
    
    - Определяет типы данных и возможные операции.
        
    - Пример: `type User { id: ID, name: String, email: String }`
        
2. **Запросы (Queries)**
    
    - Получение данных.
        
    - Пример:
        
    
    ```graphql
    query {
        user(id: "123") {
            name
            email
        }
    }
    ```
    
3. **Мутации (Mutations)**
    
    - Изменение данных на сервере.
        
    - Пример:
        
    
    ```graphql
    mutation {
        updateUser(id: "123", input: {name: "Alex"}) {
            id
            name
        }
    }
    ```
    
4. **Подписки (Subscriptions)**
    
    - Реальное время. Клиент получает обновления от сервера.
        
    - Пример: уведомления о новых сообщениях в чате.
        

---

## Преимущества GraphQL для мобильных приложений

1. **Только нужные данные**
    
    - Нет перегрузки сети: клиент получает ровно то, что нужно.
        
2. **Единая точка входа**
    
    - Все запросы идут через один endpoint `/graphql`, а не через множество REST URL.
        
3. **Сильная типизация**
    
    - Сервер заранее знает структуру данных, ошибки запросов можно отловить до выполнения.
        
4. **Поддержка оффлайн и кэша**
    
    - Клиент может кэшировать ответы (например, Apollo Client).
        
5. **Поддержка подписок**
    
    - Идеально для чатов, обновления статуса заказов, пушей в реальном времени.
        

---

## Отличие от REST

|Критерий|REST|GraphQL|
|---|---|---|
|Структура|Сервер определяет|Клиент определяет|
|URL|Много endpoints|Один endpoint `/graphql`|
|Перегрузка данных|Возможна (overfetching/underfetching)|Только нужные поля|
|Подписки|Не встроено|Есть Subscription (real-time)|
|Типизация|Нет встроенной|Сильная типизация через схему|

## Использование GraphQL в iOS

### Клиентские библиотеки

- **Apollo iOS** — наиболее популярная библиотека.
    
- **GraphQL.swift**, **Combine+GraphQL** — альтернативы.
    

### Пример запроса через Apollo

```swift
import Apollo

class UserService {
    let apollo = ApolloClient(url: URL(string: "https://example.com/graphql")!)
    
    func fetchUser(id: String) {
        let query = GetUserQuery(id: id)
        apollo.fetch(query: query) { result in
            switch result {
            case .success(let graphQLResult):
                if let user = graphQLResult.data?.user {
                    print("User name: \(user.name)")
                }
            case .failure(let error):
                print("Ошибка запроса: \(error)")
            }
        }
    }
}
```

## Кэширование

- **Apollo** поддерживает встроенный **normalized cache**.
    
- Пример: если пользователь открыл профиль раньше, данные можно показать мгновенно, а затем обновить с сервера.
    

---

## Подписки (Subscriptions) в iOS

- Реализуются через **WebSockets**.
    
- Пример: уведомления о новых сообщениях:
    

```graphql
subscription {
    newMessage(chatId: "123") {
        id
        text
        sender {
            name
        }
    }
}
```

- Apollo iOS умеет подписываться и автоматически обновлять UI.

## Пример применения

Приложение доставки еды:

- REST: нужно несколько endpoint `/users`, `/orders`, `/restaurants`.
    
- GraphQL: один запрос:
    

```graphql
query {
    user(id: "123") {
        name
        orders {
            id
            status
            restaurant { name }
        }
    }
}
```

- Меньше сетевых запросов → быстрее работа приложения → экономия батареи и трафика.
    

---

## Итог

- **GraphQL** — современный способ взаимодействия мобильного приложения с сервером.
    
- Позволяет **запрашивать ровно нужные данные**, уменьшать сетевой трафик и объединять множество REST-запросов в один.
    
- Поддерживает **подписки** для реального времени, идеально для чатов, обновлений заказов, уведомлений.
    
- В iOS удобно использовать через **Apollo Client** с кэшированием и подписками.
