#system_design
## Определение

**Инструменты мониторинга ошибок и сбоев** позволяют **отслеживать падения приложений, исключения и критические ошибки**, анализировать их и оповещать команду разработки.

- **Crashlytics (Firebase)** → инструмент для отслеживания крашей, интегрируется с Firebase.
    
- **Sentry** → мониторинг ошибок и исключений, продвинутая трассировка и аналитика событий.
    
- **Instabug** → фокус на баг-репортах от пользователей, крашах и обратной связи.
    

> Основная цель всех инструментов — **повышение стабильности приложения и улучшение UX**.

---

## 1. Crashlytics

### Особенности

- Бесплатный инструмент, часть **[[Firebase]] Suite**.
    
- Автоматическое логирование крашей и сбоев.
    
- Поддержка **custom logs** для добавления контекста ошибки.
    
- Интеграция с **Crash Reporting, Analytics, Remote Config**.
    

### Интеграция в [[iOS]]

```swift
import FirebaseCore
import FirebaseCrashlytics

FirebaseApp.configure()

// Логирование ошибки
Crashlytics.crashlytics().record(error: NSError(domain: "com.app", code: 500, userInfo: nil))

// Логирование пользовательских событий для крашей
Crashlytics.crashlytics().log("User tapped Buy button")
```

### Возможности

- Трассировка стеков вызова.
    
- Группировка схожих крашей.
    
- Настройка уведомлений для команды разработки.

## 2. Sentry

### Особенности

- Продвинутая платформа для **мониторинга ошибок и исключений**.
    
- Поддержка **[[Swift]] и [[Objective-C]]**, а также кроссплатформенно (React Native, Flutter).
    
- Реализация **Breadcrumbs** → отслеживание последовательности действий пользователя перед ошибкой.
    
- Настройка алертов по критичности ошибок.
    

### Интеграция в iOS

```swift
import Sentry

SentrySDK.start { options in
    options.dsn = "YOUR_DSN"
    options.debug = true // только для разработки
}

// Логирование ошибки
SentrySDK.capture(error: NSError(domain: "com.app", code: 500, userInfo: nil))

// Добавление контекста
SentrySDK.configureScope { scope in
    scope.setUser(Sentry.User(userId: "12345"))
    scope.setTag(value: "checkout", key: "module")
}
```

### Возможности

- Breadcrumbs → последовательность действий пользователя.
    
- User context → идентификация пользователей, с которыми произошла ошибка.
    
- Интеграция с Slack, Jira, [[GitHub]] для алертов и тикетов.

## 3. Instabug

### Особенности

- Инструмент **для баг-репортов от пользователей и крашей**.
    
- Поддержка **screenshot и screen recording**, чтобы увидеть шаги перед ошибкой.
    
- Возможность интеграции с аналитикой и инструментами баг-трекинга.
    

### Интеграция в iOS

```swift
import Instabug

Instabug.start(withToken: "YOUR_TOKEN", invocationEvents: [.shake, .screenshot])

// Логирование ошибки
Instabug.log(.error, message: "Payment failed")

// Отправка пользовательского отчета
Instabug.show()
```

### Возможности

- Баг-репорты с действиями пользователя и скриншотами.
    
- Crash reporting и детальная информация о сессии.
    
- Настройка алертов для команды.

## Best Practices при использовании инструментов

1. **Логировать критические ошибки и исключения** → автоматические краши + ручные исключения.
    
2. **Добавлять контекст** → пользователь, версия приложения, устройство, шаги действия.
    
3. **Настраивать алерты** → уведомление команды о критических крашах.
    
4. **Использовать Breadcrumbs** → отслеживание последовательности действий перед ошибкой.
    
5. **Комбинировать инструменты** → Crashlytics для автоматических крашей, Instabug для баг-репортов, Sentry для детальной трассировки.
    

---

## Итог

- **Crashlytics** → быстрый старт, простая интеграция, автоматические краши.
    
- **Sentry** → продвинутая аналитика ошибок, Breadcrumbs, User context.
    
- **Instabug** → фокус на пользовательский фидбэк, скриншоты и запись экрана.
    
- Использование этих инструментов помогает **быстро выявлять ошибки, минимизировать влияние на пользователей и повышать качество приложения**.
