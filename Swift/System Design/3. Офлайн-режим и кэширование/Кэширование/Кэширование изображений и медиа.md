#system_design
## Определение

**Кэширование изображений и медиа** — это сохранение изображений, видео или других медиа-файлов локально на устройстве для **ускорения загрузки, уменьшения трафика и повышения производительности приложения**.

> Главная цель — повторно использовать уже загруженные данные без повторного обращения к серверу.

---

## Почему важно для мобильных приложений

1. **Снижение сетевого трафика**
    
    - Пользователь не загружает одни и те же изображения повторно.
        
2. **Быстрая загрузка интерфейса**
    
    - Экраны открываются мгновенно с кэша.
        
3. **Повышение отзывчивости приложения**
    
    - Меньше задержки при отображении фото, видео или аватарок.
        
4. **Поддержка оффлайн-режима**
    
    - Пользователь видит кэшированные изображения без подключения к сети.
        

---

## Типы кэша

| Тип кэша         | Хранение             | Пример                          |
| ---------------- | -------------------- | ------------------------------- |
| **Memory Cache** | В оперативной памяти | [[UIImage]] в [[NSCache]]       |
| **Disk Cache**   | На диске устройства  | Файлы в Documents / Caches      |
| **Hybrid Cache** | Комбинация           | [[SDWebImage]] / [[Kingfisher]] |

---

## Варианты реализации в iOS

### 1. **Memory Cache**

- Используется для **быстрой загрузки** изображений во время работы приложения.
    
- Обычно через **NSCache**:
    

```swift
let imageCache = NSCache<NSString, UIImage>()

func loadImage(url: String) -> UIImage? {
    if let cachedImage = imageCache.object(forKey: url as NSString) {
        return cachedImage
    }
    return nil
}
```

- Плюсы: очень быстро.
    
- Минусы: очищается при нехватке памяти.
    

---

### 2. **Disk Cache**

- Сохраняет изображения на диске для долгосрочного хранения.
    
- Можно использовать директории:
    
    - **Caches** — очищается системой при нехватке места
        
    - **Documents** — сохраняется навсегда
        
- Пример:
    

```swift
func saveImageToDisk(image: UIImage, fileName: String) {
    if let data = image.pngData() {
        let url = FileManager.default.urls(for: .cachesDirectory, in: .userDomainMask)[0].appendingPathComponent(fileName)
        try? data.write(to: url)
    }
}

func loadImageFromDisk(fileName: String) -> UIImage? {
    let url = FileManager.default.urls(for: .cachesDirectory, in: .userDomainMask)[0].appendingPathComponent(fileName)
    if let data = try? Data(contentsOf: url) {
        return UIImage(data: data)
    }
    return nil
}
```

---

### 3. **Сторонние библиотеки**

- **SDWebImage**
    
    - Автоматическое кэширование в память и на диск.
        
    - Поддержка загрузки по [[URL]] и асинхронного отображения.
        
- **Kingfisher**
    
    - Похожий функционал с поддержкой [[SwiftUI]].
        
    - Легкая интеграция, кэширование, фоновая загрузка.
        

Пример с SDWebImage:

```swift
import SDWebImage

imageView.sd_setImage(with: URL(string: imageUrl), placeholderImage: UIImage(named: "placeholder"))
```

## Стратегии кэширования

|Стратегия|Описание|
|---|---|
|**LRU (Least Recently Used)**|Удаляются наименее используемые элементы при нехватке памяти|
|**Expiration Time**|Сохраняем кэш только определённое время|
|**Hybrid**|Комбинация памяти и диска для баланса скорости и объёма|

## Кэширование видео и других медиа

- Используются аналогичные подходы:
    
    - **Memory cache** для небольших превью.
        
    - **Disk cache** для локального хранения больших файлов.
        
- Для потокового видео можно использовать **AVAssetDownloadURLSession** для загрузки и хранения локально.
    

Пример сохранения видео:

```swift
let url = URL(string: videoUrl)!
let destination = FileManager.default.urls(for: .cachesDirectory, in: .userDomainMask)[0].appendingPathComponent("video.mp4")

URLSession.shared.downloadTask(with: url) { tempUrl, _, _ in
    if let tempUrl = tempUrl {
        try? FileManager.default.moveItem(at: tempUrl, to: destination)
    }
}.resume()
```

## Best Practices

1. Кэшировать только нужные изображения (не весь список товаров).
    
2. Использовать **placeholder** при загрузке.
    
3. Контролировать размер памяти и диска (например, ограничение на 50 MB).
    
4. Обновлять кэш при изменении контента на сервере.
    
5. Использовать асинхронную загрузку, чтобы UI не блокировался.

## Итог

- Кэширование изображений и медиа — ключевой элемент **оптимизации мобильного приложения**.
    
- Memory Cache → быстрый доступ во время работы приложения.
    
- Disk Cache → долгосрочное хранение, оффлайн-доступ.
    
- Сторонние библиотеки (SDWebImage, Kingfisher) упрощают работу.
    
- Позволяет **ускорить интерфейс, снизить трафик и улучшить UX**.
