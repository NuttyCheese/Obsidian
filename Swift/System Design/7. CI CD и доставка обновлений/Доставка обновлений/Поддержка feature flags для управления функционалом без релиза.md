#system_design
## Определение

**Feature Flags (Feature Toggles)** — это техника разработки, которая позволяет **включать или отключать функциональность приложения без выпуска нового релиза в App Store**.

Цель:

- Быстро тестировать новые функции в продакшене.
    
- Проводить **A/B-тестирование**.
    
- Ограничивать доступ к экспериментальным функциям для определённых пользователей.
    

---

## 1. Основные виды feature flags

|Вид|Описание|Пример использования|
|---|---|---|
|**Release Toggle**|Включение или отключение функции для постепенного релиза|Новая кнопка в интерфейсе|
|**Experiment Toggle**|A/B-тестирование разных версий функции|Разные варианты onboarding flow|
|**Ops Toggle**|Управление поведением приложения в продакшене|Включение логирования или throttling|
|**Permission Toggle**|Управление доступом на основе ролей или подписки|Функция доступна только премиум-пользователям|

---

## 2. Архитектура feature flags

1. **Серверная конфигурация** → флаги хранятся на сервере, приложение запрашивает актуальное состояние.
    
2. **Локальное кеширование** → для офлайн-режима и уменьшения нагрузки на сервер.
    
3. **SDK / менеджер флагов** → централизованное управление включением/отключением функций.
    

```text
+------------------+
|      Server      | <- конфигурация feature flags
+------------------+
          |
          v
+------------------+
| FeatureFlagManager | <- кеш, проверка флагов
+------------------+
          |
          v
+------------------+
|  Mobile App UI   | <- включение/отключение функционала
+------------------+
```

---

## 3. Пример реализации в [[iOS]] ([[Swift]])

### FeatureFlagManager

```swift
class FeatureFlagManager {
    static let shared = FeatureFlagManager()
    
    private var flags: [String: Bool] = [:]
    
    private init() { }
    
    func isEnabled(_ feature: String) -> Bool {
        return flags[feature] ?? false
    }
    
    func updateFlags(_ newFlags: [String: Bool]) {
        flags = newFlags
    }
}
```

### Использование в коде

```swift
if FeatureFlagManager.shared.isEnabled("NewOnboarding") {
    showNewOnboardingFlow()
} else {
    showLegacyOnboardingFlow()
}
```

## 4. Интеграция с [[CI]]/[[CD]]

1. **Сборка и деплой** → новые функции можно включать через флаги без пересборки.
    
2. **Тестирование** → тестовые пользователи получают включённые функции, остальные нет.
    
3. **Мониторинг** → отслеживание ошибок и метрик для новых функций.
    
4. **Откат функций** → флаг просто выключается → мгновенное отключение функционала без нового релиза.
    

---

## 5. Best Practices

1. **Централизованный менеджер флагов** → проще управлять и обновлять.
    
2. **Локальный кеш + серверная конфигурация** → скорость работы и поддержка офлайн.
    
3. **Разделение флагов по типам** → release, experiment, ops, permission.
    
4. **Логирование и аналитика** → отслеживать использование включённых функций.
    
5. **Регулярное удаление устаревших флагов** → предотвращает накопление «мертвого кода».
    
6. **Безопасность** → серверная проверка прав доступа для критических флагов.
    

---

## 6. Итог

- **Feature Flags** позволяют **управлять функционалом без пересборки приложения**.
    
- Используются для: тестирования, постепенного релиза, A/B-тестов, ограничения функций по ролям.
    
- В [[iOS]] реализуются через **централизованный менеджер флагов** с серверной конфигурацией и локальным кешем.
    
- **CI/CD + Feature Flags** дают гибкость в выпуске функций и минимизируют риск ошибок в продакшене.
