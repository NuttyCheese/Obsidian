#system_design
## Определение

**Network Link Conditioner (NLC)** — это инструмент для **эмуляции различных сетевых условий** на [[iOS]]-устройствах и симуляторах.

Цель: проверить, как мобильное приложение ведёт себя при:

- Слабом или нестабильном соединении.
    
- Высокой задержке (latency).
    
- Потере пакетов или ограниченной пропускной способности.
    

Это особенно важно для iOS-приложений, которые активно работают с сетью ([[Swift/Теория/Сетевое взаимодействие/REST API]], [[GraphQL]], [[WebSocket]], медиа-контент).

---

## 1. Основные параметры эмуляции

|Параметр|Описание|
|---|---|
|**Bandwidth**|Пропускная способность сети (например, 1 Mbps, 512 Kbps)|
|**Latency**|Задержка сети в миллисекундах (например, 100 ms)|
|**Packet Loss**|Процент потерянных пакетов (например, 2%)|
|**Jitter**|Варьирование задержки между пакетами|

---

## 2. Настройка Network Link Conditioner

### На симуляторе и устройстве iOS

1. **Устройство**: Включить в **Settings → Developer → Network Link Conditioner**.
    
2. **Выбрать профиль**:
    
    - **3G**, **Edge**, **LTE**, **WiFi Lossy**, **Custom**.
        
3. **Custom Profile**:
    
    - Настроить **Bandwidth**, **Latency**, **Packet Loss**, **Jitter** вручную.
        

> ⚠️ Для использования NLC на реальных устройствах нужен **Developer Mode**.

---

## 3. Примеры сценариев тестирования

1. **Медленный 3G**
    
    - Пропускная способность: 750 Kbps
        
    - Задержка: 100–150 ms
        
    - Проверка: загрузка изображений, API запросов
        
2. **Потеря пакетов (WiFi Lossy)**
    
    - Packet loss: 2–5%
        
    - Проверка: повторные попытки, таймауты, корректная обработка ошибок
        
3. **Высокая задержка**
    
    - Latency: 500–1000 ms
        
    - Проверка: UI-отклик, индикаторы загрузки, timeout для сетевых запросов
        

---

## 4. Best Practices для iOS

1. **Обработка таймаутов**
    
    - [[URLSession]]: `timeoutIntervalForRequest`, `timeoutIntervalForResource`.
        
2. **Retry и backoff**
    
    - Повторные попытки с экспоненциальной задержкой.
        
3. **Индикация состояния сети**
    
    - Показывать пользователю индикаторы загрузки или ошибки соединения.
        
4. **Кэширование**
    
    - Загруженные данные хранить локально для офлайн-доступа.
        
5. **Тестирование разных сценариев**
    
    - Комбинации низкой скорости, потери пакетов и высокой задержки.

## 5. Пример кода для обработки слабого соединения

```swift
let url = URL(string: "https://api.example.com/data")!
let config = URLSessionConfiguration.default
config.timeoutIntervalForRequest = 15
config.timeoutIntervalForResource = 30

let session = URLSession(configuration: config)
let task = session.dataTask(with: url) { data, response, error in
    if let error = error {
        print("Ошибка сети: \(error.localizedDescription)")
        // Реализовать retry с экспоненциальной задержкой
    } else {
        print("Данные получены успешно")
    }
}
task.resume()
```

## 6. Итог

- **Network Link Conditioner** позволяет эмулировать любые сетевые условия и проверять стабильность iOS-приложений.
    
- Позволяет выявлять проблемы с **UI откликом, таймаутами, повторными запросами и кэшированием**.
    
- Рекомендуется тестировать:
    
    - Медленный интернет (2G/3G)
        
    - Потерю пакетов
        
    - Высокую задержку
        
- В сочетании с **retry, timeout и кэшированием** помогает сделать приложение более устойчивым к нестабильным сетям.
