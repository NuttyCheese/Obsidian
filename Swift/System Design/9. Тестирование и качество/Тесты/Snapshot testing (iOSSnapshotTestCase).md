#system_design
## Определение

**Snapshot testing** — это метод тестирования UI, при котором **визуальное состояние компонента или экрана сравнивается с эталонным снимком (snapshot)**.

Цель: убедиться, что изменения кода **не нарушают внешний вид интерфейса**.

- Особенно полезно для сложных кастомных UI-компонентов и динамических экранов.
    
- Эффективно для обнаружения регрессий в дизайне после изменений.
    

---

## 1. Принцип работы

1. Генерируется **snapshot** текущего состояния [[UIView]]/[[UIViewController]].
    
2. Snapshot сохраняется как эталонный в проекте (Reference Image).
    
3. При следующем запуске теста новый snapshot сравнивается с эталоном.
    
4. Если изображения различаются → тест падает, показывая визуальные отличия.
    

```text
Текущий вид → Снимок → Сравнение с эталоном → Pass/Fail
```

---

## 2. Инструменты

| Инструмент                                       | Описание                                                                                    |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------- |
| **[[iOSSnapshotTestCase]] (FBSnapshotTestCase)** | Классический инструмент, поддерживает XCTest и генерацию эталонных снимков                  |
| **SnapshotTesting (pointfreeco)**                | Более современный [[Swift]]-фреймворк с поддержкой разных платформ, в том числе [[SwiftUI]] |
| **Quick + Nimble**                               | Могут использоваться совместно для более читаемых BDD-style тестов                          |

---

## 3. Пример использования iOSSnapshotTestCase

```swift
import FBSnapshotTestCase
@testable import MyApp

class MyViewSnapshotTests: FBSnapshotTestCase {

    override func setUp() {
        super.setUp()
        self.recordMode = false // true для генерации эталонного снимка
    }

    func testMyViewAppearance() {
        let view = MyCustomView(frame: CGRect(x: 0, y: 0, width: 300, height: 500))
        view.configure(with: someData)
        
        FBSnapshotVerifyView(view)
    }
}
```

- `recordMode = true` → создаёт эталонный снимок.
    
- `recordMode = false` → сравнивает с эталонным.
    
- Если UI изменился непреднамеренно → тест падает.

## 4. Best Practices

1. **Изоляция компонентов**
    
    - Тестировать отдельные UIView или ViewController, а не весь экран сразу.
        
2. **Использование фиксированных размеров**
    
    - Избегать случайных размеров, чтобы результаты теста были стабильными.
        
3. **Поддержка разных устройств**
    
    - Проверять iPhone SE, iPhone 14 Pro, iPad и разные ориентации.
        
4. **Обновление эталонов**
    
    - При намеренных изменениях UI → пересоздавать эталонные снимки (`recordMode = true`).
        
5. **Интеграция с CI**
    
    - Запускать snapshot тесты в CI/CD для своевременного обнаружения регрессий.
        

---

## 5. Особенности iOSSnapshotTestCase

- Работает как **XCTest**, легко интегрируется в существующие тестовые схемы.
    
- Поддерживает сравнение не только UIView, но и **UIViewController.view**.
    
- Сохраняет эталонные изображения рядом с тестами в репозитории.
    
- Позволяет увидеть **визуальные отличия**, чтобы быстро исправить баги в UI.
    

---

## 6. Итог

- **Snapshot testing** → инструмент для обнаружения визуальных регрессий в iOS-приложениях.
    
- **iOSSnapshotTestCase** или **SnapshotTesting** позволяют сохранять эталоны и сравнивать текущий UI с ними.
    
- Best practices: фиксированные размеры, тестирование отдельных компонентов, интеграция с CI/CD.
    
- Идеально подходит для **сложных кастомных интерфейсов и динамического контента**, где обычные unit-тесты не могут проверить внешний вид.
