#system_design
## Определение

**CDN (Content Delivery Network)** — это сеть серверов, расположенных в разных регионах, предназначенная для **быстрой и надёжной доставки статического и медиа-контента** пользователям.

Для [[iOS]]-приложений CDN используется для:

- Быстрой загрузки изображений, видео и других медиа.
    
- Снижения задержки и нагрузки на основной сервер.
    
- Обеспечения высокой доступности контента по всему миру.
    

---

## 1. Принцип работы CDN

1. Контент (изображения, видео, JS/CSS) загружается на CDN-серверы (Edge Servers).
    
2. Пользователь делает запрос через iOS-приложение.
    
3. CDN перенаправляет запрос на ближайший сервер.
    
4. Если контента нет на ближайшем сервере, он подтягивается с origin-сервера и кэшируется для последующих запросов.
    

```text
iOS App → Edge Server (CDN) → [Origin Server при необходимости] → iOS App
```

---

## 2. Основные возможности CDN

| Возможность                      | Описание                                                         |
| -------------------------------- | ---------------------------------------------------------------- |
| **Географическое распределение** | Контент доставляется с ближайшего сервера, сокращая latency      |
| **Кэширование**                  | Статические файлы хранятся на edge-серверах для быстрой загрузки |
| **Балансировка нагрузки**        | Перенаправляет трафик, снижая нагрузку на origin-сервер          |
| **Защита от DDoS**               | Некоторые CDN включают встроенные механизмы защиты               |
| **SSL/TLS**                      | Безопасная доставка через [[HTTPS]]                              |
| **Оптимизация медиа**            | Сжатие, ресайз, адаптивные форматы изображений и видео           |

---

## 3. Примеры CDN

|CDN|Особенности|
|---|---|
|**Cloudflare**|Быстрое кэширование, DDoS защита, SSL, оптимизация изображений (Polish, Mirage)|
|**Akamai**|Широкая глобальная сеть, высокое покрытие, аналитика и мониторинг|
|**Fastly**|Высокая скорость доставки, настройка кастомных правил кэширования|
|**AWS CloudFront**|Интеграция с S3 и другими AWS сервисами, поддержка приватного контента|

## 4. Best Practices для iOS

1. **Кэширование на стороне приложения**
    
    - Использовать NSCache или сторонние библиотеки ([[Kingfisher]], [[SDWebImage]]) для локального кэша изображений.
        
2. **Оптимизация форматов**
    
    - WebP/HEIC для изображений, HLS для видео.
        
3. **Минимизация запросов**
    
    - Объединять медиа в пакеты или использовать lazy-loading.
        
4. **Мониторинг CDN**
    
    - Отслеживать метрики latency, hit ratio и ошибки доставки.
        
5. **Fallback и офлайн**
    
    - Загружать критичный контент в локальный кэш для офлайн-режима.

## 5. Интеграция с iOS

### Пример загрузки изображения с CDN (SDWebImage)

```swift
import SDWebImage

let imageView = UIImageView(frame: CGRect(x: 0, y: 0, width: 200, height: 200))
view.addSubview(imageView)

let imageURL = URL(string: "https://cdn.example.com/images/photo.jpg")
imageView.sd_setImage(with: imageURL, placeholderImage: UIImage(named: "placeholder"))
```

### Пример загрузки видео через HLS

```swift
import AVKit

let url = URL(string: "https://cdn.example.com/videos/video.m3u8")!
let player = AVPlayer(url: url)
let playerViewController = AVPlayerViewController()
playerViewController.player = player
present(playerViewController, animated: true)
player.play()
```

## 6. Итог

- **CDN** значительно ускоряет доставку медиа, снижает нагрузку на сервер и повышает доступность контента.
    
- Основные функции: географическое распределение, кэширование, оптимизация медиа, DDoS защита.
    
- В iOS-приложениях CDN используется совместно с **кэшированием на устройстве и оптимизированными форматами контента**.
    
- Популярные решения: **Cloudflare, Akamai, Fastly, AWS CloudFront**.
