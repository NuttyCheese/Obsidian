#system_design
## Определение

**API Gateway** — это **серверный компонент**, который выступает **единым входом для всех клиентских запросов** к микросервисам.

Основные функции:

- Управление нагрузкой между сервисами.
    
- Агрегация и маршрутизация запросов.
    
- Безопасность и аутентификация.
    
- Мониторинг и логирование.
    

Для [[iOS]]-приложений API Gateway является точкой взаимодействия с сервером, скрывая внутреннюю структуру микросервисов.

---

## 1. Основные задачи API Gateway

|Задача|Описание|
|---|---|
|**Балансировка нагрузки**|Распределяет запросы между экземплярами сервисов для повышения производительности и отказоустойчивости|
|**Маршрутизация**|Определяет, какой микросервис обрабатывает конкретный запрос|
|**Агрегация**|Объединяет ответы нескольких микросервисов в один для клиента|
|**Безопасность**|Проверка токенов, шифрование, защита от DDoS|
|**Кэширование**|Кэширует часто используемые ответы для ускорения отклика|
|**Логирование и мониторинг**|Сбор метрик и ошибок, трассировка запросов|

---

## 2. Принцип работы

1. iOS-приложение делает [[HTTP]]/[[HTTPS]] запрос к **API Gateway**.
    
2. Gateway проверяет токен, права доступа и определяет маршрут.
    
3. Запрос перенаправляется к нужному микросервису (или к нескольким сервисам одновременно).
    
4. Результаты объединяются (если нужно) и возвращаются клиенту.
    

```text
iOS App → API Gateway → [Service A, Service B, Service C] → API Gateway → iOS App
```

---

## 3. Балансировка нагрузки

**Цель:** равномерно распределить запросы, чтобы серверы не перегружались.

### Методы балансировки

|Метод|Описание|
|---|---|
|**Round Robin**|Запросы распределяются по очереди между сервисами|
|**Least Connections**|Запрос отправляется к сервису с наименьшим числом активных соединений|
|**IP Hash**|Запросы от одного клиента всегда идут на один и тот же сервер|
|**Weighted**|Сервисы с большей мощностью получают больше запросов|

|Gateway|Особенности|

## 5. Best Practices

1. **Кэширование на Gateway**
    
    - Часто запрашиваемые данные (списки, справочники) можно кэшировать для ускорения ответов.
        
2. **Мониторинг и метрики**
    
    - Вести логирование, метрики latency и ошибок для быстрого реагирования.
        
3. **Аутентификация**
    
    - API Gateway проверяет JWT или OAuth токены перед передачей запроса микросервисам.
        
4. **Версионирование API**
    
    - Поддерживать несколько версий [[API]] для плавного обновления клиентов.
        
5. **Failover и Retry**
    
    - Настроить повторные попытки при временных сбоях и fallback-сервисы.
        

---

## 6. Итог

- **API Gateway** — ключевой компонент для масштабируемой архитектуры.
    
- Для iOS-приложений это **единая точка входа** к серверной инфраструктуре.
    
- Обеспечивает **балансировку нагрузки, безопасность, кэширование и мониторинг**.
    
- Позволяет упрощать работу мобильного клиента, скрывая сложность микросервисной архитектуры.
