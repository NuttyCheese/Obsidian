# Фоновая работа и обновление данных

## Определение

**Фоновая работа (Background Tasks)** в [[iOS]] — это механизм, позволяющий приложению **выполнять задачи и обновлять данные**, даже когда оно **не активно или находится в фоне**.

> Обновление данных может происходить:
> 
> - при получении **push-уведомления** (UI push или silent push),
>     
> - по **расписанию**, используя системные возможности Background Tasks.
>     

---

## Основные сценарии

1. **Обновление данных при получении push**
    
    - Silent push уведомления (`content-available: 1`) запускают приложение в фоне.
        
    - Приложение может обновить локальные данные, подготовить UI и синхронизировать с сервером.
        
2. **Обновление данных по расписанию**
    
    - Используются **BackgroundTasks** (`BGAppRefreshTask`, `BGProcessingTask`).
        
    - Система сама решает, когда выполнить задачу, оптимизируя батарею и ресурсы.
        

---

## Работа с push для фоновой синхронизации

### 1. Silent push

```swift
func application(_ application: UIApplication,
                 didReceiveRemoteNotification userInfo: [AnyHashable: Any],
                 fetchCompletionHandler completionHandler: @escaping (UIBackgroundFetchResult) -> Void) {
    if let updateType = userInfo["updateType"] as? String, updateType == "syncOrders" {
        syncOrders { success in
            completionHandler(success ? .newData : .failed)
        }
    } else {
        completionHandler(.noData)
    }
}
```

**Особенности:**

- Ограничение по времени выполнения (~30 секунд).
    
- Silent push может быть отложен системой в зависимости от нагрузки и батареи.

## Работа по расписанию с BackgroundTasks

### 1. Регистрация задач

```swift
import BackgroundTasks

func registerBackgroundTasks() {
    BGTaskScheduler.shared.register(forTaskWithIdentifier: "com.app.refresh", using: nil) { task in
        self.handleAppRefresh(task: task as! BGAppRefreshTask)
    }
}
```

### 2. Планирование обновления

```swift
func scheduleAppRefresh() {
    let request = BGAppRefreshTaskRequest(identifier: "com.app.refresh")
    request.earliestBeginDate = Date(timeIntervalSinceNow: 15 * 60) // минимум 15 минут
    try? BGTaskScheduler.shared.submit(request)
}
```

### 3. Обработка задачи

```swift
func handleAppRefresh(task: BGAppRefreshTask) {
    scheduleAppRefresh() // планируем следующую задачу
    
    let queue = OperationQueue()
    queue.maxConcurrentOperationCount = 1
    
    let operation = DataSyncOperation()
    
    task.expirationHandler = {
        queue.cancelAllOperations()
    }
    
    operation.completionBlock = {
        task.setTaskCompleted(success: !operation.isCancelled)
    }
    
    queue.addOperation(operation)
}
```

## Комбинирование push и расписания

- **Silent push** → триггер мгновенной синхронизации.
    
- **BackgroundTasks** → гарантирует периодическое обновление данных, даже если push не пришёл.
    
- Можно комбинировать: silent push запускает немедленное обновление, а BackgroundTasks поддерживает регулярную синхронизацию.
    

---

## Best Practices

1. **Минимизировать время выполнения задач** → использовать локальный кэш и быстрые операции.
    
2. **Использовать очереди задач** → для надежной синхронизации нескольких изменений.
    
3. **Обновлять UI после синхронизации** → уведомлять пользователя только при необходимости.
    
4. **Комбинировать стратегии** → push + расписание для надежной актуализации данных.
    
5. **Обрабатывать ошибки и retry** → сохранять неуспешные изменения для повторной отправки.
    

---

## Применение в мобильных приложениях

- Чат-приложения → обновление сообщений в фоне.
    
- Приложения доставки → обновление статусов заказов без открытия приложения.
    
- Новостные приложения → prefetch контента перед открытием.
    
- Финансовые приложения → обновление курсов валют, уведомления о транзакциях.
    

---

## Итог

- **Фоновая работа** позволяет поддерживать актуальность данных без вмешательства пользователя.
    
- **Silent push** → триггер мгновенной синхронизации.
    
- **BackgroundTasks** → периодическое обновление по расписанию, управляемое системой.
    
- **Комбинированный подход** → повышает надежность и отзывчивость приложения.
