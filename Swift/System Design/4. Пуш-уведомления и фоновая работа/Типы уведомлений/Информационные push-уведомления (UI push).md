#system_design
## Определение

**Информационные push-уведомления (UI push)** — это уведомления, которые **отображаются пользователю на экране устройства** с текстом, значком и/или звуком.

- Основная цель — **информировать пользователя о событиях, обновлениях или действиях** в приложении.
    
- Отображаются даже если приложение **не активно или закрыто**.
    

> UI push — это стандартный тип уведомлений, в отличие от silent push, которые используются только для фоновой синхронизации данных.

---

## Основные характеристики

|Характеристика|Описание|
|---|---|
|Видимость|Пользователь видит уведомление на экране|
|Взаимодействие|Можно нажать, чтобы открыть приложение или выполнить действие|
|Компоненты|Title, Body, Sound, Badge, Action Buttons|
|Контекст|Используется для оповещения о новых событиях, сообщениях, изменениях статуса|

---

## Пример использования

- Чат-приложение → уведомление о новом сообщении:
    
    - Title: "Новый чат"
        
    - Body: "Алекс написал вам сообщение"
        
- Приложение доставки → уведомление о готовности заказа:
    
    - Title: "Заказ готов"
        
    - Body: "Ваш заказ #123 готов к доставке"

## Payload для UI push

### Через APNs ([[iOS]])

```json
{
  "aps": {
    "alert": {
      "title": "Новый заказ",
      "body": "Ваш заказ #123 готов к доставке"
    },
    "badge": 1,
    "sound": "default"
  },
  "customData": {
    "orderId": "123"
  }
}
```

### Через FCM

```json
{
  "to": "DEVICE_FCM_TOKEN",
  "notification": {
    "title": "Новый заказ",
    "body": "Ваш заказ #123 готов к доставке",
    "sound": "default"
  },
  "data": {
    "orderId": "123"
  }
}
```

**Ключевые моменты:**

- `alert` / `notification` → текст уведомления.
    
- `badge` → обновление значка на иконке приложения.
    
- `sound` → проигрывание звука уведомления.
    
- `customData` / `data` → передача дополнительной информации для обработки при открытии приложения.

## Особенности реализации UI push в iOS

1. **Запрос разрешений пользователя**
    

```swift
UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .sound, .badge]) { granted, error in
    if granted {
        DispatchQueue.main.async {
            UIApplication.shared.registerForRemoteNotifications()
        }
    }
}
```

2. **Обработка уведомления при открытии**
    

```swift
func userNotificationCenter(_ center: UNUserNotificationCenter,
                            didReceive response: UNNotificationResponse,
                            withCompletionHandler completionHandler: @escaping () -> Void) {
    let userInfo = response.notification.request.content.userInfo
    if let orderId = userInfo["orderId"] as? String {
        // открыть экран заказа
    }
    completionHandler()
}
```

3. **Настройка интерактивных кнопок**
    

- Можно добавить кнопки действий (Action Buttons) в уведомление.
    
- Пользователь может взаимодействовать, не открывая приложение полностью.

## Best Practices для UI push

1. **Не перегружать пользователя уведомлениями** — важно отправлять только релевантные события.
    
2. **Использовать понятные title и body** — коротко и ясно.
    
3. **Использовать badge и sound аккуратно** — избегать раздражающих звуков и частых обновлений значка.
    
4. **Добавлять custom data** — для обработки и навигации внутри приложения при открытии.
    
5. **Сегментация аудитории** — отправлять уведомления только тем пользователям, для которых они актуальны.

## Применение

- Новости и обновления → push с текстом и картинкой.
    
- Социальные приложения → уведомления о лайках, сообщениях, комментариях.
    
- Доставка и e-commerce → уведомления о статусе заказа, акциях и скидках.
    
- Финансовые приложения → оповещения о транзакциях, курсах валют.
    

---

## Итог

- **UI push** — основной способ информирования пользователя через **видимые уведомления**.
    
- Важные элементы: **title, body, badge, sound, actions, custom data**.
    
- Требует **разрешения пользователя** и правильной обработки на устройстве.
    
- Отличается от silent push, который используется для фонового обновления данных без отображения уведомления.
