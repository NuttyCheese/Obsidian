 Задача 1: Исследование ARC и Strong Reference Cycles
Создай класс `Person`, который содержит свойство с типом класса `Car`. Класс `Car` должен содержать свойство типа `Person`. Сымитируй ситуацию, когда возникает сильная ссылка (strong reference cycle), из-за которой объекты не освобождаются из памяти.

Задание:
1. Реализуй классы так, чтобы между ними образовался цикл сильных ссылок.
2. Используй `weak` и `unowned`, чтобы исправить ситуацию, и объясни, в каком случае лучше использовать один или другой.

Пример:
```swift
class Person {
    var name: String
    var car: Car?
    
    init(name: String) {
        self.name = name
    }
    
    deinit {
        print("\(name) is being deallocated")
    }
}

class Car {
    var model: String
    var owner: Person?
    
    init(model: String) {
        self.model = model
    }
    
    deinit {
        print("\(model) is being deallocated")
    }
}
```

 Задача 2: Использование замыканий и учка памяти
Создай класс `NetworkManager`, который выполняет запрос данных и передает их обратно через замыкание. Покажи, как замыкания могут привести к утечке памяти из-за сильной ссылки на `self`.

Задание:
1. Реализуй метод в `NetworkManager`, который использует замыкание для передачи данных обратно.
2. Найди утечку памяти и исправь ее с помощью [capture list].

Пример:
```swift
class NetworkManager {
    var data: Data?
    
    func fetchData(completion: @escaping (Data?) -> Void) {
        // Симуляция асинхронного запроса
        DispatchQueue.global().asyncAfter(deadline: .now() + 2) {
            self.data = Data()
            completion(self.data)
        }
    }
    
    deinit {
        print("NetworkManager is being deallocated")
    }
}
```

 Задача 6: Утечки памяти в многопоточном окружении
Приложение использует параллельную обработку данных с помощью GCD или OperationQueue для загрузки изображений. Однажды заметили, что приложение начинает потреблять слишком много памяти, и оперативная память не освобождается даже после завершения всех операций.

Задание:
1. Реализуй код, который создает много параллельных операций по загрузке изображений.
2. Найди проблему с утечкой памяти, вызванную неправильно захваченными ссылками на объекты в многопоточном коде.
3. Исправь проблему, добавив контроль за ссылками (например, через [weak self] или инкапсуляцию данных в отдельные объекты для освобождения).
```swift
class ImageLoader {
    var images = [UIImage]()

    func loadImages(urls: [URL]) {
        let queue = DispatchQueue.global(qos: .userInitiated)
        urls.forEach { url in
            queue.async { [weak self] in
                guard let self = self else { return }
                if let data = try? Data(contentsOf: url), let image = UIImage(data: data) {
                    self.images.append(image)
                }
            }
        }
    }
}
```
