Компилятор [[Swift]] сообщает, что **несколько объявлений (переменных, констант, функций и т.д.) находятся на одной строке без разделителя `;`**.

- В Swift обычно **каждое объявление находится на своей строке**.
    
- Если вы хотите объявить несколько элементов на одной строке, **нужно разделять их точкой с запятой (`;`)**.
    
- Ошибка возникает только при **несоблюдении синтаксиса**.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Несколько переменных на одной строке без `;`**

```swift
let a = 5 let b = 10
// ❌ Consecutive declarations on a line must be separated by ';'
```

---

**Пример 2: Несколько функций на одной строке без разделителя**

```swift
func foo() {} func bar() {}
// ❌ Consecutive declarations on a line must be separated by ';'
```

---

**Пример 3: Смешанные объявления**

```swift
var x = 1 var y = 2
// ❌ ошибка компиляции
```

---

### Как исправить

#### 1️⃣ Разделять объявления на разные строки

```swift
let a = 5
let b = 10

func foo() {}
func bar() {}
```

---

#### 2️⃣ Использовать точку с запятой для нескольких объявлений на одной строке

```swift
let a = 5; let b = 10
func foo() {}; func bar() {}
```

---

### Резюме

- Ошибка возникает, когда **несколько объявлений находятся на одной строке без `;`**.
    
- Исправляется через:
    
    1. Перенос каждого объявления на отдельную строку.
        
    2. Добавление `;` между объявлениями на одной строке.
        
- Позволяет компилятору правильно интерпретировать код и успешно компилировать проект.
