Компилятор [[Swift]] сообщает, что **он ожидал объявление (declaration) в текущей области видимости**, но встретил что-то другое.

- В Swift **вне тела функций и классов можно писать только объявления**: переменные ([[var]]/[[let]]), константы, функции, классы, структуры, перечисления, протоколы и т.д.
    
- Ошибка часто возникает, когда:
    
    1. Вне класса или структуры написан **инструктивный код** (например, `print()` или вызов функции).
        
    2. Неправильно закрыт блок кода.
        
    3. Ошибка вложенности или лишний оператор.
        

---

### Примеры кода/сценариев возникновения

**Пример 1: Инструкции вне функций**

```swift
print("Hello, world!") 
// ❌ Expected declaration
```

- Вне класса/структуры/функции нельзя просто писать инструкции.
    

---

**Пример 2: Лишний код после закрывающей скобки**

```swift
struct Person {
    let name: String
}

name = "Alice" // ❌ Expected declaration
```

- `name = "Alice"` находится вне контекста инициализации или метода.
    

---

**Пример 3: Ошибка вложенности**

```swift
func greet() {
    print("Hello")
}
print("World") // ✅ если в глобальной области видимости в Playground — нормально
// ❌ в проекте Swift, где глобальный код недопустим, вызовет Expected declaration
```

---

### Как исправить

#### 1️⃣ Перенести код внутрь функции, метода или инициализатора

```swift
struct Person {
    let name: String

    func greet() {
        print("Hello, \(name)")
    }
}
```

---

#### 2️⃣ Проверить лишние операторы или вызовы вне блоков

- Удалить инструкции, которые нельзя писать в глобальной области видимости проекта.
    

---

#### 3️⃣ Проверить правильность вложенности `{}`

```swift
func greet() {
    print("Hello")
} // ✅ закрывающая скобка на месте
```

---

### Резюме

- **Expected declaration** означает, что **компилятор ожидал объявление, а получил что-то другое**.
    
- Исправляется через:
    
    1. Перенос кода внутрь функций или методов.
        
    2. Удаление инструкций, недопустимых в текущей области видимости.
        
    3. Проверку правильности скобок и вложенности.
        
- Позволяет компилятору корректно интерпретировать код и успешно собрать проект.
