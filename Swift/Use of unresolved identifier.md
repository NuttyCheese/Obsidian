Компилятор [[Swift]] сообщает, что **он не может найти идентификатор**, который используется в коде.

- Идентификатор может быть:
    
    - переменной (`x`, `name`)
        
    - функцией (`foo()`)
        
    - типом (`User`, `Person`)
        
    - константой (`pi`, `maxValue`)
        
- Ошибка возникает, когда:
    
    1. Идентификатор не объявлен.
        
    2. Есть опечатка в имени.
        
    3. Идентификатор объявлен в другой области видимости.
        
    4. Пропущен `import` для модуля, где определён идентификатор.
        

---

### Примеры кода/сценариев возникновения

**Пример 1: Опечатка в имени переменной**

```swift
let userName = "Alice"
print(userNme) // ❌ Use of unresolved identifier 'userNme'
```

---

**Пример 2: Использование несуществующей функции**

```swift
greet() // ❌ Use of unresolved identifier 'greet'
```

---

**Пример 3: Переменная вне области видимости**

```swift
func foo() {
    let x = 10
}
print(x) // ❌ Use of unresolved identifier 'x'
```

---

**Пример 4: Отсутствие импорта модуля**

```swift
let url = URL(string: "https://example.com") 
// ❌ Use of unresolved identifier 'URL' если не импортирован Foundation

import Foundation // ✅ исправление
```

---

### Как исправить

#### 1️⃣ Проверить правильность имени

```swift
print(userName) // ✅ исправлено
```

---

#### 2️⃣ Проверить область видимости

```swift
func foo() {
    let x = 10
    print(x) // ✅ x доступна внутри функции
}
```

---

#### 3️⃣ Импортировать нужный модуль

```swift
import Foundation
let url = URL(string: "https://example.com") // ✅ исправлено
```

---

#### 4️⃣ Объявить идентификатор перед использованием

```swift
func greet() {
    print("Hello")
}

greet() // ✅ исправлено
```

---

### Резюме

- **Use of unresolved identifier** возникает, когда компилятор не видит используемый идентификатор.
    
- Исправляется через:
    
    1. Проверку имени на опечатки.
        
    2. Проверку области видимости переменной или функции.
        
    3. Импорт необходимых модулей.
        
    4. Объявление идентификатора до использования.
        
- Позволяет компилятору правильно интерпретировать код и собрать проект.
