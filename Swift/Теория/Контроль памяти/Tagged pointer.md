**Tagged Pointer** (тегированный указатель) — это **оптимизация времени выполнения**, при которой **маленькие значения** (в основном [[Int]], [[NSNumber]], некоторые [[NSString]]) хранятся **непосредственно внутри указателя**, а не в отдельном объекте на куче.

Это позволяет:
- экономить память (нет аллокации объекта)
- ускорять доступ (нет косвенной адресации)
- уменьшать давление на GC/[[ARC]]

### Как это работает

В 64-битной архитектуре (arm64, x86_64) младший бит указателя обычно всегда 0 (выравнивание по 2).  
[[Swift]] (и [[Objective-C]]) использует этот бит + несколько следующих для хранения **тега** и **значения**.

**Схема (упрощённо, arm64):**

```text
Обычный указатель на объект:
0x0000000123456780   (младший бит = 0)

Tagged Pointer (Int):
1xxxxxxx...xxxxxxx1   (младший бит = 1 → тег)
                  └── значение упаковывается в оставшиеся биты
```

### Какие типы используют tagged pointers в Swift (2026)

| Тип данных                | Использует tagged pointer? | Диапазон значений (примерно)      | Примечание                    |
| ------------------------- | -------------------------- | --------------------------------- | ----------------------------- |
| `Int` / [[UInt]]          | Да                         | ±2⁶²–1 (обычно)                   | Почти всегда                  |
| `NSNumber` (из ObjC)      | Да                         | Маленькие числа                   | Через мост                    |
| `NSString` (маленькие)    | Да (иногда)                | Короткие строки (≤ 7–15 символов) | Зависит от версии             |
| [[NSDate]], [[NSValue]]   | Частично                   | Маленькие значения                | Редко                         |
| Большие `Int`, [[String]] | Нет                        | —                                 | Выделяется полноценный объект |

### Как проверить, tagged ли значение

```swift
let n = 42

// Способ 1: unsafeBitCast + проверка isa
let obj = unsafeBitCast(n, to: AnyObject.self)
print(type(of: obj) == NSNumber.self)  // true → tagged

// Способ 2: через Objective-C runtime (более надёжный)
import ObjectiveC.runtime

let isa = object_getClass(obj)
print(isa == NSNumber.self)            // true → tagged
```

**Важно:**  
Не полагайтесь на `===` или `==` для проверки tagged — это не гарантировано.

### Практическая польза (2026)

- **Экономия памяти** — нет выделения объекта под каждый маленький `Int` / `NSNumber`
- **Скорость** — доступ к значению — 1 load вместо 2–3 (load isa → load value)
- **Кэш-френдли** — меньше аллокаций → меньше промахов кэша
- **Меньше давления на ARC** — меньше объектов → меньше [[retain]]/[[release]]

### Когда tagged pointer НЕ используется

- Большие числа (`Int` > ~2⁶²)
- Длинные строки (`String` > 15–23 символа)
- Сложные объекты (`NSDate` с датой далеко в будущем/прошлом)
- 32-битная архитектура (почти исчезла)

### Короткий итог

- Tagged pointers — **оптимизация** для **маленьких значений** (`Int`, `NSNumber`, короткие `NSString`)
- Значение **упаковывается прямо в указатель** (младшие биты = тег + данные)
- **Экономия памяти и скорости** — нет объекта на куче
- В 99% случаев **не нужно** беспокоиться — компилятор и runtime всё делают сами
- Проверять можно через [[unsafeBitCast]] → `NSNumber.self` (но это редко нужно)

**Главное правило**:
> «Маленькие `Int` и `NSNumber` почти всегда tagged — это просто работает быстрее и экономнее.  
> Забудьте про это — Swift сам позаботится.»
