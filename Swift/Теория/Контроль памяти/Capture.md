# Захват (Capture) в замыканиях [[Swift]] и [[ARC]]

**Захват** (capture) — это механизм, с помощью которого **замыкание** получает доступ к переменным/константам из внешней области видимости и сохраняет их для использования даже после выхода из этой области.

### 1. Как происходит захват

Замыкание может захватывать значения **тремя способами**:

| Способ захвата              | Синтаксис                 | Когда используется                                   | Влияние на retain count | Пример                            |
| --------------------------- | ------------------------- | ---------------------------------------------------- | ----------------------- | --------------------------------- |
| **По значению**             | по умолчанию (без списка) | для [[value type]] ([[Int]], [[struct]], [[String]]) | не увеличивает          | `{ print(x) }`                    |
| **По сильной ссылке**       | `[self]` или по умолчанию | для [[reference type]] ([[class]])                   | увеличивает RC          | `{ self.do() }`                   |
| **По слабой ссылке**        | `[weak self]`             | предотвращает [[retain cycle]]                       | не увеличивает          | `{ [weak self] in self?.do() }`   |
| **По неразрушаемой ссылке** | `[unowned self]`          | когда уверены, что объект живёт дольше               | не увеличивает          | `{ [unowned self] in self.do() }` |

### 2. Самые частые паттерны (2026)

#### a) Захват value type (безопасно, копируется)

```swift
func makeCounter() -> () -> Int {
    var count = 0
    return {
        count += 1      // count захвачен по значению → своя копия
        return count
    }
}

let counter = makeCounter()
print(counter())  // 1
print(counter())  // 2
```

#### b) Классическая утечка (retain cycle)

```swift
class ViewController {
    var callback: (() -> Void)?
    
    func setup() {
        callback = {
            self.updateUI()     // сильный захват self → цикл
        }
    }
    
    deinit { print("deinit") }  // никогда не вызовется
}
```

#### c) Правильный вариант — [weak self]

```swift
callback = { [weak self] in
    guard let self else { return }
    self.updateUI()
}

// или короче (Swift 5.3+)
callback = { [weak self] in
    self?.updateUI()
}
```

#### d) [unowned self] — когда 100% уверены в жизненном цикле

```swift
class Owner {
    lazy var child: Child = Child(owner: self)
}

class Child {
    unowned let owner: Owner
    
    init(owner: Owner) {
        self.owner = owner
    }
    
    deinit { print("Child deinit") }
}
```

### 3. Ключевые правила захвата (2026)

| Ситуация                               | Как захватывать                    | Почему                                |
| -------------------------------------- | ---------------------------------- | ------------------------------------- |
| value type (`Int`, `struct`, `String`) | по умолчанию (без списка)          | копируется, безопасно                 |
| [[self]] внутри класса                 | `[weak self]`                      | предотвращает цикл                    |
| `self` в [[lazy]] var / после проверки | `[unowned self]`                   | быстрее, но опасно                    |
| Несколько объектов                     | `[weak delegate, weak dataSource]` | все ссылки weak                       |
| Захват константы один раз              | `[let value = self.value]`         | фиксирует значение на момент создания |

### 4. Типичные утечки и как их избежать

| Утечка                              | Причина                              | Решение |
|-------------------------------------|--------------------------------------|---------|
| `self` внутри замыкания             | сильный захват `self`                | `[weak self]` |
| Делегат / dataSource держит контроллер | `delegate` strong                    | `weak var delegate` |
| Замыкание в property класса         | `var callback: () -> Void`           | `[weak self]` при присваивании |
| Цикл через два класса               | `a.b = b; b.a = a`                   | один из них `weak` / `unowned` |

### 5. Короткий итог

- **Захват value types** → копируется, безопасно
- **Захват reference types** → по умолчанию [[strong]] → риск цикла
- **Главное оружие против утечек** → `[weak self]`
- **Когда можно [unowned self]** → когда жизненный цикл гарантирован (parent → child, lazy var после init)
- **Проверяй в Instruments → Leaks** — всегда, если используешь замыкания внутри классов

**Золотое правило Swift 2026**:
> «В 99% случаев внутри класса пиши `[weak self]` — это безопасно и бесплатно.  
> `[unowned self]` — только если ты **точно** знаешь, что объект не умрёт раньше замыкания.»
