Weak references (слабые ссылки) в [[Swift]] - это способ ссылаться на объект, который не увеличивает счетчик его ссылок ([[retain count]]), и не удерживает его в памяти, когда другие сильные ссылки на него отсутствуют. Если объект, на который указывает слабая ссылка, был освобожден из памяти, то слабая ссылка автоматически устанавливается в [[nil]].

Слабые ссылки часто используются для предотвращения циклических зависимостей ([[retain cycle]]) между объектами, когда два объекта сильно ссылаются друг на друга, в результате чего они не могут быть освобождены из памяти, даже если они больше не нужны. Использование слабых ссылок позволяет объектам быть освобожденными, когда больше нет сильных ссылок на них.

Пример использования слабых ссылок в Swift:
```swift
class Person {
    var name: String
    
    init(name: String) {
        self.name = name
    }
    
    deinit {
        print("\(name) освобожден из памяти")
    }
}

class Apartment {
    weak var tenant: Person?
    
    deinit {
        print("Квартира освобождена из памяти")
    }
}

var john: Person? = Person(name: "John")
var apartment: Apartment? = Apartment()

// Назначение слабой ссылки на объект Person
apartment?.tenant = john

// Освобождение объектов
john = nil
apartment = nil

```
В этом примере `apartment?.tenant` является слабой ссылкой на объект `john`. При освобождении объекта `john`, ссылка на него в объекте `apartment` автоматически становится [[nil]]. Таким образом, избегается циклическая зависимость между объектами `john` и `apartment`, и объекты могут быть освобождены из памяти, когда они больше не нужны.
