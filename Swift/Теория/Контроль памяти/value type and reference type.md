#memory_control #Swift 
Отличия между [[Value Type]] (типы значений) и [[Reference Type]] (типы ссылок) в [[Swift]] касаются того, как они хранятся в памяти и как происходит их передача.

### Value Types (Типы Значений):

1. **Хранение в памяти:**
    
    - Value types хранят свои значения напрямую в месте, где они объявлены. Например, когда создается переменная типа [[struct]] или [[enum]], её значения хранятся в стеке памяти или внутри другого объекта (например, в массиве или словаре).
2. **Передача значений:**
    
    - Передача value type осуществляется путем создания копии значений. Когда value type присваивается другой переменной, передается копия значений, а не ссылка на оригинал.
3. **Жизненный цикл:**
    
    - Жизненный цикл value type связан с областью видимости, где он создан. Когда выходит из области видимости, значение уничтожается.

### Reference Types (Типы Ссылок):

1. **Хранение в памяти:**
    
    - Reference types хранятся в куче ([[Heap]]). Когда создается экземпляр класса, его данные и методы размещаются в куче, а переменная содержит ссылку на этот объект.
2. **Передача ссылок:**
    
    - Передача reference type осуществляется по ссылке на объект. Когда reference type присваивается другой переменной, передается ссылка на тот же объект, а не копия данных.
3. **Жизненный цикл:**
    
    - Жизненный цикл reference type не связан с областью видимости. Объект существует, пока на него есть хотя бы одна ссылка. Когда все ссылки на объект отсутствуют (нет [[strong]]-ссылок), объект уничтожается автоматически.
```swift
// Value Type (Структура)
struct Point {
    var x: Double
    var y: Double
}

var point1 = Point(x: 1.0, y: 2.0)
var point2 = point1  // Создается копия значений

// Reference Type (Класс)
class Person {
    var name: String
    init(name: String) {
        self.name = name
    }
}

var person1 = Person(name: "John")
var person2 = person1  // person2 ссылается на тот же объект, что и person1

```
В примере выше `point1` и `point2` представляют value type (структуру), и при присваивании создается копия значений. В случае с `person1` и `person2`, которые представляют [[Reference Type]] (класс), при присваивании `person2` ссылается на тот же объект, что и `person1`.

Понимание этих различий важно для правильного выбора между value types и reference types в зависимости от конкретной ситуации в разработке.
