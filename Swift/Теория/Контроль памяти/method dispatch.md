**Диспетчеризация** (method dispatch) — это процесс определения, **какой именно** реализацией метода (или функции) будет выполнен вызов.  
В [[Swift]] диспетчеризация бывает **трёх основных видов**:

| Вид диспетчеризации                  | Когда происходит    | Скорость          | Полиморфизм           | Когда используется                                |     |
| ------------------------------------ | ------------------- | ----------------- | --------------------- | ------------------------------------------------- | --- |
| **Статическая** (Static / Direct)    | На этапе компиляции | Максимальная      | Нет                   | `final`, `struct`, [[enum]], [[generic]], private |     |
| **Динамическая** (Dynamic / Virtual) | Во время выполнения | Средняя           | Да (через [[vtable]]) | Обычные методы `class` без `final`                |     |
| **По протоколу** (Witness Table)     | Во время выполнения | Ниже динамической | Да (экзистенциалы)    | [[protocol]], [[any Protocol]], [[some Protocol]] |     |

### 1. Статическая диспетчеризация (Static Dispatch)

- Выбор метода происходит **на этапе компиляции**
- Самая быстрая — позволяет **инлайнинг**, удаление вызовов, оптимизации
- Нет overhead'а на runtime-поиск

**Когда компилятор выбирает статическую диспетчеризацию:**

- [[final]] методы / классы
- Методы в [[struct]], [[enum]]
- Методы в [[extension]] (если не переопределяются)
- [[private]] / [[fileprivate]] методы
- Вызовы через **конкретный тип** (не через протокол или базовый класс)

```swift
final class FastMath {
    final func square(_ x: Int) -> Int { x * x }   // всегда статическая
}

struct Vector {
    func length() -> Double { /* ... */ }         // всегда статическая
}

let v = Vector()
v.length()                                        // статическая, можно инлайнить
```

### 2. Динамическая диспетчеризация (Virtual / Dynamic Dispatch)

- Выбор метода происходит **во время выполнения** через **таблицу виртуальных методов** (vtable)
- Поддерживает **полиморфизм времени выполнения**

**Когда используется:**

- Обычные методы класса без `final`
- Методы, помеченные [[override]]
- Вызов через переменную базового типа

```swift
class Animal {
    func makeSound() { print("Some sound") }     // динамическая
}

class Dog: Animal {
    override func makeSound() { print("Woof!") }
}

let pet: Animal = Dog()
pet.makeSound()  // → "Woof!" (динамическая диспетчеризация через vtable)
```

### 3. Диспетчеризация по протоколу (Witness Table)

- Самая медленная из трёх
- Используется при работе с **экзистенциалами** (`any` [[Protocol]]) или **[[generic]]** с протоколами
- Каждый conforming-тип имеет **таблицу свидетелей** (witness table), где хранятся указатели на реализации требований протокола

```swift
protocol Drawable {
    func draw()
}

struct Circle: Drawable {
    func draw() { print("○") }
}

struct Square: Drawable {
    func draw() { print("□") }
}

let shapes: [any Drawable] = [Circle(), Square()]
shapes.forEach { $0.draw() }  // witness table dispatch → медленнее
```

**Оптимизация** — использовать **generics** вместо `any`:

```swift
func drawShapes<T: Drawable>(_ shapes: [T]) {
    shapes.forEach { $0.draw() }  // статическая диспетчеризация внутри generics
}
```

### 4. Сравнение производительности (примерные значения)

| Диспетчеризация       | Относительная скорость | Overhead на вызов | Полиморфизм | Примечание |
|-----------------------|------------------------|-------------------|-------------|------------|
| Статическая           | 1× (максимум)          | 0                 | Нет         | Инлайнинг возможен |
| Динамическая (vtable) | ~1.5–2× медленнее      | ~5–10 тактов      | Да          | Классы без final |
| Протокольная (witness table) | ~3–8× медленнее   | ~15–40 тактов     | Да          | `any Protocol` — самый дорогой |

### 5. Практические советы и паттерны (2026)

- Хочешь **максимальную скорость** → добавляй `final` к классам и методам, которые не нужно переопределять
- Хочешь **полиморфизм** → используй `class` без `final` (динамическая диспетчеризация)
- Хочешь **протоколы без overhead** → используй **generics** + `some Protocol` вместо `any Protocol`
- Избегай `any Protocol` в горячих путях (массивы, циклы) — лучше конкретный тип или generic

```swift
// Медленно (witness table dispatch в цикле)
let views: [any View] = [...]
views.forEach { $0.render() }

// Быстро (статическая диспетчеризация)
func renderViews<T: View>(_ views: [T]) {
    views.forEach { $0.render() }
}
```

### Итог — коротко

- **Статическая** — самая быстрая, нет полиморфизма, `final` / struct / generics
- **Динамическая** — полиморфизм времени выполнения, обычные методы `class`
- **Протокольная** — самая дорогая, `any Protocol`, избегай в горячих циклах

**Главное правило 2026**:
> «По умолчанию делай методы `final` или используй `struct` / generics.  
> Переходи на динамическую диспетчеризацию только там, где действительно нужен полиморфизм.»
