В [[Swift]] все типы делятся на две большие группы по семантике и поведению при копировании/присваивании:

| Группа              | Ключевые типы                                  | Семантика копирования | Где хранится (обычно)       | Поддерживает наследование? | Поддерживает `weak`/`unowned`? | Примеры                                                                                  |
| ------------------- | ---------------------------------------------- | --------------------- | --------------------------- | -------------------------- | ------------------------------ | ---------------------------------------------------------------------------------------- |
| **Value Types**     | [[struct]], [[enum]], [[tuple]], все примитивы | Копируется значение   | [[Stack]] или inline в куче | Нет                        | Нет                            | [[Int]], [[Double]], [[Bool]], [[String]], [[Array]], [[Dictionary]], [[Set]], [[Point]] |
| **Reference Types** | [[class]], замыкания ([[closure]])             | Копируется ссылка     | Куча ([[Heap]])             | Да                         | Да                             | [[UIViewController]], [[NSObject]], [[URLSession]]                                       |

### 1. Value Types (типы-значения)

**Копируется значение** — при присваивании или передаче в функцию создаётся **независимая копия**.

```swift
struct Point {
    var x: Int
    var y: Int
}

var p1 = Point(x: 10, y: 20)
var p2 = p1           // полноценная копия
p2.x = 50

print(p1.x)           // 10 — оригинал не изменился
print(p2.x)           // 50
```

**Типичные value types в Swift**:
- Все примитивы: `Int`, `UInt`, `Double`, `Float`, `Bool`, `Character`
- `String`, `Array`, `Dictionary`, `Set` — value types с **[[Copy-On-Write]]** (COW)
- `struct`, `enum`, `tuple`
- Все типы из стандартной библиотеки, не являющиеся классами

**Преимущества value types**:
- Независимые копии — безопасно передавать и мутировать
- Нет [[retain cycle]] → нет утечек памяти
- Часто хранятся inline в стеке → быстрее доступ
- Поддерживают **Copy-on-Write** для больших коллекций → экономия памяти

### 2. Reference Types (типы-ссылки)

**Копируется ссылка** — все переменные указывают на **один и тот же** объект в куче.

```swift
class Person {
    var name: String
    init(name: String) { self.name = name }
}

var alice1 = Person(name: "Алиса")
var alice2 = alice1           // та же ссылка
alice2.name = "Алиса 2.0"

print(alice1.name)            // "Алиса 2.0"
print(alice1 === alice2)      // true — один объект
```

**Типичные reference types**:
- Все классы (`class`)
- Замыкания (closures)
- [[AnyObject]], `NSObject` и их подклассы

**Преимущества reference types**:
- Общая изменяемая сущность (один объект — много владельцев)
- Поддержка наследования и полиморфизма
- Возможность [[weak]] / [[unowned]] для разрыва циклов
- [[deinit]] для очистки ресурсов

### 3. Сравнение value types и reference types

| Свойство                 | Value Types (`struct`, `enum`) | Reference Types (`class`)    |
| ------------------------ | ------------------------------ | ---------------------------- |
| Копирование              | Значение полностью копируется  | Копируется только ссылка     |
| Изменение одной копии    | Не влияет на другие            | Влияет на все ссылки         |
| Сравнение                | По значению (`==`)             | По идентичности (`===`)      |
| Наследование             | Нет                            | Да                           |
| `deinit`                 | Нет                            | Есть (вызывается при RC = 0) |
| `weak` / `unowned`       | Нет                            | Есть                         |
| Возможность retain cycle | Нет                            | Да → утечки памяти           |
| Хранение (обычно)        | Стек / [[Inline]] в куче       | Куча                         |
| Copy-on-Write            | Есть у коллекций               | Нет                          |

### 4. Реальные сценарии в iOS (2026)

- **Value types** — модели данных, DTO, состояние экрана, ViewModel (кроме ObservableObject), конфигурации
- **Reference types** — контроллеры экранов, view-модели с жизненным циклом, делегаты, кэши, сетевые сервисы

**Пример выбора типа**:

```swift
// Value type — независимые копии
struct Message {
    let text: String
    let isFromMe: Bool
}

// Reference type — общий кэш изображений
class ImageCache {
    private var cache: [URL: UIImage] = [:]
    
    func image(for url: URL) -> UIImage? {
        cache[url]
    }
    
    func set(_ image: UIImage, for url: URL) {
        cache[url] = image
    }
}
```

### Короткий итог и правило Apple (2026)

- **Value types** — безопасные, независимые, быстрые, без утечек
- **Reference types** — общие, изменяемые, с идентичностью, наследованием и жизненным циклом

**Официальное правило Apple**:
> «Используйте структуры по умолчанию.  
> Переходите на классы только если вам действительно нужна:  
> • общая идентичность (identity)  
> • наследование  
> • `deinit` или наблюдение за жизненным циклом»
