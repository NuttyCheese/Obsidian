#system_design
## Определение

**Reverse engineering** — процесс анализа приложения с целью **извлечения исходного кода, логики, ключей или токенов**.

**Цель защиты:** предотвратить кражу интеллектуальной собственности, взлом авторизации, модификацию приложения или обход защиты.

Основные подходы в [[iOS]]:

- **Обфускация кода** → усложняет анализ и декомпиляцию.
    
- **Anti-Jailbreak** → проверка устройства на наличие jailbreak для ограничения функционала.
    

---

## 1. Обфускация кода

### Цель

- Сделать **чтение кода сложным** для атакующих.
    
- Скрыть **имена методов, переменных, структур данных**.
    

### Подходы в iOS

1. **Swift Obfuscation**
    
    - Инструменты: [SwiftShield](https://github.com/rockbruno/swiftshield), ProGuard-подобные решения для [[Swift]].
        
    - Меняет имена классов, функций и переменных.
        
    - Можно зашифровать строки (например, [[API]]-ключи).
        
2. **C/[[Objective-C]] Obfuscation**
    
    - Используются препроцессорные макросы для переименования функций.
        
    - Применяются инструменты типа [LLVM obfuscator](https://github.com/obfuscator-llvm/obfuscator).
        

### Пример защиты строк

```swift
func obfuscatedString(_ key: String) -> String {
    // XOR или Base64 кодирование
    let data = Data(key.utf8)
    return data.base64EncodedString()
}

let secret = obfuscatedString("API_KEY_123")
```

## 2. Anti-Jailbreak защита

### Цель

- Предотвратить запуск приложения на **компрометированном устройстве**.
    
- Ограничить доступ к API, токенам и конфиденциальным данным.
    

### Методы проверки

1. **Проверка системных файлов**
    

```swift
let jailbreakPaths = [
    "/Applications/Cydia.app",
    "/usr/sbin/sshd",
    "/bin/bash"
]

for path in jailbreakPaths {
    if FileManager.default.fileExists(atPath: path) {
        print("Device is jailbroken")
    }
}
```

2. **Проверка разрешений**
    

```swift
let canEditSystemFiles = FileManager.default.isWritableFile(atPath: "/")
```

3. **Проверка sandbox**
    
    - Попытка выйти за пределы песочницы.
        
    - Если возможно → устройство jailbroken.
        
4. **Проверка динамических библиотек**
    

```swift
let loadedLibraries = _dyld_image_count()
```

- Наличие Cydia Substrate или tweak библиотек → jailbreak.
    

### Действия при jailbreak

- Ограничение функционала.
    
- Предупреждение пользователя.
    
- Блокировка приложения.

## 3. Best Practices

1. **Комбинировать методы защиты** → обфускация + anti-jailbreak + защита токенов.
    
2. **Обфускация строк и ключей** → минимизирует риск утечки API-ключей.
    
3. **Регулярные проверки устройства** → защита от новых методов взлома.
    
4. **Использовать Secure Enclave и Keychain** → хранение критически важных данных безопасно.
    
5. **Мониторинг аномальной активности** → запросы с jailbroken устройств могут быть заблокированы сервером.
    

---

## 4. Итог

- **Обфускация кода** → усложняет чтение и анализ приложения.
    
- **Anti-Jailbreak** → защита от запуска на небезопасных устройствах.
    
- **Важные меры в iOS:** SwiftShield, проверка системных файлов, [[sandbox]], динамических библиотек.
    
- Комбинация с **Secure Enclave, [[Keychain]] и TLS** создаёт комплексную защиту данных и логики приложения.
