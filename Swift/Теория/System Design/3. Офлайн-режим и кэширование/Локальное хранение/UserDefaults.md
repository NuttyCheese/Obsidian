#system_design
## Определение

**UserDefaults** — это встроенный фреймворк iOS для хранения **небольших настроек и данных пользователя** локально на устройстве.

> UserDefaults идеально подходит для хранения настроек приложения, флагов, простых значений и небольших массивов, но **не предназначен для больших объёмов данных**.

---

## Основные особенности

1. **Локальное хранение**
    
    - Данные сохраняются в plist-файле в песочнице приложения.
        
2. **Простота использования**
    
    - Легко сохранить и получить данные по ключу.
        
3. **Синхронизация по iCloud (опционально)**
    
    - Можно использовать `NSUbiquitousKeyValueStore` для синхронизации настроек между устройствами.
        
4. **Подходит для небольших данных**
    
    - Строки, числа, флаги, массивы, словари.
        
    - Не подходит для хранения изображений, больших JSON или базы данных.
        

---

## Типы данных, которые можно хранить

|Тип данных|Пример|
|---|---|
|String|`"user_name"`|
|Int|`123`|
|Bool|`true`|
|Double|`12.34`|
|Array|`[1, 2, 3]`|
|Dictionary|`["key": "value"]`|
|Data|`UIImage` → `Data`|

---

## Основные операции

### 1. Сохранение данных

```swift
let defaults = UserDefaults.standard
defaults.set("Alex", forKey: "userName")
defaults.set(true, forKey: "isLoggedIn")
defaults.set(25, forKey: "userAge")
```

### 2. Чтение данных

```swift
let userName = defaults.string(forKey: "userName") ?? "Unknown"
let isLoggedIn = defaults.bool(forKey: "isLoggedIn")
let userAge = defaults.integer(forKey: "userAge")
```

### 3. Удаление данных

```swift
defaults.removeObject(forKey: "userName")
```

---

## Советы по использованию

1. Использовать для **настроек и флагов**, а не для больших данных.
    
2. Не хранить чувствительные данные (лучше Keychain для паролей и токенов).
    
3. Минимизировать количество чтений/записей → кэшировать в переменных при необходимости.
    
4. Для сложных структур (например, модели пользователя) можно использовать **JSON и Data**:
    

```swift
struct User: Codable {
    let id: String
    let name: String
}

let user = User(id: "123", name: "Alex")
if let encoded = try? JSONEncoder().encode(user) {
    defaults.set(encoded, forKey: "currentUser")
}

if let data = defaults.data(forKey: "currentUser"),
   let decodedUser = try? JSONDecoder().decode(User.self, from: data) {
    print(decodedUser.name)
}
```

---

## Применение UserDefaults

- Хранение **настроек интерфейса** (тема, язык, размер шрифта).
    
- Хранение **флагов авторизации** (`isLoggedIn`).
    
- Кэширование **малых данных** между сессиями.
    
- Быстрая проверка состояния приложения (например, первый запуск).
    

---

## Ограничения

- Не подходит для больших объемов данных.
    
- Не защищает данные (не хранить пароли и токены).
    
- Данные сохраняются **в одном plist**, что может замедлить работу при чрезмерном использовании.
    

---

## Итог

- **UserDefaults** — простой и удобный способ хранить небольшие данные и настройки пользователя.
    
- Используется для флагов, конфигураций, мелких кэшей и состояния приложения.
    
- Для больших или чувствительных данных следует использовать **[[Core Data]], [[Swift/Теория/System Design/3. Офлайн-режим и кэширование/Локальное хранение/Realm]], [[SQLite]] или [[KeyChain]]**.
