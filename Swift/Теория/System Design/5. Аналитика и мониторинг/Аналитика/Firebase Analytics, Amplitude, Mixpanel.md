#system_design
## Определение

**Инструменты аналитики** помогают **собирать, анализировать и визуализировать данные о поведении пользователей** в мобильных приложениях.

- **Firebase Analytics** → бесплатная платформа Google для сбора событий и аналитики мобильных приложений.
    
- **Amplitude** → продвинутая аналитика поведения пользователей с упором на когортный анализ, воронки и retention.
    
- **Mixpanel** → фокус на событиях и воронках, удобная визуализация и сегментация пользователей.
    

> Выбор платформы зависит от требований проекта: бюджет, сложность аналитики, интеграция с другими сервисами.

---

## 1. Firebase Analytics

### Особенности

- Бесплатная, интеграция с Firebase Suite (Crashlytics, Remote Config, FCM).
    
- Автоматически отслеживает базовые события (сессии, установка приложения, открытие экранов).
    
- Позволяет создавать **custom events** для специфичных действий пользователя.
    

### Интеграция в [[iOS]]

```swift
import FirebaseCore
import FirebaseAnalytics

FirebaseApp.configure()

// Логирование пользовательского события
Analytics.logEvent("purchase_complete", parameters: [
    "product_id": "12345",
    "price": 599
])
```

### Возможности

- Автоматическое отслеживание экрана и сессий.
    
- Воронки и retention через Firebase Console.
    
- Сегментация по свойствам пользователя.

## 2. Amplitude

### Особенности

- Фокус на **поведенческую аналитику** и когортный анализ.
    
- Подходит для анализа **пользовательских потоков и воронок**.
    
- Поддерживает **real-time analytics** и визуализацию событий.
    

### Интеграция в iOS

```swift
import Amplitude

Amplitude.instance().initializeApiKey("YOUR_API_KEY")

// Логирование события
Amplitude.instance().logEvent("purchase_complete", withEventProperties: [
    "product_id": "12345",
    "price": 599
])
```

### Возможности

- Создание пользовательских сегментов.
    
- Анализ retention и пользовательских воронок.
    
- A/B тестирование и интеграция с другими сервисами.
    

---

## 3. Mixpanel

### Особенности

- Фокус на **событийной аналитике** и **воронках конверсий**.
    
- Возможность строить **событийные последовательности** и анализировать путь пользователя.
    
- Поддержка push-уведомлений и триггеров.
    

### Интеграция в iOS

```swift
import Mixpanel

Mixpanel.initialize(token: "YOUR_TOKEN")

// Логирование события
Mixpanel.mainInstance().track(event: "purchase_complete", properties: [
    "product_id": "12345",
    "price": 599
])
```

### Возможности

- Сегментация пользователей и анализ воронок.
    
- Retention и когортный анализ.
    
- Интеграция с push-уведомлениями и A/B тестированием.

## Best Practices при использовании аналитики

1. **Стандартизировать события и свойства** → единообразие для удобного анализа.
    
2. **Отслеживать только ключевые действия** → не перегружать систему лишними событиями.
    
3. **Использовать сегментацию и воронки** → для понимания поведения различных групп пользователей.
    
4. **Комбинировать с push и background tasks** → для анализа эффективности уведомлений и обновлений данных.
    
5. **Регулярно анализировать retention и конверсии** → выявление проблем UX и повышение удержания пользователей.

## Итог

- **Firebase Analytics** → бесплатный, простая интеграция, базовая аналитика и воронки.
    
- **Amplitude** → продвинутая поведенческая аналитика, когортный анализ, real-time.
    
- **Mixpanel** → событийная аналитика и воронки, поддержка push и триггеров.
    
- Все три инструмента помогают **понимать поведение пользователей, улучшать UX и повышать удержание**.
