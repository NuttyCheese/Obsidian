#system_design
## Определение

**Аналитика в мобильных приложениях** — это процесс **сбора, обработки и анализа данных о действиях пользователей**, чтобы понять поведение, улучшить продукт и повысить удержание.

**Ключевые аспекты аналитики:**

- **События (Events)** — действия пользователя внутри приложения.
    
- **Воронки (Funnels)** — последовательность действий, ведущих к целевому результату.
    
- **Retention** — показатель удержания пользователей во времени.
    

---

## 1. События (Events)

### Определение

Событие — это **фиксированное действие пользователя**, которое имеет смысл для анализа, например:

- просмотр экрана (`screen_view`)
    
- клик по кнопке (`button_click`)
    
- завершение покупки (`purchase_complete`)
    

### Пример событий в [[iOS]] (Firebase Analytics)

```swift
import FirebaseAnalytics

// Просмотр экрана
Analytics.logEvent(AnalyticsEventScreenView, parameters: [
    AnalyticsParameterScreenName: "ProductDetail",
    AnalyticsParameterScreenClass: "ProductDetailViewController"
])

// Клик по кнопке
Analytics.logEvent("add_to_cart", parameters: [
    "product_id": "12345",
    "price": 599
])
```

**Советы по событиям:**

- Использовать понятные названия.
    
- Передавать ключевые параметры (ID, категория, стоимость).
    
- Не логировать личные данные пользователя.

## 2. Воронки (Funnels)

### Определение

**Воронка** — это последовательность действий, которые должен совершить пользователь, чтобы достичь цели.  
Пример:

1. Открытие приложения →
    
2. Просмотр продукта →
    
3. Добавление в корзину →
    
4. Покупка
    

### Применение

- Определение точек, где пользователи **выходят из процесса**.
    
- Оптимизация интерфейса и UX.
    
- Оценка эффективности маркетинговых кампаний.
    

### Пример в Firebase Analytics

- Создание последовательности событий для анализа конверсии.
    
- Метрики: процент пользователей, прошедших каждый шаг воронки.
    

---

## 3. Retention (удержание)

### Определение

**Retention** — это процент пользователей, которые **возвращаются в приложение через определённое время** после установки.

- **Day 1 retention** → пользователи, вернувшиеся на следующий день.
    
- **Day 7 retention** → пользователи, вернувшиеся через неделю.
    
- **Day 30 retention** → пользователи, вернувшиеся через месяц.
    

### Применение

- Понимание, насколько приложение **интересно и полезно** пользователям.
    
- Выявление проблем с функционалом, интерфейсом или ценностью продукта.

## 4. Интеграция с iOS приложением

1. **Firebase Analytics**
    

```swift
import FirebaseCore
import FirebaseAnalytics

FirebaseApp.configure()

// Логирование события
Analytics.logEvent("purchase_complete", parameters: [
    "product_id": "12345",
    "price": 599
])
```

2. **Custom events**
    

- Создавать события для внутренних метрик (клики, ошибки, завершение процессов).
    

3. **Анализ retention и воронок**
    

- Через **Firebase Console**, Google Analytics, Amplitude или Mixpanel.
    
- Можно строить графики, отслеживать сегменты пользователей и выявлять проблемные шаги.
    

---

## 5. Best Practices

1. **Стандартизировать названия событий и параметров** — единообразие важно для анализа.
    
2. **Не перегружать событиями** — фиксировать только ключевые действия.
    
3. **Использовать user properties** — сегментация пользователей по демографическим и поведенческим характеристикам.
    
4. **Сегментировать данные** — по источникам трафика, версии приложения, устройству.
    
5. **Соблюдать конфиденциальность** — не логировать персональные данные.
    

---

## Итог

- **События** → отслеживание действий пользователя.
    
- **Воронки** → последовательность действий для достижения цели, анализ конверсий.
    
- **Retention** → процент возврата пользователей, показатель удержания.
    
- Интеграция через **Firebase, Amplitude, Mixpanel** позволяет визуализировать данные и принимать решения.
    
- Аналитика помогает **повышать конверсию, улучшать UX и удержание пользователей**.
