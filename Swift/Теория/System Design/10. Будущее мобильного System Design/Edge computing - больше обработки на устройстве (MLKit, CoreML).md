#system_design
## Определение

**Edge computing** — это концепция, при которой **обработка данных выполняется ближе к источнику данных**, то есть на устройстве пользователя, а не на удалённом сервере.

Для [[iOS]] это означает, что данные приложения могут **обрабатываться локально**, без постоянного обращения к серверу, что обеспечивает:

- Высокую скорость отклика.
    
- Снижение нагрузки на сервер и трафика.
    
- Повышение конфиденциальности и безопасности данных.
    

---

## 1. Основные преимущества для iOS

|Преимущество|Описание|
|---|---|
|**Скорость и отзывчивость**|Данные обрабатываются локально, уменьшается задержка сетевых запросов.|
|**Безопасность и конфиденциальность**|Пользовательские данные не отправляются на сервер, что снижает риски утечки.|
|**Снижение нагрузки на сервер**|Обработка ML, фильтров и аналитики выполняется на устройстве.|
|**Работа офлайн**|Приложение может выполнять критические функции без сети.|

## 2. MLKit и CoreML

### CoreML

- **CoreML** — встроенный фреймворк Apple для **интеграции машинного обучения на устройстве**.
    
- Поддерживает модели в формате `.mlmodel` (классификация, регрессия, NLP, обработка изображений).
    

Пример использования CoreML в iOS:

```swift
import CoreML
import Vision

guard let model = try? VNCoreMLModel(for: MyImageClassifier().model) else { return }

let request = VNCoreMLRequest(model: model) { request, error in
    guard let results = request.results as? [VNClassificationObservation] else { return }
    print(results.first?.identifier ?? "No result")
}

let handler = VNImageRequestHandler(cgImage: image, options: [:])
try? handler.perform([request])
```

### MLKit

- **MLKit** — библиотека Google для **мобильного машинного обучения**, работает как локально, так и с сервером.
    
- Поддерживает:
    
    - Распознавание текста (Text Recognition)
        
    - Распознавание объектов (Object Detection)
        
    - Сканы лиц (Face Detection)
        
    - Бар-коды (Barcode Scanning)
        

Пример использования MLKit для распознавания текста:

```swift
import MLKitTextRecognition

let textRecognizer = TextRecognizer.textRecognizer()
let visionImage = VisionImage(image: uiImage)

textRecognizer.process(visionImage) { result, error in
    guard error == nil, let result = result else { return }
    print(result.text)
}
```

## 3. Подходы Edge computing в мобильных приложениях

1. **ML на устройстве**
    
    - Классификация изображений, видеоанализ, NLP.
        
    - Примеры: CoreML, MLKit, TensorFlow Lite.
        
2. **Локальная обработка данных**
    
    - Фильтрация, агрегирование, валидация до отправки на сервер.
        
    - Экономия трафика и ускорение работы приложения.
        
3. **Комбинированный подход**
    
    - Часть обработки локально (Edge), часть на сервере (Cloud).
        
    - Пример: предсказание ML локально, отправка только результата на сервер для статистики.

## 4. Best Practices

1. **Оптимизация моделей**
    
    - Использовать модели с небольшим размером и быстрым inference.
        
    - Применять `Quantization` или `Pruning` для уменьшения нагрузки на CPU/Memory.
        
2. **Асинхронная обработка**
    
    - ML задачи лучше выполнять на фоновых потоках (`DispatchQueue.global()`), чтобы UI оставался отзывчивым.
        
3. **Хранение моделей локально**
    
    - CoreML позволяет хранить `.mlmodel` в приложении.
        
    - MLKit позволяет загружать модели из облака и кэшировать их локально.
        
4. **Обновление моделей**
    
    - Возможность обновлять модели через сервер без обновления приложения (MLKit Remote Model).
        

---

## 5. Итог

- **Edge computing** позволяет переносить обработку данных **на iOS-устройство**, обеспечивая скорость, безопасность и работу офлайн.
    
- **CoreML и MLKit** — основные инструменты для мобильного машинного обучения.
    
- Подход идеально подходит для:
    
    - Распознавания изображений и текста.
        
    - Прогнозирования и аналитики на устройстве.
        
    - Улучшения отзывчивости UI и снижения нагрузки на сервер.
        
- Комбинация **Edge + Cloud** позволяет достичь оптимального баланса между производительностью и сложностью.
