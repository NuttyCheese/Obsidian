# (Короткое замыкание логических выражений)

**Определение:**  
Short-circuit evaluation (короткое замыкание) — это поведение логических операторов, при котором выражение **вычисляется только до того момента, пока результат уже известен**. Если результат логической операции можно определить, не вычисляя все части выражения, [[Swift]] **не будет вычислять оставшиеся части**.

Это особенно важно при работе с **условиями, где есть побочные эффекты** (например, вызов функций) или дорогие операции.

---

## Логические операторы в Swift

|Оператор|Описание|Тип|
|---|---|---|
|`&&`|Логическое И (AND)|short-circuit|
|`||`|
|`!`|Логическое НЕ (NOT)|не short-circuit, применимо к одному операнду|

---

## Основной принцип

1. **AND (`&&`)**
    
    - Если **левый операнд = false**, результат всегда `false` → правый операнд **не вычисляется**.
        
2. **OR (`||`)**
    
    - Если **левый операнд = true**, результат всегда `true` → правый операнд **не вычисляется**.
        

---

## Примеры на Swift

### 1. Простое условие

```swift
let a = false
let b = true

if a && b {
    print("Оба true")
} else {
    print("Хотя бы один false") // <- Выполнится
}
```

**Объяснение:**  
`a && b` → `a` = `false`, значит Swift **не проверяет `b`**, сразу возвращает `false`.

---

### 2. С функциями и побочными эффектами

```swift
func expensiveCheck() -> Bool {
    print("Проверка выполнена")
    return true
}

let flag = false

if flag && expensiveCheck() {
    print("Все верно")
} else {
    print("Short-circuit сработал") // <- Выполнится
}
```

**Вывод:**

```
Short-circuit сработал
```

`expensiveCheck()` **не вызовется**, потому что левый операнд `false` уже определяет результат.

---

### 3. OR и short-circuit

```swift
func expensiveOperation() -> Bool {
    print("Вызов expensiveOperation")
    return true
}

let condition = true

if condition || expensiveOperation() {
    print("Short-circuit для OR сработал") // <- Выполнится
}
```

**Вывод:**

```
Short-circuit для OR сработал
```

`expensiveOperation()` **не вызовется**, так как `condition = true` уже определяет результат.

---

### 4. Использование с опционалами

Short-circuit полезен при **разворачивании опционалов**:

```swift
var text: String? = nil

if text != nil && text!.count > 0 {
    print("Строка не пуста")
} else {
    print("Строка nil или пуста") // <- Выполнится
}
```

**Объяснение:**

- `text != nil` = `false`
    
- Swift **не выполняет `text!.count`**, предотвращая **crash** при `nil`.
    

---

### 5. Комбинация AND и OR

```swift
let x = false
let y = true
let z = false

if x && y || z {
    print("True")
} else {
    print("False") // <- Выполнится
}
```

**Пошаговое вычисление:**

1. `x && y` → `x` = `false`, пропускаем `y`, результат = `false`
    
2. `false || z` → `z` = `false`, результат = `false`
    

---

## Полезные советы

1. **Используйте short-circuit для безопасности**
    
    - Проверка на `nil` перед доступом к опционалу:
        
        ```swift
        if user != nil && user!.isActive { ... }
        ```
        
        Без short-circuit приложение бы падало при [[nil]].
        
2. **Оптимизация дорогих вычислений**
    
    - Вызывайте функции с тяжелыми вычислениями только если это необходимо.
        
3. **Не злоупотребляйте сложными цепочками**
    
    - Слишком длинные выражения с `&&` и `||` ухудшают читаемость кода.
        
    - Лушче разбить на несколько проверок.
        

---

## Визуальная схема (псевдографика)

```
AND (&&)            OR (||)
------------        ------------
left false  -----> false  left true  -----> true
left true   -----> check right  left false -----> check right
```

---

## Итог

- **Short-circuit** экономит вычислительные ресурсы и предотвращает ошибки (`nil`, out-of-bounds и др.)
    
- Применяется с `&&` и `||`
    
- Особенно полезно при **функциях с побочными эффектами**, проверках опционалов и сложных условиях
