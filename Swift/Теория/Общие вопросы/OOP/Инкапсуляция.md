## Инкапсуляция в программировании

### Определение
**Инкапсуляция** — это принцип объектно-ориентированного программирования, который объединяет данные и методы, работающие с этими данными, внутри одного объекта и скрывает внутреннее состояние от внешнего мира. Инкапсуляция защищает данные от некорректного использования и изменений, предоставляя контролируемый интерфейс для взаимодействия.

Простыми словами: инкапсуляция — это **сокрытие внутренностей объекта** и предоставление доступа только через публичные методы.

### Зачем это знать [[iOS]]-разработчику?
1.  **Защита данных:** Предотвращает некорректное изменение состояния объекта.
2.  **Снижение сложности:** Пользователь объекта видит только публичный [[API]], не отвлекаясь на детали реализации.
3.  **Гибкость изменения:** Внутреннюю реализацию можно менять, не затрагивая клиентский код.
4.  **Контроль доступа:** Модификаторы доступа ([[public]], [[private]], [[fileprivate]], [[internal]], [[open]]) — инструменты инкапсуляции в [[Swift]].
5.  **Соблюдение принципов [[SOLID]]:** В частности, принципа единственной ответственности.

---

### Уровни доступа в Swift

| Модификатор   | Доступ                                                 | Когда использовать                |
| ------------- | ------------------------------------------------------ | --------------------------------- |
| `open`        | Любой модуль + наследование вне модуля                 | Публичное API для переопределения |
| `public`      | Любой модуль (без наследования)                        | Публичное API                     |
| `internal`    | Внутри модуля (по умолчанию)                           | Внутренние компоненты             |
| `package`     | Внутри всего пакета (Swift 5.9+)                       | Библиотеки в рамках пакета        |
| `fileprivate` | Внутри файла                                           | Утилиты одного файла              |
| `private`     | Внутри объявления ([[class]]/[[struct]]/[[extension]]) | Максимальное сокрытие             |

### Примеры инкапсуляции

#### 1. **Сокрытие свойств через `private`**

```swift
class BankAccount {
    // Скрытое состояние
    private var balance: Double = 0
    
    // Публичный интерфейс
    func deposit(amount: Double) {
        guard amount > 0 else { return }
        balance += amount
        logTransaction(.deposit, amount: amount)
    }
    
    func withdraw(amount: Double) -> Bool {
        guard amount > 0, balance >= amount else { return false }
        balance -= amount
        logTransaction(.withdrawal, amount: amount)
        return true
    }
    
    func getBalance() -> Double {
        return balance
    }
    
    // Скрытая логика
    private enum TransactionType {
        case deposit, withdrawal
    }
    
    private func logTransaction(_ type: TransactionType, amount: Double) {
        print("\(type): \(amount)")
    }
}

let account = BankAccount()
account.deposit(amount: 100)
account.withdraw(amount: 30)
print(account.getBalance())  // 70.0

// ❌ Нет прямого доступа к балансу
// account.balance = 1000  // Ошибка: 'balance' is private
```

#### 2. **[[getter|Геттеры]] и [[setter|сеттеры]] с контролем**

```swift
class User {
    private var _age: Int = 0
    
    var age: Int {
        get {
            return _age
        }
        set {
            if newValue >= 0 && newValue <= 150 {
                _age = newValue
            } else {
                print("Некорректный возраст")
            }
        }
    }
}

let user = User()
user.age = 25          // OK
user.age = -5          // Некорректный возраст
user.age = 200         // Некорректный возраст
```

#### 3. **`private(set)` — доступ на чтение публичный, на запись приватный**

```swift
struct ShoppingCart {
    private(set) var items: [String] = []
    
    mutating func addItem(_ item: String) {
        items.append(item)
    }
    
    mutating func removeItem(at index: Int) {
        guard items.indices.contains(index) else { return }
        items.remove(at: index)
    }
}

var cart = ShoppingCart()
cart.addItem("Apple")
cart.addItem("Banana")

print(cart.items)  // ["Apple", "Banana"] — можно прочитать

// ❌ Нельзя напрямую изменить
// cart.items = []  // Ошибка: 'items' setter is inaccessible

cart.removeItem(at: 0)
print(cart.items)  // ["Banana"]
```

#### 4. **Сокрытие инициализатора**

```swift
class Singleton {
    static let shared = Singleton()
    
    private init() {
        // Приватный инициализатор — нельзя создать экземпляр снаружи
        print("Singleton created")
    }
    
    func doSomething() {
        print("Doing something")
    }
}

let instance = Singleton.shared  // OK
instance.doSomething()

// ❌ Нельзя создать второй экземпляр
// let another = Singleton()  // Ошибка: 'init()' is private
```

#### 5. **Инкапсуляция через протоколы**

```swift
// Публичный протокол (что)
protocol DataServiceProtocol {
    func fetchData() async -> [String]
}

// Скрытая реализация (как)
internal class ConcreteDataService: DataServiceProtocol {
    private let apiKey = "secret123"  // скрыто
    private let baseURL = "https://api.example.com"
    
    func fetchData() async -> [String] {
        // сложная логика с apiKey и baseURL
        return ["data1", "data2"]
    }
    
    private func logRequest() {
        // внутреннее логирование
    }
}

// Публичная фабрика
class ServiceFactory {
    static func makeDataService() -> DataServiceProtocol {
        return ConcreteDataService()
    }
}
```

#### 6. **Инкапсуляция в [[SwiftUI]]**

```swift
import SwiftUI

struct CounterView: View {
    // @State — инкапсулированное состояние, доступное только View
    @State private var count = 0
    
    var body: some View {
        VStack {
            Text("Count: \(count)")
            
            Button("Increment") {
                count += 1
            }
        }
    }
}

// Использование
struct ParentView: View {
    var body: some View {
        CounterView()  // родитель не имеет доступа к count
    }
}
```

### Инкапсуляция и [[наследование]]

```swift
class Animal {
    private var privateSecret = "Hidden"
    fileprivate var fileprivateSecret = "Visible in file"
    internal var internalProperty = "Visible in module"
    public var publicProperty = "Visible everywhere"
    open var openProperty = "Visible everywhere + overridable"
    
    final func cannotOverride() {
        print("This method cannot be overridden")
    }
}

class Dog: Animal {
    // ❌ Нельзя — private
    // override var privateSecret: String { ... }
    
    // ✅ Можно — fileprivate (в том же файле)
    override var fileprivateSecret: String {
        get { "Overridden" }
        set { }
    }
    
    // ✅ Можно
    override var openProperty: String {
        get { "Overridden" }
        set { }
    }
    
    // ❌ Нельзя — final
    // override func cannotOverride() { }
}
```

### Инкапсуляция в архитектурах

#### 1. **[[MVVM (Model-View-ViewModel) Architecture|MVVM]]**

```swift
// ViewModel инкапсулирует логику и состояние
class LoginViewModel: ObservableObject {
    @Published private(set) var isLoggedIn = false
    @Published private(set) var errorMessage: String?
    
    private let authService: AuthService
    
    init(authService: AuthService) {
        self.authService = authService
    }
    
    func login(email: String, password: String) {
        // Скрытая логика валидации и запроса
        guard validateEmail(email) else {
            errorMessage = "Invalid email"
            return
        }
        
        authService.login(email: email, password: password) { [weak self] result in
            DispatchQueue.main.async {
                switch result {
                case .success:
                    self?.isLoggedIn = true
                case .failure(let error):
                    self?.errorMessage = error.localizedDescription
                }
            }
        }
    }
    
    private func validateEmail(_ email: String) -> Bool {
        return email.contains("@")
    }
}
```

#### 2. **[[VIPER Architecture|VIPER]]**

```swift
// Interactor инкапсулирует бизнес-логику
protocol UserInteractorProtocol {
    func fetchUser(id: Int)
}

class UserInteractor: UserInteractorProtocol {
    private let userService: UserService
    private weak var presenter: UserPresenterProtocol?
    
    init(userService: UserService, presenter: UserPresenterProtocol) {
        self.userService = userService
        self.presenter = presenter
    }
    
    func fetchUser(id: Int) {
        userService.fetchUser(id: id) { [weak self] result in
            switch result {
            case .success(let user):
                self?.presenter?.didFetchUser(user)
            case .failure(let error):
                self?.presenter?.didFailWithError(error)
            }
        }
    }
}
```

### Антипаттерны инкапсуляции

#### 1. **Переэкспозиция (Over-exposure)**

```swift
// ❌ Плохо — внутренняя структура раскрыта
class BankAccount {
    var balance: Double = 0  // public setter — опасно!
    
    func deposit(amount: Double) {
        balance += amount
    }
}

// ✅ Хорошо — баланс защищён
class BankAccount {
    private(set) var balance: Double = 0
    
    func deposit(amount: Double) {
        balance += amount
    }
}
```

#### 2. **Утечка ссылок (Reference Leak)**

```swift
// ❌ Плохо — возврат внутренней коллекции
class DataStore {
    private var items: [String] = []
    
    func getItems() -> [String] {
        return items  // копия — безопасно
    }
}

// ✅ Хорошо — возврат копии или неизменяемого представления
class DataStore {
    private var items: [String] = []
    
    func getItems() -> [String] {
        return items  // Array — value type, копируется
    }
}
```

#### 3. **God Object (антипаттерн)**

```swift
// ❌ Плохо — класс делает всё, нет инкапсуляции
class GodClass {
    var userData: [String: Any] = [:]
    var networkManager = NetworkManager()
    var database = Database()
    var uiHelper = UIHelper()
    
    func doEverything() {
        // тысячи строк кода
    }
}

// ✅ Хорошо — разделение ответственности
class UserService { ... }
class NetworkManager { ... }
class DatabaseService { ... }
```

### Преимущества инкапсуляции

| Преимущество | Описание |
|--------------|----------|
| **Защита данных** | Предотвращает некорректное изменение состояния |
| **Снижение сложности** | Пользователь видит только публичный API |
| **Гибкость изменения** | Внутреннюю реализацию можно менять без последствий |
| **Контроль доступа** | Можно ограничить чтение/запись свойств |
| **Тестируемость** | Легче изолировать и тестировать компоненты |

---

### Короткое правило

> **Инкапсуляция** скрывает внутреннее состояние и детали реализации.  
> Используйте `private` по умолчанию, открывайте доступ только когда нужно.  
> `private(set)` для read-only свойств — ваш друг.

---

### Итог

**Инкапсуляция** в Swift:

1.  **Реализуется через** модификаторы доступа (`private`, `fileprivate`, `internal`, `public`, `open`)
2.  **Позволяет** скрывать состояние и детали реализации
3.  **Предотвращает** некорректное использование объекта
4.  **Даёт гибкость** для изменения внутренней реализации
5.  **Является** одним из трёх столпов ООП (наряду с наследованием и полиморфизмом)

Понимание инкапсуляции — ключ к созданию надёжных, поддерживаемых и безопасных систем.
