Dependency Manager (менеджер зависимостей) в контексте [[Swift]] - это инструмент, предназначенный для управления внешними зависимостями в проектах, написанных на языке [[Swift]]. Dependency Manager облегчает процесс добавления, управления и обновления библиотек и фреймворков, необходимых для разработки приложений.

В Swift два основных менеджера зависимостей - это [[CocoaPods]] и [[Carthage]]. Вот краткое описание каждого из них:

1. **[[CocoaPods]]:**
    
    - **Особенности:**
        
        - Подходит для проектов, написанных на [[Swift]] и [[Objective-C]].
        - Использует файл подключения `Podfile` для определения зависимостей.
        - Позволяет быстро интегрировать библиотеки и фреймворки.
        - Автоматически разрешает зависимости и управляет версиями.
    - **Пример использования [[CocoaPods]]:**
```swift
# Ваш Podfile
platform :ios, '9.0'
use_frameworks!

target 'YourApp' do
  pod 'Alamofire', '~> 5.0'
end

```
**Carthage:**

- **Особенности:**
    
    - Подходит для проектов, написанных на [[Swift]].
    - Сосредоточен на сборке библиотек, не изменяя структуру проекта.
    - Не создает единый файл проекта, как [[CocoaPods]].
    - Позволяет более гибкое управление зависимостями.
- **Пример использования [[Carthage]]:**
```swift
# Ваш Cartfile
github "Alamofire/Alamofire" ~> 5.0

```
**Swift Package Manager ([[SPM]]):**

- **Особенности:**
    
    - Встроен в [[Swift]] и предоставляет нативное управление зависимостями.
    - Поддерживает только проекты, написанные на [[Swift]].
    - Имеет ограниченную поддержку платформы (пока не поддерживает [[iOS]] и macOS).
- **Пример использования [[SPM]]:**
```swift
// Ваш Package.swift
// swift-tools-version:5.5
import PackageDescription

let package = Package(
    name: "YourApp",
    dependencies: [
        .package(url: "https://github.com/Alamofire/Alamofire.git", from: "5.0.0"),
    ],
    targets: [
        .target(name: "YourApp", dependencies: ["Alamofire"]),
    ]
)

```
