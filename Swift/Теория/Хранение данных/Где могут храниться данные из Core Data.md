#save_data #swift
Данные в [[Core Data]] могут храниться в различных местах, в зависимости от конфигурации вашего приложения:

1. **[[SQLite]]**: По умолчанию CoreData использует [[SQLite]] в качестве хранилища данных. [[SQLite]] - это легковесная реляционная база данных, которая хранится в виде файла на диске. Это наиболее распространенный и рекомендуемый способ хранения данных [[Core Data]].

2. **In-Memory Store**: [[Core Data]] также может хранить данные только в оперативной памяти (In-Memory Store). Этот вариант хранения данных полезен в ситуациях, когда нужно обеспечить максимальную производительность за счет потери данных после закрытия приложения.

3. **Binary Store**: Данные могут храниться в бинарном формате (Binary Store), который также сохраняется в виде файла на диске. Этот формат полезен для хранения больших блобов данных, таких как изображения или аудиофайлы.

4. **XML Store**: [[Core Data]] поддерживает хранение данных в формате [[XML]] (XML Store), что может быть полезно для совместимости с другими системами или обмена данными.

5. **Custom Store**: [[Core Data]] позволяет создавать собственные типы хранилищ данных, что дает большую гибкость в выборе места хранения и формата данных.

В зависимости от требований вашего приложения, вы можете выбрать наиболее подходящий тип хранилища для вашей модели данных и сценария использования.
