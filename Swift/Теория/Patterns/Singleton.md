В Swift реализация `Singleton` обычно основывается на использовании статического свойства и приватного инициализатора. Вот пример реализации `Singleton` в [[Swift]]:
```swift
class MySingleton {
    // Статическое свойство, содержащее единственный экземпляр класса
    static let shared = MySingleton()
    
    // Приватный конструктор, чтобы предотвратить создание экземпляров вне класса
    private init() {
        // Инициализация
    }
    
    // Дополнительные методы и свойства класса
    func someMethod() {
        print("Singleton method called")
    }
}

```
В этом примере:

- `shared` - это статическое свойство, которое содержит единственный экземпляр класса. Оно создается и инициализируется статически при первом обращении к этому свойству.
    
- `private init()` - это приватный конструктор, который предотвращает создание экземпляров класса вне его самого.
    

Этот подход обеспечивает гарантию, что в приложении существует только один экземпляр класса `MySingleton`, и он может быть получен через `MySingleton.shared`.

Пример использования:
```swift
// Получение экземпляра Singleton
let singletonInstance = MySingleton.shared

// Вызов метода Singleton
singletonInstance.someMethod()

```
Такая реализация `Singleton` является потокобезопасной в [[Swift]], так как статические свойства и инициализаторы инициализируются только один раз в многопоточной среде.
