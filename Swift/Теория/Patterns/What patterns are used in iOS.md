В [[iOS]]-разработке часто используются различные шаблоны проектирования (паттерны), чтобы создавать структурированный и удобный для сопровождения код. Ниже представлены некоторые из наиболее распространенных шаблонов проектирования, которые применяются в [[iOS]]:

1. **[[MVC (Model-View-Controller) Architecture]] (Model-View-Controller):**
    
    - Это один из базовых паттернов iOS. Модель (`Model`) представляет данные и бизнес-логику, Представление (`View`) отображает данные пользователю, а Контроллер (`Controller`) управляет взаимодействием между Моделью и Представлением.
2. **[[Delegate]](Делегирование):**
    
    - Шаблон делегирования используется для передачи событий и данных между объектами. Примеры включают [[UITableViewDelegate]] и [[UITextFieldDelegate]].
3. **[[Singleton]] (Одиночка):**
    
    - Одиночка гарантирует, что класс имеет только один экземпляр и предоставляет глобальную точку доступа к этому экземпляру. Часто используется для управления общими ресурсами.
4. **[[Observer]] (Наблюдатель):**
    
    - Паттерн Observer позволяет объектам подписываться и реагировать на изменения в других объектах. Например, `NotificationCenter` в [[iOS]] использует паттерн [[Observer]] для передачи уведомлений о событиях.
5. **[[Builder]] (Строитель):**
    
    - Строитель используется для пошагового построения сложных объектов. Например, при создании сложных объектов [[NSAttributedString]] или при построении объекта конфигурации.
6. **[[Factory Method]] (Фабричный метод):**
    
    - Фабричный метод предоставляет интерфейс для создания экземпляров класса, но позволяет подклассам изменять тип создаваемых экземпляров. Пример - [[UIControl]] в [[iOS]].
7. **[[Strategy]] (Стратегия):**
    
    - Паттерн Стратегия определяет семейство алгоритмов, инкапсулирует каждый из них и делает их взаимозаменяемыми. Пример - использование стратегии для смены поведения объекта.
8. **[[Decorator]] (Декоратор):**
    
    - Декоратор предоставляет гибкую альтернативу наследованию для расширения функциональности класса. Например, `NSFileManager` в [[iOS]] использует декоратор для добавления дополнительной функциональности.
9. **[[Command]] (Команда):**
    
    - Команда инкапсулирует запрос как объект, что позволяет параметризовать клиентов с различными запросами. Например, использование паттерна команды в [[UIKit]] для обработки пользовательских действий.

Эти паттерны помогают улучшить структуру кода, сделать его более поддерживаемым и гибким, а также обеспечить лучшую расширяемость при разработке [[iOS]]-приложений.
