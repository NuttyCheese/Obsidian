## 1. Что такое `willSet`

**`willSet`** — это **наблюдатель свойства**, который вызывается **непосредственно перед изменением значения свойства**.

- Используется с **[[var]]**, так как [[let]] нельзя менять
    
- Позволяет **реагировать на изменения** и выполнять дополнительные действия до присвоения нового значения
    

> Проще говоря: `willSet` = «что будет происходить с новым значением перед его установкой».

---

## 2. Основные термины

| Термин                | Описание                                                                 |
| --------------------- | ------------------------------------------------------------------------ |
| **Property Observer** | Наблюдатель свойства, позволяющий отслеживать изменения значения         |
| **willSet**           | Вызывается **перед присвоением нового значения**                         |
| **[[didSet]]**        | Вызывается **после присвоения нового значения**                          |
| **newValue**          | Неявная переменная внутри willSet, содержит новое присваиваемое значение |
| **oldValue**          | Неявная переменная внутри didSet, содержит предыдущее значение           |

---

## 3. Основной синтаксис

```swift
var property: Type {
    willSet {
        print("Старое значение: \(property), новое значение: \(newValue)")
    }
}
```

- `newValue` доступно **автоматически**, можно использовать для проверки или логирования
    

---

## 4. Примеры от простого к сложному

### Пример 1. Логирование нового значения

```swift
var score: Int = 0 {
    willSet {
        print("Скоро изменится с \(score) на \(newValue)")
    }
}

score = 10
// Вывод: Скоро изменится с 0 на 10
```

- Показываем старое и новое значение
    

---

### Пример 2. Проверка нового значения

```swift
var age: Int = 18 {
    willSet {
        if newValue < 0 {
            print("Возраст не может быть отрицательным!")
        }
    }
}

age = -5
// Вывод: Возраст не может быть отрицательным!
```

- Можно проверять корректность присваиваемого значения
    

---

### Пример 3. Ограничение диапазона значений

```swift
var temperature: Int = 20 {
    willSet {
        if newValue > 100 {
            print("Температура слишком высокая")
        }
    }
}

temperature = 120
// Вывод: Температура слишком высокая
```

- Контролируем диапазон значений **до их установки**
    

---

### Пример 4. Сложное взаимодействие с другим свойством

```swift
var width: Int = 50 {
    willSet {
        print("Ширина меняется с \(width) на \(newValue)")
        height = newValue / 2
    }
}

var height: Int = 25

width = 80
// width -> 80, height -> 40
```

- Изменение одного свойства влияет на другое
    

---

### Пример 5. Использование willSet в классе с UI

```swift
class ProgressView {
    var progress: Double = 0 {
        willSet {
            print("Обновляем индикатор с \(progress) на \(newValue)")
            progressBarWidth = newValue * 200 // ширина полосы прогресса
        }
    }
    
    var progressBarWidth: Double = 0
}

let view = ProgressView()
view.progress = 0.5
// Обновляем индикатор с 0 на 0.5
// progressBarWidth = 100
```

- Реакция на изменение свойства до его установки для обновления UI
    

---

## 5. Особенности willSet

1. Работает **только с var**, нельзя с let
    
2. Вызывается **перед присвоением нового значения**
    
3. `newValue` — доступное значение, которое будет присвоено
    
4. Может использоваться для:
    
    - Логирования
        
    - Валидации
        
    - Автоматического обновления связанных свойств
        
5. Отличие от **didSet**: didSet вызывается после присвоения
    

---

## 6. Итог

- **willSet** = наблюдатель свойства, который срабатывает **до изменения**
    
- Позволяет:
    
    - Контролировать новое значение
        
    - Логировать изменения
        
    - Обновлять другие свойства или UI
        
- Совместно с **didSet** можно полностью управлять изменением значения
