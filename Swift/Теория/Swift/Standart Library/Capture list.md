## 1. Что такое Capture List

**Capture List** — это **список переменных, которые замыкание захватывает из внешнего контекста**.

- Позволяет **управлять сильными/слабыми ссылками**, чтобы избежать **[[retain cycle]]**
    
- Используется внутри **closure**, особенно при работе с классами и [[self]]
    
- Определяется в квадратных скобках `[]` перед списком параметров замыкания
    

> Проще говоря: Capture List = «список переменных, которые замыкание берёт снаружи, с возможностью указать, как именно их хранить».

---

## 2. Основные термины

| Термин                | Описание                                                                         |
| --------------------- | -------------------------------------------------------------------------------- |
| **[[closure]]**       | Анонимная функция, которая может захватывать переменные из внешнего контекста    |
| **Capture**           | Захват переменной для использования внутри замыкания                             |
| **Strong reference**  | Сильная ссылка, удерживает объект в памяти                                       |
| **Weak reference**    | Слабая ссылка, может стать nil                                                   |
| **Unowned reference** | Слабая ссылка, предполагается что объект существует всегда, но не удерживает его |
| **Retain cycle**      | Замкнутый цикл сильных ссылок между объектами и замыканиями                      |

---

## 3. Основной синтаксис

```swift
class MyClass {
    var value = 10
    func doSomething() {
        let closure = { [weak self] in
            print(self?.value ?? 0)
        }
        closure()
    }
}
```

- [[Weak self]] → захватываем self как слабую ссылку
    
- `[unowned self]` → захватываем self как неблокирующую ссылку, которая предполагает существование объекта
    

---

## 4. Примеры от простого к сложному

### Пример 1. Простое замыкание с захватом

```swift
var x = 10
let closure = { [x] in
    print(x)
}
x = 20
closure() // 10
```

- `[x]` → замыкание захватывает **текущее значение x**, а не ссылку
    

---

### Пример 2. Weak self для классов

```swift
class Person {
    var name = "Alice"
    func greet() {
        DispatchQueue.main.async { [weak self] in
            print("Hello, \(self?.name ?? "Guest")")
        }
    }
}
```

- Предотвращает **retain cycle**, если замыкание удерживает self
    

---

### Пример 3. Unowned self

```swift
class Manager {
    var id = 123
    func printID() {
        let closure = { [unowned self] in
            print("Manager ID: \(self.id)")
        }
        closure()
    }
}
```

- `unowned self` безопасно, если объект гарантированно существует в момент вызова
    

---

### Пример 4. Захват нескольких переменных

```swift
var a = 5, b = 10
let closure = { [a, b] in
    print("a = \(a), b = \(b)")
}
a = 50
b = 100
closure() // a = 5, b = 10
```

- Можно захватывать несколько значений одновременно
    

---

### Пример 5. Комбинация weak и value

```swift
class MyClass {
    var count = 0
    func run() {
        var x = 10
        let closure = { [weak self, x] in
            self?.count += 1
            print("x = \(x), count = \(self?.count ?? 0)")
        }
        closure()
    }
}
```

- Захватывает **self слабой ссылкой** и **x по значению**
    

---

## 5. Особенности Capture List

1. Позволяет **предотвратить retain cycles** в замыканиях
    
2. Можно захватывать **слабые ([[weak]]) или неблокирующие ([[unowned]]) ссылки**
    
3. Можно захватывать **значения по копии**
    
4. Захват происходит **однократно при создании замыкания**
    
5. Синтаксис: `[capture1, capture2, ...]` перед параметрами замыкания
    

---

## 6. Итог

- **Capture List** = механизм контроля захвата переменных в замыкании
    
- Используется для:
    
    - Предотвращения retain cycles (`weak` / `unowned`)
        
    - Захвата значений по копии (`[x, y]`)
        
- Позволяет писать **безопасный и эффективный код с замыканиями**
