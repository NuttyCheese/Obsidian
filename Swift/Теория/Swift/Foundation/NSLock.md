## 1. Что такое NSLock

**`NSLock`** — это класс, который позволяет **защищать критические участки кода от одновременного доступа нескольких потоков**.

- Используется для **избежания гонок данных** ([[data races]])
    
- Блокирует поток, который пытается войти в защищённый участок кода, если другой поток уже там
    
- После выполнения критической секции нужно **разблокировать lock**
    

> Проще говоря: NSLock = «замок для кода», который может держать один поток за раз.

---

## 2. Основные свойства и методы

|Метод / Свойство|Описание|
|---|---|
|`lock()`|Захватывает замок. Если замок уже занят, поток блокируется до освобождения|
|`unlock()`|Освобождает замок|
|`try()`|Попытка захватить замок без блокировки. Возвращает `true`, если получилось|
|`name`|Имя замка, для отладки|

---

## 3. Схема работы

1. Поток A вызывает `lock()` → замок захвачен
    
2. Поток B вызывает `lock()` → блокируется до освобождения замка
    
3. Поток A выполняет критическую секцию → вызывает `unlock()`
    
4. Поток B получает доступ и выполняет код
    

---

## 4. Примеры от простого к сложному

### Пример 1. Простейшее использование

```swift
let lock = NSLock()
var counter = 0

lock.lock()
counter += 1
lock.unlock()
```

- Гарантирует, что изменение `counter` безопасно в многопоточной среде
    

---

### Пример 2. Использование try

```swift
if lock.try() {
    counter += 1
    lock.unlock()
} else {
    print("Не удалось захватить lock")
}
```

- Поток не блокируется, если замок уже занят
    
- Полезно для неблокирующих операций
    

---

### Пример 3. NSLock с многопоточными задачами

```swift
let lock = NSLock()
var sharedArray: [Int] = []

DispatchQueue.concurrentPerform(iterations: 10) { i in
    lock.lock()
    sharedArray.append(i)
    lock.unlock()
}

print(sharedArray)
```

- Все потоки безопасно добавляют элементы в массив
    
- Без lock возможны гонки данных и крэш
    

---

### Пример 4. NSLock + defer

```swift
lock.lock()
defer { lock.unlock() }

counter += 1
counter += 2
```

- [[defer]] гарантирует, что замок будет освобождён даже если внутри блока произойдёт ошибка или return
    
- Очень удобно для безопасного захвата и освобождения
    

---

### Пример 5. NSLock для защиты сложной структуры данных

```swift
class ThreadSafeSet<T: Hashable> {
    private var set = Set<T>()
    private let lock = NSLock()
    
    func insert(_ value: T) {
        lock.lock()
        defer { lock.unlock() }
        set.insert(value)
    }
    
    func contains(_ value: T) -> Bool {
        lock.lock()
        defer { lock.unlock() }
        return set.contains(value)
    }
}
```

- Позволяет создавать **потокобезопасные коллекции**
    

---

## 5. Особенности NSLock

1. **Простейший примитив синхронизации**
    
2. **Блокирует поток до освобождения замка** → может вызвать [[deadlock]], если забыть unlock
    
3. **Можно использовать [[try]]() для неблокирующего доступа**
    
4. **Хорошо подходит для защиты небольших критических секций**
    
5. **Альтернатива GCD serial queue для синхронизации**
    

---

## 6. Итог

- `NSLock` = объект, который **разрешает только одному потоку выполнять критический код**
    
- Методы: `lock()`, `unlock()`, `try()`
    
- Обычно используется с `defer` для безопасного освобождения
    
- Полезен для защиты простых переменных, массивов, словарей или любых shared ресурсов
