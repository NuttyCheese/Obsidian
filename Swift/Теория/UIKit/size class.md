## Какие они бывают. Для чего их используют
Size Classes (Размерные классы) в [[iOS]] представляют собой механизм, предназначенный для управления размещением и поведением интерфейса в зависимости от размера экрана и ориентации устройства. Size Classes были введены в iOS 8 и позволяют разработчикам легко адаптировать интерфейс к различным размерам устройств и ориентациям.

**Основные аспекты Size Classes:**

1. **Compact и Regular:**
   - Size Classes определяются двумя основными значениями: Compact и Regular. Эти значения применяются к ширине и высоте экрана.

2. **Комбинации Size Classes:**
   - Всего существует четыре комбинации Size Classes:
     - Compact Width / Compact Height (или wC-hC)
     - Compact Width / Regular Height (или wC-hR)
     - Regular Width / Compact Height (или wR-hC)
     - Regular Width / Regular Height (или wR-hR)

3. **Использование в Interface Builder:**
   - Size Classes интегрированы в [[Interface Builder]], что позволяет создавать различные версии макетов для разных комбинаций Size Classes.

4. **Adaptive Layout:**
   - Size Classes являются частью концепции Adaptive Layout, которая позволяет создавать универсальные интерфейсы, которые могут автоматически подстраиваться под различные размеры экранов и устройств.

**Пример использования Size Classes:**

1. **Распределение элементов:**
   - Например, можно задать различное распределение элементов на экране в зависимости от размера устройства. На iPhone (compact width) можно показывать элементы в столбце, а на iPad (regular width) - в двух столбцах.

2. **Скрытие/Отображение элементов:**
   - Можно скрывать или отображать определенные элементы в зависимости от Size Classes. Например, скрыть боковую панель на узких экранах.

3. **Адаптация для различных ориентаций:**
   - Size Classes также используются для адаптации интерфейса к различным ориентациям устройства (портрет или ландшафт).

4. **Управление констрейнтами:**
   - Можно создавать и редактировать констрейнты в зависимости от Size Classes, чтобы обеспечить оптимальное распределение элементов.

**Примечание:**
- Size Classes упрощают процесс создания адаптивных интерфейсов, позволяя разработчикам более гибко управлять отображением и поведением приложения на различных устройствах и экранах.
