**hitTest** и **Responder Chain** — это два **последовательных**, но **совершенно разных** механизма в [[UIKit]], которые вместе решают задачу:

«Куда пользователь ткнул → кто именно это обработает».

Они работают **строго по очереди** и выполняют разные роли.

### Короткое сравнение (2026 взгляд)

| Аспект                   | [[hitTest]](_:with:)                                           | [[Responder Chain]]                                              | Кто первый?               |
| ------------------------ | -------------------------------------------------------------- | ---------------------------------------------------------------- | ------------------------- |
| **Что делает**           | Геометрический поиск: «какая вью под пальцем?»                 | Передача события: «кто из responder’ов может обработать?»        | hitTest → Responder Chain |
| **Возвращает**           | [[UIView]]? (самую вложенную подходящую вью)                   | UIResponder? (первый, кто отреагирует)                           | —                         |
| **Когда вызывается**     | При каждом касании/клике/перетаскивании                        | **После** успешного hitTest                                      | —                         |
| **Зависит от**           | [[frame]]/[[bounds]], isUserInteractionEnabled, point(inside:) | Иерархии responder (nextResponder)                               | —                         |
| **Можно переопределить** | Да (в UIView)                                                  | Да (nextResponder, touchesBegan и т.д.)                          | —                         |
| **Цель**                 | Найти первую вью-кандидата                                     | Доставить событие до объекта, который его обработает             | —                         |
| **Пример**               | Тап по кнопке внутри scroll view → hitTest вернёт кнопку       | Кнопка не обработала → событие идёт в scroll view → в контроллер | —                         |

### Последовательность работы (самая важная схема 2026)

```
Пользователь ткнул пальцем
          ↓
iOS → UIWindow
          ↓
window.hitTest(point, with: event)
          ↓
hitTest рекурсивно идёт по иерархии view сверху вниз
          ↓
Нашёл самую вложенную вью под пальцем → вернул её
          ↓
Эта вью становится first responder (если может)
          ↓
Responder Chain начинается:
firstResponder → touchesBegan / gesture recognizers
          ↓
Если не обработано → nextResponder (супервиу, контроллер, окно, приложение)
          ↓
Если никто не обработал → событие потеряно
```

### Ключевые отличия в одном предложении

- **hitTest** отвечает на вопрос **«ГДЕ произошло касание?»** — чистая геометрия  
- **Responder Chain** отвечает на вопрос **«КТО это обработает?»** — архитектурная цепочка ответственности

### Самые частые сценарии, где их переопределяют вместе (2026)

| Ситуация                                  | Что переопределяют в hitTest                       | Что в Responder Chain                      |
| ----------------------------------------- | -------------------------------------------------- | ------------------------------------------ |
| Кнопка с увеличенной зоной касания        | point(inside:) → расширяем [[bounds]]              | — (оставляем touchesBegan как есть)        |
| Пропуск касаний сквозь overlay            | hitTest → [[return]] [[nil]] если не в нужной зоне | —                                          |
| Перехват всех касаний в контроллере       | —                                                  | touchesBegan в контроллере → перехватываем |
| Кастомный drag внутри scroll view         | hitTest → возвращаем нужную вью даже вне границ    | touchesMoved в той вью → двигаем объект    |
| Отключение касаний в определённой области | point(inside:) → return false для зоны             | —                                          |

### Короткий пример: кнопка с увеличенной зоной касания

```swift
final class BigHitButton: UIButton {
    
    private let extraTouchArea: CGFloat = 20
    
    override func point(inside point: CGPoint, with event: UIEvent?) -> Bool {
        let expanded = bounds.insetBy(dx: -extraTouchArea, dy: -extraTouchArea)
        return expanded.contains(point)
    }
}
```

Теперь даже если палец промахнулся на 20 pt — кнопка всё равно сработает.

### Короткий девиз 2026

> **hitTest** — «географ», ищет, **где** лежит вью под пальцем.  
> **Responder Chain** — «почтальон», доставляет событие **кому** его обработать.  
> hitTest находит кандидата → Responder Chain решает, кто возьмёт событие на себя.
