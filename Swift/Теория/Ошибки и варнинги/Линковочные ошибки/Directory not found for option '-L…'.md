[[Xcode]] не может найти каталог, который указан в **Library Search Paths** (`-L` флаг указывает путь к директории с библиотеками).  
Чаще всего это предупреждение возникает, когда:

1. Каталог удалён или переименован.
    
2. Неправильно указан путь в Build Settings → `Library Search Paths`.
    
3. Используются устаревшие пути от [[CocoaPods]], [[Carthage]] или [[SPM]].
    
4. В проекте остались ссылки на несуществующие библиотеки.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: старый путь к библиотеке**

- В Build Settings → `Library Search Paths` добавлен путь:
    

```
$(PROJECT_DIR)/Pods/SomeLibrary/lib
```

- Библиотека была удалена из Pods → при сборке появится предупреждение:
    

```
Directory not found for option '-L/Users/username/Project/Pods/SomeLibrary/lib'
```

---

**Пример 2: кастомный фреймворк**

- Добавлен путь к локальному фреймворку:
    

```
$(PROJECT_DIR)/Frameworks/CustomFramework
```

- Каталог удалён или перемещён → Xcode выводит то же предупреждение.
    

---

### Как исправить

#### 1️⃣ Проверка существования каталога

- Убедитесь, что путь в `Library Search Paths` реально существует на диске.
    
- Если библиотека была удалена, уберите путь из Build Settings.
    

#### 2️⃣ Проверка переменных проекта

- Если используется `$(PROJECT_DIR)` или `$(SRCROOT)`, убедитесь, что они корректно указывают на проект.
    
- Для нескольких таргетов иногда нужно указать путь конкретно для каждого таргета.
    

#### 3️⃣ Проверка зависимостей

- Для CocoaPods: выполните `pod install` и убедитесь, что Pods директории существуют.
    
- Для Carthage: выполните `carthage update --platform iOS`.
    
- Для Swift Package Manager: File → Swift Packages → Update to Latest Package Versions.
    

#### 4️⃣ Удаление неиспользуемых путей

- Target → Build Settings → Library Search Paths → удалить устаревшие пути.
    

---

### Пример исправления

**Старый путь:**

```
$(PROJECT_DIR)/Pods/SomeLibrary/lib
```

**Исправление:**

1. Проверяем Pods: `SomeLibrary` удалена → путь убираем.
    
2. Library Search Paths после исправления:
    

```
$(inherited)
```

3. После этого Xcode перестаёт выдавать предупреждение при сборке.
