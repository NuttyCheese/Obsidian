Компилятор [[Swift]] сообщает, что **он ожидал закрывающую фигурную скобку `}`**, но её не нашёл.

- Ошибка возникает, когда **структуры кода (классы, функции, условия, циклы и т.д.) не закрыты корректно**.
    
- Компилятор пытается понять, где должна завершаться блоковая конструкция, и если не находит, выдаёт эту ошибку.
    
- Часто встречается при:
    
    1. Забытой закрывающей скобке.
        
    2. Случайном удалении скобки.
        
    3. Несовпадении открывающих и закрывающих скобок (`{}`) в сложных конструкциях.
        

---

### Примеры кода/сценариев возникновения

**Пример 1: Забытая закрывающая скобка функции**

```swift
func greet() {
    print("Hello")
// ❌ Expected '}' in function
```

---

**Пример 2: Несовпадение скобок в классе**

```swift
class Person {
    let name: String
    init(name: String) {
        self.name = name
// ❌ Expected '}' in class
```

---

**Пример 3: Цикл без закрывающей скобки**

```swift
for i in 1...5 {
    print(i)
// ❌ Expected '}' in for loop
```

---

### Как исправить

#### 1️⃣ Проверить и добавить закрывающую скобку

```swift
func greet() {
    print("Hello")
} // ✅ исправлено
```

---

#### 2️⃣ Использовать выравнивание кода и автоформатирование

- В [[Xcode]]: **Editor → Structure → Re-Indent** (⌃I)
    
- Это помогает визуально увидеть несоответствие скобок.
    

---

#### 3️⃣ Проверять вложенные конструкции

```swift
class Person {
    let name: String
    init(name: String) {
        self.name = name
    }
} // ✅ исправлено
```

---

### Резюме

- Ошибка возникает, когда **компилятор ожидает закрывающую скобку `}`**, но её нет.
    
- Исправляется через:
    
    1. Добавление недостающей скобки.
        
    2. Проверку вложенности блоков кода.
        
    3. Использование автоформатирования в Xcode.
        
- Позволяет компилятору правильно интерпретировать структуру кода и успешно собрать проект.
