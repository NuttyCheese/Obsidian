#crash #fata_error #xcode #swift
### Что это значит

Компилятор [[Swift]] сообщает, что **строковый литерал ([[String]]) не был корректно закрыт кавычками** (`"`).

- Ошибка возникает, когда:
    
    1. Отсутствует закрывающая кавычка `"`.
        
    2. Строка случайно разделена на несколько строк без использования `"""` для многострочных литералов.
        
    3. Экранирование символов нарушено.
        
- Компилятор ожидает **полностью закрытую строку**, иначе он не может корректно парсить остальной код.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Пропущена закрывающая кавычка**

```swift
let name = "Alice
// ❌ Unterminated string literal
```

---

**Пример 2: Строка случайно разделена на несколько строк**

```swift
let greeting = "Hello,
world"
// ❌ Unterminated string literal
```

---

**Пример 3: Нарушено экранирование кавычек**

```swift
let quote = "He said "Hello""
// ❌ Unterminated string literal
```

- Внутренние кавычки нужно экранировать: `\"`.
    

---

### Как исправить

#### 1️⃣ Добавить закрывающую кавычку

```swift
let name = "Alice" // ✅ исправлено
```

---

#### 2️⃣ Для многострочных строк использовать `"""`

```swift
let greeting = """
Hello,
world
"""
```

---

#### 3️⃣ Экранировать внутренние кавычки

```swift
let quote = "He said \"Hello\"" // ✅ исправлено
```

---

### Резюме

- **Unterminated string literal** возникает, когда строка не закрыта кавычками.
    
- Исправляется через:
    
    1. Добавление закрывающей кавычки.
        
    2. Использование многострочных литералов `"""` при необходимости.
        
    3. Экранирование внутренних кавычек через `\"`.
        
- Позволяет компилятору корректно распознавать строковые литералы и собирать проект.
