#crash #fata_error #xcode #swift
### Что это значит

[[Xcode]] сообщает, что **одна из фаз сборки с выполнением пользовательского скрипта завершилась с ошибкой**.

- В Xcode можно добавлять **Run Script Build Phases** для выполнения скриптов (например, генерация кода, обработка ресурсов, проверка версий).
    
- Ошибка `PhaseScriptExecution failed` означает, что **скрипт вернул ненулевой код возврата** → сборка остановлена.
    
- Причины могут быть связаны как с самим скриптом, так и с окружением сборки.
    

---

### Причины возникновения

1. **Ошибка в самом скрипте**
    
    - Синтаксическая ошибка bash/zsh, неправильный путь к файлу, команды, которые не выполняются.
        
2. **Неправильные пути или переменные окружения**
    
    - `$SRCROOT`, `$TARGET_NAME` и другие переменные могут быть некорректными.
        
3. **Отсутствующие зависимости или инструменты**
    
    - Например, вызывается `swiftlint`, `xcodegen`, `fastlane` без установки.
        
4. **Проблемы с правами доступа**
    
    - Скрипт не имеет права на выполнение: `chmod +x script.sh`.
        
5. **Конфликты в настройках сборки**
    
    - Несовпадение платформ, архитектур или target, особенно при копировании файлов.
        

---

### Примеры

**Пример 1: Синтаксическая ошибка в bash**

```bash
echo "Hello World
```

- Отсутствует закрывающая кавычка → скрипт падает → PhaseScriptExecution failed.
    

**Пример 2: Отсутствие команды**

```bash
swiftlint
```

- SwiftLint не установлен → скрипт завершился с кодом != 0.
    

**Пример 3: Ошибка при копировании ресурсов**

```bash
cp "$SRCROOT/Resources" "$BUILT_PRODUCTS_DIR/$PRODUCT_NAME"
```

- Путь не существует → код возврата 1 → ошибка.
    

---

### Как исправить

#### 1️⃣ Проверить скрипт на ошибки

- Выполнить скрипт вручную в терминале.
    
- Исправить синтаксис и пути.
    

#### 2️⃣ Проверить переменные окружения

- Использовать `echo $SRCROOT` и другие переменные внутри скрипта для диагностики.
    

#### 3️⃣ Установить необходимые инструменты

```bash
brew install swiftlint
```

- Или убедиться, что все инструменты доступны в PATH.
    

#### 4️⃣ Дать права на выполнение скрипта

```bash
chmod +x script.sh
```

#### 5️⃣ Настроить скрипт в Xcode

- Target → Build Phases → Run Script → убедиться, что скрипт указан корректно.
    
- Можно добавить `set -e` в начале скрипта, чтобы сразу останавливать выполнение при ошибке.
    

---

### Резюме

- Ошибка возникает, когда **Run Script Build Phase завершился с ненулевым кодом возврата**.
    
- Исправляется через:
    
    1. Проверку и исправление скрипта.
        
    2. Проверку переменных окружения.
        
    3. Установку необходимых инструментов.
        
    4. Настройку прав доступа.
        
- Позволяет корректно выполнять пользовательские скрипты и успешно завершать сборку проекта.
