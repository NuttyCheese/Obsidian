[[Xcode]] предупреждает, что ваш **Run Script Build Phase** будет выполняться **при каждой сборке**, потому что в настройках **не указаны output-файлы**.

- Run Script — это фаза сборки, где можно запускать кастомные скрипты (например, генерацию версий, копирование ресурсов, проверку lint).
    
- Xcode оптимизирует сборку, пропуская скрипты, если их **outputs** не изменились.
    
- Если outputs не указаны → Xcode не знает, изменились ли результаты скрипта → выполняет его каждый раз.
    

Причины появления:

1. Скрипт не создаёт файл, который можно указать как output.
    
2. Output файлы не прописаны в Build Phase → Xcode считает, что нужно выполнять скрипт каждый раз.
    
3. Скрипт используется для генерации версий, ресурсов, но outputs не нужны → предупреждение появляется «для информации».
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Скрипт обновления версии приложения**

```bash
#!/bin/sh
echo "Incrementing build number..."
/usr/libexec/PlistBuddy -c "Set :CFBundleVersion $(( $CURRENT_BUILD_NUMBER + 1 ))" "${PROJECT_DIR}/${INFOPLIST_FILE}"
```

- В Build Phase → Output Files пусто → предупреждение:
    

```
Run script build phase 'Increment Build Number' will be run during every build because it does not specify any outputs
```

---

**Пример 2: Скрипт проверки кода (lint)**

```bash
#!/bin/sh
swiftlint
```

- Скрипт запускается каждый раз, даже если код не изменился → предупреждение появляется.
    

---

### Как исправить

#### 1️⃣ Указать Output Files

- Если скрипт генерирует файл, можно добавить путь к нему в **Output Files**:
    

```
$(SRCROOT)/Generated/Version.swift
```

- Тогда Xcode будет запускать скрипт только при изменении inputs или отсутствия output.
    

#### 2️⃣ Игнорировать предупреждение

- Если скрипт не создаёт файл, а нужен при каждой сборке (например, lint, проверка версий) → предупреждение безопасно игнорировать.
    

#### 3️⃣ Оптимизация через timestamp

- Можно записывать дату/время выполнения скрипта в файл и указать его как output → скрипт запускается только при изменении.
    

```bash
#!/bin/sh
date > "${PROJECT_DIR}/Generated/last_build.txt"
```

- Output Files → `$(SRCROOT)/Generated/last_build.txt`
    

---

### Пример исправления

**Было:**

- Скрипт увеличивает build number, Output Files пусто → предупреждение.
    

**Исправлено:**

- Добавили файл Output:
    

```
$(SRCROOT)/Generated/BuildNumber.txt
```

- Скрипт обновляет этот файл, Xcode выполняет фазу только при изменении → предупреждение исчезает.
