[[Xcode]] сообщает, что вы **передали аргументы функции или метода, которая их не принимает**.

- В [[Swift]] функция/метод имеет определённую **сигнатуру**: количество и типы параметров.
    
- Если вы передаете аргумент там, где его **нет в определении функции**, компилятор выдаёт ошибку.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Вызов функции без параметров с аргументом**

```swift
func greet() {
    print("Hello")
}

greet("Alice") // ❌ Argument passed to call that takes no arguments
```

- Функция `greet()` не принимает параметров → аргумент `"Alice"` лишний.
    

---

**Пример 2: Метод класса без параметров**

```swift
class Person {
    func sayHello() {
        print("Hi")
    }
}

let person = Person()
person.sayHello(name: "Bob") // ❌ Argument passed to call that takes no arguments
```

- Метод `sayHello()` не имеет параметров, но передан аргумент с меткой `name`.
    

---

### Как исправить

#### 1️⃣ Убрать лишние аргументы

```swift
greet() // ✅ корректно
person.sayHello() // ✅ корректно
```

---

#### 2️⃣ Добавить параметр в функцию (если нужен)

```swift
func greet(_ name: String) {
    print("Hello, \(name)")
}

greet("Alice") // ✅ теперь аргумент корректен
```

```swift
class Person {
    func sayHello(name: String) {
        print("Hi, \(name)")
    }
}

person.sayHello(name: "Bob") // ✅ теперь работает
```

---

#### 3️⃣ Проверить сигнатуры при перегрузках

- Иногда есть несколько перегрузок функции, и выбранная версия **не принимает аргументы**.
    
- Нужно явно использовать корректную перегрузку.
    

---

### Резюме

- Ошибка возникает, когда **переданы аргументы в функцию/метод, которые не принимают параметры**.
    
- Исправляется через:
    
    1. Удаление лишних аргументов.
        
    2. Добавление параметров в определение функции.
        
    3. Проверку правильной перегрузки метода.
        
- Помогает компилятору точно сопоставить вызов функции с её сигнатурой.
