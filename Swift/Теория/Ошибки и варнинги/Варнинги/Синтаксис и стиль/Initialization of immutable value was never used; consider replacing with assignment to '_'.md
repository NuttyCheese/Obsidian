#crash #warning #xcode #swift
### Что это значит

[[Xcode]] предупреждает, что **вы создали константу ([[let]]) или переменную ([[var]])**, но **никогда не использовали её** в коде.

- Компилятор предлагает либо **использовать значение**, либо **заменить его на `_`**, если оно нужно только для побочного эффекта (например, вызова функции).
    
- Обычно это указывает на **лишний код** или **неполностью реализованную логику**.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Константа не используется**

```swift
let value = 42  // ⚠️ Initialization of immutable value was never used
```

- `value` создана, но ни разу не используется → предупреждение.
    

---

**Пример 2: Вызов функции с возвращаемым значением**

```swift
func compute() -> Int {
    return 10
}

let result = compute()  // ⚠️ Initialization of immutable value was never used
```

- `compute()` возвращает значение, но `result` не используется → предупреждение.
    

---

### Как исправить

#### 1️⃣ Использовать значение

```swift
let value = 42
print(value)  // ✅ теперь значение используется
```

```swift
let result = compute()
print(result) // ✅ используем результат функции
```

---

#### 2️⃣ Заменить на `_` (если возвращаемое значение не нужно)

```swift
_ = compute()  // ✅ значение игнорируется, побочный эффект вызывается
```

- Используется, когда важен **только вызов функции**, а результат не нужен.
    

---

#### 3️⃣ Удалить переменную, если она не нужна

```swift
compute()  // ✅ вызываем функцию напрямую без присваивания
```

- Простейший способ убрать предупреждение, если переменная не нужна.
    

---

### Резюме

- Предупреждение указывает на **неиспользуемую переменную или константу**.
    
- Исправляется использованием переменной, заменой на `_`, или удалением лишнего кода.
    
- Помогает сделать код чище и уменьшает шум от компилятора.
