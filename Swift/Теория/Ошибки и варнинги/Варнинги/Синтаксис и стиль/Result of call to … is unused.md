#crash #warning #xcode #swift
### Что это значит

[[Xcode]] предупреждает, что **вы вызвали функцию или метод, который возвращает значение**, но **результат вызова нигде не используется**.

- Обычно это указывает на **потенциально забытый результат**, который может содержать важные данные или ошибки.
    
- Предупреждение помогает избежать **логических ошибок**, когда разработчик ожидает, что результат будет обработан.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Вызов функции с возвращаемым значением**

```swift
func computeValue() -> Int {
    return 42
}

computeValue() // ⚠️ Result of call to 'computeValue()' is unused
```

- Функция возвращает `Int`, но значение игнорируется → предупреждение.
    

---

**Пример 2: Вызов метода, который возвращает Optional**

```swift
let text = "Hello"
text.uppercased()  // ⚠️ Result of call to 'uppercased()' is unused
```

- Метод возвращает новое значение строки, но мы не используем результат → предупреждение.
    

---

### Как исправить

#### 1️⃣ Использовать результат вызова

```swift
let value = computeValue()  // ✅ сохраняем результат
print(value)
```

```swift
let upper = text.uppercased()  // ✅ сохраняем и используем
print(upper)
```

---

#### 2️⃣ Игнорировать результат явно через `_`

```swift
_ = computeValue()  // ✅ явно игнорируем результат
```

- Используется, если важен только **побочный эффект** вызова функции.
    

---

#### 3️⃣ Проверить необходимость вызова

```swift
computeValue() // ❌ если результат не нужен, возможно вызов функции лишний
```

- Можно удалить вызов, если он не нужен → код станет чище.
    

---

### Резюме

- Предупреждение указывает на **неиспользуемый результат вызова функции или метода**.
    
- Исправляется: использование результата, явное игнорирование через `_`, или удаление лишнего вызова.
    
- Позволяет избежать логических ошибок и сделать код более читабельным.
