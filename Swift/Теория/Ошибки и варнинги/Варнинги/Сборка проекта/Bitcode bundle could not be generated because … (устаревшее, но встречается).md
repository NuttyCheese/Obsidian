#crash #warning #xcode #swift
### Что это значит

Это предупреждение указывает, что [[Xcode]] **не смог сгенерировать Bitcode** для вашего проекта или одной из библиотек.

- **Bitcode** — промежуточное представление кода, используемое для оптимизации и динамического компилирования на серверах Apple (App Thinning, App Store, WatchOS).
    
- Предупреждение не блокирует сборку, но может препятствовать публикации на App Store с включённым Bitcode.
    

Причины появления:

1. **Используются библиотеки без Bitcode** (например, сторонние `.framework`).
    
2. **Bitcode отключён в настройках проекта**, но используется библиотека, требующая его.
    
3. Конфликт архитектур (`arm64`, `x86_64`, `armv7` и т.д.).
    
4. Устаревшие SDK или библиотеки.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Сторонний framework без Bitcode**

- Добавлен `SomeFramework.framework` без поддержки Bitcode.
    
- В Build Settings → `Enable Bitcode = YES`
    

Предупреждение:

```
Bitcode bundle could not be generated because 'SomeFramework.framework/SomeFramework' does not contain bitcode.
```

---

**Пример 2: Включён Bitcode в таргете, но библиотека собрана без него**

- Используется [[CocoaPods]] или Carthage: pod или framework собран без Bitcode → появляется предупреждение.
    

---

### Как исправить

#### 1️⃣ Отключить Bitcode для проекта / таргета

- Target → Build Settings → **Enable Bitcode = NO**
    
- Часто самое простое решение, если библиотека не поддерживает Bitcode.
    

#### 2️⃣ Использовать версии библиотек с Bitcode

- Проверить документацию поставщика framework.
    
- Обновить Pod / [[Carthage]] / [[SPM]] до версии с поддержкой Bitcode.
    

#### 3️⃣ Пересобрать кастомные фреймворки с Bitcode

- Для собственных библиотек:
    
    - Xcode → Build Settings → **Enable Bitcode = YES**
        
    - Собрать заново.
        

#### 4️⃣ Игнорировать предупреждение, если Bitcode не нужен

- Bitcode больше не обязателен для [[iOS]] приложений, начиная с iOS 14+.
    
- Можно безопасно оставлять предупреждение, если не используете WatchOS или специфические оптимизации App Store.
    

---

### Пример исправления

**Было:**

- Target → Enable Bitcode = YES
    
- Использован SomeFramework.framework без Bitcode → предупреждение.
    

**Исправлено:**

- Target → Enable Bitcode = NO
    
- Сборка проходит без предупреждений.
