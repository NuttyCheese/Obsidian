#crash #warning #xcode #swift
### Что это значит

[[Xcode]] предупреждает, что вы **переопределяете метод родительского класса**, но **не указали ключевое слово [[override]]**.

- В [[Swift]] **все переопределяемые методы должны быть явно помечены `override`**.
    
- Без `override` компилятор не может точно понять, что метод переопределяет родительский → предупреждение.
    
- Это предотвращает **случайное дублирование методов** или **ошибки при наследовании**.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Переопределение метода без `override`**

```swift
class Animal {
    func makeSound() {
        print("Animal sound")
    }
}

class Dog: Animal {
    func makeSound() { // ⚠️ Method override is not marked with 'override'
        print("Bark")
    }
}
```

- Компилятор видит, что `Dog.makeSound()` совпадает с методом `Animal.makeSound()`, но `override` отсутствует → предупреждение.
    

---

**Пример 2: Переопределение свойства без `override`**

```swift
class Vehicle {
    var speed: Int { return 0 }
}

class Car: Vehicle {
    var speed: Int { return 100 } // ⚠️ Method override is not marked with 'override'
}
```

- Свойство `speed` переопределяет родительское → нужно `override`.
    

---

### Как исправить

#### 1️⃣ Добавить `override` перед методом

```swift
class Dog: Animal {
    override func makeSound() {
        print("Bark") // ✅ теперь компилятор понимает, что метод переопределяет родительский
    }
}
```

---

#### 2️⃣ Для свойств — тоже использовать `override`

```swift
class Car: Vehicle {
    override var speed: Int { return 100 } // ✅ правильно
}
```

---

### Резюме

- Предупреждение указывает на **отсутствие ключевого слова `override` при переопределении метода или свойства**.
    
- Исправляется добавлением `override` перед методом/свойством.
    
- Помогает компилятору проверять наследование и предотвращает случайные ошибки при изменении базового класса.
