Ошибка `EXC_BAD_ACCESS` указывает, что **программа попыталась обратиться к памяти, которая недоступна или уже освобождена**.

- Обычно это **ошибка управления памятью**.
    
- Часто встречается при работе с:
    
    1. [[unowned]] ссылками на уже деаллоцированные объекты.
        
    2. Unsafe pointers (`UnsafePointer`, `UnsafeMutablePointer`).
        
    3. Объектами [[Objective-C]], которые были удалены, но ссылка на них осталась.
        
- В [[Swift]] это **runtime crash**, [[Xcode]] обычно показывает: `Thread 1: EXC_BAD_ACCESS`.
    
- В коде сложно точно определить причину без анализа стека вызовов.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Unowned reference на деаллоцированный объект**

```swift
class Person {
    var closure: (() -> Void)?
}

var p: Person? = Person()
p?.closure = { [unowned p!] in
    print("Hello")
}

p = nil
p?.closure?() // ❌ EXC_BAD_ACCESS
```

- `p` уже деаллоцирован → обращение через `unowned` → ошибка.
    

---

**Пример 2: Unsafe pointer**

```swift
var x = 10
let ptr = UnsafePointer<Int>(&x)
x = 20
let y = ptr.pointee // ✅ безопасно, но если ptr уже невалиден → EXC_BAD_ACCESS
```

- Если указатель указывает на память, которая уже освобождена → crash.
    

---

**Пример 3: Использование объектов Objective-C после деаллоцирования**

```swift
@objc class Person: NSObject {
    @objc func greet() { print("Hello") }
}

var obj: Person? = Person()
let closure = { [unowned obj!] in obj!.greet() }
obj = nil
closure() // ❌ EXC_BAD_ACCESS
```

---

### Как исправить

#### 1️⃣ Использовать `weak` вместо `unowned` там, где объект может быть деаллоцирован

```swift
p?.closure = { [weak p] in
    p?.greet() // ✅ безопасно
}
```

---

#### 2️⃣ Проверять указатели и память при работе с UnsafePointer

- Убедиться, что память ещё валидна во время обращения.
    

---

#### 3️⃣ Контроль жизненного цикла объектов

- Не использовать объекты после их удаления.
    
- Для Objective-C объектов проверять [[nil]] перед вызовом.
    

---

#### 4️⃣ Использовать инструменты диагностики

- [[Xcode]]: Thread Sanitizer (TSAN)
    
- Instruments → Zombies, чтобы отслеживать использование уже деаллоцированных объектов.
    

---

### Резюме

- **EXC_BAD_ACCESS** — это ошибка доступа к памяти, обычно при обращении к деаллоцированному объекту или некорректному указателю.
    
- Исправляется через:
    
    1. Замена `unowned` на `weak`, если объект может быть уничтожен.
        
    2. Проверку указателей и управление памятью.
        
    3. Использование Thread Sanitizer и Instruments для диагностики.
        
- Позволяет предотвратить **[[Runtime]] crash** и ошибки управления памятью.
