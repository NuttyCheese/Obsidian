[[Xcode]] предупреждает, что **вы присвоили значение переменной**, но **никогда не использовали её** в коде.

- Переменная создаётся и ей что-то присваивается (`var x = 10` или `x = 20`), но это значение **не читается нигде**.
    
- Это часто указывает на **лишний код** или **недоделанную логику**.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Присвоение, но переменная не используется**

```swift
var counter = 0
counter = 5  // ⚠️ Variable 'counter' was written to, but never read
```

- Значение присвоено, но нигде не используется → предупреждение.
    

---

**Пример 2: Присвоение внутри цикла, без использования**

```swift
var sum = 0
for i in 1...5 {
    sum = i * 2  // ⚠️ Variable 'sum' was written to, but never read
}
```

- Каждый раз присваиваем `sum`, но оно не используется после цикла.
    

---

### Как исправить

#### 1️⃣ Использовать переменную

```swift
var counter = 0
counter = 5
print(counter) // ✅ теперь значение читается
```

```swift
var sum = 0
for i in 1...5 {
    sum += i * 2 // ✅ используем значение
}
print(sum)
```

---

#### 2️⃣ Игнорировать значение через `_`

- Если присваивание нужно только для побочного эффекта функции или вычисления:
    

```swift
_ = computeValue()  // ✅ игнорируем результат
```

- Применимо, когда переменная не нужна, но важно, что код выполняется.
    

---

#### 3️⃣ Удалить лишнюю переменную

```swift
computeValue() // ✅ вызываем функцию напрямую без присваивания
```

- Самый простой способ, если переменная не нужна → предупреждение исчезает.
    

---

### Резюме

- Предупреждение указывает на **неиспользуемую переменную, которой присваивали значение**.
    
- Исправляется использованием переменной, удалением или явным игнорированием через `_`.
    
- Помогает очистить код и уменьшить шум от компилятора.
