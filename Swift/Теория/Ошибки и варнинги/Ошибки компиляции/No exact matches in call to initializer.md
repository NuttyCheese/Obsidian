#crash #fata_error #xcode #swift
### Что это значит

[[Xcode]] сообщает, что **вы вызываете инициализатор класса или структуры с аргументами, которые не соответствуют ни одной существующей сигнатуре инициализатора**.

- В [[Swift]] **каждый инициализатор имеет строго определённые параметры и их типы**.
    
- Если вы вызываете инициализатор с **неправильными типами аргументов или неверным количеством аргументов**, компилятор не может сопоставить вызов → ошибка.
    

---

### Примеры кода/сценариев возникновения

**Пример 1: Неправильные типы аргументов**

```swift
struct Person {
    let name: String
    let age: Int
}

let p = Person(name: "Alice", age: "25") // ❌ No exact matches in call to initializer
```

- `age` должен быть [[Int]], а передан [[String]].
    

---

**Пример 2: Неправильное количество аргументов**

```swift
struct Point {
    let x: Double
    let y: Double
}

let point = Point(x: 1.0) // ❌ No exact matches in call to initializer
```

- Ожидается два аргумента (`x` и `y`), передан только один.
    

---

**Пример 3: Несоответствие именованных параметров**

```swift
struct Rectangle {
    let width: Double
    let height: Double
}

let rect = Rectangle(w: 10, h: 20) // ❌ No exact matches in call to initializer
```

- Параметры `w` и `h` не существуют → нужно `width` и `height`.
    

---

### Как исправить

#### 1️⃣ Проверить количество и имена аргументов

```swift
let rect = Rectangle(width: 10, height: 20) // ✅ правильно
```

---

#### 2️⃣ Привести типы аргументов к ожидаемым

```swift
let p = Person(name: "Alice", age: 25) // ✅ age теперь Int
```

---

#### 3️⃣ Создать собственный инициализатор, если требуется другая сигнатура

```swift
struct Person {
    let name: String
    let age: Int
    
    init(name: String, ageString: String) {
        self.name = name
        self.age = Int(ageString) ?? 0
    }
}

let p = Person(name: "Alice", ageString: "25") // ✅ теперь можно вызвать
```

---

### Резюме

- Ошибка возникает, когда **вызов инициализатора не совпадает ни с одной сигнатурой**.
    
- Исправляется через:
    
    1. Проверку именованных параметров и их количества.
        
    2. Приведение типов аргументов к ожидаемым.
        
    3. Создание дополнительного инициализатора с нужной сигнатурой.
        
- Позволяет компилятору точно сопоставлять вызовы с определением инициализаторов и предотвращает runtime ошибки.
