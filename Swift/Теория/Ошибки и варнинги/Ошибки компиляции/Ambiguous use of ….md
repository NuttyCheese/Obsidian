#crash #fata_error #xcode #swift
### Что это значит

[[Xcode]] сообщает, что **компилятор не может однозначно выбрать метод или оператор**, который вы пытаетесь использовать.

- В [[Swift]] может быть несколько **функций, методов или операторов с похожей сигнатурой**.
    
- Компилятор **не может понять, какую именно перегрузку выбрать** → выдаёт ошибку.
    
- Обычно возникает при:
    
    1. Перегрузке функций с похожими параметрами.
        
    2. Приведении типов, когда Swift не может вывести тип.
        
    3. Использовании операторов для типов, которые имеют несколько подходящих реализаций.
        

---

### Примеры кода/сценариев возникновения

**Пример 1: Перегрузка метода**

```swift
func printValue(_ value: Int) { print("Int: \(value)") }
func printValue(_ value: Double) { print("Double: \(value)") }

let number: Numeric = 5
printValue(number) // ⚠️ Ambiguous use of 'printValue'
```

- `number` может быть [[Int]] или [[Double]] → компилятор не может выбрать перегрузку.
    

---

**Пример 2: Операторы для разных типов**

```swift
let a: Int8 = 10
let b: Int16 = 20
let c = a + b // ⚠️ Ambiguous use of '+'
```

- Swift не может выбрать правильный оператор `+` для разных целочисленных типов.
    

---

**Пример 3: [[Generic]] функции**

```swift
func sum<T: Numeric>(_ a: T, _ b: T) -> T { a + b }
func sum<T: BinaryInteger>(_ a: T, _ b: T) -> T { a + b }

let value = sum(5, 10) // ⚠️ Ambiguous use of 'sum'
```

- Компилятор не может выбрать, какой generic использовать ([[Numeric]] или `BinaryInteger`).
    

---

### Как исправить

#### 1️⃣ Явно указать типы

```swift
let number: Int = 5
printValue(number) // ✅ теперь компилятор знает, какую перегрузку использовать
```

```swift
let c = Int(a) + Int(b) // ✅ приведение типов решает проблему
```

---

#### 2️⃣ Использовать полные имена методов/параметры

```swift
sum(Int(5), Int(10)) // ✅ однозначный вызов
```

---

#### 3️⃣ Изменить сигнатуру метода, чтобы убрать пересечение

```swift
func printIntValue(_ value: Int) { ... }
func printDoubleValue(_ value: Double) { ... }
```

- Убирает пересечение и делает вызов однозначным.
    

---

### Резюме

- `Ambiguous use of …` возникает, когда [[Swift]] **не может выбрать перегрузку функции/оператора/метода**.
    
- Исправляется через:
    
    1. Явное указание типов.
        
    2. Приведение значений к нужному типу.
        
    3. Изменение сигнатуры методов для устранения пересечений.
        
- Помогает избежать непредсказуемого поведения и делает код однозначным.
