#crash #fata_error #xcode #swift
### Что это значит

[[Xcode]] сообщает, что вы **пытаетесь вызвать переменную или константу как функцию**, хотя она **не является функцией**.

- В [[Swift]] **только функции или замыкания могут быть вызваны через `()`**.
    
- Часто ошибка возникает, когда:
    
    1. Вы перепутали переменную с функцией.
        
    2. Пытаетесь вызвать результат вычисления, которое не является функцией.
        
    3. Ошибочно используете имя свойства вместо метода.
        

---

### Примеры кода/сценариев возникновения

**Пример 1: Вызов переменной как функции**

```swift
let number = 5
number() // ❌ Cannot call value of non-function type 'Int'
```

- `number` — это [[Int]], а не функция → нельзя использовать `()`.
    

---

**Пример 2: Вызов свойства вместо метода**

```swift
class Person {
    var name: String = "Alice"
}

let person = Person()
person.name() // ❌ Cannot call value of non-function type 'String'
```

- `name` — свойство типа [[String]], а не метод.
    

---

**Пример 3: Ошибка с замыканием/функцией**

```swift
var closure: Int = 10
closure() // ❌ Cannot call value of non-function type 'Int'
```

- Если хотели создать замыкание, нужно объявлять как `() -> Int`.
    

```swift
var closure: () -> Int = { 10 }
closure() // ✅ корректно
```

---

### Как исправить

#### 1️⃣ Проверить тип вызываемого значения

```swift
let number = 5
print(number) // ✅ вывод переменной вместо вызова
```

---

#### 2️⃣ Убедиться, что вызываемое значение — функция/замыкание

```swift
let greet: () -> Void = { print("Hello") }
greet() // ✅ корректно
```

---

#### 3️⃣ Использовать правильное имя метода

```swift
class Person {
    func name() -> String { "Alice" }
}

let person = Person()
person.name() // ✅ вызываем метод, а не свойство
```

---

### Резюме

- Ошибка возникает, когда **вы используете `()` на значении, которое не является функцией**.
    
- Исправляется через:
    
    1. Проверку типа переменной/свойства.
        
    2. Приведение значения к функции/замыканию.
        
    3. Использование правильного метода вместо свойства.
        
- Помогает компилятору избежать неправильного вызова и [[runtime]] ошибок.
