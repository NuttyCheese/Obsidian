#crash #fata_error #xcode #swift
### Что это значит

[[Xcode]] сообщает, что вы **повторно объявили сущность (класс, структуру, функцию, переменную, константу или enum) с тем же именем в одной области видимости**.

- [[Swift]] **не разрешает дважды объявлять одну и ту же сущность в одной области видимости**.
    
- Причины появления ошибки:
    
    1. Дублирование функций или переменных в одном файле/классе.
        
    2. Дублирование между extension и основной реализацией.
        
    3. Импорт нескольких модулей с одинаковыми именами (редко).
        
    4. Ошибки при копировании/вставке кода.
        

---

### Примеры кода/сценариев возникновения

**Пример 1: Дублирование функции**

```swift
func greet() {
    print("Hello")
}

func greet() {
    print("Hi") // ❌ Invalid redeclaration of 'greet()'
}
```

---

**Пример 2: Дублирование переменной**

```swift
let number = 5
let number = 10 // ❌ Invalid redeclaration of 'number'
```

---

**Пример 3: Дублирование метода в классе/[[extension]]**

```swift
class Person {
    func sayHello() { print("Hello") }
}

extension Person {
    func sayHello() { print("Hi") } // ❌ Invalid redeclaration of 'sayHello()'
}
```

---

### Как исправить

#### 1️⃣ Удалить или переименовать дублирующую сущность

```swift
func greetUser() {
    print("Hi")
}
```

---

#### 2️⃣ Проверить области видимости

- Иногда одна и та же функция объявлена в классе и extension → переименовать метод или использовать `override` при наследовании.
    

---

#### 3️⃣ Для переменных и констант

- Использовать разные имена или объединять в одну переменную/константу.
    

---

#### 4️⃣ Проверить импорты модулей

- Иногда одинаковые имена приходят из разных модулей → использовать **module.name** для уточнения.
    

```swift
import ModuleA
import ModuleB

let a = ModuleA.SomeType()
let b = ModuleB.SomeType()
```

---

### Резюме

- Ошибка возникает, когда **одна и та же сущность объявлена дважды в одной области видимости**.
    
- Исправляется через:
    
    1. Переименование дублирующей функции, класса или переменной.
        
    2. Проверку extension и наследования.
        
    3. Проверку импортированных модулей.
        
- Позволяет избежать конфликтов имен и неоднозначности для компилятора.
