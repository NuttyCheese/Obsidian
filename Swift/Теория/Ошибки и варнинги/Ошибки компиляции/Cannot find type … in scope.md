Компилятор [[Swift]] сообщает, что **не может найти указанный тип в текущей области видимости**.

- Причины чаще всего связаны с:
    
    1. **Опечаткой** в названии типа.
        
    2. **Отсутствием импорта модуля**, в котором объявлен тип.
        
    3. Тип ещё **не определён на момент использования** (например, объявлен ниже по коду).
        
    4. Ошибками с **генерируемым кодом** или зависимостями.
        

---

### Примеры кода/сценариев возникновения

**Пример 1: Опечатка в названии типа**

```swift
let user: USer = User(name: "Alice") 
// ❌ Cannot find type 'USer' in scope
```

---

**Пример 2: Отсутствие импорта модуля**

```swift
let url = URL(string: "https://example.com")
// ❌ Cannot find type 'URL' in scope, если не импортирован Foundation

import Foundation // ✅ исправление
```

---

**Пример 3: Использование типа до его объявления**

```swift
let person: Person = Person(name: "Bob") // ❌ Cannot find type 'Person' in scope

struct Person {
    let name: String
}
```

- Тип должен быть объявлен до использования.
    

---

**Пример 4: Зависимости или пакеты**

```swift
let decoder = JSONDecoder() 
// ❌ Cannot find type 'JSONDecoder' in scope, если не импортирован Foundation
```

---

### Как исправить

#### 1️⃣ Проверить правильность написания имени типа

```swift
let user: User = User(name: "Alice") // ✅ исправлено
```

---

#### 2️⃣ Импортировать нужный модуль

```swift
import Foundation
```

---

#### 3️⃣ Объявить тип до использования

```swift
struct Person {
    let name: String
}

let person: Person = Person(name: "Bob") // ✅ теперь компилятор видит тип
```

---

#### 4️⃣ Проверить зависимости и пакеты

- Убедиться, что [[SPM]], [[CocoaPods]] или Framework подключен.
    
- Проверить target membership для типа/файла.
    

---

### Резюме

- **Cannot find type … in scope** — компилятор не видит указанный тип.
    
- Исправляется через:
    
    1. Проверку имени типа (опечатки).
        
    2. Импорт нужного модуля.
        
    3. Правильное объявление типа до использования.
        
    4. Проверку подключённых зависимостей и target membership.
        
- Позволяет избежать ошибок компиляции и корректно использовать типы в Swift.
