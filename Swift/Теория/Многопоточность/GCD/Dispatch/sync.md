**`sync`** — это метод [[DispatchQueue]], который **синхронно** добавляет блок кода в очередь и **блокирует текущий поток** до тех пор, пока этот блок **не будет выполнен полностью**.

Ключевые характеристики `sync` (2026):

- **Блокирует** вызывающий поток — выполнение следующей строки кода ждёт завершения блока  
- Выполняется **в указанной очереди** (а не в текущем потоке)  
- **Никогда** не используй `sync` на той же очереди, из которой вызываешь — **[[deadlock]]**  
- Самый опасный метод в GCD — причина №1 всех deadlock’ов в iOS-приложениях

### 2. Основные варианты sync (таблица 2026)

| Вызов                                      | Что происходит                                   | Блокирует текущий поток? | Самый частый сценарий | Опасность |
|--------------------------------------------|--------------------------------------------------|---------------------------|------------------------|-----------|
| `queue.sync { ... }`                       | Блок выполняется в `queue`                       | Да                        | Редко — только в фоне  | Высокая   |
| `DispatchQueue.main.sync { ... }`          | Блок выполняется на главном потоке               | Да                        | Почти никогда          | Максимальная (deadlock) |
| `queue.sync(flags: .barrier) { ... }`      | Барьерная синхронная запись в concurrent queue  | Да                        | Legacy-кэш             | Высокая   |

**Самое важное правило 2026**:
> `sync` на **главной** очереди (`DispatchQueue.main.sync`) **из главного потока** → **гарантированный deadlock**.  
> `sync` на **любой** очереди из **той же самой** очереди → **гарантированный deadlock**.

### 3. Самые популярные (и почти всегда неправильные) шаблоны 2026

#### Плохо №1 — Классический deadlock (самая частая ошибка)

```swift
DispatchQueue.main.sync {
    // Этот код никогда не выполнится
    label.text = "Привет"
}
```

**Почему**: главный поток уже выполняет код → ждёт сам себя → deadlock.

#### Плохо №2 — sync внутри [[actor]] / [[@MainActor]]

```swift
@MainActor
func update() {
    DispatchQueue.main.sync {
        // deadlock — мы уже на main
    }
}
```

#### Плохо №3 — sync на той же очереди

```swift
let serialQueue = DispatchQueue(label: "serial")

serialQueue.sync {
    serialQueue.sync {  // deadlock
        print("Никогда не выполнится")
    }
}
```

#### Единственный оправданный сценарий (редкий) — sync в фоне

```swift
func loadConfigSync() -> Config {
    var config: Config?
    
    let queue = DispatchQueue(label: "config")
    
    queue.sync {
        // тяжёлая синхронная работа (например, чтение файла)
        config = loadFromDisk()
    }
    
    return config!
}
```

Но даже здесь в 2026 году лучше:

```swift
func loadConfig() async -> Config {
    await withCheckedContinuation { continuation in
        DispatchQueue.global().async {
            let config = loadFromDisk()
            continuation.resume(returning: config)
        }
    }
}
```

### 4. sync vs современные альтернативы (сравнение 2026)

| Механизм                      | Блокирует поток? | Требует await? | Потокобезопасность | Рекомендация 2026        | Когда всё ещё использовать |
| ----------------------------- | ---------------- | -------------- | ------------------ | ------------------------ | -------------------------- |
| `queue.sync { ... }`          | Да               | Нет            | Ручная             | **Никогда** в production | Почти никогда              |
| `await actor.method()`        | Нет              | Да             | Полная встроенная  | Основной выбор           | Изменяемое состояние       |
| `await MainActor.run { ... }` | Нет              | Да             | Полная ([[main]])  | Для UI                   | UI-обновления из фона      |
| `DispatchQueue.main.async`    | Нет              | Нет            | Полная (main)      | Legacy                   | Старый код                 |
| `Task { @MainActor in ... }`  | Нет              | Да             | Полная             | Современный UI           | Запуск UI-логики из фона   |

**Вывод 2026**:
- **`sync` — это уже legacy-инструмент**  
- В 2026 году его использование в production-коде считается **антипаттерном**  
- Почти всегда есть безопасная альтернатива через `await`, `actor`, `@MainActor` или `withCheckedContinuation`

### 5. Типичные ошибки и последствия sync в 2026

| Ошибка                                       | Последствия                        | Как избежать                                  |
| -------------------------------------------- | ---------------------------------- | --------------------------------------------- |
| `DispatchQueue.main.sync` из главного потока | **Deadlock** — приложение зависает | Никогда не использовать sync на main          |
| `sync` на той же очереди                     | **Deadlock**                       | Никогда не sync внутри той же очереди         |
| sync для тяжёлой работы на main              | **UI freeze** (ANR)                | Всё тяжёлое — в фон (`Task`, `actor`)         |
| Использование sync вместо await              | Устаревший, небезопасный код       | Переходить на [[async]]/[[await]] + [[actor]] |
| sync внутри @MainActor / actor               | Deadlock или ошибка                | Использовать await или [[nonisolated]]        |

### 6. Лучшие практики 2026 года

- **Никогда** не используй `sync` в новом коде  
- **sync на main** — **запрещено** даже думать об этом  
- **sync на глобальной очереди** — только в крайнем случае и только в фоне  
- **Переходи на async/await + actor** — это современный, безопасный и читаемый способ  
- **Для UI** — только `@MainActor` / `await MainActor.run`  
- **Для тестов** — используй `await` и `XCTestExpectation`  
- **Swift 6 strict concurrency** — включает предупреждения о legacy-sync — не игнорируй их  
- **Рефакторинг** — ищи все `sync` в проекте и заменяй на `await`

**Короткий девиз 2026**:
> «sync — это когда ты говоришь: «я заблокирую поток, пока не выполнится блок».  
> В 2026 году это **почти всегда ошибка**.  
> Современный Swift использует `await`, `actor` и `@MainActor` — sync остался только в legacy-коде.»
