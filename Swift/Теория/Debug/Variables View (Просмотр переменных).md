## Что такое Variables View

**Variables View** — это панель в Debug Area [[Xcode]], которая показывает:

- локальные переменные текущего фрейма
    
- параметры функции
    
- свойства `self`
    
- значения optional
    
- состояние объектов в памяти
    

Она работает вместе с [[LLDB (Low-Level Debugger)|LLDB]], но позволяет анализировать данные **без ввода команд вручную**.

Проще говоря:

> Это графическое представление текущего состояния кода в момент остановки.

---

# 📍 Где находится

1. Поставить breakpoint
    
2. Запустить приложение
    
3. Когда выполнение остановится — в нижней части Xcode появится Debug Area
    
4. Левая панель — Variables View
    

---

# 🧱 Что отображается

### 1️⃣ Local Variables

Переменные текущего метода:

```swift
let count = 10
var user: User
```

Ты увидишь:

- имя
    
- тип
    
- текущее значение
    

---

### 2️⃣ Arguments

Параметры функции:

```swift
func configure(with model: User)
```

Будет видно:

- `model`
    
- его содержимое
    

---

### 3️⃣ Self

Свойства текущего объекта:

```swift
po self
```

Но в Variables View — в виде дерева.

Можно раскрыть:

- stored properties
    
- lazy properties
    
- computed (если уже вычислены)
    

---

### 4️⃣ [[Optional]]

Показывает:

```
Optional("value")
nil
```

Можно раскрыть и увидеть wrapped значение.

---

# 🔍 Как пользоваться эффективно

---

## 🧠 1. Раскрытие объектов

Можно раскрывать:

- [[struct]]
    
- [[class]]
    
- [[array]]
    
- [[dictionary]]
    
- nested объекты
    

Например:

```
user
 ├─ id
 ├─ name
 ├─ address
     ├─ city
     ├─ street
```

Это быстрее, чем писать `po user`.

---

## 🧠 2. Изменение значения во время паузы

ПКМ → Edit Value

Можно изменить:

- Int
    
- Bool
    
- String
    

И проверить, как поведёт себя приложение.

---

## 🧠 3. Проверка состояния UIViewController

В `viewDidLoad`:

Раскрываем `self`:

- view
    
- navigationController
    
- presentedViewController
    
- childViewControllers
    

Помогает понять:

- экран действительно в иерархии?
    
- есть ли [[retain cycle]]?
    
- правильно ли переданы данные?
    

---

# 🧪 Практические сценарии

---

## 📦 1. Отладка сетевого слоя

В completion:

```swift
case .success(let response):
```

В Variables View можно:

- раскрыть response
    
- проверить декодированные поля
    
- увидеть nil значения
    

---

## 📱 2. Отладка UITableView

В `cellForRowAt`:

Проверить:

- indexPath
    
- model[indexPath.row]
    
- состояние cell
    

---

## 🔄 3. Проверка retain cycle

Если `deinit` не вызывается:

Ставим breakpoint где объект должен освобождаться  
Смотрим `self`  
Проверяем ссылки

---

## 🧵 4. Проверка потоков

Можно увидеть:

- на каком потоке остановка
    
- текущий стек вызовов
    

---

# 🧬 Особенности [[Swift]]

Swift усложняет отображение:

- [[generic]] могут отображаться странно
    
- optimized build скрывает переменные
    
- computed properties не всегда видны
    
- [[lazy]] не показывается пока не инициализирован
    

Важно отлаживать в Debug конфигурации.

---

# ⚙️ Связь с LLDB

Variables View — это визуальная оболочка над командами:

```
frame variable
po
expr
```

Если переменная не отображается — можно использовать консоль.

- сравнить ссылки

В Debug Navigator можно выбрать другой фрейм

# ⚠️ Частые проблемы

### ❌ Переменная "not available"

Причина:

- оптимизация компилятора
    
- переменная вышла из scope
    

---

### ❌ Не отображаются значения async

При [[async]]/[[await]]:

- лучше ставить breakpoint внутри [[Task]]
    
- переключаться по стеку
    

---

# 📈 Когда Variables View лучше консоли

|Console|Variables View|
|---|---|
|нужно писать команды|всё видно сразу|
|быстрее для сложных выражений|быстрее для просмотра дерева|
|подходит для изменения логики|подходит для анализа структуры|

Лучше использовать вместе.

---

# 🎯 Правильный подход

1. Остановить выполнение
    
2. Посмотреть локальные переменные
    
3. Раскрыть [[self]]
    
4. Проверить входные параметры
    
5. Проверить стек
    
6. При необходимости использовать LLDB
    

Такой подход позволяет видеть систему целиком, а не отдельные print.
