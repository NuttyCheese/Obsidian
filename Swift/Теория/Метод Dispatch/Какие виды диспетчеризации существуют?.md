**Диспетчеризация** (dispatch) — это процесс определения, **какая именно реализация** метода (или свойства) будет вызвана при обращении к нему через переменную/ссылку.

В Swift существуют **четыре основных вида диспетчеризации**:

| Вид диспетчеризации                                           | Время определения | Скорость | Полиморфизм | Переопределение | Размер кода | Примеры                                                 |
| ------------------------------------------------------------- | ----------------- | -------- | ----------- | --------------- | ----------- | ------------------------------------------------------- |
| **[[Direct Dispatch\|Direct]] / [[Static Dispatch\|Static]]** | Компиляция        | ★★★★★    | Нет         | Нет             | Минимальный | [[final]], [[static]], [[private]], [[struct]]/[[enum]] |
| **[[Table Dispatch\|Table]]** (vtable)                        | Выполнение        | ★★★★☆    | Да          | Да              | Средний     | Обычные методы классов                                  |
| **[[Witness Table Dispatch\|Witness Table]]**                 | Выполнение        | ★★★★☆    | Да          | Да              | Средний     | Протоколы без [[@objc]]                                 |
| **[[Message Dispatch\|Message]]** (objc_msgSend)              | Выполнение        | ★★☆☆☆    | Да          | Да              | Большой     | `@objc`, `dynamic`, `NSObject`                          |

### 2. Подробное сравнение всех видов (2026)

| Характеристика         | Direct / [[Static Dispatch]]     | [[Table Dispatch]] ([[vtable]]) | [[Witness Table Dispatch]] | [[Message Dispatch]](objc_msgSend) |
| ---------------------- | -------------------------------- | ------------------------------- | -------------------------- | ---------------------------------- |
| Время определения      | Компиляция                       | [[Runtime]]                     | Runtime                    | Runtime                            |
| Скорость вызова        | Максимальная (~1–2 нс)           | Высокая (~3–5 нс)               | Высокая (~3–5 нс)          | Низкая (~10–20 нс)                 |
| Полиморфизм            | Нет                              | Да                              | Да                         | Да                                 |
| Поддержка [[override]] | Нет                              | Да                              | Да                         | Да                                 |
| Поддержка struct/enum  | Да                               | Нет                             | Да (протоколы)             | Нет                                |
| Поддержка протоколов   | Да (если [[some]] / [[generic]]) | Нет                             | Да                         | Да (если @objc)                    |
| Overhead в памяти      | Минимальный                      | Средний (vtable)                | Средний (witness table)    | Большой (objc runtime)             |
| Инлайнинг компилятором | Да (часто)                       | Редко                           | Редко                      | Почти никогда                      |
| Где критично           | Горячие циклы, рендеринг         | Большинство кода                | Протоколы                  | Редко (только Obj-C совместимость) |

### 3. Подробные примеры кода

#### 3.1 Direct / Static Dispatch (самый быстрый)

```swift
class FastMath {
    final func square(_ x: Int) -> Int {  // final → Direct
        return x * x
    }
    
    static func cube(_ x: Int) -> Int {   // static → Direct
        return x * x * x
    }
}

let m = FastMath()
print(m.square(5))     // прямой вызов
print(FastMath.cube(3)) // прямой вызов
```

#### 3.2 Table Dispatch (vtable) — обычные методы классов

```swift
class Animal {
    func makeSound() {          // vtable
        print("Generic")
    }
}

class Dog: Animal {
    override func makeSound() {
        print("Woof!")
    }
}

let a: Animal = Dog()
a.makeSound()  // vtable → Woof!
```

#### 3.3 Witness Table Dispatch — протоколы без @objc

```swift
protocol Drawable {
    func draw()
}

struct Circle: Drawable {
    func draw() { print("○") }
}

let d: any Drawable = Circle()
d.draw()  // witness table → ○
```

#### 3.4 Message Dispatch — @objc / dynamic

```swift
class MyView: UIView {
    @objc dynamic func didTap() {   // objc_msgSend
        print("Tapped!")
    }
}
```

### 4. Производительность (примерные цифры 2026)

| Тип диспетчеризации       | Вызов метода (нс) | Разница с Direct | Где критично |
|----------------------------|-------------------|-------------------|--------------|
| Direct / Static            | ~1–2 нс           | 1×                | 60 fps UI, ML-инференс, аудио |
| Table (vtable/witness)     | ~3–5 нс           | 2–3× медленнее    | Большинство кода |
| Message (objc_msgSend)     | ~10–20 нс         | 5–10× медленнее   | Редко (только @objc) |

### 5. Лучшие практики и оптимизации 2026 (Swift 6+)

- **Максимальная скорость** → используйте **[[final]]**, **[[static]]**, **[[private]]**, **[[some]]**  
- **Протоколы** → возвращайте **[[some Protocol]]**, храните **[[any Protocol]]** только если нужно  
- **@objc / dynamic** → только для [[KVO]], [[Delegate]], старый [[UIKit]], совместимость с Obj-C  
- **[[SwiftUI]]** — `some View`, `some ViewModel` — стандарт (часто Direct / Witness)  
- **Горячие пути** (рендеринг, циклы, обработка кадров) — избегайте `any` и `@objc`  
- **Generics** — `<T: Protocol>` вместо `any` для статической диспетчеризации  
- **Swift 6 strict concurrency** — Static Dispatch сильно упрощает проверку потокобезопасности

**Короткий девиз 2026**:
> «Static Dispatch — это когда компилятор говорит: «Я уже всё посчитал и сделаю мгновенно».  
> final, static, private, some — ваши лучшие друзья для скорости.  
> any и @objc — только когда без них действительно нельзя.»
