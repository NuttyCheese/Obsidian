## Диспетчеризация в Swift

### Определение
**Диспетчеризация (dispatch)** — это процесс определения, **какая именно реализация** метода (или свойства) будет вызвана при обращении к нему через переменную или ссылку . В [[Swift]] существует несколько видов диспетчеризации, которые различаются временем определения (компиляция или выполнение), скоростью вызова, поддержкой полиморфизма и возможностью переопределения.

Понимание диспетчеризации критически важно для написания высокопроизводительного кода, особенно в горячих циклах, рендеринге, аудиообработке и других местах, где важна каждая наносекунда .

---

### Четыре основных вида диспетчеризации в Swift

| Вид диспетчеризации          | Время определения | Скорость | Полиморфизм | Переопределение | Размер кода | Примеры                                       |
| ---------------------------- | ----------------- | -------- | ----------- | --------------- | ----------- | --------------------------------------------- |
| **Direct / Static**          | Компиляция        | ★★★★★    | Нет         | Нет             | Минимальный | `final`, `static`, `private`, `struct`/`enum` |
| **Table (vtable)**           | Выполнение        | ★★★★☆    | Да          | Да              | Средний     | Обычные методы классов                        |
| **Witness Table**            | Выполнение        | ★★★★☆    | Да          | Да              | Средний     | Протоколы без `@objc`                         |
| **Message** (`objc_msgSend`) | Выполнение        | ★★☆☆☆    | Да          | Да              | Большой     | `@objc`, `dynamic`, `NSObject`                |

---

### Подробное сравнение всех видов

| Характеристика                    | Direct / Static                   | Table (vtable)    | Witness Table           | Message (objc_msgSend)             |
| --------------------------------- | --------------------------------- | ----------------- | ----------------------- | ---------------------------------- |
| **Время определения**             | Компиляция                        | Runtime           | Runtime                 | Runtime                            |
| **Скорость вызова**               | Максимальная (~1–2 нс)            | Высокая (~3–5 нс) | Высокая (~3–5 нс)       | Низкая (~10–20 нс)                 |
| **Полиморфизм**                   | Нет                               | Да                | Да                      | Да                                 |
| **Поддержка override**            | Нет                               | Да                | Да                      | Да                                 |
| **Поддержка [[struct]]/[[enum]]** | Да                                | Нет               | Да (протоколы)          | Нет                                |
| **Поддержка протоколов**          | Да (через [[some]] / [[generic]]) | Нет               | Да                      | Да (если `@objc`)                  |
| **[[Overhead]] в памяти**         | Минимальный                       | Средний (vtable)  | Средний (witness table) | Большой (objc [[runtime]])         |
| **Инлайнинг компилятором**        | Да (часто)                        | Редко             | Редко                   | Почти никогда                      |
| **Где критично**                  | Горячие циклы, рендеринг          | Большинство кода  | Протоколы               | Редко (только Obj-C совместимость) |

### Подробные примеры кода

#### 3.1 Direct / Static Dispatch (самый быстрый)

```swift
class FastMath {
    // final → Direct Dispatch (нельзя переопределить)
    final func square(_ x: Int) -> Int {
        return x * x
    }
    
    // static → Direct Dispatch
    static func cube(_ x: Int) -> Int {
        return x * x * x
    }
    
    // private → Direct Dispatch (виден только внутри класса)
    private func internalHelper() {
        print("helper")
    }
}

let m = FastMath()
print(m.square(5))      // прямой вызов
print(FastMath.cube(3)) // прямой вызов

// Структуры и перечисления всегда используют Static Dispatch
struct Point {
    var x: Int
    var y: Int
    
    func distance() -> Double {  // Static Dispatch
        return sqrt(Double(x * x + y * y))
    }
}
```

#### 3.2 Table Dispatch (vtable) — обычные методы классов

```swift
class Animal {
    // обычный метод → Table Dispatch (vtable)
    func makeSound() {
        print("Generic animal sound")
    }
    
    // можно переопределить в наследниках
}

class Dog: Animal {
    override func makeSound() {
        print("Woof!")
    }
}

class Cat: Animal {
    override func makeSound() {
        print("Meow!")
    }
}

// Полиморфизм через Table Dispatch
let animals: [Animal] = [Dog(), Cat(), Animal()]
for animal in animals {
    animal.makeSound()  // vtable → правильная реализация
}
```

#### 3.3 Witness Table Dispatch — протоколы без @objc

```swift
protocol Drawable {
    func draw()
}

struct Circle: Drawable {
    func draw() { print("○") }
}

struct Square: Drawable {
    func draw() { print("□") }
}

// Existential container с witness table
let shapes: [any Drawable] = [Circle(), Square()]
for shape in shapes {
    shape.draw()  // witness table dispatch
}

// Оптимизированный вариант — Static Dispatch через generic
func draw<T: Drawable>(_ shape: T) {
    shape.draw()  // Static Dispatch (инлайнится)
}

draw(Circle())   // прямой вызов
```

#### 3.4 Message Dispatch — @objc / dynamic

```swift
import UIKit

class MyView: UIView {
    // @objc dynamic → Message Dispatch (objc_msgSend)
    @objc dynamic func didTap() {
        print("Tapped!")
    }
    
    // Обычный метод → Table Dispatch
    func normalMethod() {
        print("Normal")
    }
}

// KVO работает только с Message Dispatch
class Observer: NSObject {
    @objc dynamic var value: Int = 0
}

let obj = Observer()
obj.addObserver(
    NSKeyValueObservation?,
    forKeyPath: \.value,
    options: [.new],
    context: nil
)  // Требует @objc dynamic
```

### Производительность (примерные цифры 2026)

| Тип диспетчеризации | Вызов метода (нс) | Разница с Direct | Где критично |
|---------------------|-------------------|------------------|--------------|
| **Direct / Static** | ~1–2 нс | 1× | 60 fps UI, ML-инференс, аудиообработка, горячие циклы |
| **Table (vtable/witness)** | ~3–5 нс | 2–3× медленнее | Большинство кода |
| **Message (objc_msgSend)** | ~10–20 нс | 5–10× медленнее | Редко (только @objc совместимость) |

**Измерение производительности**:
```swift
import Darwin

func measure(_ name: String, _ block: () -> Void) {
    let start = mach_absolute_time()
    block()
    let end = mach_absolute_time()
    var info = mach_timebase_info()
    mach_timebase_info(&info)
    let elapsed = (end - start) * UInt64(info.numer) / UInt64(info.denom)
    print("\(name): \(elapsed) ns")
}

class Test {
    final func direct() { }
    func table() { }
    @objc dynamic func message() { }
}

let test = Test()
measure("Direct") { (0..<1_000_000).forEach { _ in test.direct() } }
measure("Table") { (0..<1_000_000).forEach { _ in test.table() } }
measure("Message") { (0..<1_000_000).forEach { _ in test.message() } }
```

### Лучшие практики и оптимизации (Swift 6+)

#### 1. **Максимальная скорость → используйте final, static, private, some**

```swift
// ✅ Хорошо — Direct Dispatch
class FastService {
    final func process() { }  // нельзя переопределить
    static func shared() { }   // прямой вызов
    private func helper() { }  // только внутри класса
}

// ✅ Хорошо — Struct всегда Static
struct FastData {
    func transform() { }  // Direct Dispatch
}

// ✅ Хорошо — some для протоколов
func makeDrawable() -> some Drawable {
    return Circle()  // статический тип
}
```

#### 2. **Протоколы → возвращайте [[some Protocol]], храните [[any Protocol]] только если нужно**

```swift
// ❌ Плохо — existential container с witness table
let shapes: [any Drawable] = [Circle(), Square()]

// ✅ Хорошо — статический тип
let circle = Circle()
let square = Square()

// ✅ Хорошо — generic
func process<T: Drawable>(_ shape: T) {
    shape.draw()  // статическая диспетчеризация
}
```

#### 3. **@objc / dynamic → только для [[KVO]], [[Delegate]], старого [[UIKit]]**

```swift
// ✅ Только когда действительно нужно
class LegacyView: UIView {
    @objc dynamic var animated: Bool = false  // для KVO
}

// ❌ Не нужно для обычных методов
class NormalClass {
    @objc func something() { }  // избыточно, замедляет
}
```

#### 4. **[[SwiftUI]] — some View, some ViewModel — стандарт (Direct / Witness)**

```swift
struct ContentView: View {
    var body: some View {  // some → Static Dispatch
        Text("Hello")
    }
}

@Observable
class ViewModel {  // @Observable → оптимизированная диспетчеризация
    var count = 0
}
```

#### 5. **Горячие пути (рендеринг, циклы, обработка кадров) — избегайте any и @objc**

```swift
// ❌ Плохо для горячего пути
func process(_ items: [any Drawable]) {
    for item in items {
        item.draw()  // witness table на каждой итерации
    }
}

// ✅ Хорошо для горячего пути
func process<T: Drawable>(_ items: [T]) {
    for item in items {
        item.draw()  // static dispatch
    }
}
```

#### 6. **Generics — `<T: Protocol>` вместо any для статической диспетчеризации**

```swift
// ❌ Existential — медленнее
func processAny(_ shape: any Drawable) {
    shape.draw()
}

// ✅ Generic — статическая диспетчеризация
func processGeneric<T: Drawable>(_ shape: T) {
    shape.draw()
}
```

#### 7. **Swift 6 strict concurrency — Static Dispatch сильно упрощает проверку потокобезопасности**

```swift
// Swift 6 — статическая диспетчеризация легче анализируется
final class SafeClass {
    func process() { }  // статически известен
}

// Existential types сложнее для анализа
let any: any Sendable = Something()
```

### Итог

**Диспетчеризация** в Swift определяет, как вызываются методы. Понимание четырех видов диспетчеризации (Direct, Table, Witness Table, Message) позволяет:

1.  **Оптимизировать горячие пути** — использовать [[final]], [[static]], [[private]], [[some]].
2.  **Избегать избыточных динамических вызовов** — [[any]] и @objc только по необходимости.
3.  **Писать более предсказуемый код** — статическая диспетчеризация упрощает отладку.
4.  **Улучшать производительность** — особенно в UI-рендеринге, аудиообработке, ML-инференсе.

Выбор правильного вида диспетчеризации — важная часть написания высокопроизводительного Swift-кода .
