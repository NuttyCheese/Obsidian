**Бинарный поиск** — это классический алгоритм поиска элемента в **отсортированном массиве** со сложностью **O(log n)**. Он работает путём многократного деления поискового диапазона пополам.

### Основные свойства

- Требует **отсортированного** массива
- Может возвращать **индекс** первого вхождения или **[[nil]]**
- Может быть реализован **итеративно** (предпочтительно) или **рекурсивно**
- Очень эффективен для больших массивов (миллионы элементов)
- Часто используется в [[iOS]] для поиска по ID, дате, рейтингу и т.д.

## 1. Итеративная реализация (самая рекомендуемая)

```swift
/// Возвращает индекс первого найденного элемента или nil
func binarySearch<T: Comparable>(_ array: [T], target: T) -> Int? {
    var low = 0
    var high = array.count - 1
    
    while low <= high {
        let mid = low + (high - low) / 2   // защита от переполнения
        
        if array[mid] == target {
            return mid
        } else if array[mid] < target {
            low = mid + 1
        } else {
            high = mid - 1
        }
    }
    
    return nil
}
```

### Защита от переполнения (очень важно для больших массивов)

```swift
let mid = low + (high - low) / 2    // правильно
// вместо
let mid = (low + high) / 2          // может переполниться на Int.max
```

## 2. Рекурсивная реализация

```swift
func binarySearchRecursive<T: Comparable>(
    _ array: [T],
    target: T,
    low: Int = 0,
    high: Int? = nil
) -> Int? {
    let high = high ?? array.count - 1
    guard low <= high else { return nil }
    
    let mid = low + (high - low) / 2
    
    if array[mid] == target {
        return mid
    } else if array[mid] < target {
        return binarySearchRecursive(array, target: target, low: mid + 1, high: high)
    } else {
        return binarySearchRecursive(array, target: target, low: low, high: mid - 1)
    }
}
```

## 3. Возврат диапазона всех вхождений (если элементы могут повторяться)

```swift
/// Возвращает диапазон индексов всех элементов, равных target
func binarySearchRange<T: Comparable>(_ array: [T], target: T) -> Range<Int>? {
    // Находим левую границу
    var left = 0
    var right = array.count
    while left < right {
        let mid = left + (right - left) / 2
        if array[mid] < target {
            left = mid + 1
        } else {
            right = mid
        }
    }
    let start = left
    
    // Находим правую границу
    left = 0
    right = array.count
    while left < right {
        let mid = left + (right - left) / 2
        if array[mid] <= target {
            left = mid + 1
        } else {
            right = mid
        }
    }
    let end = left
    
    return start < end ? start..<end : nil
}

// Пример
let arr = [1, 2, 2, 2, 3, 4, 5, 5, 5, 6]
if let range = binarySearchRange(arr, target: 2) {
    print(range)           // 1..<4
    print(arr[range])      // [2, 2, 2]
}
```

## 4. Бинарный поиск по ключу (KeyPath) — очень популярный паттерн

```swift
extension Array where Element: Comparable {
    func binarySearch<Key: Comparable>(
        by keyPath: KeyPath<Element, Key>,
        value: Key
    ) -> Int? {
        var low = 0
        var high = count - 1
        
        while low <= high {
            let mid = low + (high - low) / 2
            let midValue = self[mid][keyPath: keyPath]
            
            if midValue == value {
                return mid
            } else if midValue < value {
                low = mid + 1
            } else {
                high = mid - 1
            }
        }
        
        return nil
    }
}

// Пример использования
struct Product {
    let id: Int
    let name: String
    let price: Double
}

let products = [
    Product(id: 1001, name: "Phone", price: 799),
    Product(id: 1005, name: "Laptop", price: 1499),
    Product(id: 1012, name: "Tablet", price: 499)
]

if let index = products.binarySearch(by: \.id, value: 1005) {
    print("Найден продукт: \(products[index].name)")  // Laptop
}
```

## 5. Реальные сценарии в [[iOS]]-разработке

```swift
// Поиск ближайшей даты в отсортированном массиве событий
struct Event {
    let date: Date
    let title: String
}

let events: [Event] = // отсортировано по date

func findNearestEvent(to targetDate: Date) -> Event? {
    var low = 0
    var high = events.count - 1
    var closestIndex = 0
    var minDiff = Double.infinity
    
    while low <= high {
        let mid = low + (high - low) / 2
        let diff = abs(events[mid].date.timeIntervalSince(targetDate))
        
        if diff < minDiff {
            minDiff = diff
            closestIndex = mid
        }
        
        if events[mid].date < targetDate {
            low = mid + 1
        } else {
            high = mid - 1
        }
    }
    
    return events[closestIndex]
}
```

## 6. Сравнение с линейным поиском

```swift
// Линейный поиск — O(n)
if let index = events.firstIndex(where: { $0.id == targetID }) { ... }

// Бинарный поиск — O(log n)
if let index = binarySearch(events.map(\.id), target: targetID) { ... }
```

## 7. Важные замечания и ловушки

- Массив **должен быть отсортирован** по тому же критерию, по которому ищем
- [[swapAt]], `sorted`, [[sort]] — ваши друзья для подготовки массива
- Для структур реализуйте `Comparable` или используйте `KeyPath`
- При поиске диапазона используйте две границы (левая и правая)
- Для очень больших массивов (> 1 млн) бинарный поиск даёт огромный выигрыш

## Итог — когда использовать бинарный поиск в 2026 году

| Сценарий                                  | Использовать бинарный поиск?   |
| ----------------------------------------- | ------------------------------ |
| Отсортированный массив моделей по ID/дате | Да                             |
| Поиск в массиве > 10 000 элементов        | Да                             |
| Часто повторяющийся поиск по одному полю  | Да (лучше подготовить индекс)  |
| Массив не отсортирован                    | Нет → сначала `sorted()`       |
| Нужен индекс первого вхождения            | Да                             |
| Нужны все вхождения                       | Да (с range-версией)           |
| Маленький массив (< 100 элементов)        | Можно [[firstIndex]]`(where:)` |

Бинарный поиск — один из тех алгоритмов, которые **всегда** стоит держать в арсенале iOS-разработчика.  
Он прост в реализации, невероятно быстр и часто встречается в задачах на интервью и реальных приложениях.
