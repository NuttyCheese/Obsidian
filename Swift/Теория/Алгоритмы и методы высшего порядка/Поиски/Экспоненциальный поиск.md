# Экспоненциальный поиск (Exponential Search) в [[Swift]]

**Экспоненциальный поиск** — это гибридный алгоритм, который сочетает **линейное удвоение диапазона** и **бинарный поиск**. Он особенно эффективен, когда искомый элемент находится **ближе к началу** большого отсортированного массива.

### Идея работы

1. Начинаем с позиции 1 и **удваиваем** индекс (1 → 2 → 4 → 8 → 16 …), пока не найдём блок, в котором может находиться элемент (т.е. `array[i] >= target`).
2. Как только блок найден — выполняем **бинарный поиск** только в пределах последнего удвоенного диапазона `[i/2 … min(i, n-1)]`.
3. Это позволяет быстро «подобраться» к нужной области, а затем точно найти элемент.

### Сложность

- **Средняя и худшая**: O(log n) — за счёт бинарного поиска в диапазоне
- **Лучшая**: O(1) — если элемент первый
- По сравнению с бинарным поиском: быстрее, если элемент в начале массива, чуть медленнее — если в конце

### Когда экспоненциальный поиск действительно полезен

- Массив **очень большой** (сотни тысяч — миллионы элементов)
- Элемент с высокой вероятностью находится **ближе к началу**
- Доступ к элементам **дорогой** (например, чтение из файла, [[API]], базы данных)
- Данные **отсортированы**, но не известно заранее, где находится искомое значение

В большинстве [[iOS]]-приложений чаще используют просто **бинарный поиск** или [[firstIndex]]`(where:)`, но экспоненциальный поиск — отличный инструмент для собеседований и специфических оптимизаций.

## 1. Полная итеративная реализация

```swift
/// Возвращает индекс первого найденного элемента или nil
func exponentialSearch<T: Comparable>(_ array: [T], target: T) -> Int? {
    if array.isEmpty { return nil }
    if array[0] == target { return 0 }
    
    // Шаг 1: экспоненциальное удвоение диапазона
    var bound = 1
    while bound < array.count && array[bound] < target {
        bound *= 2
    }
    
    // Шаг 2: бинарный поиск в диапазоне [bound/2 … min(bound, count-1)]
    let left = bound / 2
    let right = min(bound, array.count - 1)
    
    return binarySearch(array, target: target, low: left, high: right)
}

// Вспомогательный бинарный поиск (с защитой от переполнения)
func binarySearch<T: Comparable>(_ array: [T], target: T, low: Int, high: Int) -> Int? {
    var low = low
    var high = high
    
    while low <= high {
        let mid = low + (high - low) / 2
        
        if array[mid] == target {
            return mid
        } else if array[mid] < target {
            low = mid + 1
        } else {
            high = mid - 1
        }
    }
    
    return nil
}
```

## 2. Версия с KeyPath (для массивов структур)

```swift
extension Array {
    func exponentialSearch<Key: Comparable>(
        by keyPath: KeyPath<Element, Key>,
        target: Key
    ) -> Int? {
        guard !isEmpty else { return nil }
        if self[0][keyPath: keyPath] == target { return 0 }
        
        var bound = 1
        while bound < count && self[bound][keyPath: keyPath] < target {
            bound *= 2
        }
        
        let left = bound / 2
        let right = min(bound, count - 1)
        
        var low = left
        var high = right
        
        while low <= high {
            let mid = low + (high - low) / 2
            let midValue = self[mid][keyPath: keyPath]
            
            if midValue == target {
                return mid
            } else if midValue < target {
                low = mid + 1
            } else {
                high = mid - 1
            }
        }
        
        return nil
    }
}
```

## 3. Реальные примеры использования

### Пример 1 — поиск в большом массиве ID

```swift
let ids = Array(100_000...200_000)  // 100 001 элементов

if let index = exponentialSearch(ids, target: 150_342) {
    print("ID найден на позиции \(index)")  // ≈ 50342
}
```

### Пример 2 — поиск по дате в логе событий

```swift
struct LogEvent {
    let timestamp: Int  // Unix timestamp
    let message: String
}

let logs = (0..<1_000_000).map { i in
    LogEvent(timestamp: 1_700_000_000 + i * 60, message: "Event \(i)")
}

let targetTime = 1_700_030_000
if let index = logs.exponentialSearch(by: \.timestamp, target: targetTime) {
    print("Событие найдено: \(logs[index].message)")
}
```

### Пример 3 — поиск ближайшего значения (модификация)

```swift
func findClosestExponential<T: Comparable & SignedNumeric>(
    _ array: [T],
    target: T
) -> (index: Int, value: T)? {
    guard !array.isEmpty else { return nil }
    
    var bound = 1
    while bound < array.count && array[bound] < target {
        bound *= 2
    }
    
    let left = bound / 2
    let right = min(bound, array.count - 1)
    
    var closestIndex = left
    var minDiff = abs(target - array[left])
    
    for i in left...right {
        let diff = abs(target - array[i])
        if diff < minDiff {
            minDiff = diff
            closestIndex = i
        }
    }
    
    return (closestIndex, array[closestIndex])
}

let temperatures = stride(from: -50, through: 50, by: 0.5).map { $0 }
if let closest = findClosestExponential(temperatures, target: 23.7) {
    print("Ближайшая температура: \(closest.value)°C на индексе \(closest.index)")
}
```

## 4. Сравнение с другими алгоритмами

| Алгоритм              | Сложность (средняя) | Лучше всего работает, если элемент… | Требует сортировки |
|-----------------------|----------------------|---------------------------------------|---------------------|
| Линейный              | O(n)                 | в начале массива                      | Нет                 |
| Бинарный              | O(log n)             | в любой части                         | Да                  |
| Экспоненциальный      | O(log n)             | ближе к началу                        | Да                  |
| Интерполяционный      | O(log log n)         | равномерно распределён                | Да                  |

## Итог — когда использовать экспоненциальный поиск в 2026 году

| Ситуация                                      | Рекомендация                          |
|-----------------------------------------------|---------------------------------------|
| Элемент с высокой вероятностью в начале       | Да, экспоненциальный поиск            |
| Очень большой массив (>100 000 элементов)     | Да, особенно если доступ дорогой      |
| Данные отсортированы по возрастанию           | Да                                    |
| Элемент скорее всего в конце массива          | Нет → обычный бинарный поиск          |
| Массив небольшой                              | Нет → `firstIndex(where:)`            |
| Нужно максимальная надёжность                 | Бинарный поиск                        |

Экспоненциальный поиск — это **нишевый**, но очень красивый и эффективный алгоритм.  
В большинстве iOS-приложений его заменяют **бинарный поиск** или **функции высшего порядка**, но знание этого алгоритма отлично демонстрирует понимание оптимизаций поиска.
