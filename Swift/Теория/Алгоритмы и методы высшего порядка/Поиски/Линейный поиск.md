# Линейный поиск (Linear Search) в Swift

**Линейный поиск** — самый простой и универсальный алгоритм поиска элемента в коллекции. Он последовательно проверяет каждый элемент, пока не найдёт совпадение или не дойдёт до конца.

### Основные свойства

- Сложность: **O(n)** в среднем и худшем случае, **O(1)** в лучшем
- Работает на **любых** коллекциях (отсортированных и неотсортированных)
- Не требует подготовки данных
- Очень прост в реализации и понимании
- В [[Swift]] почти всегда заменяется встроенными функциями высшего порядка

### Когда использовать в iOS-разработке (2026)

- Небольшие массивы моделей (< 1000–5000 элементов)
- Поиск по неотсортированным данным
- Поиск по сложным условиям (несколько полей, fuzzy-логика)
- Поиск в [[Dictionary]], [[Set]], маленьких списках в UI
- Когда важна простота кода, а не производительность

## 1. Классическая ручная реализация

```swift
/// Возвращает индекс первого найденного элемента или nil
func linearSearch<T: Equatable>(_ array: [T], target: T) -> Int? {
    for (index, element) in array.enumerated() {
        if element == target {
            return index
        }
    }
    return nil
}
```

## 2. Современный и рекомендуемый способ — [[firstIndex]](where:)

```swift
let numbers = [7, 3, 9, 1, 5, 8, 2]

// Поиск индекса
if let index = numbers.firstIndex(of: 5) {          // если Equatable
    print("Индекс: \(index)")                       // → 4
}

if let index = numbers.firstIndex(where: { $0 > 6 }) {
    print("Первый > 6 находится на позиции \(index)") // → 2 (9)
}
```

## 3. [[first]](where:) — когда нужен сам элемент

```swift
struct Product {
    let id: String
    let name: String
    let price: Double
    let category: String
}

let products = [
    Product(id: "P1", name: "iPhone", price: 999, category: "Phone"),
    Product(id: "P2", name: "MacBook", price: 1999, category: "Laptop"),
    Product(id: "P3", name: "AirPods", price: 199, category: "Accessories")
]

// Первый продукт дороже 1500
if let expensive = products.first(where: { $0.price > 1500 }) {
    print("Самый дорогой: \(expensive.name) за $\(expensive.price)")
    // → MacBook за $1999
}

// Первый аксессуар
if let accessory = products.first(where: { $0.category == "Accessories" }) {
    print("Найден аксессуар: \(accessory.name)")
}
```

## 4. Поиск по нескольким условиям

```swift
let users = [
    (name: "Anna", age: 28, isActive: true),
    (name: "Ben",  age: 19, isActive: false),
    (name: "Clara", age: 34, isActive: true),
    (name: "David", age: 28, isActive: true)
]

// Первый активный пользователь старше 30
if let user = users.first(where: { $0.isActive && $0.age > 30 }) {
    print("Найден: \(user.name), возраст \(user.age)")
    // → Clara, возраст 34
}

// Первый пользователь с возрастом 28, который активен
if let match = users.first(where: { $0.age == 28 && $0.isActive }) {
    print("Найден: \(match.name)")
    // → Anna или David (первый в списке)
}
```

## 5. Поиск в [[Dictionary]] и [[Set]]

```swift
let config = [
    "theme": "dark",
    "fontSize": "17",
    "notifications": "true",
    "language": "ru"
]

// Поиск по значению
if let entry = config.first(where: { $0.value == "ru" }) {
    print("Ключ для русского языка: \(entry.key)") // → "language"
}

// Поиск ключа по значению (часто встречающийся паттерн)
let keyForDarkTheme = config.first(where: { $0.value == "dark" })?.key

// В Set — проверка наличия
let activeIDs: Set<Int> = [1001, 1005, 1012, 1020]
if activeIDs.contains(1005) {
    print("Пользователь 1005 активен")
}
```

## 6. Поиск с ранним выходом (Sentinel / Guard)

```swift
func findFirstMatching<T>(_ items: [T], predicate: (T) -> Bool) -> T? {
    for item in items {
        if predicate(item) {
            return item
        }
    }
    return nil
}

// или просто
let firstAdult = people.first { $0.age >= 18 }
```

## 7. Реальные сценарии из [[iOS]]-приложений

```swift
// 1. Поиск текущего выбранного элемента в списке настроек
let settingsOptions = ["Light", "Dark", "System"]
if let selectedIndex = settingsOptions.firstIndex(of: currentTheme) {
    tableView.selectRow(at: IndexPath(row: selectedIndex, section: 0), animated: false, scrollPosition: .none)
}

// 2. Поиск первой непрочитанной нотификации
if let unread = notifications.first(where: { !$0.isRead }) {
    showNotificationBanner(unread)
}

// 3. Поиск ближайшего сервера по пингу (маленький массив)
let servers = ["eu1", "eu2", "us1", "us2", "asia1"]
let pings = [45, 38, 120, 115, 180] // мс

if let bestIndex = pings.firstIndex(of: pings.min() ?? 0) {
    connectToServer(servers[bestIndex])
}
```

## 8. Сравнение производительности

| Количество элементов | Линейный поиск (firstIndex) | Бинарный поиск (если отсортировано) |
|-----------------------|------------------------------|--------------------------------------|
| 100                   | ~0.0001 мс                   | ~0.00001 мс                          |
| 1 000                 | ~0.001 мс                    | ~0.0001 мс                           |
| 10 000                | ~0.01 мс                     | ~0.0003 мс                           |
| 1 000 000             | ~10 мс                       | ~0.001 мс                            |

**Вывод 2026 года**:  
Для массивов < 5–10 тысяч элементов разница практически незаметна.  
Поэтому в 95% случаев в iOS используют именно `firstIndex(where:)` / `first(where:)`.

## Итог — современные рекомендации

| Ситуация                                      | Что использовать                     |
|-----------------------------------------------|--------------------------------------|
| Маленький массив (< 5000 элементов)           | `firstIndex(where:)` / `first(where:)` |
| Неотсортированные данные                      | Линейный поиск (всегда)              |
| Отсортированный массив > 10 000 элементов     | Бинарный поиск                       |
| Поиск по сложному условию                     | `first(where:)`                      |
| Поиск в `Dictionary` / `Set`                  | `first(where:)` или `contains`       |
| Нужно индекс первого вхождения                | `firstIndex(where:)`                 |
| Нужен сам элемент                             | `first(where:)`                      |

Линейный поиск в Swift почти всегда выражается через **функции высшего порядка** — они безопасны, читаемы и оптимизированы компилятором.
