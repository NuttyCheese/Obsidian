В [[Swift]] существуют два основных способа итерации по элементам коллекции (массивы, множества, словари, строки, диапазоны и любые типы, реализующие [[Sequence]] или [[Collection]]):

- **Цикл `for-in`** — классический императивный цикл с полным контролем над потоком выполнения.
- **Метод `forEach`** — функциональный метод коллекции, использующий замыкание.

Несмотря на внешнюю схожесть, они имеют **существенные различия** в поведении, возможностях и рекомендуемых сценариях использования. Выбор между ними влияет на читаемость, производительность в цепочках и возможность прерывания итерации.

## 1. Синтаксис и базовое использование

### [[for-in]] (цикл)

```swift
for item in collection {
    // тело цикла
}
```

- Поддерживает **все** `Sequence` (включая ленивые последовательности).
- Работает с паттернами (destructuring), `where`-условиями, индексами через `enumerated()`.

```swift
let numbers = [10, 20, 30, 40, 50]

// Простой проход
for number in numbers {
    print(number)
}

// С индексом
for (index, number) in numbers.enumerated() {
    print("[\(index)] → \(number)")
}

// Только чётные (с where)
for number in numbers where number % 2 == 0 {
    print(number)  // 10, 30, 50
}

// Деструктуризация
let pairs = [(1, "one"), (2, "two")]
for (id, name) in pairs {
    print("\(id): \(name)")
}
```

### [[forEach]] (метод коллекции)

```swift
collection.forEach { item in
    // тело замыкания
}
```

или короче:

```swift
collection.forEach { print($0) }
```

- Доступен на всех `Sequence`.
- Замыкание **не возвращает** значения (результат — `Void`).
- Часто используется в цепочках функциональных методов.

```swift
numbers.forEach { number in
    print(number)
}

// С индексом (через enumerated)
numbers.enumerated().forEach { index, number in
    print("[\(index)] → \(number)")
}
```

## 2. Ключевые различия (2026 год)

| Аспект                         | for-in                                       | forEach                                                                                      | Комментарий / Рекомендация 2026                    |
| ------------------------------ | -------------------------------------------- | -------------------------------------------------------------------------------------------- | -------------------------------------------------- |
| **Прерывание цикла**           | Поддерживает `break`, `continue`             | Не поддерживает `break` / `continue`                                                         | Если нужен ранний выход — только `for-in`          |
| **return внутри**              | Выходит из функции / замыкания / цикла       | Выходит **только** из текущего вызова замыкания (пропускает остаток тела для этого элемента) | `return` в `forEach` **не прерывает** всю итерацию |
| **throw**                      | Можно бросить ошибку (прерывает цикл)        | Можно бросить ошибку (прерывает весь вызов)                                                  | Оба поддерживают throwing-замыкания                |
| **Изменение коллекции внутри** | Безопасно (если коллекция var)               | Опасно — часто приводит к ошибкам (изменение во время итерации)                              | Избегайте мутации внутри `forEach`                 |
| **Производительность**         | Обычно чуть быстрее (нет overhead замыкания) | Почти идентична в релизе (-O)                                                                | Разница < 1–3% — не критично                       |
| **Читаемость в цепочках**      | Требует отдельной строки                     | Идеально вписывается в .[[map]]().[[filter]]().[[forEach]]()                                 | `forEach` выигрывает в функциональном стиле        |
| **Ленивость**                  | Работает с lazy-последовательностями         | Работает, но часто материализует раньше                                                      | Оба ок, но `for-in` более предсказуем              |
| **Поддержка labeled break**    | Да (`break outerLoop`)                       | Нет                                                                                          | Для вложенных циклов — только `for-in`             |

### Примеры различий в контроле потока

```swift
// for-in: break работает
for number in 1...10 {
    if number == 5 { break }
    print(number)           // → 1 2 3 4
}

// forEach: break НЕ компилируется
numbers.forEach { number in
    if number == 5 { break }    // Ошибка компиляции!
    print(number)
}

// forEach: return только пропускает тело текущей итерации
[1, 2, 3, 4, 5].forEach { number in
    if number == 3 { return }   // Пропускает print для 3, но продолжает дальше
    print(number)               // → 1 2 4 5
}

// Прерывание forEach через throw (редкий, но рабочий способ)
do {
    try numbers.forEach { number throws in
        if number == 4 { throw CancellationError() }
        print(number)
    }
} catch {
    print("Прервано на 4")
}
```

## 3. Мутация коллекции внутри итерации

**Важно в 2026 году**: мутировать коллекцию во время итерации — **опасно** в обоих случаях, но особенно в `forEach`.

```swift
var items = [1, 2, 3, 4, 5]

// Опасно в for-in (может пропустить/повторить элементы)
for item in items where item % 2 == 0 {
    items.removeAll { $0 == item }   // → undefined behavior
}

// Ещё опаснее в forEach
items.forEach { item in
    if item > 3 {
        items.removeAll { $0 == item }   // Crash или UB почти гарантирован
    }
}
```

**Рекомендация**:  
- Хотите мутировать → используйте `removeAll(where:)`, `filter`, или собирайте индексы и удаляйте после цикла.  
- Никогда не мутируйте коллекцию внутри `forEach`.

## 4. Когда использовать что (современные рекомендации 2025–2026)

| Сценарий                                          | Рекомендация          | Почему |
|---------------------------------------------------|-----------------------|------|
| Простой проход без условий и прерываний           | `forEach`             | Короткий, функциональный стиль, хорошо в цепочках |
| Нужен `break`, `continue`, `return` из функции    | `for-in`              | Единственный вариант |
| Вложенные циклы с labeled break/continue          | `for-in`              | `forEach` не поддерживает |
| Цепочка: `.map().filter().forEach { ... }`        | `forEach`             | Естественно вписывается |
| Итерация + ранний выход при первом совпадении     | `for-in` + `break` или `first(where:)` | `forEach` неудобен |
| Логирование, сайд-эффекты без возврата            | `forEach`             | Более декларативно |
| Работа с индексами + мутация                      | `for-in` с `indices`  | Полный контроль |
| SwiftUI / декларативный UI                        | `ForEach { ... }`     | Это отдельная view-структура, не путать с `Sequence.forEach` |

## 5. Полезные паттерны и антипаттерны

```swift
// Хорошо: сайд-эффекты в цепочке
users
    .filter { $0.isActive }
    .forEach { user in
        Analytics.track(event: "active_user_view", userID: user.id)
    }

// Плохо: попытка break в forEach
items.forEach { item in
    if shouldStop { break }   // ← ошибка
    process(item)
}

// Лучше:
for item in items {
    if shouldStop { break }
    process(item)
}

// Альтернатива для прерывания без throw
let firstMatch = items.first { $0.matchesCondition }
```

## Итог — современный взгляд 2026

- **Используйте `forEach`**, когда:
  - код функциональный
  - нет необходимости в `break`/`continue`
  - вы в цепочке методов
  - хотите подчеркнуть сайд-эффекты без возврата значения

- **Используйте `for-in`**, когда:
  - нужен контроль потока (`break`, `continue`, ранний `return`)
  - мутируете коллекцию (или работаете с индексами)
  - вложенные циклы
  - производительность критична (хоть разница и мала)

В 90% повседневного кода выбор — вопрос стиля и команды. Но понимание этих различий помогает писать **безопасный**, **эффективный** и **предсказуемый** код.
