**Insertion Sort** — простой и интуитивно понятный алгоритм сортировки, который работает по принципу «строим отсортированную часть постепенно, вставляя каждый новый элемент на правильное место».

### Основные характеристики

- Сложность:
  - Лучший случай: **O(n)** — массив уже отсортирован
  - Средний и худший случай: **O(n²)**
- Память: **O(1)** дополнительной (in-place)
- **Стабильная** сортировка — сохраняет относительный порядок равных элементов
- Очень эффективна для **маленьких массивов** и **почти отсортированных данных**
- Хорошо работает с потоковыми данными (онлайн-сортировка)

### Когда Insertion Sort действительно полезна в iOS (2026)

- Маленькие списки (< 100–200 элементов)
- Почти отсортированные данные (например, новые элементы добавляются в конец)
- Анимации сортировки в учебных приложениях ([[UITableView]] / [[UICollectionView]])
- Постепенная сортировка поступающих событий / сообщений / транзакций
- Когда важна **стабильность** и простота кода

В реальных приложениях **почти всегда** используют встроенные `sorted(by:)` или [[sort]]() (TimSort), но понимание Insertion Sort полезно для обучения, собеседований и специфических задач.

## 1. Классическая in-place реализация

```swift
extension Array where Element: Comparable {
    mutating func insertionSort() {
        for i in 1..<count {
            var j = i
            let key = self[j]
            
            // Сдвигаем элементы вправо, пока не найдём место для key
            while j > 0 && self[j - 1] > key {
                self[j] = self[j - 1]
                j -= 1
            }
            
            self[j] = key
        }
    }
}

// Пример
var numbers = [64, 34, 25, 12, 22, 11, 90]
numbers.insertionSort()
print(numbers)  // [11, 12, 22, 25, 34, 64, 90]
```

## 2. Универсальная версия с замыканием (аналог sorted(by:))

```swift
extension Array {
    mutating func insertionSort<T>(by areInIncreasingOrder: (Element, Element) -> Bool) where Element == T {
        for i in 1..<count {
            var j = i
            let key = self[j]
            
            while j > 0 && !areInIncreasingOrder(self[j - 1], key) {
                self[j] = self[j - 1]
                j -= 1
            }
            
            self[j] = key
        }
    }
}
```

## 3. Реальные примеры использования

### Пример 1 — сортировка задач по дедлайну (стабильность важна)

```swift
struct Task {
    let dueDate: Date
    let title: String
    let priority: Int
}

var tasks = [
    Task(dueDate: Date().addingTimeInterval(86400), title: "Meeting A", priority: 2),
    Task(dueDate: Date().addingTimeInterval(3600),  title: "Fix bug", priority: 1),
    Task(dueDate: Date().addingTimeInterval(86400), title: "Meeting B", priority: 2),
    Task(dueDate: Date().addingTimeInterval(7200),  title: "Review PR", priority: 3)
]

tasks.insertionSort { $0.dueDate < $1.dueDate }

// Стабильность: Meeting A остаётся перед Meeting B (оба на один день)
print(tasks.map { $0.title })
// ["Fix bug", "Review PR", "Meeting A", "Meeting B"]
```

### Пример 2 — анимация сортировки в UITableView

```swift
var demoItems = [8, 3, 5, 1, 7, 2, 9]

// В контроллере можно анимировать каждый сдвиг
func animateInsertionSort() {
    for i in 1..<demoItems.count {
        var j = i
        let key = demoItems[j]
        
        while j > 0 && demoItems[j - 1] > key {
            // Анимация сдвига ячеек
            UIView.animate(withDuration: 0.3) {
                // визуально меняем местами ячейки j-1 и j
            }
            
            demoItems[j] = demoItems[j - 1]
            j -= 1
            
            tableView.reloadRows(at: [IndexPath(row: j, section: 0),
                                      IndexPath(row: j + 1, section: 0)],
                                 with: .automatic)
        }
        
        demoItems[j] = key
        tableView.reloadRows(at: [IndexPath(row: j, section: 0)], with: .automatic)
    }
}
```

### Пример 3 — сортировка словаря по значениям (через массив)

```swift
let scores = [
    "Alice": 92,
    "Bob":   78,
    "Clara": 95,
    "David": 85
]

var scorePairs = Array(scores)
scorePairs.insertionSort { $0.value < $1.value }

print(scorePairs.map { "\($0.key): \($0.value)" })
// ["Bob: 78", "David: 85", "Alice: 92", "Clara: 95"]
```

### Пример 4 — онлайн-сортировка (поступающие данные)

```swift
var sortedLog = [LogEvent]()

func addLog(_ event: LogEvent) {
    sortedLog.append(event)
    
    // Вставляем новый элемент на правильное место
    var j = sortedLog.count - 1
    while j > 0 && sortedLog[j - 1].timestamp > event.timestamp {
        sortedLog[j] = sortedLog[j - 1]
        j -= 1
    }
    sortedLog[j] = event
}
```

### Пример 5 — сортировка по нескольким критериям

```swift
struct Student {
    let grade: Int
    let name: String
}

var students = [
    Student(grade: 85, name: "Anna"),
    Student(grade: 92, name: "Ben"),
    Student(grade: 85, name: "Clara"),
    Student(grade: 78, name: "David")
]

students.insertionSort { a, b in
    if a.grade != b.grade {
        return a.grade < b.grade
    }
    return a.name < b.name
}

// Результат: сначала по оценке, внутри оценки — по имени
// David (78), Anna (85), Clara (85), Ben (92)
```

## 4. Сравнение Insertion Sort с другими алгоритмами

| Алгоритм          | Сложность (средняя) | Сложность (худшая) | Стабильность | In-place | Лучше всего работает на… |
|-------------------|----------------------|---------------------|--------------|----------|---------------------------|
| Insertion Sort    | O(n²)                | O(n²)               | Да           | Да       | Почти отсортированных     |
| Bubble Sort       | O(n²)                | O(n²)               | Да           | Да       | Маленькие массивы         |
| Quick Sort        | O(n log n)           | O(n²)               | Нет          | Да       | Случайные данные          |
| Heap Sort         | O(n log n)           | O(n log n)          | Нет          | Да       | Гарантированная сложность |
| TimSort (Swift)   | O(n log n)           | O(n log n)          | Да           | Нет      | Реальные данные           |

## Итог — современные рекомендации (2026)

| Ситуация                                      | Что использовать                     | Почему |
|-----------------------------------------------|--------------------------------------|--------|
| Обычная сортировка в приложении               | `sorted(by:)` или `sort()`           | TimSort — стабильный, быстрый, читаемый |
| Учебный проект / анимация сортировки          | Insertion Sort                       | Простота и наглядность вставок |
| Очень маленький массив (< 50 элементов)       | Insertion Sort                       | Разница незаметна |
| Почти отсортированный массив (онлайн-данные)  | Insertion Sort                       | Близко к O(n) |
| Стабильность сортировки важна                 | `sorted(by:)` (TimSort) или Insertion Sort | Оба стабильны |
| Большой массив                                | Никогда Insertion Sort               | O(n²) — неприемлемо |
| Обучение / собеседование                      | Ручная реализация Insertion Sort     | Показывает понимание базовых алгоритмов |

**Короткое правило для iOS-разработчика**:
> Insertion Sort — отличный алгоритм для **обучения**, **анимаций** и **маленьких / почти отсортированных** данных.  
> Для любых реальных приложений используйте `sorted(by:)` или `sort()` — они быстрее, стабильнее и безопаснее.
