# Быстрая сортировка (Quick Sort) в [[Swift]]

**Quick Sort** — один из самых популярных и эффективных алгоритмов сортировки на практике. Он относится к категории **divide-and-conquer** («разделяй и властвуй») и использует стратегию выбора **опорного элемента (pivot)** с последующим разделением массива.

### Основные характеристики

- Средняя сложность: **O(n log n)**
- Худшая сложность: **O(n²)** (при неудачном выборе pivot, например, уже отсортированный массив + pivot = первый/последний элемент)
- Лучшая сложность: **O(n log n)**
- **Нестабильная** сортировка (не сохраняет относительный порядок равных элементов)
- **In-place** (может сортировать без дополнительной памяти, кроме стека вызовов)
- Очень быстра на практике благодаря хорошей локальности данных и кэш-дружественности

### Когда использовать Quick Sort в iOS-разработке (2026)

- Сортировка больших массивов моделей перед отображением в таблице/коллекции
- Подготовка данных для [[UITableView]] / [[UICollectionView]] / List в SwiftUI
- Сортировка результатов поиска, логов, транзакций, событий
- Когда важна высокая скорость и память ограничена

Однако в большинстве случаев **рекомендуется** использовать встроенный метод `sorted(by:)` — он использует **TimSort** (гибрид Merge Sort + Insertion Sort) и является стабильным.

## 1. Классическая рекурсивная реализация (простая версия)

```swift
func quickSort<T: Comparable>(_ array: [T]) -> [T] {
    guard array.count > 1 else { return array }
    
    let pivot = array[array.count / 2]
    
    let less    = array.filter { $0 < pivot }
    let equal   = array.filter { $0 == pivot }
    let greater = array.filter { $0 > pivot }
    
    return quickSort(less) + equal + quickSort(greater)
}
```

### Проблема этой версии

- Создаёт **много временных массивов** → O(n log n) по времени + **O(n log n)** по памяти
- Не является in-place

## 2. In-place реализация Quick Sort (рекомендуемая для обучения)

```swift
extension Array where Element: Comparable {
    mutating func quickSortInPlace() {
        quickSortInPlaceHelper(low: 0, high: count - 1)
    }
    
    private mutating func quickSortInPlaceHelper(low: Int, high: Int) {
        guard low < high else { return }
        
        let pivotIndex = partition(low: low, high: high)
        
        quickSortInPlaceHelper(low: low, high: pivotIndex - 1)
        quickSortInPlaceHelper(low: pivotIndex + 1, high: high)
    }
    
    private mutating func partition(low: Int, high: Int) -> Int {
        let pivot = self[high]           // последний элемент как pivot
        var i = low - 1
        
        for j in low..<high {
            if self[j] <= pivot {
                i += 1
                swapAt(i, j)
            }
        }
        
        swapAt(i + 1, high)
        return i + 1
    }
}

// Пример
var numbers = [64, 34, 25, 12, 22, 11, 90]
numbers.quickSortInPlace()
print(numbers)  // [11, 12, 22, 25, 34, 64, 90]
```

## 3. Универсальная версия с замыканием (аналог sorted(by:))

```swift
extension Array {
    func quickSorted<T>(by areInIncreasingOrder: (Element, Element) -> Bool) -> [T] {
        var copy = self as! [T]
        copy.quickSortInPlace(by: areInIncreasingOrder)
        return copy
    }
    
    mutating func quickSortInPlace(by areInIncreasingOrder: (Element, Element) -> Bool) {
        quickSortHelper(low: 0, high: count - 1, by: areInIncreasingOrder)
    }
    
    private mutating func quickSortHelper(low: Int, high: Int, by areInIncreasingOrder: (Element, Element) -> Bool) {
        guard low < high else { return }
        
        let pivotIndex = partition(low: low, high: high, by: areInIncreasingOrder)
        
        quickSortHelper(low: low, high: pivotIndex - 1, by: areInIncreasingOrder)
        quickSortHelper(low: pivotIndex + 1, high: high, by: areInIncreasingOrder)
    }
    
    private mutating func partition(low: Int, high: Int, by areInIncreasingOrder: (Element, Element) -> Bool) -> Int {
        let pivot = self[high]
        var i = low - 1
        
        for j in low..<high {
            if areInIncreasingOrder(self[j], pivot) {
                i += 1
                swapAt(i, j)
            }
        }
        
        swapAt(i + 1, high)
        return i + 1
    }
}
```

## 4. Реальные примеры использования

### Пример 1 — сортировка моделей по нескольким критериям

```swift
struct Task {
    let priority: Int      // 1 — высокий, 3 — низкий
    let dueDate: Date
    let title: String
}

var tasks = [
    Task(priority: 2, dueDate: Date().addingTimeInterval(86400), title: "Meeting"),
    Task(priority: 1, dueDate: Date(), title: "Fix bug"),
    Task(priority: 3, dueDate: Date().addingTimeInterval(172800), title: "Write docs")
]

// Сортировка: сначала по приоритету (меньше — важнее), затем по дате
tasks.quickSortInPlace { a, b in
    if a.priority != b.priority {
        return a.priority < b.priority
    }
    return a.dueDate < b.dueDate
}
```

### Пример 2 — сортировка словаря по значениям

```swift
let scores = [
    "Alice": 92,
    "Bob":   78,
    "Clara": 95,
    "David": 85
]

// Сортировка по убыванию оценок
let sortedScores = scores.sorted { $0.value > $1.value }
print(sortedScores)  // [("Clara", 95), ("Alice", 92), ("David", 85), ("Bob", 78)]
```

### Пример 3 — подготовка данных для UITableView

```swift
struct Message {
    let date: Date
    let text: String
    let isFromMe: Bool
}

var messages: [Message] = // загружены из API

// Сортируем по дате (от старых к новым)
messages.quickSortInPlace { $0.date < $1.date }

// или через встроенный метод (рекомендуется)
messages = messages.sorted { $0.date < $1.date }
```

### Пример 4 — случайная выборка pivot (улучшение против худшего случая)

```swift
extension Array {
    mutating func quickSortRandomPivot<T>(by areInIncreasingOrder: (T, T) -> Bool) where Element == T {
        quickSortHelperRandom(low: 0, high: count - 1, by: areInIncreasingOrder)
    }
    
    private mutating func quickSortHelperRandom(low: Int, high: Int, by areInIncreasingOrder: (Element, Element) -> Bool) {
        guard low < high else { return }
        
        // Случайный pivot
        let pivotIndex = Int.random(in: low...high)
        swapAt(pivotIndex, high)
        
        let newPivot = partition(low: low, high: high, by: areInIncreasingOrder)
        
        quickSortHelperRandom(low: low, high: newPivot - 1, by: areInIncreasingOrder)
        quickSortHelperRandom(low: newPivot + 1, high: high, by: areInIncreasingOrder)
    }
}
```

## Итог — современные рекомендации (2026)

| Ситуация                                      | Что использовать                     | Почему |
|-----------------------------------------------|--------------------------------------|--------|
| Обычная сортировка по одному полю             | `sorted { $0 < $1 }` или `sorted(by: \.field)` | Простота, стабильность, TimSort |
| Сортировка на месте (экономия памяти)         | `sort()` или своя in-place Quick Sort | Меньше аллокаций |
| Сортировка по нескольким критериям            | `sorted { … }` с if / кортеж         | Читаемо |
| Критична максимальная скорость + большой массив | `sort()` (TimSort) или Quick Sort с random pivot | TimSort лучше в худшем случае |
| Нужно стабильность сортировки                 | `sorted(by:)` (TimSort)              | Quick Sort нестабильный |
| Обучение / собеседование                     | Ручная реализация Quick Sort         | Показывает понимание алгоритмов |

**Короткое правило для iOS-разработчика**:
> В 98% случаев используй `sorted(by:)` или `sort()`.  
> Ручную Quick Sort пиши только для обучения или очень специфичных задач с жёсткими ограничениями по памяти.
