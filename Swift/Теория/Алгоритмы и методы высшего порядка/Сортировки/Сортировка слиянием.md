# Сортировка слиянием (Merge Sort) в [[Swift]]

**Merge Sort** — классический алгоритм сортировки, основанный на принципе «разделяй и властвуй». Он делит массив на две половины, рекурсивно сортирует каждую, а затем сливает (merge) их в один отсортированный массив.

### Основные характеристики

- Сложность: **O(n log n)** во всех случаях (лучший, средний, худший)
- Память: **O(n)** дополнительной (для временных массивов при слиянии)
- **Стабильная** сортировка — сохраняет относительный порядок равных элементов
- Хорошо подходит для **больших массивов**, **внешней сортировки** (диск, поток) и **связных списков**
- Не in-place (требует дополнительной памяти)

### Когда Merge Sort действительно полезна в [[iOS]] (2026)

- Когда **стабильность сортировки критична** (например, сортировка по нескольким полям)
- Сортировка больших массивов моделей перед отображением в таблице/коллекции
- Обработка потоковых данных (онлайн-сортировка поступающих элементов)
- Работа со связными списками (Linked List), где Merge Sort особенно эффективен
- Учебные проекты, демонстрация рекурсии и divide-and-conquer

В реальных iOS-приложениях **почти всегда** используют встроенный метод `sorted(by:)` — он реализует **TimSort** (гибрид Merge Sort + Insertion Sort), который стабильный, быстрый и оптимизирован для реальных данных.

## 1. Классическая рекурсивная реализация

```swift
func mergeSort<T: Comparable>(_ array: [T]) -> [T] {
    guard array.count > 1 else { return array }
    
    let middle = array.count / 2
    let left  = mergeSort(Array(array[0..<middle]))
    let right = mergeSort(Array(array[middle..<array.count]))
    
    return merge(left, right)
}

func merge<T: Comparable>(_ left: [T], _ right: [T]) -> [T] {
    var result = [T]()
    var leftIndex = 0
    var rightIndex = 0
    
    while leftIndex < left.count && rightIndex < right.count {
        if left[leftIndex] <= right[rightIndex] {
            result.append(left[leftIndex])
            leftIndex += 1
        } else {
            result.append(right[rightIndex])
            rightIndex += 1
        }
    }
    
    // Добавляем оставшиеся элементы
    result.append(contentsOf: left[leftIndex...])
    result.append(contentsOf: right[rightIndex...])
    
    return result
}

// Пример
let numbers = [64, 34, 25, 12, 22, 11, 90]
let sorted = mergeSort(numbers)
print(sorted)  // [11, 12, 22, 25, 34, 64, 90]
```

## 2. Универсальная версия с замыканием (аналог sorted(by:))

```swift
func mergeSort<T>(_ array: [T], by areInIncreasingOrder: (T, T) -> Bool) -> [T] {
    guard array.count > 1 else { return array }
    
    let middle = array.count / 2
    let left  = mergeSort(Array(array[0..<middle]), by: areInIncreasingOrder)
    let right = mergeSort(Array(array[middle..<array.count]), by: areInIncreasingOrder)
    
    return merge(left, right, by: areInIncreasingOrder)
}

func merge<T>(_ left: [T], _ right: [T], by areInIncreasingOrder: (T, T) -> Bool) -> [T] {
    var result = [T]()
    var leftIndex = 0
    var rightIndex = 0
    
    while leftIndex < left.count && rightIndex < right.count {
        if areInIncreasingOrder(left[leftIndex], right[rightIndex]) {
            result.append(left[leftIndex])
            leftIndex += 1
        } else {
            result.append(right[rightIndex])
            rightIndex += 1
        }
    }
    
    result.append(contentsOf: left[leftIndex...])
    result.append(contentsOf: right[rightIndex...])
    
    return result
}
```

## 3. In-place версия Merge Sort (с дополнительной памятью только для временного буфера)

```swift
extension Array where Element: Comparable {
    mutating func mergeSortInPlace() {
        var temp = [Element](repeating: self[0], count: count)
        mergeSortInPlace(low: 0, high: count - 1, temp: &temp)
    }
    
    private mutating func mergeSortInPlace(low: Int, high: Int, temp: inout [Element]) {
        guard low < high else { return }
        
        let mid = low + (high - low) / 2
        
        mergeSortInPlace(low: low, high: mid, temp: &temp)
        mergeSortInPlace(low: mid + 1, high: high, temp: &temp)
        
        mergeInPlace(low: low, mid: mid, high: high, temp: &temp)
    }
    
    private mutating func mergeInPlace(low: Int, mid: Int, high: Int, temp: inout [Element]) {
        var i = low
        var j = mid + 1
        var k = low
        
        // Копируем в temp
        for idx in low...high {
            temp[idx] = self[idx]
        }
        
        while i <= mid && j <= high {
            if temp[i] <= temp[j] {
                self[k] = temp[i]
                i += 1
            } else {
                self[k] = temp[j]
                j += 1
            }
            k += 1
        }
        
        // Копируем оставшиеся элементы из левой половины
        while i <= mid {
            self[k] = temp[i]
            i += 1
            k += 1
        }
    }
}
```

## 4. Реальные примеры использования

### Пример 1 — стабильная сортировка задач по приоритету и дате

```swift
struct Task {
    let priority: Int
    let dueDate: Date
    let title: String
}

var tasks = [
    Task(priority: 2, dueDate: Date().addingTimeInterval(86400), title: "Meeting A"),
    Task(priority: 1, dueDate: Date(), title: "Fix bug"),
    Task(priority: 2, dueDate: Date().addingTimeInterval(86400), title: "Meeting B")
]

let sortedTasks = mergeSort(tasks) { a, b in
    if a.priority != b.priority {
        return a.priority < b.priority
    }
    return a.dueDate < b.dueDate
}

// Стабильность: Meeting A остаётся перед Meeting B
print(sortedTasks.map { $0.title })
// ["Fix bug", "Meeting A", "Meeting B"]
```

### Пример 2 — сортировка большого массива моделей перед отображением

```swift
struct Message {
    let timestamp: Date
    let sender: String
    let text: String
}

var messages: [Message] = // загружены из API

// Стабильная сортировка по времени (если два сообщения в одну секунду)

// Теперь безопасно отображаем в UITableView

```

### Пример 3 — сортировка словаря по значениям (через массив)

```swift
let ratings = [
    "iPhone 16": 4.8,
    "MacBook Pro": 4.9,
    "AirPods Pro": 4.7,
    "Apple Watch": 4.6

```

## 5. Сравнение Merge Sort с другими алгоритмами

| Алгоритм       | Сложность (все случаи) | Стабильность | In-place | Память    | Лучше всего работает на… |
|----------------|--------------------------|--------------|----------|-----------|---------------------------|
| Merge Sort     | O(n log n)               | Да           | Нет      | O(n)      | Большие массивы, стабильность |
| Quick Sort     | O(n log n) / O(n²)       | Нет          | Да       | O(log n)  | Случайные данные          |
| Heap Sort      | O(n log n)               | Нет          | Да       | O(1)      | Гарантированная сложность |
| TimSort (Swift)| O(n log n)               | Да           | Нет      | O(n)      | Реальные данные           |
| Insertion Sort | O(n²) / O(n)             | Да           | Да       | O(1)      | Почти отсортированные     |

## Итог — современные рекомендации (2026)

| Ситуация                                   | Что использовать                       | Почему                                  |
| ------------------------------------------ | -------------------------------------- | --------------------------------------- |
| Обычная сортировка в приложении            | `sorted(by:)` или [[sort]]()           | TimSort — стабильный, быстрый, читаемый |
| Нужна **стабильность** сортировки          | `sorted(by:)` (TimSort) или Merge Sort | Оба стабильны                           |
| Очень большой массив + внешняя память      | Merge Sort (внешняя сортировка)        | Эффективен для потоков/диска            |
| Маленький массив или почти отсортированный | Insertion Sort                         | Близко к O(n)                           |
| Обучение / собеседование                   | Ручная реализация Merge Sort           | Показывает понимание divide-and-conquer |
| Сортировка моделей в таблице/коллекции     | `sorted { … }`                         | Простота и стабильность                 |

**Короткое правило для iOS-разработчика**:
> Merge Sort — это **классика** для понимания стабильной сортировки и внешней обработки данных.  
> В реальных приложениях почти всегда используйте `sorted(by:)` — Swift внутри использует TimSort, который сочетает лучшее от Merge Sort и Insertion Sort.
