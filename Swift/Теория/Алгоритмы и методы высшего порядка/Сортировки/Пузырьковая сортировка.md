# Пузырьковая сортировка (Bubble Sort) в [[Swift]]

**Bubble Sort** — один из самых простых и понятных алгоритмов сортировки. Он многократно проходит по массиву, сравнивая соседние элементы и меняя их местами, если они стоят в неправильном порядке. Каждый проход «всплывает» самый большой (или маленький) элемент в конец (или начало) — отсюда и название «пузырьковая».

### Основные характеристики

- Сложность:
  - Лучший случай: **O(n)** — если массив уже отсортирован (с оптимизацией)
  - Средний и худший случай: **O(n²)**
- Память: **O(1)** дополнительной (in-place)
- **Стабильная** сортировка — сохраняет относительный порядок равных элементов
- Очень медленная на больших массивах
- Идеальна для обучения и демонстрации

### Когда Bubble Sort всё ещё используется в [[iOS]] (2026)

- Обучение алгоритмам и студентам
- Демонстрация анимации сортировки в учебных приложениях ([[UITableView]] с визуализацией)
- Очень маленькие массивы (< 50 элементов), где производительность не важна
- Простейшие прототипы и скрипты
- Когда важна **стабильность** и код должен быть максимально простым

В реальных приложениях **почти никогда** не используют Bubble Sort для сортировки данных — вместо него берут `sorted(by:)` или [[sort]]() (TimSort).

## 1. Классическая реализация (in-place)

```swift
extension Array where Element: Comparable {
    mutating func bubbleSort() {
        let n = count
        guard n > 1 else { return }
        
        for i in 0..<n {
            // Флаг для раннего выхода, если массив уже отсортирован
            var swapped = false
            
            for j in 0..<(n - i - 1) {
                if self[j] > self[j + 1] {
                    swapAt(j, j + 1)
                    swapped = true
                }
            }
            
            // Оптимизация: если за проход не было обменов → массив отсортирован
            if !swapped { break }
        }
    }
}

// Пример
var numbers = [64, 34, 25, 12, 22, 11, 90]
numbers.bubbleSort()
print(numbers)  // [11, 12, 22, 25, 34, 64, 90]
```

## 2. Универсальная версия с замыканием (аналог sorted(by:))

```swift
extension Array {
    mutating func bubbleSort<T>(by areInIncreasingOrder: (Element, Element) -> Bool) where Element == T {
        let n = count
        guard n > 1 else { return }
        
        for i in 0..<n {
            var swapped = false
            
            for j in 0..<(n - i - 1) {
                if !areInIncreasingOrder(self[j], self[j + 1]) {
                    swapAt(j, j + 1)
                    swapped = true
                }
            }
            
            if !swapped { break }
        }
    }
}
```

## 3. Реальные примеры использования

### Пример 1 — сортировка задач по приоритету

```swift
struct Task {
    let priority: Int      // меньшее число = выше приоритет
    let title: String
}

var tasks = [
    Task(priority: 3, title: "Write docs"),
    Task(priority: 1, title: "Fix critical bug"),
    Task(priority: 2, title: "Review PR"),
    Task(priority: 1, title: "Deploy hotfix")  // два с приоритетом 1
]

tasks.bubbleSort { $0.priority < $1.priority }

// Стабильность: "Fix critical bug" остаётся перед "Deploy hotfix"
print(tasks.map { $0.title })
// ["Fix critical bug", "Deploy hotfix", "Review PR", "Write docs"]
```

### Пример 2 — демонстрация анимации в UITableView

```swift
var demoArray = [5, 3, 8, 4, 2]

// В контроллере можно анимировать каждый swap
func animateBubbleSort() {
    for i in 0..<demoArray.count {
        for j in 0..<(demoArray.count - i - 1) {
            if demoArray[j] > demoArray[j + 1] {
                // Анимация ячеек в таблице
                UIView.animate(withDuration: 0.3) {
                    // меняем местами визуальные ячейки
                }
                
                demoArray.swapAt(j, j + 1)
                
                // Обновляем таблицу
                tableView.reloadRows(at: [IndexPath(row: j, section: 0),
                                          IndexPath(row: j + 1, section: 0)],
                                     with: .automatic)
            }
        }
    }
}
```

### Пример 3 — сортировка словаря по значениям

```swift
let scores = [
    "Alice": 92,
    "Bob":   78,
    "Clara": 95,
    "David": 85
]

// Преобразуем в массив и сортируем
var scorePairs = Array(scores)
scorePairs.bubbleSort { $0.value < $1.value }

print(scorePairs.map { "\($0.key): \($0.value)" })
// ["Bob: 78", "David: 85", "Alice: 92", "Clara: 95"]
```

### Пример 4 — оптимизированная версия с флагом и уменьшением диапазона

```swift
extension Array where Element: Comparable {
    mutating func bubbleSortOptimized() {
        let n = count
        var swapped: Bool
        var lastSwapIndex = n - 1
        
        repeat {
            swapped = false
            let currentEnd = lastSwapIndex
            
            for j in 0..<currentEnd {
                if self[j] > self[j + 1] {
                    swapAt(j, j + 1)
                    swapped = true
                    lastSwapIndex = j  // запоминаем последний обмен
                }
            }
        } while swapped
    }
}
```

## 4. Сравнение Bubble Sort с другими алгоритмами

| Алгоритм       | Сложность (средняя) | Сложность (худшая) | Стабильность | In-place | Практическая скорость |
|----------------|----------------------|---------------------|--------------|----------|------------------------|
| Bubble Sort    | O(n²)                | O(n²)               | Да           | Да       | Очень медленная        |
| Quick Sort     | O(n log n)           | O(n²)               | Нет          | Да       | Очень быстрая          |
| Heap Sort      | O(n log n)           | O(n log n)          | Нет          | Да       | Хорошая                |
| TimSort (Swift)| O(n log n)           | O(n log n)          | Да           | Нет      | Лучшая на практике     |

## Итог — современные рекомендации (2026)

| Ситуация                                      | Что использовать                     | Почему |
|-----------------------------------------------|--------------------------------------|--------|
| Обычная сортировка в приложении               | `sorted(by:)` или `sort()`           | TimSort — стабильный, быстрый, читаемый |
| Учебный проект / демонстрация анимации        | Bubble Sort                          | Простота и наглядность обменов |
| Очень маленький массив (< 20 элементов)       | Bubble Sort                          | Разница незаметна |
| Стабильность сортировки важна                 | `sorted(by:)` (TimSort)              | Bubble Sort тоже стабилен, но медленный |
| Большой массив                                | Никогда Bubble Sort                  | O(n²) — неприемлемо |
| Обучение / собеседование                      | Ручная реализация Bubble Sort        | Показывает понимание базовых алгоритмов |

**Короткое правило для iOS-разработчика**:
> Bubble Sort — это алгоритм для **учёбы и демонстрации**, а не для реальных приложений.  
> Для любых реальных данных используйте `sorted(by:)` или `sort()` — они быстрее, стабильнее и безопаснее.
