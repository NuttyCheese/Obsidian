# Пирамидальная сортировка (Heap Sort) в [[Swift]]

**Heap Sort** — алгоритм сортировки, основанный на структуре данных **бинарная куча** (heap). Он гарантированно работает за **O(n log n)** во всех случаях и выполняется **на месте** (in-place), не требуя дополнительной памяти (кроме рекурсивного стека).

### Основные характеристики

- Сложность: **O(n log n)** во всех случаях (лучший, средний, худший)
- Память: **O(1)** дополнительной (in-place)
- **Нестабильная** сортировка
- Не требует случайного доступа к памяти (хорошо работает на внешних носителях)
- Часто используется как основа для **Priority [[Queue]]** (очередь с приоритетом)

### Когда Heap Sort действительно полезен в iOS-разработке

- Сортировка очень больших массивов в памяти с жёсткими ограничениями по дополнительной памяти
- Реализация **очереди с приоритетом** (например, задачи по приоритету, события по времени)
- Игровые механики (очереди рендеринга, событий, AI)
- Когда важна **гарантированная O(n log n)** сложность (в отличие от Quick Sort)

Однако в 95% случаев в iOS-разработке лучше использовать встроенный метод `sorted(by:)` или [[sort]]() — они используют **TimSort** (стабильный, гибридный, очень быстрый на практике).

## 1. Классическая in-place реализация (max-[[heap]])

```swift
extension Array where Element: Comparable {
    mutating func heapSort() {
        let n = count
        
        // Шаг 1: Построение max-heap
        for i in stride(from: n / 2 - 1, through: 0, by: -1) {
            heapify(n: n, i: i)
        }
        
        // Шаг 2: Извлечение максимумов
        for i in stride(from: n - 1, through: 1, by: -1) {
            swapAt(0, i)              // максимум в конец
            heapify(n: i, i: 0)       // восстанавливаем кучу для оставшихся элементов
        }
    }
    
    private mutating func heapify(n: Int, i: Int) {
        var largest = i
        let left    = 2 * i + 1
        let right   = 2 * i + 2
        
        if left < n && self[left] > self[largest] {
            largest = left
        }
        if right < n && self[right] > self[largest] {
            largest = right
        }
        
        if largest != i {
            swapAt(i, largest)
            heapify(n: n, i: largest)
        }
    }
}

// Пример использования
var numbers = [64, 34, 25, 12, 22, 11, 90]
numbers.heapSort()
print(numbers)  // [11, 12, 22, 25, 34, 64, 90]
```

## 2. Универсальная версия с замыканием (аналог sorted(by:))

```swift
extension Array {
    mutating func heapSort<T>(by areInIncreasingOrder: (Element, Element) -> Bool) where Element == T {
        let n = count
        
        // Построение max-heap (по убыванию при areInIncreasingOrder)
        for i in stride(from: n / 2 - 1, through: 0, by: -1) {
            heapify(n: n, i: i, by: areInIncreasingOrder)
        }
        
        // Извлечение
        for i in stride(from: n - 1, through: 1, by: -1) {
            swapAt(0, i)
            heapify(n: i, i: 0, by: areInIncreasingOrder)
        }
    }
    
    private mutating func heapify<T>(n: Int, i: Int, by areInIncreasingOrder: (Element, Element) -> Bool) where Element == T {
        var largest = i
        let left  = 2 * i + 1
        let right = 2 * i + 2
        
        if left < n && areInIncreasingOrder(self[largest], self[left]) {
            largest = left
        }
        if right < n && areInIncreasingOrder(self[largest], self[right]) {
            largest = right
        }
        
        if largest != i {
            swapAt(i, largest)
            heapify(n: n, i: largest, by: areInIncreasingOrder)
        }
    }
}
```

## 3. Реальные примеры использования

### Пример 1 — сортировка задач по приоритету и дедлайну

```swift
struct Task {
    let priority: Int      // 1 — критично, 5 — низкий
    let dueDate: Date
    let title: String
}

var tasks = [
    Task(priority: 3, dueDate: Date().addingTimeInterval(86400*2), title: "Write docs"),
    Task(priority: 1, dueDate: Date(), title: "Fix critical bug"),
    Task(priority: 2, dueDate: Date().addingTimeInterval(3600), title: "Review PR")
]

tasks.heapSort { a, b in
    if a.priority != b.priority {
        return a.priority < b.priority   // меньший приоритет — важнее
    }
    return a.dueDate < b.dueDate
}
```

### Пример 2 — сортировка транзакций по сумме (от больших к маленьким)

```swift
struct Transaction {
```

### Пример 3 — использование Heap Sort для Priority Queue (очередь с приоритетом)

```swift
struct Event: Comparable {
    let time: Date
    let priority: Int
    let description: String
    
    static func < (lhs: Event, rhs: Event) -> Bool {
        if lhs.priority != rhs.priority {
            return lhs.priority < rhs.priority
        }
        return lhs.time < rhs.time
    }
}

var eventQueue = [Event]()

// Добавление событий
eventQueue.append(Event(time: Date().addingTimeInterval(3600), priority: 2, description: "Meeting"))
eventQueue.append(Event(time: Date(), priority: 1, description: "Critical alert"))

// Сортировка → самый приоритетный и ближайший будет первым
eventQueue.heapSort()

// Обработка ближайшего события
if let next = eventQueue.first {
    print("Следующее событие: \(next.description)")
}
```

### Пример 4 — сортировка словаря по значениям (через массив)

```swift
let scores = [
    "Alice": 92,
    "Bob":   78,
    "Clara": 95,
    "David": 85
]

// Преобразуем в массив кортежей и сортируем
let sortedScores = scores.sorted { $0.value > $1.value }

```

## 4. Сравнение Heap Sort с другими алгоритмами

| Алгоритм       | Сложность (средняя) | Сложность (худшая) | Стабильность | In-place | Практическая скорость |
|----------------|----------------------|---------------------|--------------|----------|------------------------|
| Heap Sort      | O(n log n)           | O(n log n)          | Нет          | Да       | Хорошая                |
| Quick Sort     | O(n log n)           | O(n²)               | Нет          | Да       | Очень быстрая          |
| Merge Sort     | O(n log n)           | O(n log n)          | Да           | Нет      | Стабильно высокая      |
| TimSort (Swift)| O(n log n)           | O(n log n)          | Да           | Нет      | Лучшая на практике     |

## Итог — современные рекомендации (2026)

| Ситуация                                      | Что использовать                     | Почему |
|-----------------------------------------------|--------------------------------------|--------|
| Обычная сортировка в приложении               | `sorted(by:)` или `sort()`           | TimSort — стабильный, быстрый, читаемый |
| Жёсткие ограничения по памяти                 | `heapSort()` (in-place)              | Не создаёт копий |
| Нужна гарантированная O(n log n)              | Heap Sort                            | Нет O(n²) худшего случая |
| Сортировка по приоритету (очередь)            | Heap Sort или PriorityQueue          | Логично для кучи |
| Сортировка моделей в таблице/коллекции        | `sorted { … }`                       | Простота и стабильность |
| Обучение / собеседование                      | Ручная реализация Heap Sort          | Показывает понимание кучи |

**Короткое правило для iOS-разработчика**:
> В 99% случаев используй `sorted(by:)` или `sort()`.  
> Heap Sort пиши вручную только для понимания кучи, Priority Queue или очень специфичных задач с ограничением памяти.
