# Нотация сложности алгоритмов (Big O, Big Θ, Big Ω) в Swift

**Нотация асимптотической сложности** — это способ описать, как растёт время выполнения или потребление памяти алгоритма при увеличении размера входных данных **n**.

Она абстрагирует нас от конкретного железа, языка и констант, позволяя сравнивать алгоритмы «в большом».

### Основные обозначения

| Нотация       | Что означает                              | Про что говорит                          | Пример роста при n=1 000 000 |
|---------------|-------------------------------------------|------------------------------------------|-------------------------------|
| **O(f(n))**   | Верхняя граница (худший случай)           | «Не хуже, чем…»                          | O(n) → ≤ 1 млн операций       |
| **Ω(f(n))**   | Нижняя граница (лучший случай)            | «Не лучше, чем…»                         | Ω(n log n) → ≥ ~20 млн операций |
| **Θ(f(n))**   | Точная граница (средний / типичный случай)| «Ровно такого порядка»                   | Θ(n log n) → ≈ 20 млн операций |
| **o(f(n))**   | Строго медленнее                          | «Значительно медленнее»                  | o(n²) → сильно быстрее n²     |
| **ω(f(n))**   | Строго быстрее                            | «Значительно быстрее»                    | ω(n) → сильно быстрее n       |

### Практические примеры сложности в Swift

#### O(1) — константное время

```swift
let users = ["alice": User(id: 1), "bob": User(id: 2)]
let user = users["alice"]          // O(1)
let first = array[0]               // O(1)
let count = array.count            // O(1)
let contains = set.contains(42)   // O(1) в среднем
```

#### O(log n) — логарифмическое

```swift
// Бинарный поиск
func binarySearch<T: Comparable>(_ arr: [T], target: T) -> Int? {
    var low = 0, high = arr.count - 1
    while low <= high {
        let mid = low + (high - low) / 2
        if arr[mid] == target { return mid }
        else if arr[mid] < target { low = mid + 1 }
        else { high = mid - 1 }
    }
    return nil
}

// Поиск в отсортированном Set / Dictionary (внутренне — красно-чёрное дерево)
let sortedSet = Set(1...1_000_000)
sortedSet.contains(777_777)  // O(log n)
```

#### O(n) — линейное

```swift
array.forEach { print($0) }               // O(n)
array.map { $0 * 2 }                      // O(n)
array.filter { $0 > 0 }                   // O(n)
array.first(where: { $0 == target })      // O(n) в худшем
array.contains(target)                    // O(n) для Array
```

#### O(n log n) — линейно-логарифмическое

```swift
array.sorted()                            // TimSort → O(n log n)
array.sorted { $0.name < $1.name }        // O(n log n)
array.sort()                              // in-place TimSort
```

#### O(n²) — квадратичное

```swift
for i in array {
    for j in array {
        if i + j == target { /* ... */ }
    }
}  // O(n²)

for i in 0..<n {
    for j in 0..<n {
        print(i, j)
    }
}
```

#### O(n³) — кубическое

```swift
for i in 0..<n {
    for j in 0..<n {
        for k in 0..<n {
            matrix[i][j] += a[i][k] * b[k][j]
        }
    }
}  // умножение матриц O(n³)
```

#### O(2ⁿ) — экспоненциальное

```swift
func subsets<T>(_ set: [T]) -> [[T]] {
    if set.isEmpty { return [[]] }
    let first = set[0]
    let rest = Array(set.dropFirst())
    let without = subsets(rest)
    let with = without.map { [first] + $0 }
    return without + with
}  // 2ⁿ подмножеств

// Рекурсивный Фибоначчи без мемоизации
func fib(_ n: Int) -> Int {
    n <= 1 ? n : fib(n-1) + fib(n-2)
}
fib(40)  // очень долго
```

#### O(n!) — факториальное

```swift
func permutations<T>(_ array: [T]) -> [[T]] {
    guard !array.isEmpty else { return [[]] }
    var result = [[T]]()
    for (i, elem) in array.enumerated() {
        var rest = array
        rest.remove(at: i)
        for perm in permutations(rest) {
            result.append([elem] + perm)
        }
    }
    return result
}  // n! перестановок
```

### Практические тесты времени выполнения

```swift
import Foundation

func measureTime(_ title: String, operation: () -> Void) {
    let start = CFAbsoluteTimeGetCurrent()
    operation()
    let duration = CFAbsoluteTimeGetCurrent() - start
    print("\(title): \(String(format: "%.6f", duration)) сек")
}

let n = 10_000

// O(n log n)
measureTime("sorted() — O(n log n)") {
    var arr = (0..<n).map { _ in Int.random(in: 0...1_000_000) }
    arr.sort()
}

// O(n²)
measureTime("вложенные циклы — O(n²)") {
    var count = 0
    for _ in 0..<500 {
        for _ in 0..<500 { count += 1 }
    }
}

// O(2ⁿ) — маленький n!
measureTime("Fibonacci(35) — O(2ⁿ)") {
    func fib(_ n: Int) -> Int { n <= 1 ? n : fib(n-1) + fib(n-2) }
    _ = fib(35)
}
```

### Советы для iOS-разработчика (2026)

| Ситуация                                      | Допустимая сложность | Что использовать                     |
|-----------------------------------------------|----------------------|--------------------------------------|
| Поиск по ключу / ID                           | O(1)                 | `Dictionary[key]`, `Set.contains`    |
| Фильтрация / map / contains                   | O(n)                 | `filter`, `map`, `first(where:)`     |
| Сортировка массива моделей                    | O(n log n)           | `sorted(by:)`, `sort()`              |
| Вложенные циклы по массиву                    | O(n²) — только если n < 1 000 | Переосмыслить задачу, использовать индексы |
| Рекурсивный перебор комбинаций                | O(2ⁿ), O(n!)         | Только если n ≤ 15–20                |
| Бинарный / интерполяционный поиск             | O(log n)             | Отсортированный массив + свой алгоритм |

**Короткое правило**:
- **O(1)** → всегда хорошо
- **O(log n)** → отлично для поиска
- **O(n)** → нормально до 1–10 млн элементов
- **O(n log n)** → стандарт для сортировки
- **O(n²)** → только если n ≤ 1 000–5 000
- **O(2ⁿ), O(n!)** → только для очень маленьких n (≤ 20)
