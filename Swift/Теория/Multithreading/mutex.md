Мьютекс (Mutex) - это средство синхронизации, которое обеспечивает эксклюзивный доступ к общему ресурсу из нескольких потоков. Он представляет собой объект, который может находиться в двух состояниях: заблокированном (захваченном) и разблокированном (освобожденном). Поток, желающий получить доступ к защищенному ресурсу, должен сначала захватить мьютекс, а затем освободить его после завершения работы с ресурсом. Мьютексы широко используются для предотвращения гонок данных и других проблем согласованности данных в многопоточных приложениях. Давайте рассмотрим более подробно, что это такое, как они работают и как их использовать в [[Swift]] с примерами.

1. **Что такое мьютекс (Mutex)**:
   - Мьютекс (от "mutual exclusion") - это средство синхронизации, которое обеспечивает эксклюзивный доступ к общему ресурсу из нескольких потоков.
   - Когда поток захватывает мьютекс, он блокирует доступ к защищенному ресурсу для других потоков. После завершения работы с ресурсом, поток освобождает мьютекс.

2. **Принцип работы**:
   - Мьютексы работают по принципу захвата и освобождения.
   - Когда поток хочет получить доступ к защищенному ресурсу, он пытается захватить мьютекс. Если мьютекс доступен (разблокирован), поток захватывает его и продолжает работу. Если мьютекс уже заблокирован другим потоком, поток ожидает его освобождения.
   - После завершения работы с ресурсом, поток освобождает мьютекс, позволяя другим потокам получить доступ к защищенному ресурсу.

3. **Использование в Swift**:
   - В Swift мьютексы могут быть реализованы с использованием класса `NSLock` из [[Foundation]] Framework.
   - Пример использования мьютекса:

```swift
import Foundation

let mutex = NSLock()
var sharedResource = 0

// Поток 1
mutex.lock()
sharedResource += 1
mutex.unlock()

// Поток 2
mutex.lock()
sharedResource -= 1
mutex.unlock()
```

4. **Пример гонки данных без мьютекса**:
   - Рассмотрим пример, когда два потока пытаются изменить общий ресурс без использования мьютекса:

```swift
var sharedResource = 0

DispatchQueue.global().async {
    sharedResource += 1
}

DispatchQueue.global().async {
    sharedResource -= 1
}
```

В этом примере, если оба потока выполняются параллельно, то значение `sharedResource` может быть непредсказуемым из-за гонки данных.

5. **Преимущества и недостатки**:
   - Преимущества мьютексов:
     - Обеспечивают безопасный доступ к общим ресурсам из нескольких потоков.
     - Предотвращают гонки данных и другие проблемы согласованности данных.
   - Недостатки мьютексов:
     - Могут вызывать блокировку потоков и снижать производительность приложения при частом использовании.
     - Могут быть источником потенциальных проблем, таких как взаимная блокировка ([[deadlock]]) или голодание потоков ([[starvation]]).
