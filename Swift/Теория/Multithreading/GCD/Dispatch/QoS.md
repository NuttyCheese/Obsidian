## 1. Что такое QoS

**QoS (Quality of Service)** — это **приоритет выполнения задачи или потока**, который сообщает системе **насколько критична задача** и сколько ресурсов ей выделять.

- Используется с **GCD ([[DispatchQueue]])**, **OperationQueue**, и **[[Swift]] Concurrency ([[Task]])**.
    
- Помогает системе **распределять CPU и память** между задачами с разной важностью.
    

> Проще: QoS = «важность задачи» для системы.

---

## 2. Основные уровни QoS

|QoS|Использование|Пример|
|---|---|---|
|`.userInteractive`|Для UI и анимаций, критично, должно быть мгновенно|Обновление кнопок, анимация, скролл|
|`.userInitiated`|Для задач, которые пользователь ожидает сразу|Загрузка данных по клику, открытие экрана|
|`.utility`|Для долгих фоновых задач, не критично для UI|Скачивание файлов, синхронизация данных|
|`.background`|Для задач, которые не нужны немедленно|Очистка кеша, резервное копирование|
|`.default`|Стандартный уровень|По умолчанию|

---

## 3. Использование с DispatchQueue

```swift
let queue = DispatchQueue.global(qos: .userInitiated)

queue.async {
    print("Выполняется высокоприоритетная задача")
}
```

- Система старается выполнить **высокоприоритетные задачи быстрее**, низкоприоритетные могут ждать.
    

---

## 4. Использование с Swift Concurrency

```swift
Task.detached(priority: .userInitiated) {
    print("Задача с высоким приоритетом")
}

Task.detached(priority: .background) {
    print("Фоновая задача с низким приоритетом")
}
```

- Задачи с разным QoS могут выполняться параллельно, но система **отдаёт больше ресурсов задачам с высоким приоритетом**.
    

---

## 5. Важные моменты

- **QoS влияет на производительность и отзывчивость приложения**, но не гарантирует точный порядок выполнения.
    
- **Неправильное использование** может привести к:
    
    - **Priority Inversion** (высокий поток ждёт низкий),
        
    - **Priority Starvation** (низкий поток долго не выполняется).
        
- QoS ≠ жесткий приоритет CPU, это **подсказка системе о важности задачи**.
    

---

## 6. Итог

- **QoS** = уровень важности задачи для системы.
    
- Используется с **DispatchQueue**, **OperationQueue**, **Task**.
    
- Основные уровни:
    
    - `.userInteractive` — UI, критично, мгновенно
        
    - `.userInitiated` — пользователь ждёт результат
        
    - `.utility` — долгие задачи
        
    - `.background` — фоновые задачи
        
- Правильное использование QoS помогает **балансировать нагрузку и отзывчивость приложения**.
