### 1. Что такое Thread в Swift (реальность 2026 года)

**Thread** — это **низкоуровневый объект**, представляющий **отдельный поток выполнения** (native thread) в iOS/macOS.

Каждый `Thread` имеет:

- собственный **стек** (обычно 512 КБ – 1 МБ)  
- собственный **RunLoop** (если вы его запустите)  
- собственные **регистры** и **контекст выполнения**  
- идентификатор в ядре (kernel thread)

**Важнейший факт 2026 года**:
> `Thread` — это **legacy-инструмент**.  
> Apple уже **много лет** (с 2014–2015) рекомендует **не использовать** `Thread` напрямую в большинстве случаев.  
> Вместо этого нужно использовать:
> - **GCD** (`DispatchQueue`) — до 2021 года  
> - **Swift Concurrency** (`Task`, `actor`, `TaskGroup`) — с 2021 и особенно в 2026 году

**Когда `Thread` всё ещё оправдан в 2026** (очень редкие случаи):

- Низкоуровневая работа с **C-библиотеками** (Core Audio, Metal, OpenGL ES legacy)  
- Специфические real-time задачи (аудио-обработка, где нужен полный контроль над потоком)  
- Очень старый код / поддержка iOS 9–12 (крайне редко)  
- Тестирование / отладка поведения потоков на уровне ядра

### 2. Сравнение Thread с современными альтернативами (2026)

| Характеристика                     | Thread (ручное создание)                | DispatchQueue / GCD                     | Swift Concurrency (Task / actor)        | Что выбрать в 2026 |
|------------------------------------|------------------------------------------|------------------------------------------|------------------------------------------|---------------------|
| Управление потоками                | Ручное (start, cancel, join)             | Автоматическое (пул потоков)             | Автоматическое + структурированное      | Swift Concurrency   |
| Отмена задач                       | Сложно (Thread.cancel не работает)       | DispatchWorkItem (ограниченно)           | Нативная (Task.cancel)                   | Swift Concurrency   |
| Потокобезопасность                 | Нужно вручную (lock, semaphore)          | Нужно вручную (barrier, sync)            | Встроенная (actor, Sendable)             | actor               |
| UI-обновления                      | Запрещено напрямую                       | DispatchQueue.main.async                 | @MainActor / await MainActor.run         | @MainActor          |
| Читаемость / современность         | Очень низкая                             | Средняя                                  | Высокая                                  | Swift Concurrency   |
| Производительность                 | Максимальная (если правильно)            | Хорошая                                  | Отличная (часто лучше GCD)               | Swift Concurrency   |
| Рекомендация Apple                 | Не рекомендуется                         | Legacy, но допустимо                     | Основной выбор                           | Swift Concurrency   |

**Вывод 2026**:
> `Thread` — это **почти антипаттерн** в новом коде.  
> Если вы пишете `Thread { ... }.start()` в 2026 году — это почти всегда **ошибка выбора инструмента**.

### 3. Как выглядит создание и запуск Thread (и почему это плохо)

#### Вариант 1 — Через замыкание (самый частый)

```swift
let thread = Thread {
    for i in 1...5 {
        print("Фоновый поток: \(i)")
        Thread.sleep(forTimeInterval: 1)
    }
}
thread.name = "MyBackgroundThread"
thread.qualityOfService = .utility
thread.start()
```

**Проблемы**:
- Нет встроенной отмены  
- Нет RunLoop по умолчанию (таймеры, порты не работают)  
- Нельзя безопасно обновлять UI  
- Сложно синхронизировать с другими частями приложения

#### Вариант 2 — Через selector (ещё более legacy)

```swift
class MyClass: NSObject {
    @objc func backgroundTask() {
        print("Работа в потоке")
    }
}

let thread = Thread(target: self, selector: #selector(backgroundTask), object: nil)
thread.start()
```

**Проблемы**: ещё хуже читаемость, Objective-C стиль, нет замыканий.

### 4. RunLoop в Thread — важная особенность

**Каждый Thread может иметь свой RunLoop**, но он **не запускается автоматически**.

```swift
let thread = Thread {
    // Без RunLoop.current.run() таймеры и порты НЕ будут работать
    
    let timer = Timer(timeInterval: 1.0, repeats: true) { _ in
        print("Таймер в фоновом потоке")
    }
    
    RunLoop.current.add(timer, forMode: .default)
    RunLoop.current.run()  // ← обязательно, иначе таймер не сработает
}
thread.start()
```

**Важно**:
> Запуск `RunLoop.current.run()` **блокирует поток навсегда**, пока кто-то не вызовет `RunLoop.current.invalidate()` или `CFRunLoopStop`.

Это одна из самых частых причин **зависания** при использовании `Thread`.

### 5. Типичные ошибки при использовании Thread в 2026

| Ошибка                                      | Последствия                              | Как избежать |
|---------------------------------------------|------------------------------------------|--------------|
| Обновление UI напрямую из Thread            | Main Thread Violation / краш             | Всегда `DispatchQueue.main.async` или `@MainActor` |
| Запуск RunLoop без возможности остановки    | Поток зависает навсегда                  | Использовать `RunLoop.current.run(mode:until:)` или не запускать RunLoop |
| Отсутствие синхронизации доступа к данным   | Race Condition / Data Race               | Использовать `actor` / сериальную очередь |
| Создание потоков в цикле                    | Thread Explosion                         | Использовать `TaskGroup` / `OperationQueue` |
| Использование `Thread` вместо `Task`        | Устаревший код, сложно поддерживать      | Переписывать на `Task` / `actor` |

### 7. Лучшие практики 2026 года

- **Переходите на Swift Concurrency** — `Thread` уже считается legacy  
- **Никогда** не создавайте `Thread` в цикле или рекурсии — это Thread Explosion  
- **Для UI** — только `@MainActor` / `MainActor.run`  
- **Для фоновых задач** — `Task` / `TaskGroup` / `actor`  
- **Для тестов** — используйте `Task` + `XCTestExpectation`  
- **Мониторинг** — включайте **Thread Sanitizer** и **Instruments → Threads**  
- **Swift 6 strict concurrency** — почти полностью исключает необходимость в `Thread`
