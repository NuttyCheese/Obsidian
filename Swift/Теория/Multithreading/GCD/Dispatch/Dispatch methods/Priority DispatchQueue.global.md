Вот **полное, подробное и максимально актуальное** (на февраль 2026 года) руководство по **приоритетам (QoS)** в `DispatchQueue.global` в Swift и Grand Central Dispatch (GCD).

### 1. Что такое QoS и зачем он нужен в 2026 году

**QoS** (Quality of Service) — это **приоритет выполнения задач** в глобальных очередях GCD.

Он отвечает на вопрос:  
«Насколько **важно** и **срочно** выполнить эту задачу по сравнению с другими задачами в системе?»

От QoS зависит:

- **Как быстро** задача получит процессорное время  
- **Будет ли она вытеснять** другие задачи  
- **Сможет ли низкоприоритетная задача вообще выполниться**, если CPU занят высокоприоритетными задачами  
- **Сколько энергии** потребует задача (очень важно для автономности iPhone)

**Ключевой факт 2026 года**:
> Apple активно продвигает **Swift Concurrency** (`Task(priority:)`) как основной способ управления приоритетами.  
> `DispatchQueue.global(qos:)` остаётся актуальным только для:
> - поддержки legacy-кода  
> - интеграции с GCD-библиотеками  
> - очень простых фоновых операций

### 2. Все доступные QoS в 2026 году (с реальными рекомендациями)

| QoS                     | Когда использовать (рекомендация Apple 2026)                          | Примеры задач в iOS-приложении 2026                          | Относительная скорость | Влияние на батарею | Приоритет в планировщике |
|-------------------------|------------------------------------------------------------------------|---------------------------------------------------------------|--------------------------|---------------------|---------------------------|
| `.userInteractive`      | Анимации, реакция на касание, скролл, рендеринг кадров                | Обновление UI во время жеста, анимации, Metal rendering       | Самый высокий            | Высокое             | Максимальный              |
| `.userInitiated`        | Задачи, инициированные пользователем, но не критичные для отзывчивости | Загрузка данных для текущего экрана, отправка формы           | Очень высокий            | Высокое             | Очень высокий             |
| `.default`              | Задачи без явного приоритета (по умолчанию)                           | Большинство фоновых операций без явного требования            | Средний                  | Среднее             | Средний                   |
| `.utility`              | Долгие, некритичные задачи                                            | Обработка фото, аналитика, индексация, сжатие видео           | Средний-низкий           | Низкое              | Средний-низкий            |
| `.background`           | Очень низкий приоритет, задачи, которые могут ждать                   | Резервное копирование, очистка кэша, отправка статистики     | Самый низкий             | Минимальное         | Низкий                    |

**Важное правило 2026**:
> Никогда не используйте `.userInteractive` или `.userInitiated` для задач дольше 1–2 секунд — это приводит к **Priority Starvation** фоновых задач и быстрому разряду батареи.

### 3. Как правильно выбирать QoS в 2026 году

| Тип задачи                              | Рекомендуемый QoS 2026                  | Почему именно этот |
|-----------------------------------------|------------------------------------------|--------------------|
| Анимации, скролл, реакция на касание    | `.userInteractive`                       | Максимальная отзывчивость UI |
| Загрузка данных для текущего экрана     | `.userInitiated`                         | Пользователь ждёт результат |
| Обработка фото, аналитика, индексация   | `.utility`                               | Долгая задача, но не критичная |
| Резервное копирование, очистка кэша    | `.background`                            | Может ждать, минимальное влияние на батарею |
| Отправка логов / статистики             | `.background`                            | Самый низкий приоритет |
| Большинство фоновых операций без явной важности | `.default`                         | Сбалансированный вариант |

### 4. Примеры правильного использования в 2026 году

#### Пример 1 — Загрузка данных для экрана (userInitiated)

```swift
func loadFeed() {
    DispatchQueue.global(qos: .userInitiated).async {
        let posts = fetchPostsFromNetwork()
        
        DispatchQueue.main.async {
            self.posts = posts
            self.tableView.reloadData()
        }
    }
}
```

#### Пример 2 — Фоновая аналитика (utility)

```swift
func trackScreenView(screen: String) {
    DispatchQueue.global(qos: .utility).async {
        let event = AnalyticsEvent(screenView: screen, timestamp: Date())
        sendAnalytics(event)
    }
}
```

#### Пример 3 — Очистка кэша при выходе из приложения (background)

```swift
func applicationDidEnterBackground(_ application: UIApplication) {
    DispatchQueue.global(qos: .background).async {
        clearOldCache()
        removeUnusedFiles()
    }
}
```

### 5. Типичные ошибки и ловушки с DispatchQueue.global(qos:) в 2026

| Ошибка                                      | Последствия                              | Как избежать |
|---------------------------------------------|------------------------------------------|--------------|
| Все задачи с `.userInitiated` / `.userInteractive` | Priority Starvation фоновых задач        | Использовать правильные QoS |
| Забыть указать QoS                          | По умолчанию `.default` — может быть слишком низко/высоко | Всегда явно задавать |
| `sync` на глобальной очереди из главного потока | Deadlock / сильное замедление            | Никогда не использовать `sync` на global из main |
| Обновление UI без `.main.async`             | Main Thread Violation                    | Всегда `@MainActor` или `DispatchQueue.main.async` |
| Перегруженная очередь `.userInitiated`      | Фоновые задачи почти не запускаются      | Ограничивать concurrency (TaskGroup) |

### 6. DispatchQueue.global vs Swift Concurrency (честное сравнение 2026)

| Характеристика                     | DispatchQueue.global(qos:)             | Swift Concurrency (Task(priority:))    | Что выбрать в 2026 |
|------------------------------------|----------------------------------------|----------------------------------------|---------------------|
| Потокобезопасность                 | Нужно вручную (lock, barrier)          | Встроенная (actor, Sendable)           | Swift Concurrency   |
| Приоритеты                         | QoS                                    | Task priority                          | Оба хороши, Task проще |
| Отмена задач                       | DispatchWorkItem (сложно)              | Нативная (Task.cancel())               | Swift Concurrency   |
| UI-обновления                      | DispatchQueue.main.async               | @MainActor / await MainActor.run       | @MainActor лучше    |
| Читаемость                         | Старый стиль                           | Современный, меньше кода               | Swift Concurrency   |
| Совместимость с legacy             | Отличная                               | Требует адаптации                      | GCD для старого кода |

**Вывод 2026**:
- **Новый код** → **Task(priority:)** + `@MainActor` + `actor`  
- **Поддержка старого кода** → `DispatchQueue.global(qos:)`  
- `DispatchQueue.concurrent(label:)` — почти никогда не используйте

### 7. Лучшие практики 2026 года

- **Переходите на Swift Concurrency** — DispatchQueue.global уже считается legacy  
- **QoS** — всегда указывайте явно (`.userInteractive`, `.userInitiated`, `.utility`, `.background`)  
- **Не используйте** `sync` в production — почти всегда это ошибка  
- **Для UI** — только `@MainActor` / `MainActor.run` / `DispatchQueue.main.async`  
- **Для тестов** — используйте `XCTestExpectation` + `queue.async` или `Task`  
- **Мониторинг** — добавляйте `os_signpost` для отслеживания длительности задач в Instruments  
- **Swift 6 strict concurrency** — часто подсвечивает небезопасное использование глобальных очередей

**Короткий девиз 2026**:
> «DispatchQueue.global(qos:) — это когда нужно быстро запустить задачу в фоне с правильным приоритетом.  
> В 2026 году это уже legacy-инструмент.  
> Новый стандарт — Task(priority:) + actor + @MainActor.  
> Используй GCD только для поддержки старого кода.»
