**isAsynchronous** — это свойство класса [[Operation]] (и его подклассов, включая [[BlockOperation]]) в [[Swift]].

Оно определяет, является ли операция **асинхронной** (т.е. выполняется ли она в фоновом режиме и сама управляет своим жизненным циклом) или **синхронной** (выполняется в текущем потоке и завершается сразу после выполнения основного кода).

### Ключевые моменты про `isAsynchronous` (актуально на 2026 год)

| Значение `isAsynchronous` | Что означает                                                                      | Когда операция считается завершённой?                  | Как управлять состоянием                                       | Самый частый use-case в 2026                                                         |
| ------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------ | -------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| `false` (по умолчанию)    | Операция **синхронная** — выполняется в потоке очереди, блокирует его до конца    | Когда `main()` завершается                             | [[GCD]] / [[OperationQueue]] делает всё автоматически          | Простые вычисления, обработка данных, [[BlockOperation]] без асинхронных вызовов     |
| `true`                    | Операция **асинхронная** — запускает свою работу и **не блокирует** поток очереди | Только когда вы вручную установите `isFinished = true` | Нужно вручную менять `isExecuting`, `isFinished` через [[KVO]] | Сетевые запросы, загрузка файлов, операции с [[RunLoop]], [[Metal]] / [[Core Audio]] |

### Важные правила (2026)

- По умолчанию **все** операции (`BlockOperation`, кастомные `Operation`) имеют `isAsynchronous = false`  
- Если вы используете `isAsynchronous = true`, то **обязаны**:
  - самостоятельно управлять состояниями `isExecuting` / `isFinished`  
  - отправлять KVO-уведомления (`willChangeValue`, `didChangeValue`)  
  - вызывать `isFinished = true` только после **полного** завершения асинхронной работы  
- Если забыть это сделать — операция **зависнет** в очереди навсегда (`isFinished` никогда не станет `true`)

### Примеры (сравнение синхронного и асинхронного поведения)

#### 1. Синхронная операция (по умолчанию — самый частый случай)

```swift
let queue = OperationQueue()
queue.maxConcurrentOperationCount = 1

let op = BlockOperation {
    print("Начало синхронной операции")
    Thread.sleep(forTimeInterval: 2) // имитация тяжёлой работы
    print("Конец синхронной операции")
}

op.completionBlock = {
    print("completionBlock вызван")
}

queue.addOperation(op)
print("Операция добавлена в очередь")
```

**Вывод** (примерный порядок):
```
Операция добавлена в очередь
Начало синхронной операции
(пауза 2 секунды)
Конец синхронной операции
completionBlock вызван
```

→ `completionBlock` вызывается **сразу после** завершения блока.

#### 2. Асинхронная операция (нужно вручную управлять состоянием)

```swift
class AsyncDownloadOperation: Operation {
    override var isAsynchronous: Bool { true }
    
    private var _isExecuting: Bool = false {
        willSet { willChangeValue(forKey: "isExecuting") }
        didSet { didChangeValue(forKey: "isExecuting") }
    }
    
    override var isExecuting: Bool { _isExecuting }
    
    private var _isFinished: Bool = false {
        willSet { willChangeValue(forKey: "isFinished") }
        didSet { didChangeValue(forKey: "isFinished") }
    }
    
    override var isFinished: Bool { _isFinished }
    
    override func start() {
        if isCancelled {
            _isFinished = true
            return
        }
        
        _isExecuting = true
        
        // Асинхронная работа
        URLSession.shared.dataTask(with: url) { data, _, error in
            // обработка результата...
            
            self._isExecuting = false
            self._isFinished = true
        }.resume()
    }
}
```

**Важно**: если не управлять `_isExecuting` и `_isFinished` + KVO — операция **никогда не завершится** в глазах OperationQueue.

### 3. Сравнение: синхронная vs асинхронная операция (2026)

| Характеристика                | isAsynchronous = false (по умолчанию)         | isAsynchronous = true                        | Что выбрать в 2026          |
| ----------------------------- | --------------------------------------------- | -------------------------------------------- | --------------------------- |
| Управление состоянием         | Автоматическое ([[GCD]] / [[OperationQueue]]) | Ручное ([[KVO]], isExecuting/isFinished)     | false (если не асинхронная) |
| Сложность реализации          | Очень простая                                 | Высокая (много boilerplate)                  | false в 95% случаев         |
| Подходит для сетевых запросов | Нет (блокирует поток очереди)                 | Да                                           | async/await + Task лучше    |
| Подходит для вычислений       | Да                                            | Да, но избыточно                             | false или actor             |
| Отмена во время выполнения    | Частично (нужно проверять isCancelled)        | Полная (можно остановить асинхронную работу) | Task.cancel лучше           |
| Рекомендация Apple            | Использовать по умолчанию                     | Только если действительно асинхронная        | async/await в новом коде    |

### 4. Лучшие практики 2026 года

- **isAsynchronous = false** — используйте почти всегда (особенно с `BlockOperation`)  
- **isAsynchronous = true** — только если операция запускает **асинхронную работу** (сеть, I/O, RunLoop) и вы готовы писать KVO  
- **Для асинхронных операций** — лучше использовать `async` / `await` + `Task` / `actor`  
- **Для UI** — всегда обновляйте через `@MainActor` / `MainActor.run`  
- **Отмена** — проверяйте `isCancelled` в начале и в тяжёлых циклах  
- **Swift 6 strict concurrency** — часто подсвечивает небезопасную передачу данных в операции  
- **Мониторинг** — используйте `os_signpost` для отслеживания длительности операций в Instruments

**Короткий девиз 2026**:
> «isAsynchronous = false — почти всегда.  
> isAsynchronous = true — только если вы действительно делаете асинхронную работу и готовы к KVO.  
> В 2026 году почти все асинхронные операции лучше делать через Task / async / await, а не через Operation.»
