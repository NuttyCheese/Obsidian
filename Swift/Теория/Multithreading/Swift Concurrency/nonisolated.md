Вот **полное, подробное и максимально актуальное** (на февраль 2026 года) руководство по атрибуту **`nonisolated`** в Swift.

### 1. Что такое nonisolated простыми словами

**`nonisolated`** — это атрибут, который **отключает изоляцию** для отдельного метода, свойства или функции внутри `actor` (или `@MainActor`).

Обычно всё внутри `actor` **изолировано** — доступ извне требует `await`, а внутри — безопасен.

`nonisolated` говорит компилятору:

> «Этот метод / свойство **не трогает изолированное состояние** актёра, поэтому его можно вызывать **без await** из любого контекста, и он не требует переключения на executor актёра».

**Коротко**:
- `nonisolated` = «этот кусок кода **не зависит** от данных актёра → можно вызывать как обычную функцию, без `await` и без риска гонок данных».

### 2. Когда и зачем использовать nonisolated (2026 актуальные сценарии)

| Ситуация                                      | Почему именно nonisolated                   | Альтернатива без nonisolated |
|-----------------------------------------------|---------------------------------------------|------------------------------|
| Константы / утилитарные методы актёра         | Не трогают состояние → нет смысла в await   | Дублировать код вне актёра   |
| Статические методы / свойства                 | Нет доступа к экземплярному состоянию       | Выносить в отдельный тип     |
| Делегаты / callback / уведомления             | Вызываются из любого потока, без await      | Потеря контекста или лишний await |
| Чистые вычисления (pure functions)            | Не изменяют состояние → безопасны           | Дублирование логики          |
| Логирование / отладка / метрики без состояния | Вызываются часто, не должны блокировать     | Синхронные вызовы → гонка    |
| Совместимость с протоколами / старым кодом   | Протокол требует синхронного метода         | Принудительный `await` → неудобно |

**Золотое правило 2026**:
> `nonisolated` можно использовать **только** если метод/свойство **не читает и не пишет** изолированные свойства актёра.  
> Компилятор **строго проверяет** это правило.

### 3. Самые популярные шаблоны nonisolated 2026 года

#### Шаблон 1 — Константы и утилиты внутри актёра (самый частый)

```swift
actor SessionManager {
    private var token: String?
    
    // nonisolated — можно вызывать без await
    nonisolated var appVersion: String {
        "2.1.4"
    }
    
    nonisolated func isProductionEnvironment() -> Bool {
        #if PRODUCTION
        return true
        #else
        return false
        #endif
    }
    
    func login() async {
        // ...
        print("Версия приложения:", appVersion)           // без await
        print("Продакшн?:", isProductionEnvironment())    // без await
    }
}

let manager = SessionManager()
print(manager.appVersion)  // без await — безопасно
```

#### Шаблон 2 — Делегат / callback без await

```swift
protocol AnalyticsDelegate: AnyObject {
    func trackEvent(_ name: String)
}

actor AnalyticsService {
    weak var delegate: AnalyticsDelegate?
    
    // nonisolated — можно вызывать из любого потока без await
    nonisolated func notifyDelegate(event: String) {
        delegate?.trackEvent(event)
    }
    
    func track(_ event: String) {
        // ... внутренняя логика ...
        notifyDelegate(event: event)  // без await — безопасно
    }
}
```

#### Шаблон 3 — nonisolated + async (очень мощный паттерн)

```swift
actor ImageProcessor {
    private var cache: [URL: UIImage] = [:]
    
    // nonisolated async — можно вызывать без await, но сама функция асинхронная
    nonisolated func processImage(at url: URL) async -> UIImage? {
        // Здесь можно использовать await внутри
        if let cached = await cache[url] {
            return cached
        }
        
        let image = await downloadAndProcess(url)
        await cache[url] = image  // await обязателен для записи
        return image
    }
}

let processor = ImageProcessor()

// Можно вызвать без await, но результат требует await
Task {
    let image = await processor.processImage(at: url)
}
```

**Важно**: `nonisolated async` — это **не отменяет await** внутри функции, но **сам вызов** функции не требует `await` извне.

### 4. Типичные ошибки и ловушки 2026 года

| Ошибка                                      | Последствия                              | Как избежать |
|---------------------------------------------|------------------------------------------|--------------|
| Использовать `self.value` внутри nonisolated метода | Ошибка компиляции: "actor-isolated property" | nonisolated метод **не может** трогать изолированное состояние |
| Думать, что nonisolated делает метод потокобезопасным | Нет — только отключает await             | Самостоятельно обеспечивать безопасность |
| nonisolated на методе, который меняет состояние | Ошибка компиляции в strict concurrency   | Компилятор ловит |
| Забыть, что nonisolated async всё равно требует await на вызове результата | Ошибка при использовании результата      | `let result = await actor.nonisolatedAsyncMethod()` |
| Использовать nonisolated для тяжёлых вычислений | Задержка на вызывающем потоке            | Тяжёлое — в `Task.detached` или обычный `async` |

### 5. nonisolated vs другие механизмы (2026 сравнение)

| Механизм                  | Требуется await извне | Доступ к состоянию актёра | Потокобезопасность | Рекомендация 2026 | Когда использовать |
|---------------------------|-------------------------|----------------------------|---------------------|-------------------|---------------------|
| обычный метод актёра      | Да                      | Да                         | Полная              | Основной          | Всё, что трогает состояние |
| `nonisolated` метод       | Нет                     | Нет                        | Отсутствует          | Для утилит        | Константы, чистые вычисления |
| `nonisolated(unsafe)`     | Нет                     | Да (unsafe)                | Отсутствует          | Редко             | Низкоуровневый код |
| `@MainActor` метод        | Нет (если уже на main)  | Да (main thread)           | Полная (main)       | UI                | ViewModel, контроллеры |
| `isolated(any)` параметр  | Нет внутри функции      | Да (от параметра)          | Полная              | Универсальные функции | Функции для актёров |

### 6. Лучшие практики 2026 года

- **nonisolated** — используй **только** для методов, которые **не трогают** изолированное состояние актёра  
- **nonisolated async** — мощный паттерн для асинхронных утилит, но помни про `await` на вызове результата  
- **Всегда проверяй** — если метод трогает `self.что-то` → он **не может** быть nonisolated  
- **Для UI** — `@MainActor` предпочтительнее nonisolated  
- **Для тестов** — nonisolated методы удобно мокать и вызывать синхронно  
- **Swift 6 strict concurrency** — включай полную проверку — она ловит попытки доступа к состоянию в nonisolated методах
