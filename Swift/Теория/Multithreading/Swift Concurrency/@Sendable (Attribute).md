## 1. Что такое `@Sendable`

`@Sendable` — это **атрибут для замыканий (closures)**, который говорит:

> «Это замыкание безопасно для использования в многопоточности,  
> его можно передавать между task’ами и actor’ами без гонок данных».

---

## 2. Зачем нужен `@Sendable`

- В Swift Concurrency задачи ([[Task]]) и акторы ([[actor]]) могут выполняться на разных потоках.
    
- Если замыкание **не гарантирует потокобезопасность**, это может привести к гонкам данных ([[race condition]]).
    
- `@Sendable` заставляет компилятор проверять замыкание:
    
    - захваченные переменные должны быть `Sendable` (значимые типы или `@Sendable` объекты).
        
    - внутри замыкания нельзя «мутировать» что попало без синхронизации.
        

---

## 3. Примеры от простого к сложному

### Пример 1. Простое замыкание

```swift
let closure: @Sendable () -> Void = {
    print("Безопасное замыкание")
}
```

👉 Здесь `@Sendable` указывает, что замыкание можно запускать из разных потоков.

---

### Пример 2. Использование в Task

```swift
Task(priority: .high, operation: { @Sendable in
    print("Запускается в Task")
})
```

👉 Атрибут гарантирует, что замыкание можно безопасно передать в Task.

---

### Пример 3. Ошибка без `Sendable`

```swift
var counter = 0

// ❌ ошибка: захват переменной, не безопасной для многопоточности
let closure = { 
    counter += 1
}
```

С `@Sendable`:

```swift
var counter = 0

let closure: @Sendable () -> Void = {
    // ❌ Ошибка компиляции: нельзя мутировать counter, так как он не изолирован
    // counter += 1
    print("Можно только читать:", counter)
}
```

👉 `@Sendable` помогает отлавливать потенциальные гонки данных.

---

### Пример 4. Передача Sendable-значений

```swift
struct User: Sendable {
    let name: String
}

let user = User(name: "Alex")

let closure: @Sendable () -> Void = {
    print("Пользователь:", user.name) // ✅ можно
}
```

👉 `User` — безопасный (`Sendable`), значит, его можно захватывать.

---

### Пример 5. Класс без Sendable

```swift
class Logger {
    var text = ""
}

let logger = Logger()

let closure: @Sendable () -> Void = {
    // ❌ Ошибка: Logger не Sendable, нельзя безопасно использовать
    print(logger.text)
}
```

---

### Пример 6. Исправляем с помощью `@unchecked Sendable`

```swift
final class Logger: @unchecked Sendable {
    var text = ""
}

let logger = Logger()

let closure: @Sendable () -> Void = {
    // ✅ теперь компилятор разрешает
    print(logger.text)
}
```

👉 `@unchecked Sendable` говорит: «Я ручаюсь за потокобезопасность сам». Но это **опасно**, нужно использовать с осторожностью.

---

### Пример 7. Combine с Sendable

```swift
import Combine

func performWork(handler: @escaping @Sendable (String) -> Void) {
    Task {
        handler("Hello")
    }
}

performWork { text in
    print("Результат:", text)
}
```

---

## 4. Когда использовать `@Sendable`

- Всегда, когда пишете замыкание, которое будет передаваться в `Task`, `Task.detached`, async API, или может выполняться параллельно.
    
- Для API, где вы хотите **явно гарантировать** потокобезопасность.
    
- Чтобы компилятор проверял корректность (лучше поймать баг при компиляции, чем ловить крэш в рантайме).
    

---

## 5. Сравнение

|Атрибут|Что делает|
|---|---|
|`@Sendable`|Делает замыкание потокобезопасным, требует чтобы захваченные данные были Sendable|
|`Sendable` (протокол)|Протокол для типов, которые можно безопасно передавать между потоками|
|`@unchecked Sendable`|Разрешает типу быть Sendable без проверки компилятора (ответственность на вас)|

---

## 6. Итог

- `@Sendable` = безопасные замыкания для concurrency.
    
- Он защищает от гонок данных и делает код более предсказуемым.
    
- Если компилятор ругается на `@Sendable` замыкание → значит, у вас потенциальная проблема с потокобезопасностью.
