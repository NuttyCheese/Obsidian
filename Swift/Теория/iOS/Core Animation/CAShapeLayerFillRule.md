**`CAShapeLayerFillRule`** — это перечисление ([[enum]]), которое определяет **правило заливки** (fill rule) для сложных путей в **[[CAShapeLayer]]**.

Когда путь содержит **пересечения**, **вложенные контуры** или **дырки** (например, буква "O", пончик, многоугольник с вырезом), правило заливки решает, какие области будут **закрашены**, а какие останутся **прозрачными**.

### Возможные значения (два варианта)

| Значение   | Константа в [[Objective-C]] / [[Swift]] | Как работает (правило)                                                           | Когда использовать (самые частые случаи)                                              |
| ---------- | --------------------------------------- | -------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| `.nonZero` | `kCAFillRuleNonzero`                    | Заливает область, если количество пересечений пути с лучом из точки **нечётное** | Простые формы, вложенные контуры без дырок, большинство стандартных случаев           |
| `.evenOdd` | `kCAFillRuleEvenOdd`                    | Заливает область, если количество пересечений пути с лучом из точки **чётное**   | **Формы с дырками**, буквы с отверстиями (O, B, 8, a, e), пончики, вырезанные области |

### Визуальное сравнение (самый понятный пример)

Представим путь в форме буквы "O" (внешний круг + внутренний круг):

```swift
let outer = UIBezierPath(ovalIn: CGRect(x: 20, y: 20, width: 200, height: 200))
let inner = UIBezierPath(ovalIn: CGRect(x: 80, y: 80, width: 80, height: 80))

let combined = outer
combined.append(inner)  // ← добавляем внутренний контур
```

Теперь смотрим, как заливается:

| fillRule       | Результат заливки                              | Объяснение                                                                 |
|----------------|------------------------------------------------|----------------------------------------------------------------------------|
| `.nonZero`     | Весь "пончик" залит (внутренний круг тоже)     | Оба контура идут в одну сторону → счётчик нечётный везде → всё залито     |
| `.evenOdd`     | Только внешний круг залит, центр — дырка       | Внутренний контур создаёт чётное пересечение → область остаётся пустой    |

### Рекомендуемый паттерн использования в 2026 году

```swift
let shapeLayer = CAShapeLayer()

let outer = UIBezierPath(ovalIn: CGRect(x: 20, y: 20, width: 200, height: 200))
let inner = UIBezierPath(ovalIn: CGRect(x: 80, y: 80, width: 80, height: 80))

let path = outer
path.append(inner)  // ← внутренний контур добавлен

shapeLayer.path = path.cgPath
shapeLayer.fillColor = UIColor.systemBlue.cgColor
shapeLayer.strokeColor = UIColor.black.cgColor
shapeLayer.lineWidth = 4

// Самое важное — выбор правила заливки
shapeLayer.fillRule = .evenOdd  // ← для дырки внутри
// shapeLayer.fillRule = .nonZero   // ← весь круг залит

layer.addSublayer(shapeLayer)
```

### Когда какой fillRule использовать (практические рекомендации)

| Ситуация / желаемый результат                     | Рекомендуемый fillRule | Почему именно этот |
|---------------------------------------------------|-------------------------|--------------------|
| Простая форма без дырок (круг, квадрат, звезда)   | `.nonZero` (по умолчанию) | Всё заливается корректно |
| Форма с отверстием (бублик, буква O, B, 8, a, e)  | `.evenOdd`              | Внутренний контур создаёт дырку |
| Многоугольник с вырезом (дом с окнами)            | `.evenOdd`              | Все внутренние контуры становятся прозрачными |
| Сложная иконка с несколькими вложенными областями | `.evenOdd`              | Самое предсказуемое поведение |
| Контур идёт в разные стороны (clockwise / counterclockwise) | `.nonZero`              | Счётчик направления учитывается |

### Важные нюансы и лучшие практики 2026

- **По умолчанию** — `fillRule = .nonZero` (самое безопасное для простых форм)
- **Для большинства иконок и UI-элементов с дырками** — переключайтесь на `.evenOdd`
- **Направление контура важно** только для `.nonZero` (по часовой / против)
- **Не забывайте** `path.closeSubpath()` для замкнутых контуров (хотя `UIBezierPath` часто делает это автоматически)
- **В SwiftUI** — эквивалент — `Path` + `.fill(style: FillStyle(eoFill: true))` для `.evenOdd`
- **Swift 6 strict concurrency** — `CAShapeLayer` и `UIBezierPath` — только на главном потоке (`@MainActor`)
- **Документируйте** — пишите комментарий «fillRule = .evenOdd — для создания дырок в форме (внутренние контуры прозрачные)»

**Короткий итог 2026**:
> `CAShapeLayer.fillRule` решает, как заливать **сложные пути с пересечениями и дырками**.  
> В 2026 году:  
> - `.nonZero` — по умолчанию, для простых форм  
> - `.evenOdd` — почти всегда, когда есть отверстия (O, B, пончик, вырезы)  
> - используйте с [[UIBezierPath]] + `CAShapeLayer.path = path.cgPath`  
> Это **маленькое, но критическое** свойство, которое спасает от "почему у меня всё залито, а не должно быть"
