**`CAShapeLayerLineCap`** — это перечисление ([[enum]]), которое определяет **стиль концов линий** (line cap style) для обводки (`stroke`) в **[[CAShapeLayer]]**.

Оно управляет тем, как выглядят **концы** открытого или незакрытого пути при отрисовке линии.

### Возможные значения (все варианты)

| Значение в Swift                        | Константа в Objective-C          | Как выглядит конец линии                              | Когда использовать (самые частые случаи) |
|-----------------------------------------|----------------------------------|-------------------------------------------------------|-------------------------------------------|
| `.butt`                                 | `kCALineCapButt`                 | Плоский срез ровно по конечной точке                 | По умолчанию — самый резкий и аккуратный  |
| `.round`                                | `kCALineCapRound`                | Полукруглый (радиус = половине толщины линии)         | Самый популярный — плавные, мягкие концы  |
| `.square`                               | `kCALineCapSquare`               | Квадратный выступ (длина = половине толщины линии)    | Редко — когда нужен чёткий квадратный конец |

### Визуальное сравнение (самый понятный пример)

Представьте горизонтальную линию толщиной 20 pt:

```swift
let path = UIBezierPath()
path.move(to: CGPoint(x: 50, y: 100))
path.addLine(to: CGPoint(x: 300, y: 100))

let layer = CAShapeLayer()
layer.path = path.cgPath
layer.lineWidth = 20
layer.strokeColor = UIColor.systemBlue.cgColor
layer.fillColor = nil
```

Теперь меняем только `lineCap`:

| lineCap     | Визуальный результат (концы линии)                     | Описание |
|-------------|--------------------------------------------------------|----------|
| `.butt`     | Резкий плоский срез                                    | Самый строгий и точный |
| `.round`    | Полукруглые "шапочки" на концах                        | Самый мягкий и часто используемый |
| `.square`   | Квадратные выступы (как маленькие прямоугольники)      | Редко, когда нужен "квадратный" конец |

### Рекомендуемый паттерн использования в 2026 году

```swift
let path = UIBezierPath(arcCenter: CGPoint(x: 100, y: 100),
                        radius: 80,
                        startAngle: 0,
                        endAngle: .pi * 1.5,
                        clockwise: true)

let shapeLayer = CAShapeLayer()
shapeLayer.path = path.cgPath
shapeLayer.fillColor = nil
shapeLayer.strokeColor = UIColor.systemPurple.cgColor
shapeLayer.lineWidth = 16
shapeLayer.lineCap = .round          // ← Самый популярный выбор
shapeLayer.strokeEnd = 0.75          // для анимированного прогресса

view.layer.addSublayer(shapeLayer)

// Анимация заполнения
let animation = CABasicAnimation(keyPath: "strokeEnd")
animation.fromValue = 0
animation.toValue = 1
animation.duration = 1.5
animation.timingFunction = CAMediaTimingFunction(name: .easeInEaseOut)
shapeLayer.add(animation, forKey: "progress")
shapeLayer.strokeEnd = 1
```

### Когда какой lineCap использовать (практические рекомендации)

| Ситуация / желаемый визуальный стиль              | Рекомендуемый lineCap | Почему именно этот |
|---------------------------------------------------|-----------------------|--------------------|
| Прогресс-бары, круговые индикаторы, лоадеры       | `.round`              | Самые плавные и приятные концы |
| Чёткие геометрические линии, сетки, графики       | `.butt`               | Максимальная точность и острота |
| Ретро-стиль, пиксель-арт, "квадратные" иконки     | `.square`             | Специфический "квадратный" выступ |
| Толстые линии с острыми углами                    | `.butt`               | Не добавляет лишних полукругов |
| Анимация "рисования" (stroke animation)           | `.round`              | Концы выглядят естественно при анимации |

### Лучшие практики CAShapeLayer.lineCap в Swift 2026

- **По умолчанию** — оставляйте `.butt` (самый нейтральный)  
- **Для большинства UI-элементов** — переключайтесь на `.round` — выглядит мягче и современнее  
- **Для `.square`** — используйте только осознанно (редкие стилизованные случаи)  
- **Комбинируйте** с `lineJoin = .round` — вместе дают самый плавный результат  
- **Для анимаций stroke** — `.round` почти всегда лучше (концы не "обрезаются" резко)  
- **В SwiftUI** — эквивалент — `StrokeStyle(lineCap: .round)` в `Path.stroke(style:)`  
- **Swift 6 strict concurrency** — [[CAShapeLayer]] — только на главном потоке ([[@MainActor]])  
- **Документируйте** — пишите комментарий «lineCap = .round — для плавных концов прогресс-бара»

**Короткий итог 2026**:
> `CAShapeLayer.lineCap` управляет **формой концов линий** при обводке пути.  
> В 2026 году:  
> - `.butt` — по умолчанию, самый точный и резкий  
> - `.round` — **самый популярный** и визуально приятный (почти всегда выбирайте его)  
> - `.square` — только для специфического стиля  
> Это **маленькое**, но **очень заметное** свойство, которое сильно влияет на эстетику линий и прогресс-индикаторов.
