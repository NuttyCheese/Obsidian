**`CAShapeLayerLineJoin`** — это перечисление (enum) в [[Core Animation]], которое определяет **стиль соединения линий** (line join style) при обводке пути в **[[CAShapeLayer]]**.

Оно управляет тем, как выглядят **углы** и **места стыка** линий в пути, особенно при толстой обводке (`lineWidth > 1`).

### Возможные значения (все три варианта)

| Значение в Swift                        | Константа в Objective-C          | Как выглядит соединение линий                          | Когда использовать (самые частые случаи) |
|-----------------------------------------|----------------------------------|--------------------------------------------------------|-------------------------------------------|
| `.miter`                                | `kCALineJoinMiter`               | Острый угол (выступающий шип)                          | По умолчанию — для чётких, острых углов   |
| `.round`                                | `kCALineJoinRound`               | Закруглённый угол (полукруглый радиус = половине толщины линии) | Самый популярный — мягкие, приятные углы |
| `.bevel`                                | `kCALineJoinBevel`               | Скошенный угол (прямой срез под 45°)                   | Редко — когда нужен "срезанный" угол      |

### Визуальное сравнение (на примере угла 90° с толщиной линии 20 pt)

Представьте ломаную линию с острым углом:

```swift
let path = UIBezierPath()
path.move(to: CGPoint(x: 100, y: 100))
path.addLine(to: CGPoint(x: 200, y: 200))
path.addLine(to: CGPoint(x: 300, y: 100))

let layer = CAShapeLayer()
layer.path = path.cgPath
layer.lineWidth = 20
layer.strokeColor = UIColor.systemBlue.cgColor
layer.fillColor = nil
```

Теперь меняем только `lineJoin`:

| lineJoin     | Визуальный результат (угол)                           | Описание |
|--------------|--------------------------------------------------------|----------|
| `.miter`     | Острый выступающий шип                                 | Самый резкий и геометрически точный |
| `.round`     | Плавный закруглённый угол                              | Самый мягкий и часто используемый |
| `.bevel`     | Скошенный угол (как срез под 45°)                      | Средний вариант, редко нужен |

### Рекомендуемый паттерн использования в 2026 году

```swift
let path = UIBezierPath()
path.move(to: CGPoint(x: 50, y: 50))
path.addLine(to: CGPoint(x: 200, y: 200))
path.addLine(to: CGPoint(x: 350, y: 80))
path.addLine(to: CGPoint(x: 500, y: 180))

let shapeLayer = CAShapeLayer()
shapeLayer.path = path.cgPath
shapeLayer.fillColor = nil
shapeLayer.strokeColor = UIColor.systemPurple.cgColor
shapeLayer.lineWidth = 24
shapeLayer.lineCap = .round      // для плавных концов
shapeLayer.lineJoin = .round     // ← Самый популярный выбор для углов
shapeLayer.strokeEnd = 0.0       // для анимации

view.layer.addSublayer(shapeLayer)

// Анимация обводки
let animation = CABasicAnimation(keyPath: "strokeEnd")
animation.fromValue = 0
animation.toValue = 1
animation.duration = 2.5
animation.timingFunction = CAMediaTimingFunction(name: .easeInEaseOut)
shapeLayer.add(animation, forKey: "stroke")
shapeLayer.strokeEnd = 1
```

### Когда какой lineJoin использовать (практические рекомендации)

| Ситуация / желаемый визуальный стиль              | Рекомендуемый lineJoin | Почему именно этот |
|---------------------------------------------------|------------------------|--------------------|
| Прогресс-бары, круговые индикаторы, лоадеры       | `.round`               | Самые плавные и современные углы |
| Чёткие геометрические формы, графики, сетки       | `.miter`               | Максимальная точность и острота углов |
| Ретро-стиль, пиксель-арт, "срезанные" углы        | `.bevel`               | Специфический "скошенный" эффект |
| Толстые линии (> 10 pt) с острыми углами          | `.round`               | Избегает слишком длинных шипов (miter) |
| Анимация "рисования" (stroke animation)           | `.round`               | Углы выглядят естественно при заполнении |

### Лучшие практики CAShapeLayer.lineJoin в Swift 2026

- **По умолчанию** — оставляйте `.miter` (самый геометрически точный)  
- **Для большинства UI-элементов** — переключайтесь на `.round` — выглядит мягче, современнее и приятнее  
- **Для `.bevel`** — используйте только осознанно (очень редкие стилизованные случаи)  
- **Всегда комбинируйте** с `lineCap = .round` — вместе дают самый плавный и профессиональный результат  
- **Контролируйте длину шипов** через `miterLimit` (по умолчанию 10) — если используете `.miter` и линии очень толстые  
- **В SwiftUI** — эквивалент — `StrokeStyle(lineJoin: .round)` в `Path.stroke(style:)`  
- **Swift 6 strict concurrency** — `CAShapeLayer` — только на главном потоке (`@MainActor`)  
- **Документируйте** — пишите комментарий «lineJoin = .round — для плавных соединений линий в прогресс-баре»

**Короткий итог 2026**:
> `CAShapeLayer.lineJoin` управляет **формой углов** при обводке пути.  
> В 2026 году:  
> - `.miter` — по умолчанию, острые и точные углы  
> - `.round` — **самый популярный** и визуально приятный (выбирайте его в 90% случаев)  
> - `.bevel` — только для специфического стиля  
> Это **маленькое**, но **очень заметное** свойство, которое сильно влияет на эстетику толстых линий и углов.
