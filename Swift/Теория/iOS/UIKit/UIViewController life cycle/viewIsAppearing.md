**`viewIsAppearing`** — это свойство жизненного цикла [[UIViewController]] в [[UIKit]], которое появилось в **iOS 17** (2023) и стало одним из самых важных дополнений к классическому циклу в 2025–2026 годах.

### Что именно делает viewIsAppearing

```swift
var viewIsAppearing: Bool { get }
```

- **true** — когда контроллер **находится в процессе появления** на экране (между [[viewWillAppear]] и [[viewDidAppear]])
- **false** — во всех остальных случаях

Это **самый точный** индикатор того, что контроллер **реально виден пользователю прямо сейчас** и находится в переходной фазе появления.

### Сравнение с другими методами жизненного цикла

| Метод / Свойство            | Когда true / вызывается                              | Можно ли использовать для одноразовых действий при появлении? | Можно ли использовать для анимаций появления? | Можно ли использовать для сетевых запросов? |
|-----------------------------|------------------------------------------------------|---------------------------------------------------------------|-----------------------------------------------|---------------------------------------------|
| `viewWillAppear`            | Перед появлением (может вызываться многократно)      | Нет (вызывается даже при возврате назад)                      | Да, но часто слишком рано                     | Да, но может быть избыточно                 |
| `viewDidAppear`             | После полного появления                              | Да (но уже поздно для некоторых анимаций)                     | Нет (анимация уже закончилась)                | Да (стандартный выбор)                      |
| `viewIsAppearing` (iOS 17+) | Только во время первого появления (один раз за цикл) | **Да** (идеально)                                             | **Да** (лучшее место)                         | Да (очень хороший выбор)                    |
| `isBeingPresented`          | Только при модальном представлении                   | Частично                                                      | Частично                                      | Частично                                    |
| `isMovingToParent`          | Только при добавлении в контейнер                    | Частично                                                      | Частично                                      | Частично                                    |

### Самые популярные и рекомендуемые паттерны viewIsAppearing в 2026 году

#### 1. Одноразовая анимация появления (самый частый и лучший кейс)

```swift
override func viewIsAppearing(_ animated: Bool) {
    super.viewIsAppearing(animated)
    
    // Этот код выполнится **только один раз** при появлении контроллера
    
    // Идеальное место для анимаций входа
    logoImageView.transform = CGAffineTransform(scaleX: 0.1, y: 0.1)
    logoImageView.alpha = 0
    
    UIView.animate(withDuration: 0.6, delay: 0.1, usingSpringWithDamping: 0.7, initialSpringVelocity: 0.5) {
        self.logoImageView.transform = .identity
        self.logoImageView.alpha = 1
    }
}
```

#### 2. Первый сетевой запрос при появлении экрана

```swift
private var hasAppeared = false

override func viewIsAppearing(_ animated: Bool) {
    super.viewIsAppearing(animated)
    
    if !hasAppeared {
        hasAppeared = true
        
        // Первый запрос данных только при первом появлении
        fetchInitialData()
    }
}
```

#### 3. Адаптация UI под текущий traitCollection / размер экрана

```swift
override func viewIsAppearing(_ animated: Bool) {
    super.viewIsAppearing(animated)
    
    // Можно комбинировать с traitCollection
    if traitCollection.horizontalSizeClass == .regular {
        // iPad или большой iPhone в ландшафте
        updateLayoutForRegularSizeClass()
    }
}
```

### Лучшие практики viewIsAppearing в Swift 2026

- **Используйте** `viewIsAppearing` для:
  - одноразовых анимаций входа
  - первого запроса данных при появлении экрана
  - настройки, которая должна выполниться **только при первом появлении**
- **Не используйте** для:
  - повторяющихся действий при возврате на экран (для этого — `viewWillAppear` / `viewDidAppear`)
  - тяжёлых операций (сеть, диск) — лучше `viewDidAppear` или асинхронно
  - layout-логики (размеры) — это всё ещё `viewDidLayoutSubviews`
- **Всегда** вызывайте `super.viewIsAppearing(animated)`
- **Для SwiftUI** — аналог — `.task { }` или `.onAppear` (но `.task` предпочтительнее для одноразовых операций)
- **Документируйте** — пишите комментарий «viewIsAppearing — одноразовая анимация появления логотипа (выполняется только при первом показе экрана)»

**Короткий итог 2026**:
> `viewIsAppearing(_:)` — это **новый (с iOS 17)** метод жизненного цикла, который вызывается **ровно один раз** за цикл появления контроллера — в момент, когда он **реально становится видимым**.  
> В 2026 году:  
> - лучшее место для **одноразовых анимаций входа**  
> - отличное место для **первого сетевого запроса** при появлении экрана  
> - вызывается **после** `viewWillAppear` и **до** `viewDidAppear`  
> - это **один из самых полезных** методов для чистого и современного кода в UIKit за последние годы
