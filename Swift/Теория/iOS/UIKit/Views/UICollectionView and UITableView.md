Вот **самое понятное и актуальное** (2026 год) сравнение **[[UITableView]]** и **[[UICollectionView]]** — без воды, с акцентом на реальные отличия и когда что выбирать.

### Короткая таблица отличий (2026)

| Критерий                              | [[UITableView]]                                            | [[UICollectionView]]                                     | Кто побеждает в 2026              |
| ------------------------------------- | ---------------------------------------------------------- | -------------------------------------------------------- | --------------------------------- |
| Основная форма отображения            | Вертикальный список (одна колонка)                         | Любая сетка, карусель, поток, кастомный layout           | UICollectionView (гибче)          |
| Колонки                               | Только 1 (или 2 в iPad с .grouped стилем)                  | Любое количество (динамически)                           | **UICollectionView**              |
| Размер ячеек                          | Обычно одинаковая высота (или динамическая)                | Каждая ячейка может иметь свой размер                    | **UICollectionView**              |
| Горизонтальная прокрутка              | Не поддерживается нативно                                  | Поддерживается нативно (scrollDirection = .horizontal)   | **UICollectionView**              |
| Supplementary views (headers/footers) | Только секционные заголовки и футеры                       | Полноценные supplementary views + decoration views       | **UICollectionView**              |
| Кастомные layouts                     | Только Flow-like ([[UITableViewFlowLayout]] не существует) | Полностью кастомные layouts ([[UICollectionViewLayout]]) | **UICollectionView**              |
| Производительность на больших данных  | Отличная (очень оптимизирована под списки)                 | Хорошая, но требует больше внимания к layout             | UITableView (для простых списков) |
| Код и сложность                       | Проще писать и поддерживать                                | Больше кода (особенно с кастомным layout)                | UITableView (для простоты)        |
| [[SwiftUI]] аналог                    | List / ScrollView + LazyVStack                             | LazyVGrid / LazyHGrid + кастомные layouts                | —                                 |

### Когда в 2026 году выбирать что

| Сценарий                                             | Лучший выбор в 2026                         | Почему именно этот                                                    |
| ---------------------------------------------------- | ------------------------------------------- | --------------------------------------------------------------------- |
| Простой список (настройки, чат, контакты, новости)   | **UITableView**                             | Меньше кода, лучше производительность, нативная оптимизация           |
| Галерея фото/видео, карточки товаров, рекомендации   | **UICollectionView**                        | Разные размеры ячеек, горизонтальный скролл, кастомные layouts        |
| Карусель, слайдер, горизонтальная лента              | **UICollectionView**                        | Нативная горизонтальная прокрутка + paging                            |
| Сетка с динамическими размерами (Pinterest-style)    | **UICollectionView** + Compositional Layout | Самый простой способ в 2026 — [[UICollectionViewCompositionalLayout]] |
| Сложный dashboard с разными типами ячеек             | **UICollectionView**                        | Легко смешивать разные размеры и типы                                 |
| Чистый SwiftUI проект                                | **LazyVGrid / LazyHGrid**                   | Нет смысла использовать [[UIKit]]                                     |
| Очень большой список (тысячи строк) без кастомизации | **UITableView**                             | Лучшая оптимизация под большие списки                                 |

### Короткий вердикт 2026

- **UITableView** → простой вертикальный список, минимум кода, максимальная скорость  
- **UICollectionView** → всё остальное: сетки, горизонтальный скролл, разные размеры ячеек, кастомные layouts, supplementary views

**Золотое правило 2026**:
> Если тебе нужна **одна колонка** и **простой список** → бери **UITableView**  
> Если хоть немного **гибкости** (колонки, размеры, горизонталь, кастомный layout) → сразу бери **UICollectionView** (особенно с **Compositional Layout**)

В новых проектах 2025–2026 большинство разработчиков уже **не используют UITableView**, если только это не классический список настроек или чат.
