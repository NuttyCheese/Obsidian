**`while`** — это **цикл с предусловием** в [[Swift]]:  
проверяет условие **перед** каждой итерацией и выполняет тело цикла **только пока условие истинно**.

Если условие изначально ложно — тело цикла **не выполнится ни разу** (в отличие от [[repeat-while]]).

### Основные отличия while от repeat-while (2025–2026)

| Характеристика           | `while`                    | `repeat-while`                    | Когда выбрать while                 |
| ------------------------ | -------------------------- | --------------------------------- | ----------------------------------- |
| Проверка условия         | **до** выполнения тела     | **после** выполнения тела         | Условие может быть ложно сразу      |
| Гарантия выполнения тела | Нет (0 или больше раз)     | Да (минимум 1 раз)                | Не нужна хотя бы одна итерация      |
| Самый частый паттерн     | Пока что-то не закончилось | Запрос ввода до корректных данных | Чтение, поиск, обработка до условия |
| Читаемость               | Логика условия видна сразу | Сначала тело, потом условие       | Условие логически стоит впереди     |

### Самые популярные и рекомендуемые паттерны while в 2026

#### 1. Классический счётчик (самый понятный)

```swift
var i = 0
while i < 5 {
    print(i)          // 0 1 2 3 4
    i += 1
}
```

#### 2. Обработка до выполнения условия (очень частый случай)

```swift
var attempts = 0
let maxAttempts = 3

while attempts < maxAttempts {
    print("Попытка \(attempts + 1)")
    
    if simulateLoginSuccess() {
        print("Успех!")
        break
    }
    
    attempts += 1
}

if attempts == maxAttempts {
    print("Превышено количество попыток")
}
```

#### 3. Чтение/обработка данных до конца (очень популярно в парсерах, консольных утилитах)

```swift
var index = 0
let lines = ["команда1", "команда2", "exit", "команда3"]

while index < lines.count {
    let command = lines[index]
    
    if command == "exit" {
        print("Выход")
        break
    }
    
    print("Выполняю:", command)
    index += 1
}
```

#### 4. while + guard / [[if let ]](современный стиль)

```swift
var optionalData: Data? = fetchSomeData()

while let data = optionalData {
    process(data)
    
    // следующая порция данных
    optionalData = fetchNextChunk()
    
    // если данных больше нет → optionalData станет nil → цикл завершится
}
```

#### 5. Бесконечный цикл с условием выхода (while true + [[break]])

```swift
while true {
    print("Ожидание ввода...")
    guard let input = readLine(), !input.isEmpty else {
        print("Пустой ввод → выход")
        break
    }
    
    if input.lowercased() == "quit" {
        break
    }
    
    process(input)
}
```

Это **самый популярный** паттерн для консольных утилит и интерактивных скриптов.

### 6. Лучшие практики while в Swift 2026

- **Предпочитай `guard` / `if let` + `break`** вместо сложных условий в `while`  
- **Избегай** `while true` без явного `break` — код становится трудно читаемым  
- **Не используй** `while` для известного количества итераций — лучше `for-in` или `for i in 0..<n`  
- **Для бесконечных задач** (сервер, игра) — чаще `Task { while !isCancelled { … } }`  
- **В [[SwiftUI]] / [[async]]** — `while` почти не используется → заменяется `Task` + `for await` или `.onReceive`  
- **Документируйте** — пишите комментарий «while — обработка строк до команды 'exit'»

**Короткий девиз 2026**:
> `while` — это «пока условие истинно — делай».  
> В 2026 году:  
> - используй, когда **количество итераций заранее неизвестно**  
> - предпочитай `guard` + `break` или `while let` для читаемости  
> - `while true` — только с чётким `break` внутри  
> - для известного диапазона → `for-in`  
> - в асинхронном коде → почти всегда `for await` или `Task.sleep`
