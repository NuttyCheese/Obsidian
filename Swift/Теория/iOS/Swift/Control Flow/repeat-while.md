**`repeat-while`** — это **цикл с постусловием** в [[Swift]]: тело цикла выполняется **минимум один раз**, а проверка условия происходит **после** каждой итерации.

Главное отличие от обычного `while`:

- `while` — проверка **до** выполнения тела  
- `repeat-while` — проверка **после** выполнения тела

Поэтому `repeat-while` гарантирует **хотя бы одно** выполнение кода.

### Когда repeat-while предпочтительнее while (2025–2026 практика)

| Ситуация                                              | Почему repeat-while лучше                                | Альтернатива (когда выбрать while) |
|-------------------------------------------------------|----------------------------------------------------------|-------------------------------------|
| Нужно выполнить действие хотя бы один раз             | Гарантия первого выполнения                              | —                                   |
| Запрос ввода пользователя до получения корректных данных | Не нужно дублировать ввод перед циклом                   | while + предварительный ввод        |
| Инициализация + повтор до выполнения условия          | Код инициализации не дублируется                         | while с предварительной инициализацией |
| Работа с опциональными значениями в цикле             | Часто сочетается с `guard let` / `if let` внутри         | —                                   |
| Генерация первого элемента + продолжение до условия   | Естественная структура                                   | —                                   |

### Полный синтаксис и все важные варианты

#### Базовый вид

```swift
repeat {
    // тело цикла — выполняется минимум 1 раз
} while условие
```

#### С break / continue

```swift
var attempts = 0

repeat {
    attempts += 1
    print("Попытка №\(attempts)")
    
    if attempts == 3 {
        print("Превышено количество попыток")
        break
    }
    
    if Bool.random() {          // симуляция успеха
        print("Успех!")
        break
    }
} while attempts < 10
```

#### С continue (пропуск итерации)

```swift
var i = 0

repeat {
    i += 1
    
    if i % 3 == 0 {
        continue  // пропускаем кратные 3
    }
    
    print(i)  // 1, 2, 4, 5, 7, 8, 10, ...
} while i < 10
```

#### Самый популярный паттерн 2026 — валидация ввода

```swift
var input: String?

repeat {
    print("Введите положительное число:")
    input = readLine()
    
    guard let text = input,
          let number = Int(text),
          number > 0 else {
        print("Некорректный ввод, попробуйте снова")
        continue
    }
    
    print("Вы ввели:", number)
    break
    
} while true
```

Этот паттерн встречается очень часто в консольных утилитах, скриптах и при отладке.

#### С опциональными значениями + [[guard let]]

```swift
var optionalValue: Int? = nil

repeat {
    print("Введите число или 'q' для выхода:")
    if let input = readLine()?.lowercased(), input == "q" {
        break
    }
    
    guard let value = Int(readLine() ?? "") else {
        print("Не число, повторите")
        continue
    }
    
    optionalValue = value
    break
    
} while true

print("Получено:", optionalValue ?? "ничего")
```

### 5. Лучшие практики repeat-while в Swift 2026

- Используй `repeat-while`, когда **гарантированно** нужно выполнить тело хотя бы один раз  
- В остальных случаях → обычный `while` (более понятная логика)  
- Очень часто сочетается с `guard let` / `if let` внутри тела  
- Для бесконечных циклов с выходом через `break` — пиши `while true` или `repeat { ... } while true`  
- Не забывай `break` / `continue` — они работают точно так же, как в `while`  
- В SwiftUI / асинхронном коде → почти не используется (заменяется `Task`, `AsyncStream`, `for await`)  
- Документируйте — пиши комментарий «repeat-while — запрашиваем ввод до получения корректного числа»

**Короткий девиз 2026**:
> `repeat-while` — это цикл, который говорит: «сначала сделай, потом проверь, нужно ли повторить».  
> Используй его именно тогда, когда **первое выполнение гарантировано** (ввод, инициализация, минимум одна попытка).  
> Во всех остальных случаях — обычный `while` читается лучше.
