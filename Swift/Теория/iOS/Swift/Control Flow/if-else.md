**`if-else`** — это базовый оператор ветвления в [[Swift]], который позволяет **выполнять разные блоки кода в зависимости от истинности условия** (логического выражения типа `Bool`).

Это один из самых часто используемых конструкций в любом языке, и в Swift он особенно удобен благодаря поддержке **Optional binding**, **pattern matching**, **guard-подобного стиля** и **короткой записи**.

### 1. Полный синтаксис и все варианты (актуально 2025–2026)

#### Вариант 1 — Классический `if`

```swift
if условие {
    // выполняется, если условие == true
}
```

#### Вариант 2 — Полный `if-else`

```swift
if условие {
    // true
} else {
    // false
}
```

#### Вариант 3 — Цепочка `if-else if-else` (самый частый в реальном коде)

```swift
if score >= 90 {
    grade = "A"
} else if score >= 80 {
    grade = "B"
} else if score >= 70 {
    grade = "C"
} else if score >= 60 {
    grade = "D"
} else {
    grade = "F"
}
```

#### Вариант 4 — `if let` — самый популярный способ работы с [[Optional]] (2026 стандарт)

```swift
let optionalName: String? = getUserName()

if let name = optionalName {
    print("Привет, \(name)!")
} else {
    print("Гость")
}

// Сокращённая запись (Swift 5.3+)
if let name {
    print("Привет, \(name)!")
}
```

#### Вариант 5 — `if let` + дополнительные условия через `,`

```swift
if let age = user?.age, age >= 18, !user.isBlocked {
    grantAdultAccess()
} else {
    showRestrictedContent()
}
```

#### Вариант 6 — `if case` — проверка enum / паттерна (очень мощно)

```swift
if case .success(let data) = networkResult {
    process(data)
} else {
    showError()
}
```

#### Вариант 7 — `if` без фигурных скобок (только одна строка)

```swift
if isPremium { showPremiumFeatures() }
```

**Правило 2026**:  
Если тело `if` или `else` — **одна строка**, фигурные скобки можно опустить.  
Но в команде часто требуют **всегда** писать скобки — для единообразия и безопасности.

### 2. Самые частые реальные сценарии (2025–2026)

| Сценарий                               | Лучший стиль 2026                            | Почему именно так      |
| -------------------------------------- | -------------------------------------------- | ---------------------- |
| Проверка Optional из [[API]] / словаря | `if let value = dict["key"]` или `guard let` | Безопасно, читаемо     |
| Валидация возраста / прав доступа      | `if age >= 18 && hasPermission`              | Линейно и понятно      |
| Обработка состояний [[enum]]           | `if case .success(let data) = result`        | Кратко, без [[switch]] |
| Проверка нескольких условий            | `if let a = a, let b = b, a > b`             | Плоский код            |
| Простое ветвление без вложенности      | `if condition { ... } else { ... }`          | Читаемость             |
| Ранний выход из функции                | `guard` вместо `if let` + `return`           | Линейный код дальше    |

### 3. Лучшие практики `if-else` в Swift 2026

- **Предпочитай `guard let`** в функциях — ранний выход делает код плоским  
- **Используй `if let`** когда логика внутри блока небольшая и локальная  
- **Комбинируй** `if let` + условия через `,` — это очень читаемо  
- **Не делай** глубокую вложенность `if-else` — лучше `guard` или `switch`  
- **Для enum** — чаще `if case` или `switch` вместо `if result == .success`  
- **В SwiftUI / Combine** — старайся избегать `if-else` в `body`, используй тернарный оператор или view-модификаторы  
- **Swift 6 strict concurrency** — `if-else` полностью безопасен  
- **Документируйте** — пиши комментарий «if let — безопасное извлечение токена»

**Короткий девиз 2026**:
> `if-else` — это когда ты говоришь: «если условие верно — делай это, иначе — вот это».  
> В 2026 году:  
> - `if let` / `guard let` — для Optional (99% случаев)  
> - `if case` — для enum и паттернов  
> - избегай глубокой вложенности → используй `guard` или `switch`  
> Это **основа** читаемого и безопасного кода.
