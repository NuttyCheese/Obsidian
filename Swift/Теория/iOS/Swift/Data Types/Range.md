**`Range`** — это структура в [[Swift]], представляющая **последовательность значений** от нижней границы (`lowerBound`) до верхней (`upperBound`).

Она используется для:
- безопасной итерации ([[for-in]]),
- взятия подпоследовательностей (slicing массивов и строк),
- проверки принадлежности ([[contains]]),
- работы с индексами и диапазонами дат, символов и других [[Comparable]] типов.

> Проще говоря: `Range` — это «от и до», с двумя главными вариантами:
> - `a..<b` — полуоткрытый (не включает b)
> - `a...b` — закрытый (включает b)

### 1. Основные виды Range (2026 актуально)

| Тип диапазона                | Синтаксис | Включает upperBound? | Тип элемента             | Самый частый сценарий 2026 |
| ---------------------------- | --------- | -------------------- | ------------------------ | -------------------------- |
| `Range<Bound>`               | `a..<b`   | Нет                  | `Bound` ([[Comparable]]) | Индексы массивов, строк    |
| `ClosedRange<Bound>`         | `a...b`   | Да                   | `Bound` (Comparable)     | Циклы `for-in`, даты       |
| `PartialRangeFrom<Bound>`    | `a...`    | —                    | `Bound`                  | От элемента и до конца     |
| `PartialRangeThrough<Bound>` | `...b`    | Да                   | `Bound`                  | До элемента включительно   |
| `PartialRangeUpTo<Bound>`    | `..<b`    | Нет                  | `Bound`                  | До элемента (исключая)     |

### 2. Самые важные свойства и методы

| Свойство / Метод           | Что возвращает / делает                                  | Пример 2026 |
|----------------------------|----------------------------------------------------------|-------------|
| `lowerBound`               | Нижняя граница                                           | `0..<5` → `0` |
| `upperBound`               | Верхняя граница (не включается для `Range`)              | `0..<5` → `5` |
| `contains(_:)`             | Проверяет, входит ли значение в диапазон                 | `(1...10).contains(5)` → `true` |
| `isEmpty`                  | Пустой ли диапазон                                       | `5..<5` → `true` |
| `count`                    | Количество элементов (только для `Range` и `ClosedRange`)| `0...5` → `6` |
| `clamped(to:)`             | Ограничивает диапазон другим диапазоном                  | `(0...10).clamped(to: 3...7)` → `3...7` |

### 3. Самые популярные и рекомендуемые паттерны 2026

#### 3.1 Итерация в цикле (самый частый случай)

```swift
// Полуоткрытый диапазон (0..<5)
for i in 0..<5 {
    print(i) // 0 1 2 3 4
}

// Закрытый диапазон (0...5)
for i in 0...5 {
    print(i) // 0 1 2 3 4 5
}
```

#### 3.2 Slicing массивов и строк (очень часто)

```swift
let fruits = ["яблоко", "банан", "апельсин", "груша", "киви"]

let firstThree = fruits[0..<3]          // ["яблоко", "банан", "апельсин"]
let lastTwo    = fruits[3...]           // ["груша", "киви"]
let upToThree  = fruits[..<3]           // ["яблоко", "банан", "апельсин"]

let text = "Hello, Swift!"
let greeting = text[..<5]               // "Hello"
```

#### 3.3 Проверка принадлежности

```swift
let validScores = 0...100
if validScores.contains(score) {
    print("Оценка валидна")
}
```

#### 3.4 Пропорциональное ограничение размеров ([[Auto Layout]] + Range)

```swift
let aspectRatio: CGFloat = 16/9

imageView.heightAnchor.constraint(
    equalTo: imageView.widthAnchor,
    multiplier: aspectRatio
).isActive = true
```

#### 3.5 Stride — проход с шагом (очень полезно для чисел)

```swift
// Чётные числа от 0 до 10 (не включая 10)
for i in stride(from: 0, to: 10, by: 2) {
    print(i) // 0 2 4 6 8
}

// До 10 включительно
for i in stride(from: 0, through: 10, by: 2) {
    print(i) // 0 2 4 6 8 10
}
```

#### 3.6 Частичные диапазоны в subscript

```swift
let array = [10, 20, 30, 40, 50]

// От индекса 2 и до конца
let tail = array[2...]      // [30, 40, 50]

// До индекса 3 (не включая)
let prefix = array[..<3]    // [10, 20, 30]
```

### 4. Лучшие практики Range в Swift 2026

- **Всегда** используй `..<` для индексов массивов и строк — это стандарт  
- **Используй** `...` для дат, счёта элементов, когда нужна включительная верхняя граница  
- **Предпочитай** `safeAreaLayoutGuide` и `layoutMarginsGuide` вместо фиксированных чисел  
- **Не используй** `Range` для отрицательных шагов — лучше `stride(from:through:by:)` с отрицательным `by`  
- **Для строк** — всегда работай с `String.Index`, а не с `Int`  
- **Документируйте** — пиши комментарий «0..<5 — индексы первых 5 элементов (не включая 5)»

**Короткий девиз 2026**:
> `Range` — это «от и до», безопасный и удобный способ работать с последовательностями.  
> В 2026 году:  
> - `..<` — для индексов и slicing  
> - `...` — для включительных диапазонов  
> - `stride` — для шага  
> - `safeAreaLayoutGuide` — для отступов  
> Это **основа** чистого и адаптивного кода в циклах, массивах и Auto Layout.
