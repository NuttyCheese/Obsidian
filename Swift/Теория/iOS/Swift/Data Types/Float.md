**`Float`** — это 32-битный тип с плавающей точкой (single precision) в [[Swift]], который используется, когда **точность ~6–7 значащих цифр** достаточна, а экономия памяти важна.

Он занимает **в 2 раза меньше** памяти, чем `Double` (4 байта против 8), но теряет в точности и диапазоне.

### 1. Сравнение Float vs [[Double]] (2026 актуально)

| Характеристика                | Float (32-bit)                        | Double (64-bit)                     | Когда выбрать Float в 2026          |
| ----------------------------- | ------------------------------------- | ----------------------------------- | ----------------------------------- |
| Размер в памяти               | 4 байта                               | 8 байт                              | Массивы/буферы больших данных       |
| Точность (значащих цифр)      | ~6–7                                  | ~15–17                              | Графика, игры, анимация (часто)     |
| Диапазон значений             | ±1.18×10⁻³⁸ … ±3.4×10³⁸               | ±1.7×10⁻³⁰⁸ … ±1.7×10³⁰⁸            | Float — для большинства UI/графики  |
| Минимальное положительное     | ≈1.4×10⁻⁴⁵                            | ≈2.225×10⁻³⁰⁸                       | —                                   |
| Производительность на CPU/GPU | Обычно быстрее (меньше данных)        | Медленнее на некоторых архитектурах | [[Metal]], SpriteKit, SceneKit      |
| Рекомендация Apple            | Использовать в графике, буферах, SIMD | По умолчанию для всех вычислений    | Float — в Metal шейдерах, текстурах |

**Вывод 2026**:
- **Double** — основной тип для большинства вычислений  
- **Float** — специализированный тип для графики, игр, больших массивов, Metal, SIMD, экономии памяти

### 2. Самые частые сценарии использования Float в 2026

| Сценарий                              | Почему именно Float                                      | Пример |
|---------------------------------------|----------------------------------------------------------|--------|
| Metal / MetalKit / шейдеры            | Metal использует half/float по умолчанию                 | vertex/fragment шейдеры |
| SpriteKit / SceneKit                  | Текстуры, позиции, углы, цвета — почти всегда Float     | SKSpriteNode.position |
| Core Animation / CALayer              | transform, opacity, cornerRadius и т.д. — Float          | CATransform3D |
| SIMD векторы / Accelerate             | SIMD4<Float>, vDSP, vForce — почти всегда Float          | обработка аудио/изображений |
| Большие массивы координат/цветов      | Экономия памяти в 2 раза                                 | [Float] для 3D-моделей |
| Игры / ARKit / RealityKit             | Производительность и память критичны                     | ARAnchor.transform |
| OpenGL ES (устаревший, но встречается)| GL_FLOAT — стандарт                                      | Legacy проекты |

### 3. Важные примеры и ловушки (2026)

#### Пример 1: Float в Metal / шейдерах

```swift
struct Vertex {
    var position: SIMD4<Float>
    var color:    SIMD4<Float>
    var texCoord: SIMD2<Float>
}
```

**Почему Float**:  
- Metal шейдеры используют `float`, `float2`, `float3`, `float4`  
- `Double` не поддерживается в шейдерах

#### Пример 2: Экономия памяти в больших массивах

```swift
// 1 млн точек 3D
let pointsDouble: [SIMD3<Double>] = ...  // 24 МБ
let pointsFloat:  [SIMD3<Float>]  = ...  // 12 МБ
```

**Разница в 2 раза** — критично для AR, 3D, игр.

#### Пример 3: Ловушка — накопление погрешности

```swift
var sum: Float = 0
for _ in 0..<1_000_000 {
    sum += 0.1
}
print(sum) // ≈ 99999.98 (погрешность!)

var sumD: Double = 0
for _ in 0..<1_000_000 {
    sumD += 0.1
}
print(sumD) // ≈ 100000.0
```

**Вывод**:  
Для суммирования, финансов, точных расчётов → **только Double** или [[Decimal]].

#### Пример 4: Float → Double и обратно (самый частый переход)

```swift
let floatValue: Float = 3.1415926535   // ≈ 3.141593
let doubleValue = Double(floatValue)   // 3.1415927410125732
let backToFloat = Float(doubleValue)   // 3.141593
```

**Потеря точности** происходит при приведении `Double → Float`.

### 4. Лучшие практики Float в Swift 2026

- **Используй Float** только там, где требуется **Metal**, **SpriteKit**, **SceneKit**, **ARKit**, **RealityKit** или **экономия памяти**  
- **По умолчанию** — **Double** для всех вычислений, координат, физики, финансов  
- **Никогда** не используй Float для денег — только `Decimal`  
- **При конверсии** — учитывай потерю точности: `Float → Double` безопасно, `Double → Float` — с осторожностью  
- **В SIMD** — почти всегда `Float` (SIMD4<Float>, float4 и т.д.)  
- **Swift 6 strict concurrency** — `Float` полностью `Sendable` и безопасен  
- **Документируйте** — пиши комментарий «Float — позиция в Metal-шейдере (экономия памяти)»

**Короткий девиз 2026**:
> `Float` — это **32-битный** тип с плавающей точкой: быстрее и экономнее памяти, но **менее точный**.  
> В 2026 году используй его **только** в:  
> - графике / шейдерах (Metal, SpriteKit, SceneKit)  
> - больших массивах координат/цветов  
> - AR/VR/игр  
> Всё остальное — **Double** или `Decimal` (для финансов).
