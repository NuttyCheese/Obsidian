**TimeZone** (в Swift — `TimeZone`, в [[Objective-C]] — `NSTimeZone`) — это класс, который представляет **часовой пояс** (timezone) и все связанные с ним правила: смещение от UTC, переходы на летнее/зимнее время (DST), исторические изменения и т.д.

В 2026 году `TimeZone` остаётся одним из самых важных классов при работе с датами, локализацией и любыми сценариями, где время должно быть **правильно отображено** или **сохранено** с учётом региона пользователя.

### Основные способы получения TimeZone

| Способ получения                             | Пример кода                                                                 | Когда использовать в 2026 | Рекомендация |
|----------------------------------------------|-----------------------------------------------------------------------------|----------------------------|--------------|
| Текущий часовой пояс устройства              | `TimeZone.current`                                                          | Почти всегда — самый частый | ★★★★★ |
| Часовой пояс по идентификатору (IANA)       | `TimeZone(identifier: "Europe/Moscow")`                                     | Когда нужен конкретный пояс | ★★★★★ |
| GMT / UTC                                    | `TimeZone(secondsFromGMT: 0)` или `TimeZone(abbreviation: "UTC")!`          | Для серверного времени, логов | ★★★★☆ |
| Автоматический GMT-смещение                  | `TimeZone(secondsFromGMT: 10800)` // +3 часа                                 | Редко (лучше identifier)   | ★★☆☆☆ |
| Системный пояс (не учитывает изменения)      | `TimeZone.system`                                                           | Только для очень редких случаев | ★☆☆☆☆ |
| Из локали (рекомендуемый для пользователя)   | `TimeZone.current` или `Locale.current.timeZone` (iOS 16+)                  | Когда важна локаль пользователя | ★★★★★ |

### Самые важные и часто используемые свойства и методы

| Свойство / Метод                   | Тип / Возвращает        | Что возвращает / зачем нужен                             | Самый частый сценарий       |
| ---------------------------------- | ----------------------- | -------------------------------------------------------- | --------------------------- |
| `identifier`                       | [[String]]              | IANA-идентификатор ("Europe/Moscow", "America/New_York") | Логи, сервер, отладка       |
| `abbreviation`                     | `String?`               | Короткое имя ("MSK", "EST", "PDT")                       | Отображение пользователю    |
| `secondsFromGMT`                   | [[Int]]                 | Текущее смещение в секундах от UTC                       | Расчёт разницы времени      |
| `isDaylightSavingTime`             | [[Bool]]                | Действует ли сейчас летнее время в этом поясе            | Показ "летнее/зимнее"       |
| `nextDaylightSavingTimeTransition` | [[Date]]`?`             | Дата следующего перехода на летнее/зимнее время          | Уведомления о смене времени |
| `localizedName(for:locale:)`       | `String?`               | Локализованное имя пояса ("Москва, стандартное время")   | Отображение в UI            |
| `knownTimeZoneIdentifiers`         | `[String]` ([[static]]) | Все доступные IANA-идентификаторы                        | Список в настройках         |

### Полный современный пример (2026 — с учётом локали и DST)

```swift
import Foundation

// 1. Текущий часовой пояс пользователя
let tz = TimeZone.current
print("Текущий пояс: \(tz.identifier)")                    // "Europe/Moscow"
print("Смещение от UTC: \(tz.secondsFromGMT() / 3600) ч")  // 3
print("Короткое имя: \(tz.abbreviation() ?? "—")")         // "MSK"
print("Летнее время сейчас? \(tz.isDaylightSavingTime)")   // false (в 2026 в РФ нет DST)

// 2. Конкретный пояс
if let moscow = TimeZone(identifier: "Europe/Moscow") {
    print("Москва: \(moscow.secondsFromGMT() / 3600) ч")
    
    // Локализованное имя
    let ruLocale = Locale(identifier: "ru_RU")
    if let name = moscow.localizedName(for: .standard, locale: ruLocale) {
        print("На русском: \(name)")  // "Москва, стандартное время"
    }
}

// 3. UTC / GMT
let utc = TimeZone(secondsFromGMT: 0)!
print("UTC offset: \(utc.secondsFromGMT()) сек")  // 0

// 4. Проверка перехода на летнее время (если пояс поддерживает DST)
if let nextDST = tz.nextDaylightSavingTimeTransition {
    print("Следующий переход DST: \(nextDST)")
} else {
    print("В этом поясе нет переходов на летнее время")
}
```

### Лучшие практики TimeZone в Swift 2026

- **Никогда** не используйте `TimeZone.system` в пользовательском коде — всегда `TimeZone.current`  
- **Всегда** используйте **IANA-идентификаторы** ("Europe/Moscow", "Asia/Tokyo") — они стабильны и не зависят от страны  
- **Для серверного времени** — храните и передавайте всё в **UTC** (`TimeZone(secondsFromGMT: 0)`), конвертируйте в локальный пояс только для отображения  
- **Для отображения** — используйте `localizedName(for:locale:)` или `DateFormatter.timeZone` с нужным локалем  
- **Для дат** — всегда явно указывайте `timeZone` в `DateFormatter` и `Calendar` — по умолчанию берётся `current`  
- **В [[SwiftUI]]** — `TimeZone.current` доступен через `@Environment(\.timeZone)` (iOS 17+)  
- **Для логов / [[API]]** — всегда сохраняйте `Date` + `TimeZone.identifier` — это единственный надёжный способ избежать путаницы  
- **Документируйте** — пишите комментарий «TimeZone.current — часовой пояс пользователя (учитывает DST и изменения)»

**Короткий итог 2026**:
> `TimeZone` — это класс, представляющий **часовой пояс** с учётом всех правил (DST, исторические изменения, локализация).  
> В 2026 году:  
> - основной способ — `TimeZone.current` (пояс пользователя)  
> - для фиксированного времени — `TimeZone(identifier: "Europe/Moscow")` или `TimeZone(secondsFromGMT: 0)` (UTC)  
> - ключевые свойства — `identifier`, `secondsFromGMT`, `isDaylightSavingTime`, `localizedName`  
> - это **самый важный** и **самый часто используемый** класс при работе с датами и временем в iOS-приложениях.
