**Immutable** (неизменяемый) в [[Swift]] — это объект, значение или коллекция, **которую нельзя изменить после создания**.  
Это один из ключевых принципов безопасного и предсказуемого кода в Swift.

### Основные правила immutable в Swift (2025–2026)

| Тип данных / Конструкция                       | Как объявить immutable            | Можно ли изменить содержимое?  | Почему это важно                 |
| ---------------------------------------------- | --------------------------------- | ------------------------------ | -------------------------------- |
| Примитивы ([[Int]], [[String]], [[Bool]])      | `let x = 42`                      | Нет                            | Нет побочных эффектов            |
| [[struct]] / [[enum]] ([[Value Type]])         | `let point = Point(x: 10, y: 20)` | Нет (даже если свойства `var`) | Value semantics + безопасность   |
| [[class]] ([[Reference Type]])                 | `let user = User()`               | Да (можно менять свойства)     | Ссылка immutable, объект mutable |
| Коллекции ([[Array]], [[Dictionary]], [[Set]]) | `let array = [1,2,3]`             | Нет                            | Защита от случайных мутаций      |
| Параметры [[func]]                             | `func process(data: Data)`        | Нет (даже если внутри `var`)   | Функция не меняет входные данные |

### 1. Почему immutable — это суперсила Swift

| Преимущество                       | Как это работает в Swift                          | Реальный выигрыш в 2026             |
| ---------------------------------- | ------------------------------------------------- | ----------------------------------- |
| Безопасность от багов              | Компилятор запрещает изменение `let`              | Нет случайных мутаций               |
| Предсказуемость                    | Значение не меняется неожиданно                   | Легче отлаживать                    |
| Thread-safety (особенно в Swift 6) | Immutable данные безопасны между акторами         | Нет [[data race]] без синхронизации |
| Оптимизация компилятора            | `let` позволяет инлайнить, константно сворачивать | Быстрее код в релизе                |
| Value semantics                    | Копирование дешёвое (Copy-on-Write)               | Нет скрытых side-effects            |
| Читаемость и намерение             | `let` говорит: «это константа, не трогай»         | Код становится декларативным        |

### 2. Самые частые и полезные паттерны immutable (2026)

#### 2.1 Константы и конфигурация

```swift
let apiBaseURL = URL(string: "https://api.example.com")!
let maxRetryCount = 3
let defaultTimeout: TimeInterval = 30
```

#### 2.2 Модели данных (DTO, [[Codable]])

```swift
struct User: Codable, Identifiable {
    let id: UUID
    let name: String
    let email: String
    let createdAt: Date
    // все свойства let → модель immutable
}
```

#### 2.3 Параметры функций (самый важный паттерн)

```swift
func processUser(_ user: User) {
    // user — immutable внутри функции
    // даже если User имеет var свойства — их нельзя менять
}
```

#### 2.4 Immutable коллекции

```swift
let allowedRoles = Set(["admin", "moderator", "editor"])
// allowedRoles.insert("guest") // Ошибка компиляции
```

#### 2.5 Immutable ссылка на mutable объект

```swift
let sharedCache = Cache<String, Any>()
// sharedCache = другой кэш // Ошибка
sharedCache.set("key", value: "value") // Разрешено — объект mutable
```

### 3. Immutable vs Mutable — когда что выбирать

| Сценарий                                | Рекомендация 2026                             | Почему                                  |
| --------------------------------------- | --------------------------------------------- | --------------------------------------- |
| Константы, конфиги, [[API]]-ключи       | `let`                                         | Никогда не меняются                     |
| Модели данных (User, Post, Product)     | `let` для всех свойств                        | Immutable данные = безопасно            |
| ViewModel / состояние в SwiftUI         | `@Published var` (mutable) + immutable модели | UI реагирует на изменения               |
| Параметры функций                       | `let` (по умолчанию)                          | Функция не должна менять входные данные |
| Локальные переменные внутри функции     | `let` везде, где значение не меняется         | Читаемость + безопасность               |
| Коллекции, которые не должны изменяться | `let`                                         | Защита от случайных append/remove       |
| Объекты, которые должны мутировать      | `var` + класс (или struct с `mutating`)       | Необходимость изменения                 |

### 4. Лучшие практики immutable в Swift 2026

- **Делай `let` по умолчанию** — меняй на `var` только если действительно нужно изменять  
- **Все модели данных** (User, Post, Order и т.д.) — **полностью immutable** (`let` свойства)  
- **Параметры функций** — почти всегда `let` (компилятор сам добавляет, если не указано)  
- **В [[SwiftUI]]** — модели и данные → `let`, `@State` / `@Published` — для изменяемого состояния  
- **В [[Combine]] / [[async]]** — отдавай immutable данные из publishers / async функций  
- **[[Swift]] 6 strict concurrency** — immutable данные автоматически безопасны между акторами  
- **Документируйте** — пиши комментарий «let — неизменяемый список разрешённых ролей»

**Короткий девиз 2026**:
> Immutable — это когда ты создаёшь значение **один раз** и **никогда** его не меняешь.  
> В 2026 году:  
> - `let` — по умолчанию везде  
> - модели данных → полностью immutable  
> - параметры функций → `let`  
> - изменяй только то, что действительно должно меняться  
> Это **основа** безопасного, предсказуемого и быстрого кода в Swift.
