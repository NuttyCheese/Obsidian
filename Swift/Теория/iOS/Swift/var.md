**`var`** — ключевое слово в Swift, которое объявляет **изменяемую переменную** ([[mutable]] variable).  
После инициализации значение такой переменной **можно менять** сколько угодно раз.

Это противоположность **`let`**, которое создаёт **неизменяемую константу** ([[immutable]]).

### Почему var так важен и когда его использовать (2026 взгляд)

| Ситуация / потребность                        | Почему нужен именно `var`                                | Альтернатива (когда лучше взять `let`) |
|-----------------------------------------------|----------------------------------------------------------|----------------------------------------|
| Счётчик, индекс, временное состояние          | Нужно увеличивать/уменьшать значение                     | —                                      |
| Коллекции, которые будут модифицироваться     | `append`, `remove`, `update` требуют mutability          | `let` — если коллекция не меняется     |
| Свойство класса/структуры, которое меняется   | Пользовательский ввод, состояние UI, данные из сети      | `let` — если значение неизменно        |
| Локальная переменная в функции, которую обновляют | Результат вычислений, аккумулятор, буфер                 | `let` — если значение присваивается 1 раз |
| Optional, который сначала nil, потом получает значение | `var name: String? = nil` → потом `name = "Alice"`       | `let` — если значение известно сразу   |

**Золотое правило 2026 года**:  
«Используй `var` только тогда, когда значение **реально будет меняться**.  
Во всех остальных случаях — `let`.»

### Самые популярные и рекомендуемые паттерны var в 2026

#### 1. Изменяемые коллекции (самый частый кейс)

```swift
var shoppingList = ["яблоки", "бананы"]
shoppingList.append("апельсины")
shoppingList[0] = "груши"
shoppingList.remove(at: 1)
print(shoppingList) // ["груши", "апельсины"]
```

#### 2. Счётчики и аккумуляторы

```swift
var total = 0
for price in cartItems {
    total += price
}
```

#### 3. Состояние в циклах и обработчиках

```swift
var isLoading = false

func fetchData() {
    isLoading = true
    // ... сетевой запрос
    isLoading = false
}
```

#### 4. Свойства в [[struct]] и [[class]] (очень важно различать)

```swift
struct Point {
    var x: Int      // можно менять, если сам экземпляр var
    var y: Int
}

var p = Point(x: 0, y: 0)
p.x = 10            // ок

let fixed = Point(x: 5, y: 5)
// fixed.x = 15     // ошибка компиляции
```

```swift
class UserViewModel {
    var username = "Гость"     // можно менять всегда
    var isLoading = false
}
```

#### 5. var + computed property + mutating (мощный паттерн)

```swift
struct Counter {
    private var count = 0
    
    var value: Int {
        count
    }
    
    mutating func increment() {
        count += 1
    }
}

var counter = Counter()
counter.increment()
print(counter.value) // 1
```

### 6. Лучшие практики var в Swift 2026

- **Делай `let` по умолчанию** — меняй на `var` только если значение **реально изменяется**  
- **В структурах** — свойства `var` можно менять **только если сам экземпляр объявлен как `var`**  
- **В классах** — свойства `var` можно менять **независимо** от того, `let` или `var` ссылка  
- **Используй `private(set) var`** когда нужно читать свойство снаружи, но писать — только внутри  
  ```swift
  struct Score {
      private(set) var points = 0
      
      mutating func addPoint() {
          points += 1
      }
  }
  ```
- **Не делай `var` для неизменяемых данных** — это антипаттерн  
- **Swift 6 strict concurrency** — `var` в `actor` требует осторожности (нужен [[nonisolated]] или [[@MainActor]])  
- **Документируйте** — пиши комментарий «var count — текущее количество попыток входа»

**Короткий девиз 2026**:
> `var` — это когда значение **должно изменяться**.  
> В 2026 году:  
> - `let` — по умолчанию везде  
> - `var` — только для счётчиков, состояний, коллекций, которые модифицируются  
> - в структурах `var` требует `var` экземпляра  
> - в классах `var` работает всегда  
> Это **основа** безопасного и предсказуемого кода в Swift.
