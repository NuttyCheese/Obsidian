**`return`** — ключевое слово в [[Swift]], которое выполняет **два действия одновременно**:

1. **Завершает** выполнение текущей функции, метода, замыкания или геттера/сеттера.
2. **Возвращает** значение (если функция/замыкание имеет возвращаемый тип).

Это **самый важный** способ управления потоком выполнения и передачи результата вызывающему коду.

### 1. Основные сценарии использования [[return]] (2026 актуально)

| Сценарий                                 | Пример кода (рекомендуемый стиль)               | Что происходит                             |
| ---------------------------------------- | ----------------------------------------------- | ------------------------------------------ |
| Возврат значения из функции              | `return a + b`                                  | Функция завершается и возвращает результат |
| Досрочный выход из функции               | `guard … else { return }`                       | Выход при невыполнении условия             |
| Возврат из замыкания                     | `{ return $0 * 2 }`                             | Замыкание возвращает значение              |
| Возврат [[nil]] из функции с Optional    | `return nil`                                    | Явно указываем отсутствие значения         |
| Возврат из геттера вычисляемого свойства | `get { return computedValue }`                  | Возвращает значение свойства               |
| Досрочный выход из цикла / switch        | `return` внутри `for` / `switch`                | Прерывает весь метод, а не только цикл     |
| Возврат из `throwing` функции            | `return .success(…)` или `return` после `throw` | Возврат значения или пропуск после ошибки  |

### 2. Полный синтаксис и все варианты

#### Вариант 1 — Обычный возврат значения

```swift
func multiply(_ a: Int, _ b: Int) -> Int {
    return a * b
}

// Сокращённая запись (однострочное тело)
func square(_ n: Int) -> Int { n * n }
```

#### Вариант 2 — Досрочный выход (самый частый паттерн)

```swift
func divide(_ a: Int, by b: Int) -> Int? {
    guard b != 0 else {
        print("Деление на ноль запрещено")
        return nil              // ранний выход
    }
    
    return a / b                // основной путь
}
```

#### Вариант 3 — return в замыканиях

```swift
let numbers = [1, 2, 3, 4, 5]
let doubled = numbers.map { number -> Int in
    return number * 2
}

// Сокращённо
let tripled = numbers.map { $0 * 3 }
```

#### Вариант 4 — return в switch / if case (очень популярно)

```swift
func describe(_ result: Result<String, Error>) -> String {
    switch result {
    case .success(let value):
        return "Успех: \(value)"
    case .failure(let error):
        return "Ошибка: \(error.localizedDescription)"
    }
}
```

#### Вариант 5 — return без значения (Void-функции)

```swift
func logEvent(_ message: String) {
    print("[LOG]", message)
    return // необязательно — можно просто выйти
}
```

### 3. Лучшие практики return в Swift 2026

- **Используй ранний [[return]]** (early return) с `guard` — это делает код **линейным** и **читаемым**  
- **Не делай глубокую вложенность** — лучше несколько `guard let` / `return` в начале функции  
- **В функциях с Optional** — возвращай `nil` при ошибке вместо force unwrap  
- **В замыканиях** — используй **неявный возврат** для однострочных выражений  
  ```swift
  map { $0 * 2 }      // неявный return
  ```
- **В switch** — всегда возвращай значение из всех кейсов (или используй `return` только в одном)  
- **Swift 6 strict concurrency** — `return` полностью безопасен, но возвращаемый тип должен быть `Sendable` при передаче между акторами  
- **Документируйте** — пиши комментарий «return nil — если пользователь не найден»

**Короткий девиз 2026**:
> `return` — это «закончил работу и отдал результат».  
> В 2026 году:  
> - ранний `return` + `guard` — основной стиль  
> - неявный возврат в однострочных замыканиях  
> - `return nil` / `return .failure(…)` — для ошибок  
> - избегай глубокой вложенности — выходи рано  
> Это **основа** плоского, читаемого и безопасного кода.
