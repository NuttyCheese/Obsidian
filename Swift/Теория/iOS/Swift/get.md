**`get`** — это часть **вычисляемого свойства** (computed property) в Swift, которая определяет, **как именно получить значение свойства** при обращении к нему.

Вычисляемые свойства **не хранят** данные напрямую (в отличие от обычных `var`/`let`), а **вычисляют** их каждый раз заново через `get` (и опционально изменяют через `set`).

### 1. Почему `get` так важен (2026 взгляд)

| Ситуация / Проблема                          | Без `get` (обычное свойство)                     | С `get` (вычисляемое свойство)                     | Выигрыш |
|----------------------------------------------|--------------------------------------------------|-----------------------------------------------------|---------|
| Значение зависит от других свойств            | Нужно вручную обновлять при каждом изменении     | Автоматически пересчитывается при обращении         | Нет дублирования логики |
| Логика получения сложная                     | Приходится писать отдельный метод                | Выглядит как обычное свойство                       | Читаемость + инкапсуляция |
| Хочу read-only свойство                      | Можно, но занимает память                        | Не занимает память, только логика                   | Экономия памяти |
| Нужно отреагировать на чтение                | Невозможно                                       | Можно добавить логику в `get`                       | Отладка, аналитика |
| Свойство — это просто преобразование         | Дублирование кода в методах                      | Одно место для всей логики                          | DRY-принцип |

### 2. Полный синтаксис и все варианты (2026)

#### Вариант 1 — Только `get` (read-only, самый частый)

```swift
struct Circle {
    var radius: Double
    
    var diameter: Double {
        get {
            radius * 2
        }
    }
    
    // Сокращённая запись (самый популярный стиль)
    var circumference: Double {
        2 * .pi * radius
    }
}

let circle = Circle(radius: 5)
print(circle.diameter)      // 10.0
print(circle.circumference) // ≈31.4159
```

#### Вариант 2 — `get` + `set` (полное свойство)

```swift
struct Circle {
    var radius: Double = 0
    
    var diameter: Double {
        get { radius * 2 }
        set { radius = newValue / 2 }
    }
}

var circle = Circle()
circle.diameter = 20
print(circle.radius)   // 10.0
```

- `newValue` — встроенная переменная, содержит новое присвоенное значение

#### Вариант 3 — `get` с логикой и Optional

```swift
struct User {
    var firstName: String?
    var lastName: String?
    
    var fullName: String {
        get {
            switch (firstName, lastName) {
            case (.some(let f), .some(let l)): return "\(f) \(l)"
            case (.some(let f), .none):        return f
            case (.none, .some(let l)):        return l
            case (.none, .none):               return "Аноним"
            }
        }
    }
}

let user = User(firstName: "Алекс", lastName: nil)
print(user.fullName) // Алекс
```

#### Вариант 4 — `get` + property observers (очень мощно)

```swift
class TemperatureViewModel: ObservableObject {
    @Published var celsius: Double = 0 {
        didSet {
            // Обновляем UI или другие свойства
            fahrenheit = celsius * 9/5 + 32
        }
    }
    
    @Published var fahrenheit: Double = 32 {
        get {
            celsius * 9/5 + 32
        }
        set {
            celsius = (newValue - 32) * 5/9
        }
    }
}
```

**Важно**:  
observers (`willSet`, `didSet`) работают **только** с хранимыми свойствами (`var celsius`), а не с чисто вычисляемыми (`var fahrenheit`).

### 3. Лучшие практики `get` в Swift 2026

- **Предпочитай сокращённую запись** для простых вычислений:
  ```swift
  var area: Double { width * height }  // вместо get { width * height }
  ```

- **Используй `get` для**:
  - преобразований (celsius → fahrenheit)
  - вычисляемых значений (area, fullName, ageInYears)
  - инкапсуляции сложной логики (форматирование, проверки)
  - read-only свойств, которые не нужно хранить

- **Не используй `get` для**:
  - тяжёлых вычислений (лучше кэшировать в хранимом свойстве)
  - сайд-эффектов (логи, сетевые запросы) — это нарушает принцип чистоты
  - мутации состояния — для этого есть `set` или методы

- **В SwiftUI / Combine** — часто комбинируй `get` с `@Published`:
  ```swift
  @Published var radius: Double = 0
  var diameter: Double {
      radius * 2
  }
  ```

- **Swift 6 strict concurrency** — `get` безопасен, если все зависимости `Sendable` или на том же акторе  
- **Документируйте** — пиши комментарий «get — вычисляет полное имя пользователя»

**Короткий девиз 2026**:
> `get` — это когда свойство **выглядит** как обычное, но **на самом деле** вычисляется каждый раз при обращении.  
> В 2026 году используй его для:  
> - преобразований и зависимых значений  
> - инкапсуляции логики  
> - read-only свойств без хранения  
> Сокращённая запись `{ ... }` — золотой стандарт.
