**`tuple`** (кортеж) в [[Swift]] — это **лёгкий, временный контейнер** для хранения **нескольких значений** (обычно 2–7) разных типов в одном объекте.

Это **не отдельный тип** (как [[struct]] или [[class]]), а **анонимная структура**, создаваемая прямо в коде. Кортежи — **[[value type]]**, копируются при передаче и очень удобны для временного объединения данных.

### 1. Почему кортежи всё ещё очень полезны в 2025–2026

| Ситуация                                | Почему кортеж здесь идеален                      | Альтернатива (когда кортеж не нужен) |
| --------------------------------------- | ------------------------------------------------ | ------------------------------------ |
| Возврат 2–4 значений из функции         | Не нужно создавать отдельную [[struct]]/[[enum]] | > 5 значений → struct                |
| Временная группировка данных            | Быстро, без объявления нового типа               | Постоянные данные → struct           |
| Распаковка в цикле / switch             | Очень читаемо                                    | —                                    |
| Координаты, размеры, min-max пары       | Классический use-case                            | —                                    |
| Временный результат нескольких операций | Не захламляет код новым типом                    | —                                    |

### 2. Основные виды кортежей и синтаксис (2026 стандарт)

#### Анонимный кортеж (без имён)

```swift
let pair = (10, "Swift")           // (Int, String)
print(pair.0)                      // 10
print(pair.1)                      // "Swift"
```

#### Именованный кортеж (самый рекомендуемый стиль)

```swift
let point = (x: 15, y: -8)         // (x: Int, y: Int)
print(point.x)                     // 15
print(point.y)                     // -8
```

#### Кортеж с типами и именами

```swift
let userInfo: (name: String, age: Int, isActive: Bool) = ("Alex", 28, true)
```

### 3. Самые популярные паттерны использования кортежей в 2026

#### 3.1 Возврат нескольких значений из функции (самый частый кейс)

```swift
func minMax(numbers: [Int]) -> (min: Int, max: Int)? {
    guard let first = numbers.first else { return nil }
    
    var minVal = first
    var maxVal = first
    
    for num in numbers.dropFirst() {
        if num < minVal { minVal = num }
        if num > maxVal { maxVal = num }
    }
    
    return (min: minVal, max: maxVal)
}

if let result = minMax(numbers: [7, 2, 9, 1, 5]) {
    print("Min:", result.min, "Max:", result.max)   // Min: 1 Max: 9
}
```

#### 3.2 Распаковка в цикле (очень читаемо)

```swift
let points: [(x: Int, y: Int)] = [(1, 2), (3, 4), (5, 6)]

for (x, y) in points {
    print("Точка: (\(x), \(y))")
}
```

#### 3.3 Временная группировка в switch

```swift
let response = (status: 200, data: Data())

switch response {
case (200..<300, let data):
    print("Успех, данные:", data.count, "байт")
case (400..<500, _):
    print("Клиентская ошибка")
case (500..., _):
    print("Серверная ошибка")
default:
    print("Неизвестный статус")
}
```

#### 3.4 Кортеж как временный результат вычислений

```swift
let (width, height) = image.size
let aspectRatio = Double(width) / Double(height)
```

### 4. Лучшие практики кортежей в Swift 2026

- **Используй именованные кортежи** (`(x: Int, y: Int)`) — код становится намного читаемее  
- **Ограничивай количество элементов** — 2–4 идеально, 5–6 уже на грани, >6 → создавай struct  
- **Не храни кортежи в свойствах классов/структур** — лучше отдельный тип  
- **Для возврата из функции** — кортеж удобен, но если результат используется в нескольких местах → struct  
- **В SwiftUI** — кортежи часто используются для `GeometryReader` и временных вычислений  
- **Swift 6 strict concurrency** — кортеж `Sendable`, если все его элементы [[Sendable]]  
- **Документируйте** — пиши комментарий «(min: Int, max: Int) — минимальное и максимальное значения»

**Короткий девиз 2026**:
> Кортеж — это **быстрый и лёгкий способ** сгруппировать 2–5 значений без создания нового типа.  
> В 2026 году:  
> - используй **именованные кортежи** для читаемости  
> - возвращай из функций 2–4 значения  
> - распаковывай в циклах и switch  
> - если кортеж используется в нескольких местах → замени на struct  
> Это **временный помощник**, а не замена полноценным типам.
