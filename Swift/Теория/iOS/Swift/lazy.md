**`lazy`** — это модификатор свойства в Swift, который откладывает **инициализацию** свойства до **первого обращения** к нему.

Это один из самых мощных и часто используемых инструментов для оптимизации памяти и производительности, особенно в 2025–2026 годах, когда приложения становятся всё сложнее и работают с большим количеством данных.

### 1. Почему lazy так важен (реальные сценарии)

| Сценарий / Проблема                              | Без lazy                                          | С lazy                                            | Выигрыш в 2026 |
|--------------------------------------------------|---------------------------------------------------|---------------------------------------------------|----------------|
| Дорогостоящая инициализация (API-клиент, БД, тяжёлые вычисления) | Создаётся сразу при создании объекта              | Создаётся только при первом использовании         | Экономия памяти и CPU |
| Большие коллекции / модели / кэш                 | Загружаются всегда, даже если не используются     | Загружаются по требованию                         | Значительная экономия RAM |
| Свойство зависит от других свойств / self       | Нужно вручную лениво инициализировать             | Swift сам гарантирует однократную инициализацию   | Безопасность + читаемость |
| Многопоточный доступ                             | Нужно вручную синхронизировать                    | Swift гарантирует **thread-safe** однократную инициализацию | Нет data race |
| SwiftUI / Combine — редко используемые свойства  | Создаются сразу, даже если view не отображается   | Создаются только при первом рендере / обращении   | Быстрее запуск экрана |

### 2. Полный синтаксис и все варианты (2026 стандарт)

#### Вариант 1 — Простой lazy var

```swift
class DataLoader {
    lazy var expensiveData: [String] = {
        print("Загружаю тяжёлые данные...")
        return ["Очень", "дорого", "стоящие", "данные"]
    }()
}

let loader = DataLoader()
// пока ничего не загружено
print(loader.expensiveData.first!) 
// → "Загружаю тяжёлые данные..."
// → "Очень"
```

#### Вариант 2 — Сокращённая запись (самый популярный стиль)

```swift
class UserProfile {
    var name = "Аноним"
    
    lazy var greeting = "Привет, \(name)!"
}

let profile = UserProfile()
profile.name = "Алексей"           // можно менять name до первого обращения
print(profile.greeting)            // → "Привет, Алексей!"
```

**Важно**:  
`lazy` захватывает **текущее состояние** `self` в момент **первого обращения**, а не в момент создания объекта.

#### Вариант 3 — Lazy с вычислением и self (очень частый паттерн)

```swift
class Settings {
    var theme: Theme = .light
    
    lazy var backgroundColor: UIColor = {
        switch theme {
        case .light: return .white
        case .dark:  return .black
        }
    }()
}
```

#### Вариант 4 — Lazy + escaping closure (многопоточный безопасный)

```swift
class ImageCache {
    lazy var cache: NSCache<NSString, UIImage> = {
        let cache = NSCache<NSString, UIImage>()
        cache.countLimit = 100
        cache.totalCostLimit = 50 * 1024 * 1024 // 50 МБ
        return cache
    }()
}
```

Swift **гарантирует**, что `lazy` инициализируется **только один раз**, даже при параллельном доступе из нескольких потоков.

### 3. Когда lazy НЕ нужен (антипаттерны 2026)

- Свойство инициализируется **дешёво** и **всегда используется** → обычный `var` быстрее  
- Свойство — это **константа** → используй `let` (компилятор может оптимизировать лучше)  
- Свойство должно быть **неизменяемым** → `let` + вычисляемое свойство `{ ... }`  
- Свойство меняется **часто** → lazy создаётся один раз, повторная инициализация невозможна  

### 4. Лучшие практики lazy в Swift 2026

- Используй `lazy var` для:
  - тяжёлых вычислений (форматирование, парсинг)
  - больших коллекций / кэшей / моделей
  - объектов, которые могут не понадобиться (API-клиент, БД, логгер)
  - зависимых свойств, которые редко читаются

- **Не используй** lazy для:
  - простых значений (Int, Bool, String)
  - свойств, которые почти всегда нужны
  - свойств, которые должны быть `let` (константы)

- **В SwiftUI** — часто комбинируй `lazy var` с `@StateObject` / `@ObservedObject`:

```swift
class ViewModel: ObservableObject {
    @Published var user: User?
    
    lazy var displayName: String = {
        user?.name ?? "Гость"
    }()
}
```

- **Swift 6 strict concurrency** — `lazy` полностью thread-safe (однократная инициализация гарантирована)  
- **Документируйте** — пиши комментарий «lazy — тяжёлая инициализация кэша изображений»

**Короткий девиз 2026**:
> `lazy` — это когда ты говоришь: «создавай это только тогда, когда оно реально понадобится».  
> В 2026 году используй его **везде**, где инициализация дорогая или не всегда нужна:  
> - кэши, коллекции, тяжёлые объекты  
> - вычисляемые свойства с зависимостью от self  
> - отложенная загрузка данных  
> Это **основа** экономии памяти и ускорения запуска экранов.
