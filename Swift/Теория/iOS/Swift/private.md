**`private`** — это модификатор доступа в Swift, который ограничивает видимость элемента **только внутри того типа** (структуры, класса, перечисления, расширения), в котором он объявлен.

Это самый строгий уровень доступа после `fileprivate`.

### Уровни доступа в Swift (2026 актуально)

| Модификатор     | Видимость                                              | Где можно использовать элемент                          | Самый частый сценарий использования |
|-----------------|--------------------------------------------------------|----------------------------------------------------------|-------------------------------------|
| `private`       | Только внутри текущего типа (struct/class/enum)        | Только внутри `{ }` этого типа                           | Внутренние свойства/методы типа     |
| `fileprivate`   | Весь файл (.swift)                                     | В любом месте этого же файла                             | Общие хелперы внутри одного файла   |
| `internal`      | Весь модуль (target/framework) — уровень по умолчанию  | Весь модуль                                              | Большинство свойств/методов         |
| `public`        | Везде, включая другие модули                           | Для публичных API                                        | Библиотеки, фреймворки              |
| `open`          | Как `public` + можно наследовать и переопределять      | Только для классов и методов                             | Базовые классы для наследования     |

### Когда использовать `private` (рекомендации 2026)

| Ситуация                                              | Почему именно `private`                                  | Альтернатива (когда не использовать) |
|-------------------------------------------------------|----------------------------------------------------------|--------------------------------------|
| Внутренние свойства, которые никто снаружи не должен видеть | Полная инкапсуляция, защита от случайного изменения      | `fileprivate` — если нужно в файле   |
| Приватные методы-хелперы внутри класса/структуры      | Логика скрыта, читаемость кода выше                     | `internal` — если нужно в модуле     |
| Константы/переменные, используемые только внутри типа | Нет смысла открывать наружу                              | `let` внутри функции — ещё лучше     |
| Свойства, которые могут сломать инварианты при изменении | Защита состояния объекта                                 | `fileprivate(set)` — если нужно читать снаружи |
| В расширениях (extension)                             | Метод/свойство видно только в этом типе                  | `fileprivate` — если нужно в файле   |

### Примеры (современный стиль 2026)

#### Пример 1. Приватные свойства в структуре

```swift
struct User {
    let id: UUID
    private var _email: String          // никто снаружи не увидит
    private var _passwordHash: String   // критично скрыто
    
    var email: String {
        _email
    }
    
    init(id: UUID, email: String, password: String) {
        self.id = id
        self._email = email
        self._passwordHash = hash(password) // приватная логика
    }
    
    private func hash(_ password: String) -> String {
        // сложный хэш
        return "hashed_\(password)"
    }
}
```

#### Пример 2. Приватный метод в классе

```swift
final class NetworkManager {
    private let session: URLSession
    
    init() {
        let config = URLSessionConfiguration.default
        config.timeoutIntervalForRequest = 30
        self.session = URLSession(configuration: config)
    }
    
    func fetchData(from url: URL) async throws -> Data {
        let (data, _) = try await session.data(from: url)
        return data
    }
    
    // приватный метод — внутренняя реализация
    private func validateResponse(_ response: URLResponse) throws {
        guard let httpResponse = response as? HTTPURLResponse,
              200...299 ~= httpResponse.statusCode else {
            throw NetworkError.invalidStatus
        }
    }
}
```

#### Пример 3. `private(set)` — часто встречающийся паттерн

```swift
struct Counter {
    private(set) var count = 0   // читать можно снаружи, писать — только внутри
    
    mutating func increment() {
        count += 1
    }
}

var counter = Counter()
print(counter.count)     // 0 — можно читать
// counter.count = 10    // Ошибка компиляции
counter.increment()
print(counter.count)     // 1
```

### 5. Лучшие практики `private` в Swift 2026

- **Делай `private` по умолчанию** для всех внутренних деталей реализации  
- **Используй `private(set)`**, когда нужно читать свойство снаружи, но писать — только внутри  
- **В расширениях** — `private` делает метод/свойство видимым только в этом типе  
- **Не используй `private`**, если логика нужна в том же файле → бери `fileprivate`  
- **Swift 6 strict concurrency** — `private` никак не влияет на Sendable/акторы — это только область видимости  
- **Документируйте** — пиши комментарий «private — внутренняя реализация хэширования пароля»

**Короткий девиз 2026**:
> `private` — это «только внутри этого типа, и нигде больше».  
> В 2026 году:  
> - `private` — для полной инкапсуляции  
> - `private(set)` — когда нужно читать снаружи, но писать внутри  
> - `fileprivate` — если нужно в том же файле  
> - `internal` — для всего остального в модуле  
> Это **основа** чистой инкапсуляции и защиты состояния в Swift.
