**`let`** — ключевое слово в [[Swift]] для объявления **константы** (неизменяемого значения).  
После присвоения значения изменить его уже **нельзя** — попытка это сделать вызовет ошибку компиляции.

Это один из самых важных элементов языка, который лежит в основе **безопасного**, **предсказуемого** и **производительного** кода в Swift.

### 1. Почему `let` — это must-have в 2025–2026

| Преимущество                              | Как это работает в Swift                              | Реальный выигрыш в 2026 |
|-------------------------------------------|-------------------------------------------------------|--------------------------|
| Безопасность от багов                     | Компилятор запрещает изменение после инициализации    | Нет случайных мутаций    |
| Предсказуемость                           | Значение гарантированно не изменится                  | Легче отлаживать         |
| Thread-safety (Swift 6+)                  | Константы безопасны между акторами без синхронизации  | Нет data race            |
| Оптимизация компилятора                   | `let` позволяет инлайнить, константно сворачивать     | Быстрее код в релизе     |
| Value semantics                           | Копирование дешёвое (COW для коллекций)               | Нет скрытых side-effects |
| Читаемость и намерение                    | `let` говорит: «это константа, не трогай»             | Код становится декларативным |
| Защита от ошибок в больших командах       | Невозможно случайно изменить значение                 | Меньше багов             |

### 2. Основные сценарии использования `let` (реальные примеры 2026)

#### 2.1 Константы и конфигурация

```swift
let apiBaseURL = URL(string: "https://api.example.com/v2")!
let maxRetryCount = 3
let defaultTimeout: TimeInterval = 30
let appVersion = Bundle.main.object(forInfoDictionaryKey: "CFBundleShortVersionString") as! String
```

#### 2.2 Модели данных ([[Codable]], [[Identifiable]])

```swift
struct User: Codable, Identifiable {
    let id: UUID
    let name: String
    let email: String
    let createdAt: Date
    // все свойства — let → модель полностью immutable
}
```

#### 2.3 Параметры функций (по умолчанию `let`)

```swift
func processUser(_ user: User, with discount: Double = 0.1) {
    // user и discount — let, их нельзя случайно изменить
    let finalPrice = user.cartTotal * (1 - discount)
    // ...
}
```

#### 2.4 [[Immutable]] коллекции

```swift
let allowedRoles = Set(["admin", "moderator", "editor"])
// allowedRoles.insert("guest") // Ошибка компиляции

let fixedConfig = ["theme": "dark", "apiKey": "abc123"]
// fixedConfig["theme"] = "light" // Ошибка
```

#### 2.5 Локальные константы внутри функций

```swift
func calculateTax(income: Double) -> Double {
    let taxRate = 0.13          // константа внутри функции
    let deductionLimit = 5000.0
    
    if income > deductionLimit {
        let taxable = income - deductionLimit
        return taxable * taxRate
    }
    
    return income * taxRate
}
```

### 3. let vs var — золотые правила 2026

| Ситуация                                      | Рекомендация 2026                                | Почему |
|-----------------------------------------------|--------------------------------------------------|--------|
| Значение **никогда** не изменяется            | `let`                                            | Безопасно, оптимизировано |
| Значение **может** измениться                 | `var`                                            | Только тогда, когда действительно нужно |
| Модель данных (DTO, Codable, Identifiable)    | **все свойства `let`**                           | Immutable данные = безопасно |
| Локальные переменные внутри функции           | `let` везде, где значение не меняется            | Читаемость + безопасность |
| Параметры функций                             | `let` (по умолчанию, даже если не пишешь)        | Функция не должна менять входные данные |
| Состояние в SwiftUI / Combine                 | `@State`, `@Published`, `@ObservedObject var`    | Изменяемое состояние UI |
| Константы конфигурации, API-ключи, URL        | `let`                                            | Никогда не меняются |

### 4. Shadowing (перекрытие) — полезная фича `let`

```swift
let value = 10

if true {
    let value = 20          // новая константа, shadowing внешней
    print(value)            // 20
}

print(value)                // 10 — внешняя осталась нетронутой
```

**Часто используется**:
- в замыканиях
- в циклах
- при распаковке Optional / enum

### 5. Лучшие практики `let` в Swift 2026

- **Делай `let` по умолчанию** — меняй на `var` только если действительно нужно изменять  
- **Все модели данных** (User, Post, Order и т.д.) — **полностью immutable** (`let` свойства)  
- **Параметры функций** — почти всегда `let` (компилятор сам добавляет, если не указано)  
- **В SwiftUI** — модели и данные → `let`, `@State` / `@Published` — для изменяемого состояния  
- **В Combine / async** — отдавай immutable данные из publishers / async функций  
- **Swift 6 strict concurrency** — immutable данные автоматически безопасны между акторами  
- **Документируйте** — пиши комментарий «let — неизменяемый список разрешённых ролей»

**Короткий девиз 2026**:
> `let` — это «создал один раз — и забыл».  
> В 2026 году:  
> - `let` везде, где значение **не меняется**  
> - модели данных → полностью immutable  
> - параметры функций → `let`  
> - изменяй только то, что действительно должно меняться  
> Это **основа** безопасного, предсказуемого и быстрого кода в Swift.
