**`willSet`** — это **наблюдатель свойства** (property observer) в [[Swift]], который вызывается **непосредственно перед** тем, как свойству присваивается новое значение.

Он позволяет:
- увидеть новое значение **до** его установки (`newValue`)
- отреагировать на предстоящее изменение
- предотвратить/ограничить нежелательные изменения
- синхронизировать другие свойства или UI **до** присваивания

### Главные правила willSet (2025–2026 актуально)

| Характеристика                      | Детали                                                                         | Важный нюанс 2026                          |
| ----------------------------------- | ------------------------------------------------------------------------------ | ------------------------------------------ |
| Работает только с                   | [[var]] свойствами (в структурах, классах, перечислениях)                      | [[let]] — запрещено                        |
| Когда вызывается                    | **Перед** присваиванием нового значения                                        | Даже если новое значение равно старому     |
| Доступные переменные                | `newValue` — автоматическая константа с новым значением                        | `oldValue` в willSet **нет**               |
| Изменение свойства внутри willSet   | Запрещено напрямую менять само свойство ([[retain cycle]] / бесконечный вызов) | Можно менять **другие** свойства           |
| Совместимость с computed properties | Не работает с computed properties (только stored properties)                   | Для computed → используйте [[Set Collection]]/[[get]] |
| Swift 6 strict concurrency          | Полностью безопасен, если свойство на правильном акторе                        | —                                          |

### Самые важные и часто используемые паттерны willSet

#### 1. Валидация / ограничение диапазона (самый популярный кейс)

```swift
var temperature: Double = 22 {
    willSet {
        if newValue < -50 || newValue > 100 {
            print("Температура вне допустимого диапазона!")
            // Можно бросить assertion в debug, но не менять newValue
        }
    }
}

temperature = 150   // → "Температура вне допустимого диапазона!"
```

**Важно**: внутри `willSet` **нельзя** присваивать значение самому свойству — это вызовет бесконечный цикл.

#### 2. Синхронизация связанных свойств (очень частый паттерн)

```swift
struct Circle {
    var radius: Double = 1 {
        willSet {
            // До изменения радиуса обновляем площадь
            area = .pi * newValue * newValue
        }
    }
    
    private(set) var area: Double = .pi
}

var circle = Circle()
circle.radius = 5
print(circle.area)          // ≈78.54 (уже обновлено до присваивания)
```

#### 3. Логирование / отладка изменений

```swift
var score: Int = 0 {
    willSet {
        print("Счёт изменяется с \(score) → \(newValue)")
    }
}
```

#### 4. willSet + [[didSet]] — полный контроль

```swift
var pageNumber: Int = 1 {
    willSet {
        print("Сейчас переходим на страницу \(newValue)")
    }
    didSet {
        print("Страница изменена. Было: \(oldValue), стало: \(pageNumber)")
        updateUI()
    }
}
```

### Чего НЕЛЬЗЯ делать в willSet (критические ошибки)

```swift
var badCounter: Int = 0 {
    willSet {
        badCounter = newValue + 1   // ← БЕСКОНЕЧНЫЙ ЦИКЛ!
    }
}
```

```swift
var temperature: Double = 20 {
    willSet {
        temperature = max(newValue, -50)   // ← Тоже бесконечный цикл
    }
}
```

### Лучшие практики willSet в 2026

- **Не меняйте** само свойство внутри `willSet` — это приведёт к краху или бесконечному циклу  
- **Используйте** `willSet` для:
  - валидации диапазона
  - синхронизации связанных свойств
  - логирования/отладки
  - подготовки UI/состояния до изменения
- **Для пост-обработки** (обновление UI, сохранение, уведомления) → предпочтительнее `didSet`  
- **Не злоупотребляйте** — наблюдатели слегка замедляют присваивание  
- **В SwiftUI / [[Combine]]** — вместо willSet чаще используют `@Published` + `.sink` / `.onChange`  
- **Swift 6 strict concurrency** — `willSet` безопасен, но само свойство должно быть на правильном акторе  
- **Документируйте** — пишите комментарий «willSet — синхронизация площади до изменения радиуса»

**Короткий девиз 2026**:
> `willSet` — это «последний шанс посмотреть/подготовиться перед тем, как значение изменится».  
> В 2026 году:  
> - используй для **валидации**, **синхронизации связанных свойств**, **логирования**  
> - **не меняй** само свойство внутри willSet  
> - для UI/сохранения/уведомлений → чаще `didSet`  
> - если логика сложная → лучше отдельный метод или computed property
