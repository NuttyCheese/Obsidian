**`Equatable`** — это фундаментальный протокол в [[Swift]], который позволяет сравнивать два значения одного типа на равенство с помощью оператора `==` (и автоматически `!=`).

- Если тип соответствует `Equatable`, то `a == b` означает «значения полностью идентичны».
- `Equatable` **наследуется** всеми стандартными типами: [[Int]], [[Double]], [[String]], [[Bool]], [[Array]], [[Set]], [[Dictionary]], [[Optional]], [[Date]], [[URL]] и т.д.
- Swift **автоматически синтезирует** реализацию `Equatable` для структур и перечислений, если все их поля/ассоциированные значения сами Equatable.

### 1. Требования протокола (официально на 2026 год)

```swift
public protocol Equatable {
    static func == (lhs: Self, rhs: Self) -> Bool
}
```

- Нужно реализовать **только** `==`.
- `!=` генерируется автоматически: `a != b` ⇔ `!(a == b)`.

### 2. Автоматическая синтезировка (самый частый случай)

Swift синтезирует `==` для структур и [[enum]], если:

- Все свойства/associated values сами Equatable
- Нет [[let]] свойств с разными значениями по умолчанию (редкий случай)

```swift
// Автоматически Equatable
struct Point: Equatable {
    let x: Int
    let y: Int
    let z: Double?
}

let p1 = Point(x: 1, y: 2, z: 3.0)
let p2 = Point(x: 1, y: 2, z: 3.0)
let p3 = Point(x: 1, y: 2, z: nil)

print(p1 == p2)  // true
print(p1 == p3)  // false
```

```swift
// Enum с associated values — тоже синтезируется
enum Result<Value: Equatable>: Equatable {
    case success(Value)
    case failure(String)
}

let r1 = Result.success(200)
let r2 = Result.success(200)
let r3 = Result.failure("error")

print(r1 == r2)  // true
print(r1 == r3)  // false
```

### 3. Ручная реализация Equatable (когда нужно кастомное сравнение)

```swift
struct Person: Equatable {
    let name: String
    let age: Int
    let id: UUID  // уникальный идентификатор, но не участвует в сравнении

    static func == (lhs: Person, rhs: Person) -> Bool {
        lhs.name == rhs.name && lhs.age == rhs.age
        // id не сравнивается — два человека с одинаковыми именем и возрастом считаются равными
    }
}

let alice1 = Person(name: "Alice", age: 30, id: UUID())
let alice2 = Person(name: "Alice", age: 30, id: UUID())

print(alice1 == alice2)  // true, несмотря на разные id
```

### 4. Equatable для классов (ссылочная семантика по умолчанию)

Классы **не** получают синтезированное `==` — по умолчанию `==` сравнивает **ссылки** (identity), а не содержимое.

```swift
class User {
    var name: String
    var age: Int
    
    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }
}

let u1 = User(name: "Bob", age: 25)
let u2 = User(name: "Bob", age: 25)

print(u1 == u2)  // false — разные объекты в памяти
print(u1 === u2) // false — разные ссылки
```

Чтобы сравнивать **по содержимому**, нужно вручную реализовать `Equatable`:

```swift
extension User: Equatable {
    static func == (lhs: User, rhs: User) -> Bool {
        lhs.name == rhs.name && lhs.age == rhs.age
    }
}

print(u1 == u2)  // теперь true
```

**Альтернатива**: использовать `===` / `!==` для проверки идентичности объектов (ссылок).

### 5. Equatable в коллекциях и стандартной библиотеке

| Тип коллекции      | Требование к Element | Поведение `==` для коллекций                                     |
| ------------------ | -------------------- | ---------------------------------------------------------------- |
| `Array<T>`         | `T: Equatable`       | Сравнивает поэлементно по индексу                                |
| `Set<T>`           | `T: Hashable`        | `[[Hashable]]` наследует `Equatable` → сравнивает по содержимому |
| `Dictionary<K, V>` | `K: Hashable`        | Ключи сравниваются по `==`, значения — по `==`                   |
| `String`           | —                    | Сравнивает по символам ([[Character]])                           |

Пример:

```swift
let a1 = [1, 2, 3]
let a2 = [1, 2, 3]
let a3 = [3, 2, 1]

print(a1 == a2)  // true
print(a1 == a3)  // false
```

### 6. Производительность Equatable (примерные замеры 2026)

| Операция                     | Простой тип (Int, Bool) | Struct с 3–5 полями | Класс с ручным == | Коллекция (Array 100 элементов) |
|------------------------------|--------------------------|----------------------|-------------------|----------------------------------|
| Операция `==`                | ~1 нс                    | ~3–8 нс              | ~5–12 нс          | ~100–500 нс                      |
| Синтезированное ==           | —                        | ~2–5 нс              | —                 | —                                |
| Ручная реализация ==         | —                        | ~5–10 нс             | ~8–15 нс          | —                                |
| Использование в Set.contains | ~2–5 нс                  | ~10–20 нс            | ~15–30 нс         | —                                |

**Вывод**:  
Синтезированное `==` для структур — самое быстрое и безопасное.  
Ручная реализация для классов — дороже, но необходима для сравнения по содержимому.

### 7. Лучшие практики Equatable в Swift 2026

- **Для структур и enum** — почти всегда полагайтесь на **автоматическую синтезировку** — она быстрая и безошибочная  
- **Для классов** — реализуйте `==` вручную **только** если нужно сравнивать содержимое (не ссылки)  
- **Соблюдайте контракт**: `a == a` всегда true, `a == b` ⇔ `b == a`, транзитивность  
- **Не нарушайте** `Hashable` ⇒ `Equatable` — если тип Hashable, `==` должен быть согласован с `hashValue`/`hash(into:)`  
- **Swift 6 strict concurrency** — `Equatable` полностью `Sendable`, если `Self` — `Sendable`  
- **Документируйте** — пиши комментарий «Equatable — сравнение по имени и возрасту (id не учитывается)»

**Короткий девиз 2026**:
> `Equatable` — это когда ты хочешь сказать: «два значения равны, если их содержимое совпадает».  
> В 2026 году:  
> - для struct/enum — оставляй синтез (быстро и безопасно)  
> - для class — реализуй вручную, если нужно сравнивать содержимое  
> - используй `==` в коллекциях, `Set`, `Dictionary`, `min`/`max`  
> Это **основа** типобезопасного сравнения в Swift.
