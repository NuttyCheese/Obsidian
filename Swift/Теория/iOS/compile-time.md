**Compile-time** — это **время, когда компилятор анализирует и преобразует твой код ДО запуска программы**.

Проще:

> compile-time — это всё, что происходит **пока ты жмёшь Build**, а приложение ещё **не запущено**.

### На compile-time Swift:

- проверяет типы
    
- проверяет синтаксис
    
- выводит типы (`type inference`)
    
- подставляет литералы (`42`, `"text"`, [[nil]])
    
- проверяет протоколы (`ExpressibleBy…`)
    
- проверяет диапазоны (`Int8 = 1000 ❌`)
    
- генерирует SIL → LLVM IR → машинный код
    
- оптимизирует код
    

❗️Если ошибка найдена тут — **приложение даже не соберётся**.

---

## 2. Что такое **runtime**

**[[Runtime]]** — это **время, когда программа уже запущена и выполняется**.

Проще:

> runtime начинается, когда ты нажал ▶ Run, и код реально работает на устройстве.

### На runtime происходит:

- создание объектов
    
- выделение памяти
    
- выполнение [[if-else|if]], [[for-in|for]], [[switch]]
    
- вызовы функций
    
- network-запросы
    
- работа с UI
    
- `print`, `fatalError`, `precondition`
    

Если ошибка тут — приложение **падает во время работы**.

---

## 3. Ключевая разница (коротко)

|           | Compile-time  | Runtime                   |
| --------- | ------------- | ------------------------- |
| Когда     | До запуска    | Во время выполнения       |
| Кто       | Компилятор    | Процессор + runtime среды |
| Ошибки    | Compile error | Crash / exception         |
| Стоимость | Бесплатно     | Дорого                    |
| Цель      | Безопасность  | Работа                    |

💡 **Хороший код = максимум ошибок на compile-time, минимум на runtime.**

---

## 4. Что такое runtime в Swift (важный момент)

[[Swift]] — **не интерпретируемый язык**.

Это значит:

- нет «Swift-движка»
    
- код → нативный машинный код
    
- runtime — это:
    
    - Swift Runtime ([[ARC]], [[generic]], [[Protocol]])
        
    - Objective-C runtime (частично)
        
    - ОС ([[iOS]], macOS)
        

---

## 5. Что происходит раньше: compile-time или runtime?

Ответ строго по порядку:

```
1️⃣ Compile-time
2️⃣ Linking
3️⃣ Launch
4️⃣ Runtime
```

### Распишем:

#### 1️⃣ Compile-time

- проверка кода
    
- генерация бинарника
    

#### 2️⃣ Linking

- связывание модулей
    
- Swift + ObjC + System frameworks
    

#### 3️⃣ Launch

- загрузка приложения в память
    
- инициализация runtime
    

#### 4️⃣ Runtime

- выполнение `main`
    
- запуск UI
    
- работа приложения
    

👉 **Runtime невозможен без compile-time.**

---

## 6. Где тут «compile-time evaluation»?

Иногда говорят:

> «это работает на compile-time»

Обычно имеют в виду:

- проверку
    
- вывод типов
    
- подстановку литералов
    
- оптимизацию
    

❗️Но Swift **почти ничего не исполняет** на compile-time.  
Он **анализирует**, но не «выполняет код» (кроме макросов).

---

## 7. Примеры: compile-time vs runtime

### ❌ Compile-time ошибка

```swift
let x: Int = "hello"
```

Ошибка:

> Cannot convert value of type '[[String]]' to '[[Int]]'

Программа **не соберётся**.

---

### ❌ Runtime ошибка

```swift
let x: Int! = nil
print(x + 1)
```

Компилируется ❗️  
Падает ❗️

---

## 8. Литералы и compile-time

Когда ты пишешь:

```swift
let x: Int8 = 100
```

✔️ compile-time:

- проверка диапазона
    
- вызов `init(integerLiteral:)`
    

```swift
let x: Int8 = 1000
```

❌ compile-time error  
👉 **даже не доходит до runtime**

---

## 9. Почему compile-time важнее (философия Swift)

Swift сознательно:

- усложняет компилятор
    
- упрощает runtime
    
- двигает ошибки «влево» (раньше)
    

Это:

- безопаснее
    
- быстрее
    
- предсказуемее
    
- ближе к C/C++ традиции, чем к динамическим языкам
    

---

## 10. Частая путаница

❌ «compile-time = сборка проекта»

Частично да, но:

- сборка = compile-time + linking
    
- compile-time — это **логический этап анализа**

---

## 12. Короткая метафора (чтобы закрепить)

- **Compile-time** — архитектор + чертёж
    
- **Runtime** — стройка + эксплуатация
    

Если ошибка в чертеже — дом не построят.  
Если ошибка в эксплуатации — дом рухнет.

---

## 13. Итог (коротко и чётко)

- Compile-time — до запуска
    
- Runtime — во время работы
    
- Compile-time всегда раньше
    
- Swift старается ловить максимум ошибок заранее
    
- Литералы и протоколы — часть compile-time контракта
