**NSCameraUsageDescription** — это обязательный ключ в файле **Info.plist** вашего [[iOS]]-приложения, который объясняет пользователю, **зачем** приложению нужен доступ к камере.

Без этого ключа (или с пустым/неинформативным значением) приложение **крашнется** при попытке использовать камеру (включая [[UIImagePickerController]] с `sourceType = .camera`, [[AVCaptureSession]], [[AVCaptureDevice]] и т.д.), начиная с **iOS 10** (2016) и по сегодняшний день (2026).

### Почему ключ обязателен

Apple ввела это требование в iOS 10 для повышения приватности:  
пользователь должен **чётко понимать**, зачем приложение хочет доступ к камере.

Если ключ отсутствует или значение пустое → при первом запросе доступа к камере приложение **крашнется** с ошибкой:

```
This app has crashed because it attempted to access privacy-sensitive data without a usage description.  
The app's Info.plist must contain an NSCameraUsageDescription key with a string value explaining to the user how the app uses this data.
```

### Как правильно добавить NSCameraUsageDescription

#### 1. Через [[Xcode]] (самый удобный способ)

1. Откройте проект → выберите target → вкладка **Info**
2. Нажмите **+** в разделе **Custom iOS Target Properties**
3. Выберите **Privacy - Camera Usage Description** (или начните печатать «NSCamera» — Xcode подскажет)
4. В поле **Value** напишите понятное объяснение на языке пользователя (на английском и/или русском)

Примеры хороших значений (2026 стандарт):

```text
"Приложению нужен доступ к камере, чтобы вы могли сделать фото профиля или отсканировать QR-код."
```

```text
"Доступ к камере требуется для съёмки фото и видео внутри приложения."
```

```text
"Мы используем камеру для создания аватарки и сканирования документов."
```

Плохой пример (приведёт к отказу в App Review):

```text
"Нужен доступ к камере"
"Camera access"
"Для работы приложения"
```

#### 2. Через Info.plist напрямую (XML-вид)

```xml
<key>NSCameraUsageDescription</key>
<string>Приложению нужен доступ к камере, чтобы вы могли сделать фото профиля или отсканировать QR-код.</string>
```

#### 3. Локализация (если приложение на нескольких языках)

Создайте файл **InfoPlist.strings** в папке локализации (например ru.lproj):

```text
/* ru.lproj/InfoPlist.strings */
"NSCameraUsageDescription" = "Приложению нужен доступ к камере, чтобы вы могли сделать фото профиля или отсканировать QR-код.";
```

А в **Base.lproj/InfoPlist.strings**:

```text
"NSCameraUsageDescription" = "The app needs camera access to take profile photos or scan QR codes.";
```

### Связанные ключи (если используете и другие функции)

| Ключ                              | Когда нужен                                      | Пример значения |
|-----------------------------------|--------------------------------------------------|-----------------|
| **NSCameraUsageDescription**      | Доступ к камере (фото/видео)                     | Обязателен |
| **NSMicrophoneUsageDescription**  | Запись звука при съёмке видео                    | "Доступ к микрофону нужен для записи видео" |
| **NSPhotoLibraryUsageDescription**| Доступ к галерее (чтение фото/видео)             | "Доступ к галерее нужен для выбора фото профиля" |
| **NSPhotoLibraryAddUsageDescription** | Только запись в галерею (iOS 11+)             | "Разрешите сохранять сделанные фото в галерею" |

### Лучшие практики NSCameraUsageDescription в 2026 году

- **Текст должен быть понятным и конкретным** — пользователь должен сразу понять, зачем именно камера нужна  
- **Не пишите** "Для работы приложения" или "Camera access" — App Review **отклонит**  
- **Добавляйте** ключ **даже если** пока не используете камеру — если планируете в будущем  
- **Для [[SwiftUI]]** — ключ всё равно нужен в Info.plist (SwiftUI не меняет требования приватности)  
- **Для TestFlight / App Store** — обязательно тестируйте запрос разрешения на устройстве без настроенной камеры  
- **Privacy Manifest** — **не требуется** для NSCameraUsageDescription (это UI-описание, а не сбор данных)  
- **Документируйте** — пишите комментарий в Info.plist:  
  `"NSCameraUsageDescription" = "Доступ к камере нужен для съёмки фото профиля и сканирования QR-кодов."`

**Короткий итог 2026**:
> `NSCameraUsageDescription` — **обязательный** ключ в Info.plist, объясняющий пользователю, зачем приложению нужна камера.  
> В 2026 году:  
> - без него приложение **крашнется** при попытке открыть камеру  
> - текст должен быть **конкретным** и **человеческим**  
> - добавляется в Xcode → Info → Privacy - Camera Usage Description  
> - если используете видео → добавьте ещё и NSMicrophoneUsageDescription  
> Это **одно из самых частых** причин отказа в App Review и крашей на устройстве.
