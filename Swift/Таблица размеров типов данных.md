## 1. Объяснение столбцов

- **Size** — фактический размер данных внутри структуры
    
- **Stride** — размер с учётом выравнивания CPU (каждая структура занимает кратное stride)
    
- **Alignment** — выравнивание (адрес должен делиться на это число)
    
- **Heap allocation** — хранится ли структура/данные в heap (для классов, массивов, словарей, захваченных замыканий)
    

---

## 2. Примеры проверки в Swift

```swift
import Foundation

print("Int size:", MemoryLayout<Int>.size)          // 8
print("Int stride:", MemoryLayout<Int>.stride)      // 8
print("Int alignment:", MemoryLayout<Int>.alignment)// 8

print("Bool size:", MemoryLayout<Bool>.size)        // 1
print("Bool stride:", MemoryLayout<Bool>.stride)    // 8
print("Bool alignment:", MemoryLayout<Bool>.alignment) // 1

print("Tuple<Int8, Int32> size:", MemoryLayout<(Int8, Int32)>.size) // 8
print("Tuple stride:", MemoryLayout<(Int8, Int32)>.stride)          // 8
print("Tuple alignment:", MemoryLayout<(Int8, Int32)>.alignment)    // 4
```

---

## 3. Small buffer оптимизация

- **String**, **Array**, **Dictionary**, **Set** используют **inline storage** для маленьких коллекций:
    

```swift
var smallArray = [1,2,3] // до 3-5 элементов inline
var smallDict = ["a": 1, "b": 2] // inline bucket storage
```

- При превышении small buffer → выделяется heap, size не меняется (struct остаётся 16 байт), но данные уходят в heap.
    

---

## 4. Важные выводы

1. **[[Value Type]] ≠ heap allocation**
    
    - Int, Bool, Tuple → хранятся inline
        
2. **[[Reference Type]] = pointer в структуре**
    
    - 8 байт на 64-битных платформах
        
3. **Stride > Size**
    
    - padding для выравнивания под CPU
        
4. **Optional и spare bit optimization**
    
    - [[Optional]] может не занимать больше места
