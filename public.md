**public** в Swift — это один из уровней доступа (access level modifier), который делает сущность (класс, структуру, функцию, свойство, инициализатор и т.д.) **видимой и доступной** для использования **вне текущего модуля** (т.е. в любом другом модуле, который импортирует данный).

По состоянию на 2026 год (Swift 6+) `public` остаётся **основным** уровнем доступа для создания **публичного API** библиотек, фреймворков, SPM-пакетов и модулей, которые будут использоваться в других проектах.

### Полная таблица уровней доступа в Swift 6 (2026)

| Уровень доступа     | Видимость                                      | Наследование / переопределение из других модулей | Когда использовать в 2026 |
|----------------------|------------------------------------------------|----------------------------------------------------|----------------------------|
| `private`            | Только внутри текущего файла                   | Нет                                                | Вспомогательные свойства/методы внутри файла |
| `fileprivate`        | Весь текущий файл                              | Нет                                                | Редко (делить между типами в одном файле) |
| `internal` (по умолчанию) | Весь текущий модуль (таргет)                | Нет                                                | Всё, что не нужно экспортировать наружу |
| `public`             | Весь модуль + любой импортирующий модуль       | Нет (нельзя наследовать/переопределять)            | Публичный API библиотек (методы, свойства, init, структуры, enum) |
| `open`               | То же, что public + **разрешено наследование и переопределение** в других модулях | **Да**                                             | Базовые классы и методы, которые должны расширяться |

### Когда использовать именно `public` (а не `open`) в 2026 году

| Ситуация                                      | Почему именно `public` (а не `open`) | Пример |
|-----------------------------------------------|---------------------------------------|--------|
| **Структуры**, **enum**, **функции**, **инициализаторы**, **свойства** | Эти типы **нельзя наследовать** — `open` для них запрещён | `public struct User: Codable { ... }` |
| **Методы**, **свойства**, **init**, которые **не предназначены для переопределения** | Пользователи должны использовать, но не менять поведение | `public func fetchData() async throws -> Data { ... }` |
| **Публичный API библиотеки**, где важна **стабильность** | `public` защищает от случайных переопределений | `public class NetworkClient { public func request(...) }` |
| **Типы без наследования**                     | `open` работает только для классов               | `public enum Status { case success, failure }` |

### Когда использовать `open` вместо `public`

```swift
// В библиотеке
open class BaseViewModel: ObservableObject {
    @Published open var title: String = ""
    
    open func loadData() async {
        // базовая реализация
    }
}

// В приложении пользователя
class CustomViewModel: BaseViewModel {
    override func loadData() async {
        await super.loadData()
        // кастомная логика
    }
}
```

**Правило 2026**:
- `open` — только для **классов** и их **методов/свойств**, которые **реально должны быть переопределены**  
- `public` — для **всего остального** публичного API

### Лучшие практики `public` / `open` в Swift 2026

- **Минимизируй public / open** — каждый публичный API — это обязательство на всю жизнь  
- **public** — для всего, что должно быть **использовано**, но **не изменено**  
- **open** — только для точек расширения (базовые классы, методы жизненного цикла)  
- **internal по умолчанию** — всё, что не нужно экспортировать, оставляй internal  
- **Swift 6 strict concurrency** — public / open классы и методы должны быть `@MainActor` или Sendable, если используются в UI/Concurrency  
- **Документируйте** — пиши `/// public` / `/// open` + объяснение, зачем это открыто  
- **Тестирование** — public / open API тестируй в отдельном тест-таргете с `@testable import`  
- **SPM-пакеты** — public / open — это твой контракт с пользователями пакета

**Короткий девиз 2026**:
> «public — это когда ты говоришь: «это можно использовать из любого модуля».  
> open — это когда ты говоришь: «это можно использовать **и** наследовать/переопределять из любого модуля».  
> В 2026 году public — для 90% публичного API, open — только для базовых классов и методов, которые должны расширяться.»
