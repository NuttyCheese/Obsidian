**non-optional** (неопциональный) в Swift — это термин, который описывает свойства, параметры, возвращаемые значения или переменные, которые **не могут быть nil** и **обязательно имеют значение**.

В Swift это противоположность **опциональным типам** (`?` / `Optional<T>`). Non-optional типы — это те, которые гарантированно содержат значение и не требуют распаковки или проверки на nil.

### Основные проявления non-optional в Swift (2026 актуальные)

| Контекст                          | Пример синтаксиса                          | Что означает                              | Когда обязательно использовать |
|-----------------------------------|--------------------------------------------|-------------------------------------------|---------------------------------|
| Свойства в struct / class / actor | `var name: String` (без ?)                 | Свойство **не может быть nil**            | Почти всегда (кроме редких случаев) |
| Параметры функций                 | `func greet(name: String)`                 | Аргумент **обязательно должен быть передан** и не nil | Почти всегда (кроме опциональных параметров) |
| Возвращаемый тип функции          | `func getUser() -> User`                   | Функция **гарантирует**, что вернёт не-nil значение | Когда nil невозможен по логике |
| Константы (`let`)                 | `let id: UUID = UUID()`                    | Значение **неизменно** и **не nil**       | Всегда для неизменяемых значений |
| Non-optional в протоколах         | `associatedtype ID: Hashable`              | Конкретный тип должен быть non-optional   | В протоколах с сильными требованиями |

### Самые частые ошибки и ловушки с non-optional (2026)

1. **Забыли инициализировать non-optional свойство в init**

```swift
struct User {
    var name: String     // non-optional
    var age: Int         // non-optional
    
    init(name: String) { // Ошибка компиляции!
        self.name = name
        // age не инициализировано → ошибка
    }
}
```

Решение: либо инициализируй все non-optional свойства, либо сделай их опциональными.

2. **Попытка присвоить nil non-optional свойству**

```swift
var user = User(name: "Alice", age: 30)
user.age = nil  // Ошибка компиляции!
```

3. **Использование force-unwrap (!) на non-optional**

```swift
let name: String = "Bob"
let length = name!.count  // Ошибка компилятора: Force-unwrap of non-optional value
```

4. **Non-optional в возвращаемом типе, но функция может вернуть nil**

```swift
func findUser(id: UUID) -> User {  // Ошибка: функция может вернуть nil
    return users.first { $0.id == id }  // nil возможен
}
```

Правильно:

```swift
func findUser(id: UUID) -> User? { ... }  // или throw
```

### Лучшие практики работы с non-optional в Swift 2026

- **Делай свойства non-optional по умолчанию** — это самый безопасный и выразительный способ  
- **Используй опционалы только когда nil реально возможен**  
- **Инициализируй все non-optional свойства в init** (или в default-значении)  
- **Избегай force-unwrap (!)** — даже если компилятор позволяет, это опасно  
- **Swift 6 strict concurrency** — non-optional типы безопаснее для Sendable (меньше Optional-обёрток)  
- **Протоколы** — предпочитай non-optional требования, если nil не имеет смысла  
- **Документируйте** — пиши комментарий «non-optional — значение всегда присутствует»

**Короткий девиз 2026**:

> В 2026 году это **основной** способ писать безопасный и выразительный код в Swift.  
> Опционалы — только там, где nil действительно возможен и имеет смысл.»
